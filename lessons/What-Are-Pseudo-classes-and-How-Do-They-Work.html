What Are Pseudo-classes?

Pseudo-classes are special CSS keywords that allow you to select an element based on its specific state or position.

Pseudo-classes target elements in particular states or positions within the document structure

Element States & Positions

Pseudo-classes can select elements based on various conditions:

When it's active

Element being clicked

When it's hovered

Mouse pointer over it

First child

First child of parent

Last child

Last child of parent

Link visited

Previously clicked link

When disabled

Disabled form element

Pseudo-class Syntax

To use a pseudo-class, add it to the selector using a colon (:) followed by the pseudo-class name.

Basic Syntax:

selector:pseudo-class {
  /* CSS properties */
}

Structure Breakdown:

  • selector: The element to target
  • : Colon separator (required)
  • pseudo-class: The state or position keyword
  • { }: CSS properties to apply

:active Pseudo-class

The :active pseudo-class selects the active state of an element, such as when clicking on a button.

CSS Example:

button:active {
  background-color: #660f12;
  transform: scale(0.98);
}

The button changes when you click it

Try It:

Click and hold the button to see the :active state in action!

:hover Pseudo-class

The :hover pseudo-class defines the hover state of an element when a mouse pointer is over it.

CSS Example:

a:hover {
  color: #a01a1e;
  text-decoration: underline;
}
Hover over me!

The link changes color and style on hover

Common Uses:

  • Button hover effects
  • Link styling
  • Menu item highlighting
  • Image overlays

:first-child Pseudo-class

The :first-child pseudo-class selects an element that is the first child of its parent element.

CSS Example:

.container p:first-child {
  background-color: lightcoral;
  padding: 0.4rem;
}

HTML Example:

<div class="container">
  <p>first child</p>
  <p>second child</p>
  <p>third child</p>
  <p>last child</p>
</div>

Result:

first child

second child

third child

last child

:last-child Pseudo-class

The :last-child pseudo-class targets the last element that is the child of its parent.

CSS Example:

.container p:last-child {
  background-color: lightblue;
  padding: 0.4rem;
}

Result:

first child

second child

third child

last child

Use Cases:

  • Remove bottom borders from last items
  • Add special styling to final list items
  • Adjust spacing for last elements

:visited Pseudo-class

The :visited pseudo-class lets you style a link the user has already visited.

CSS Example:

a:visited {
  color: #9c27b0;
}
Visit Example.com

After clicking, the link will change to purple

Important Note:

Due to privacy concerns, browsers limit which CSS properties can be modified with :visited (typically just color-related properties).

:disabled Pseudo-class

The :disabled pseudo-class lets you style an interactive element in disabled mode.

CSS Example:

button:disabled {
  background-color: #ccc;
  color: #666;
  cursor: not-allowed;
}

HTML Example:

<button disabled>Disabled Button</button>

This button cannot be clicked

Why Use Pseudo-classes?

Pseudo-classes let you style elements based on user interactions, making your website feel more interactive

Benefits:

  • Enhanced Interactivity: Respond to user actions
  • Better UX: Provide visual feedback
  • Cleaner HTML: No need for extra classes
  • Dynamic Styling: Style based on element position or state
  • Accessibility: Help users understand interactive elements

Other Common Pseudo-classes

Beyond the ones we've covered, there are many more pseudo-classes available:

:focus
:first-of-type
:last-of-type
:nth-of-type
:modal
:enabled
:checked
:required

Quick Overview:

  • :focus - Element with keyboard focus
  • :first-of-type - First element of its type
  • :nth-of-type - Elements by formula (even, odd, 3n)
  • :checked - Checked checkboxes or radio buttons
  • :enabled - Enabled form elements

Pseudo-classes Summary

Key Takeaways:

  • Definition: Special CSS keywords for selecting elements based on state or position
  • Syntax: selector:pseudo-class { }
  • :active - Element being clicked
  • :hover - Element being hovered over
  • :first-child - First child of parent
  • :last-child - Last child of parent
  • :visited - Previously visited links
  • :disabled - Disabled form elements
  • Purpose: Create interactive, dynamic user experiences
Pseudo-classes are essential for creating responsive, interactive web designs without extra JavaScript
