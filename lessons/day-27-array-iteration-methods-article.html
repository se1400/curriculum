Array Iteration Methods

What You'll Learn in This Lesson

  • Why array iteration methods exist and what problem they solve compared to writing loops manually
  • How forEach() runs a function on each element for side effects like updating the DOM
  • How map() transforms every element and returns a new array — the most important tool for rendering UI
  • How filter() keeps only elements that pass a test — the core of search and category filtering
  • How find() and findIndex() locate the first matching element
  • How findLast() and findLastIndex() search from the end of the array (ES2023)
  • How some() and every() answer "does any item pass?" and "do all items pass?" questions
  • How reduce() collapses an array into a single value — sums, counts, totals, or grouped objects
  • How flat() and flatMap() handle nested arrays
  • How to chain multiple methods into a single readable pipeline
  • How Object.groupBy() groups array items by category (ES2024)

You now know how to store data in arrays and objects, and how to format it with strings and numbers. This lesson gives you the most powerful tools for turning that data into visible, interactive pages. map() is how modern JavaScript renders UI — it is what React, Vue, and Svelte use internally. filter() is how search and category filtering works. reduce() is how shopping carts compute totals.

These methods replace common loop patterns with cleaner, more expressive code. A for loop that builds a new array is almost always a map(). A loop that conditionally keeps some items is almost always a filter(). Learning to think in these methods is one of the biggest upgrades in your JavaScript skill.

Every concept in this lesson is demonstrated with visible output. By the end, you will build a complete filterable, sortable product grid — one of the most common patterns in real-world front-end development.

Understanding the Callback Signature

Every iteration method accepts a callback function — a function that gets called once for each element in the array. The callback receives up to three arguments:

array.someMethod((element, index, array) => {
  // element — the current item
  // index   — its zero-based position in the array
  // array   — the original array (rarely needed)
});

In practice, most callbacks only use element. Use index when you need to number items or access adjacent items. The array argument is available for advanced use cases but rarely needed.

LISTING 27.1 — Callback Argument Examples 
const fruits = ['apple', 'banana', 'cherry'];

// Only element needed
fruits.forEach(fruit => {
  console.log(fruit);
});

// Element and index — useful for numbered lists
fruits.forEach((fruit, index) => {
  listEl.innerHTML += `<li>${index + 1}. ${fruit}</li>`;
});

forEach()

forEach() calls a function for each element and returns undefined. Use it when you want to do something with each item — update the DOM, log to the console, trigger an action. It is not for building new arrays.

LISTING 27.2 — forEach() Updating the DOM 
const notifications = [
  { text: 'New message from Sam',     type: 'message' },
  { text: 'Your order has shipped',   type: 'order'   },
  { text: 'Friend request from Lee',  type: 'social'  },
];

const list = document.querySelector('#notification-list');

notifications.forEach(notif => {
  const li = document.createElement('li');
  li.textContent = notif.text;
  li.className = `notif-${notif.type}`;
  list.appendChild(li);
});
NOTE: forEach() always returns undefined — you cannot chain another method after it. If you need the result, use map() instead.

map()

map() transforms every element of an array and returns a new array of the same length. The original array is unchanged. The callback must return the new value for each element.

[IMG - map method transforms each element]

map() is the most important rendering tool in JavaScript. To display an array of objects on screen, you map over it to produce an array of HTML strings, join them into one string, and set that as innerHTML.

LISTING 27.3 — map() Rendering a Product Grid 
const products = [
  { name: 'Wireless Headphones', price: 79.99,  category: 'Audio'  },
  { name: 'USB-C Hub',           price: 34.99,  category: 'Tech'   },
  { name: 'Notebook',            price: 12.99,  category: 'Office' },
];

const grid = document.querySelector('#product-grid');

grid.innerHTML = products.map(product => `
  <div class="card">
    <h3>${product.name}</h3>
    <span class="badge">${product.category}</span>
    <strong>$${product.price.toFixed(2)}</strong>
  </div>
`).join('');

Read this like a pipeline: take each product, turn it into an HTML string, collect those strings into an array, join them into one string, and set the result as the grid's HTML content.

LISTING 27.4 — map() Transforming Data 
const prices = [10, 20, 30, 40];

// Double every price
const doubled = prices.map(price => price * 2);
// [20, 40, 60, 80] — new array, original unchanged

// Extract one property from each object
const names = products.map(p => p.name);
// ['Wireless Headphones', 'USB-C Hub', 'Notebook']

filter()

filter() returns a new array containing only the elements where the callback returns true. The original array is unchanged. If no elements pass, an empty array is returned.

LISTING 27.5 — filter() for Category Filtering 
const products = [
  { name: 'Headphones', category: 'Audio',  inStock: true  },
  { name: 'USB Hub',    category: 'Tech',   inStock: false },
  { name: 'Speaker',    category: 'Audio',  inStock: true  },
  { name: 'Notebook',   category: 'Office', inStock: true  },
];

// Keep only Audio products
const audioProducts = products.filter(p => p.category === 'Audio');
// [{name:'Headphones',...}, {name:'Speaker',...}]

// Keep only in-stock items
const available = products.filter(p => p.inStock);

// Combine: in-stock Audio products
const availableAudio = products.filter(p => p.category === 'Audio' && p.inStock);
LISTING 27.6 — Live Category Toggle with filter() 
let selectedCategory = 'All';

function renderProducts() {
  const visible = selectedCategory === 'All'
    ? products
    : products.filter(p => p.category === selectedCategory);

  document.querySelector('#grid').innerHTML = visible.map(p => `
    <div class="card">${p.name}</div>
  `).join('');

  document.querySelector('#count').textContent =
    `Showing ${visible.length} of ${products.length}`;
}

document.querySelectorAll('.filter-btn').forEach(btn => {
  btn.addEventListener('click', () => {
    selectedCategory = btn.dataset.category;
    renderProducts();
  });
});

find() and findIndex()

find() returns the first element where the callback returns true. If nothing matches, it returns undefined. Unlike filter(), it stops searching as soon as it finds a match.

findIndex() works the same way but returns the index of the first match, or -1 if nothing matches.

LISTING 27.7 — find() and findIndex() 
const users = [
  { id: 1, name: 'Alice', role: 'admin'  },
  { id: 2, name: 'Bob',   role: 'editor' },
  { id: 3, name: 'Carol', role: 'viewer' },
];

// Find the admin user
const admin = users.find(u => u.role === 'admin');
// { id: 1, name: 'Alice', role: 'admin' }

// Find a user by ID from a URL parameter
const selectedId = 2;
const user = users.find(u => u.id === selectedId);
// { id: 2, name: 'Bob', role: 'editor' }

// Find the index to update the user later
const idx = users.findIndex(u => u.id === selectedId);
// 1
users[idx].role = 'admin'; // update by index

findLast() and findLastIndex()

findLast() and findLastIndex() are ES2023 additions that work exactly like find() and findIndex(), but search from the end of the array. They are ideal for time-sorted data where the most recent matching item is what you need.

LISTING 27.8 — findLast() for Most Recent Match 
const logEntries = [
  { type: 'info',    message: 'App started',    time: '09:00' },
  { type: 'warning', message: 'Slow response',  time: '09:15' },
  { type: 'error',   message: 'Timeout',        time: '09:22' },
  { type: 'info',    message: 'Retry succeeded',time: '09:23' },
  { type: 'error',   message: 'DB connection',  time: '09:31' },
];

// Most recent error
const lastError = logEntries.findLast(e => e.type === 'error');
// { type: 'error', message: 'DB connection', time: '09:31' }

const lastErrorIdx = logEntries.findLastIndex(e => e.type === 'error');
// 4

some() and every()

some() returns true if at least one element passes the test. every() returns true only if all elements pass. Both return false otherwise. These are perfect for controlling UI state.

LISTING 27.9 — some() and every() for UI State 
const cartItems = [
  { name: 'Headphones', price: 79.99, inStock: true  },
  { name: 'USB Hub',    price: 34.99, inStock: false },
  { name: 'Speaker',    price: 49.99, inStock: true  },
];

// some(): is ANY item out of stock? — show warning
if (cartItems.some(item => !item.inStock)) {
  warningEl.textContent = 'Some items are unavailable.';
  checkoutBtn.disabled = true;
}

// every(): are ALL items in stock? — safe to check out
if (cartItems.every(item => item.inStock)) {
  checkoutBtn.disabled = false;
}

// Validating a form: are all required fields filled?
const fields = document.querySelectorAll('input[required]');
const allFilled = [...fields].every(f => f.value.trim() !== '');
submitBtn.disabled = !allFilled;
NOTE: some() stops at the first true result. every() stops at the first false. Both are more efficient than looping through every element when you just need a yes/no answer.

reduce()

reduce() is the most flexible iteration method. It collapses an array into a single value of any type — a number, a string, an object, or even another array. The callback receives an accumulator (the running result) and the current element, and must return the new accumulator value. The second argument to reduce() is the initial value of the accumulator.

[IMG - reduce accumulator visual flow]

LISTING 27.10 — reduce() for a Shopping Cart Total 
const cart = [
  { name: 'Headphones', price: 79.99, qty: 1 },
  { name: 'USB Hub',    price: 34.99, qty: 2 },
  { name: 'Notebook',   price: 12.99, qty: 3 },
];

// Start at 0. Add each item's price × quantity. End with the total.
const total = cart.reduce((sum, item) => sum + item.price * item.qty, 0);
// 0 + 79.99 + 69.98 + 38.97 = 188.94

document.querySelector('#total').textContent =
  new Intl.NumberFormat('en-US', { style: 'currency', currency: 'USD' }).format(total);

Think of it step by step: you start with 0, then add 79.99 (headphones), then add 69.98 (two hubs), then add 38.97 (three notebooks), and end with 188.94. The accumulator carries the running total through each step.

LISTING 27.11 — reduce() to Count Items by Category 
const products = [
  { name: 'Headphones', category: 'Audio'  },
  { name: 'Speaker',    category: 'Audio'  },
  { name: 'USB Hub',    category: 'Tech'   },
  { name: 'Notebook',   category: 'Office' },
  { name: 'Pen Set',    category: 'Office' },
];

const counts = products.reduce((acc, product) => {
  const cat = product.category;
  acc[cat] = (acc[cat] || 0) + 1;
  return acc;
}, {});
// { Audio: 2, Tech: 1, Office: 2 }

flat() and flatMap()

flat(depth) flattens nested arrays. Without an argument (or with depth 1), it flattens one level deep. Pass Infinity to flatten all levels.

flatMap(callback) combines map() and flat(1) in one step — it maps over the array and flattens the result one level. It is useful when your callback returns arrays.

LISTING 27.12 — flat() and flatMap() 
const nested = [1, [2, 3], [4, [5, 6]]];
nested.flat();    // [1, 2, 3, 4, [5, 6]]   — one level
nested.flat(2);   // [1, 2, 3, 4, 5, 6]     — two levels
nested.flat(Infinity); // [1, 2, 3, 4, 5, 6] — all levels

// flatMap: each post has a tags array — collect all tags
const posts = [
  { title: 'Intro to JS',    tags: ['js', 'beginner'] },
  { title: 'CSS Grid Guide', tags: ['css', 'layout']  },
];

const allTags = posts.flatMap(post => post.tags);
// ['js', 'beginner', 'css', 'layout']

// Using map then flat separately produces the same result:
// posts.map(p => p.tags).flat()

Chaining Methods

Because most iteration methods return new arrays, you can chain them together into a single, readable pipeline. Each method receives the output of the previous one.

LISTING 27.13 — Chaining filter(), sort(), map(), join() 
const products = [/* array of product objects */];

const html = products
  .filter(p => p.inStock)                    // keep only in-stock
  .sort((a, b) => a.price - b.price)         // sort by price ascending
  .map(p => `<div class="card">
    <h3>${p.name}</h3>
    <strong>$${p.price.toFixed(2)}</strong>
  </div>`)                                  // render each as HTML
  .join('');                                  // combine into one string

document.querySelector('#grid').innerHTML = html;

Read it left to right: first keep in-stock items, then sort by price, then render each as a card, then combine into HTML. This pipeline produces a different result every time inStock or price changes in the data.

NOTE: Each method in a chain creates a new array. The original array is never mutated. For long chains over very large arrays, performance may matter — but for the data sizes typical in front-end work (hundreds or low thousands of items), chaining is always the right choice for readability.

Object.groupBy() — ES2024

Object.groupBy(array, keyFn) groups array items into an object. The keyFn is called on each item; its return value becomes the group key. The result is an object where each key maps to an array of matching items.

LISTING 27.14 — Object.groupBy() for Tab Navigation 
const products = [
  { name: 'Headphones', category: 'Audio'  },
  { name: 'Speaker',    category: 'Audio'  },
  { name: 'USB Hub',    category: 'Tech'   },
  { name: 'Notebook',   category: 'Office' },
];

const grouped = Object.groupBy(products, p => p.category);
// {
//   Audio:  [{ name:'Headphones',...}, { name:'Speaker',...}],
//   Tech:   [{ name:'USB Hub',...}],
//   Office: [{ name:'Notebook',...}]
// }

// Render the active tab's products
function renderTab(tabName) {
  const items = grouped[tabName] || [];
  document.querySelector('#tab-content').innerHTML =
    items.map(p => `<div class="card">${p.name}</div>`).join('');
}
NOTE: Object.groupBy() is available in all modern browsers as of 2024. Check caniuse.com if you need to support older browsers.

Rendering HTML with Iteration Methods — Full Example

Here is the complete, real-world pattern: an array of objects, filtered and sorted, rendered as cards on screen.

LISTING 27.15 — Complete Filter-Sort-Render Pipeline 
const products = [
  { name: 'Wireless Headphones', price: 79.99, category: 'Audio',  inStock: true  },
  { name: 'Portable Speaker',    price: 49.99, category: 'Audio',  inStock: true  },
  { name: 'USB-C Hub',           price: 34.99, category: 'Tech',   inStock: false },
  { name: 'Mechanical Keyboard', price: 129.99,category: 'Tech',   inStock: true  },
  { name: 'Notebook Set',        price: 12.99, category: 'Office', inStock: true  },
  { name: 'Desk Organiser',      price: 19.99, category: 'Office', inStock: true  },
];

let currentCategory = 'All';
let sortMode = 'price-asc';

function renderGrid() {
  const filtered = currentCategory === 'All'
    ? products
    : products.filter(p => p.category === currentCategory);

  const sorted = filtered.toSorted((a, b) =>
    sortMode === 'price-asc'  ? a.price - b.price :
    sortMode === 'price-desc' ? b.price - a.price :
    a.name.localeCompare(b.name)
  );

  document.querySelector('#grid').innerHTML = sorted
    .map(p => `
      <div class="card ${!p.inStock ? 'out-of-stock' : ''}">
        <h3>${p.name}</h3>
        <span class="badge">${p.category}</span>
        <strong>$${p.price.toFixed(2)}</strong>
        <span class="stock">${p.inStock ? '✓ In Stock' : '✗ Out of Stock'}</span>
      </div>`)
    .join('');

  document.querySelector('#count').textContent =
    `Showing ${sorted.length} of ${products.length} products`;
}

forEach vs map — When to Use Which

This is one of the most common sources of confusion for beginners:

  • Use forEach() when you want to perform an action for each item and you do not need a result — appending to the DOM, sending API requests, logging.
  • Use map() when you want to transform each item and use the resulting array — generating HTML strings, converting currencies, extracting a single property.
  • Never use forEach() to build a new array by pushing inside the callback — that is what map() is for.
LISTING 27.16 — forEach vs map 
// WRONG: using forEach to build an array
const names = [];
products.forEach(p => names.push(p.name)); // anti-pattern

// RIGHT: use map
const names = products.map(p => p.name);

// forEach is correct for DOM side effects
products.forEach(p => {
  const card = document.createElement('div');
  card.textContent = p.name;
  grid.appendChild(card); // side effect: mutating the DOM
});

Summary

  • forEach() runs a callback on each element and returns undefined — use it for side effects like DOM updates.
  • map() transforms every element and returns a new array of the same length — it is the primary tool for generating HTML from data.
  • filter() returns a new array containing only items where the callback returns true — use it for search, category filtering, and removing unwanted items.
  • find() and findIndex() locate the first matching item. findLast() and findLastIndex() (ES2023) search from the end.
  • some() returns true if any item passes. every() returns true only if all items pass. Both are perfect for toggling UI state.
  • reduce() collapses an array into a single value of any type. The initial value sets the accumulator's starting state.
  • Methods can be chained: filter().sort().map().join('') is a complete render pipeline.
  • Object.groupBy() (ES2024) groups items by a key function, producing an object of categorized arrays.

Q&A

What is the difference between forEach() and map()?
forEach() is for performing actions — it runs your callback for each item and returns undefined. You cannot chain further methods after it. map() is for transforming data — it also runs a callback for each item but returns a new array containing each return value. Use forEach() when you only need the side effect (like appending to the DOM). Use map() when you need the transformed data — generating HTML strings is the most common use.
When should I use filter() instead of writing a loop?
Always — unless there is a performance-critical reason not to. filter(p => p.category === 'Audio') is shorter, more expressive, and communicates intent immediately. A for loop that conditionally pushes to a new array does the same thing in four times as many lines. The key advantage is declarative code: filter() says what you want; a loop says how to get it.
How does reduce() work? What is the accumulator?
The accumulator is the running result that your callback updates on each iteration. You set its starting value as the second argument to reduce(). On each call, your callback receives the current accumulator and the current item, and must return the new accumulator. After the last item, the final accumulator value is returned. For summing cart totals: start accumulator at 0, add each item's price, finish with the total. For grouping by category: start with an empty object {}, add each item to its category key, finish with a grouped object.
What is the difference between find() and filter()?
find() returns a single item — the first one that matches — and stops searching. If nothing matches, it returns undefined. filter() always scans the entire array and returns all matching items as a new array (which may be empty). Use find() when you know there is one target (finding a user by ID). Use filter() when you want all matching items (all products in a category).
When should I use some() vs every()?
Use some() when you need to know if at least one item meets a condition — "is any item out of stock?", "does the cart have any items?" Use every() when all items must pass — "have all required fields been filled?", "are all selected items available?" Both return a boolean, making them ideal for enabling or disabling UI elements based on state.
What does chaining array methods mean?
Chaining means calling one method immediately after another on the result of the previous call. Because filter(), map(), sort(), and similar methods all return new arrays, you can write products.filter(...).sort(...).map(...).join('') as one continuous expression. Each step transforms the data and passes the result to the next. The original array is never mutated.
How do I use map() to generate HTML?
Map over your array, returning a template literal HTML string for each item. Then call .join('') on the resulting array of strings to concatenate them into one HTML string. Finally, assign that string to an element's innerHTML. The pattern is: container.innerHTML = data.map(item => `<div>${item.name}</div>`).join('');
What is Object.groupBy() and when is it useful?
Object.groupBy(array, keyFn) groups array items into an object where each key is a category and each value is an array of matching items. It is most useful when you need tab-based or category-based navigation — click a category tab and immediately have access to grouped[category] to render that group. Introduced in ES2024, it is available in all modern browsers.
