What Are Paths?

A path is a string that specifies the location of a file or directory in a file system.

In web development, paths let developers link to resources like images, stylesheets, scripts, and other web pages.

There are two types of paths: absolute paths and relative paths. Both are essential when specifying file locations within a file system.

Path Examples
<!-- Absolute path -->
<a href="https://example.com/image.jpg">

<!-- Relative path -->
<a href="image.jpg">

Absolute Paths

An absolute path is a complete link to a resource. It starts from the root directory, includes every other directory, and finally the filename and extension.

The "root directory" refers to the top-level directory or folder in a hierarchy.

An absolute path includes the protocol (such as http, https, or file) and the domain name if the resource is on the web.

Web Resource Example
<a href="https://design-style-guide.
       freecodecamp.org/img/
       fcc_secondary_small.svg">
  View fCC Logo
</a>

Protocol: https
Domain: design-style-guide.freecodecamp.org
Filename: fcc_secondary_small.svg

Absolute Paths on Local Machine

When linking to a resource on your local machine using an absolute path, you start from the root directory and traverse through every folder.

The URL in the browser includes the file:// protocol, followed by the complete path to the file.

Local Absolute Path
<p>
  Read more on the
  <a href="/Users/user/Desktop/fCC/
            script-code/absolute-vs-
            relative-paths/pages/
            about.html">
    About Page
  </a>
</p>
Browser Address Bar
file:///Users/user/Desktop/fCC/
script-code/absolute-vs-relative-
paths/pages/about.html

Protocol: file://
Path: /Users/user/Desktop/fCC/...
Filename: about.html

Relative Paths

A relative path specifies the location of a file relative to the directory of the current file.

It does not include the protocol or the domain name, making it shorter and more flexible for internal links within the same website.

If both files are in the same folder, you simply use the filename.

Same Folder Example
<!-- From contact.html to about.html
     (both in same folder) -->
<p>
  Read more on the
  <a href="about.html">About Page</a>
</p>

No protocol, no domain name
Just the filename!

When to Use Absolute Paths

Use absolute paths in the following situations:

  • When linking to a resource hosted on an external website
  • When you need the link to a page or resource to work consistently regardless of the document location within the site
External Resource
<!-- Linking to external site -->
<a href="https://github.com/user/repo">
  View on GitHub
</a>
Consistent Link
<!-- Works from any page location -->
<a href="https://mysite.com/resources/
       guide.pdf">
  Download Guide
</a>

When to Use Relative Paths

Use relative paths in the following situations:

  • When linking to resources within the same website
  • If you want to keep your code cleaner and easier to maintain during development
  • During local testing to ensure links work without an internet connection
Internal Website Links
<!-- Same folder -->
<a href="about.html">About</a>

<!-- Subfolder -->
<a href="pages/contact.html">Contact</a>

<!-- Parent folder -->
<a href="../index.html">Home</a>
Benefits
✓ Shorter and cleaner code
✓ Works offline during development
✓ Easier to move or rename folders
✓ No hardcoded domain names
✓ Portable across environments
