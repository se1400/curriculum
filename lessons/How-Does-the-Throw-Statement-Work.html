The throw Statement

What is the throw Statement?

The throw statement in JavaScript is used to throw a user-defined exception.

What is an Exception?

An exception in programming is when an unexpected event happens and disrupts the normal flow of the program.

Examples of Exceptions

  • Dividing by zero
  • Invalid user input
  • Accessing undefined properties
  • Network errors
  • File not found

Why Handle Exceptions?

Preventing Crashes

As programmers, it's important to handle exceptions so your programs don't crash unexpectedly when errors occur.

Benefits of Exception Handling

  • Prevent Crashes: Keep your application running smoothly
  • Better User Experience: Show helpful error messages
  • Debugging: Easier to identify and fix problems
  • Control Flow: Manage error scenarios gracefully
  • Robustness: Build more reliable applications

Professional Code

Proper exception handling is a mark of professional, production-ready code.

throw Statement Syntax

throw expression;

The Expression

The expression is the object or value that represents the exception you want to throw.

What Can You Throw?

  • Built-in Error classes: Error, TypeError, RangeError, etc.
  • Custom Error objects: Your own error classes
  • Strings: Simple error messages (not recommended)
  • Any value: Though objects are preferred

Best Practice: Use Error objects or classes for better error information and stack traces.

Choosing the Right Error

  • Error: Generic error (most common)
  • TypeError: Value is not of expected type
  • RangeError: Value is outside allowed range
  • ReferenceError: Invalid reference

Example: TypeError

Scenario

Create a function that validates if the input is a number, and throw a TypeError if it's not.

The Function

function validateNumber(input) {
  if (typeof input !== "number") {
    throw new TypeError("Expected a number, but received " + typeof input);
  }
  return input * 2;
}

Function Purpose

This function doubles a number, but first validates that the input is actually a number.

TypeError Example Breakdown

Step-by-Step Analysis

1. Type Check

if (typeof input !== "number")

Check if the type of input is NOT a number

2. Throw TypeError

throw new TypeError("Expected a number, but received " + typeof input);

Throw a TypeError with a custom, descriptive message

3. Return Result

return input * 2;

If the input is valid, return the doubled value

TypeError Example in Action

Valid Input

const result = validateNumber(5);
console.log(result); // Output: 10

✓ Input is a number, function returns 5 × 2 = 10

Invalid Input

const result = validateNumber("hello");
// Throws: TypeError: Expected a number, but received string

✗ Input is a string, TypeError is thrown with the message:
"Expected a number, but received string"

Generic Error Example

Using the Error Constructor

If you want to throw a more generic error message, you can use the Error constructor.

Divide Function

function divide(numerator, denominator) {
  if (denominator === 0) {
    throw new Error("Cannot divide by zero");
  }
  return numerator / denominator;
}

What This Does

This function checks if the denominator is 0. If it is, it throws an error with the message "Cannot divide by zero".

Divide Function Breakdown

Logic Flow

1. Check for Zero

if (denominator === 0)

Validate that we're not dividing by zero

2. Throw Error

throw new Error("Cannot divide by zero");

Stop execution and throw a clear error message

3. Perform Division

return numerator / denominator;

If valid, perform and return the division

Divide Function in Action

Valid Division

const result = divide(10, 2);
console.log(result); // Output: 5

✓ Denominator is not zero, returns 10 ÷ 2 = 5

Invalid Division

const result = divide(10, 0);
// Throws: Error: Cannot divide by zero

✗ Denominator is zero, Error is thrown with the message:
"Cannot divide by zero"

What Happens When You Throw?

Immediate Effects

When you use the throw statement:

Program Behavior

  • Execution Stops: The current function stops executing immediately
  • Error Propagates: The error bubbles up through the call stack
  • Looking for Handler: JavaScript looks for error handlers (try/catch)
  • No Handler Found: If no handler exists, the program crashes
  • Handler Found: If a handler exists, control transfers to it

⚠️ Important

Code after a throw statement will not execute unless the error is caught.

Different throw Syntaxes

Using new Error()

throw new Error("Something went wrong");

✓ Recommended: Creates an Error object with message and stack trace

Using new TypeError()

throw new TypeError("Expected a different type");

✓ Specific: Use when you know the error type

Throwing a String (Not Recommended)

throw "Error message";

✗ Avoid: No error object, no stack trace

Creating Custom Error Messages

Descriptive Messages

Always provide clear, descriptive error messages that help identify the problem.

Good vs Bad Error Messages

❌ Bad Messages

throw new Error("Error");
throw new Error("Bad input");
throw new Error("Failed");

Too vague, not helpful

✅ Good Messages

throw new Error("Cannot divide by zero");
throw new TypeError("Expected number, got string");
throw new RangeError("Age must be between 0 and 120");

Clear, specific, actionable

What is try/catch?

The try/catch syntax is used to gracefully handle exceptions in JavaScript.

It allows you to:

  • Catch errors that are thrown
  • Handle them without crashing
  • Continue program execution
  • Provide user-friendly error messages

Preview Example

try {
  const result = divide(10, 0);
} catch (error) {
  console.log("Caught error:", error.message);
}
