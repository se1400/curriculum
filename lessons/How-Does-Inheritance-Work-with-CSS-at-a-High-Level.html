CSS Inheritance: The Basics

Inheritance determines how styles are passed from parent elements to their child elements.

Like real-world inheritance, children inherit traits from their parents. In CSS, certain properties can be inherited by child elements from their parent elements.

Key Benefit

Allows for efficient and consistent styling across an entire document without repeating code.

Properties That Inherit by Default

Not all CSS properties inherit automatically. Properties that do inherit include:

  • color - Text color
  • font-family - Font type
  • line-height - Line spacing

When you set these properties on a parent element, all child elements will inherit them unless specifically overridden.

Example: Inheriting Color

The Code

<div style="color: blue;">
  This is the parent element.
  <p>This is the child element 
  inheriting the color.</p>
</div>

What Happens

Both the parent div and the child p will display their text in blue.

This is the parent element.
This is the child element inheriting the color.

Why? The color property is inherited automatically.

Properties That Don't Inherit

The following properties are not inherited by default:

  • margin - Outer spacing
  • padding - Inner spacing
  • border - Element borders
  • background - Background styling

To make a child element use these styles from its parent, you must set them explicitly.

Forcing Inheritance with inherit

The inherit keyword forces a child element to inherit a property from its parent, even if that property doesn't normally inherit.

Use Case

When you want a child element to have the same non-inheriting property value as its parent (like padding or margin).

Example: Using inherit

The Code

<div style="padding: 20px;">
  This is the parent element 
  with padding.
  <p style="padding: inherit;">
    This is the child element 
    inheriting the padding.
  </p>
</div>

Result

The child p element will inherit the 20px of padding from its parent div.

Without padding: inherit, the child would have no padding since padding doesn't inherit by default.

Why Inheritance Matters

Consistency

Maintain uniform styling across your document without repeating rules.

Less Redundancy

Define a style once on a parent element, and child elements inherit it automatically.

Easier Maintenance

Makes CSS more concise and manageable. Change one parent style to update all children.

Important: One-Directional Flow

Parent → Child Only

Inheritance works in one direction only: from parent to child.

If you override a style on a child element, it won't affect the parent element.

Child elements can break the inheritance chain by defining their own values, but this doesn't propagate upward.
