Three Ways to Apply CSS

CSS can be applied to a webpage in three main ways: inline, internal, or external. Each method has its own use cases, advantages, and limitations.

Why This Matters

Understanding when to use each type of CSS is essential for writing clean, efficient, and maintainable code. Choosing the right method impacts code organization, reusability, and project scalability.

Inline CSS

Written directly within an HTML element using the style attribute. Applies styles to a specific element only.

Use Cases

  • Quick, one-off styles
  • Overriding other styles for specific elements
  • Testing styles during development

⚠️ Important Note

Should be avoided in most cases because it clutters HTML and makes code harder to maintain.

HTML with Inline CSS
<p style="color: green;">
  This is an inline-styled paragraph.
</p>
Browser Output

This is an inline-styled paragraph.

Internal CSS

Written within <style> tags inside the <head> section. Applies styles to the entire page.

Best For

  • Styling a single document
  • Single-page websites
  • Page-specific styles

Limitations

  • Not reusable across multiple pages
  • Mixes HTML and CSS
  • Harder to maintain in larger projects
HTML with Internal CSS
<head>
  <style>
    p {
      color: blue;
    }
  </style>
</head>
<body>
  <p>This paragraph is styled using 
  internal CSS.</p>
</body>
Browser Output

This paragraph is styled using internal CSS.

External CSS

Written in a separate .css file and linked to HTML using the <link> element in the head section.

The Link Element

  • rel attribute: Specifies the relationship (e.g., "stylesheet")
  • href attribute: Specifies the URL of the CSS file

✓ Preferred Method

External CSS is the preferred method in professional web development for maintaining consistent styles across multiple pages.

index.html
styles.css
index.html
<head>
  <link rel="stylesheet" 
        href="styles.css">
</head>
<body>
  <p>This paragraph is styled using 
  external CSS.</p>
</body>
styles.css
p {
  color: purple;
}
Browser Output

This paragraph is styled using external CSS.

Why External CSS is Ideal

External CSS is best for large projects where you want to maintain consistent styles across multiple pages.

Key Benefits

  • Separation of Concerns: HTML handles structure, CSS handles styling
  • Maintainability: Changes in one file update all pages
  • Scalability: Easy to manage as projects grow
  • Reusability: One stylesheet for multiple pages

Comparison: When to Use Each Method

Inline CSS

  • Quick one-off styles
  • Overriding specific elements
  • Testing during development

Internal CSS

  • Single-page websites
  • Page-specific styles
  • Styling one document

External CSS

  • Large projects
  • Multiple pages
  • Professional development

Best Practice Guidelines

Understanding when to use each type of CSS is crucial for efficient and effective web development.

The Golden Rule

In most cases, external CSS should be your go-to approach, especially for larger and more complex projects.

Key Takeaways

  • Inline CSS: Use sparingly for quick overrides
  • Internal CSS: Good for single-page applications
  • External CSS: Best for maintainability and scalability
  • Separation of concerns leads to cleaner, more maintainable code
