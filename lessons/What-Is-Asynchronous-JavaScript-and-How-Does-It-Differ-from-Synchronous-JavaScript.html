Asynchronous JavaScript

What is Asynchronous JavaScript?

Let's learn about asynchronous JavaScript.

Asynchronous events occur at different times, independently from each other.

Definition

In the context of software development, "asynchronous" refers to tasks that run in the background, independently from the main flow of the program.

‚úì Main Advantage

The main advantage of asynchronous processes is that they don't block the execution of the main program.

That's particularly helpful for tasks that may take a long time to run, such as:

‚Ä¢ Fetching data from a remote server

‚Ä¢ Processing large files

‚Ä¢ Handling user input

‚Ä¢ Performing complex calculations

Asynchronous Programming

This is what we know as asynchronous programming.

Synchronous Programming

Traditional Approach

This approach contrasts with the traditional synchronous programming technique that you have been working with so far.

Predictable Execution

When you write multiple lines of code, you can usually predict what will happen and when it will happen.

The first line will be executed, then the second line, and so on in the order that you write them.

Example

const topic = "JavaScript"; const learning = `I'm learning ${topic}!`; console.log(learning);

Sequential Execution

In this example, the first line where you define the variable will be executed first, then the second line, and finally the third line where the string is logged to the console.

Everything is sequential and predictable. Each line is completed before the next one starts.

Single-Threaded

This type of JavaScript program is known as single-threaded.

Understanding Threads

What is a Thread?

The concept of threads is very important.

A thread is a unit of execution within a process. It's like a separate flow of control within the program.

Synchronous Threading

In synchronous programming, threads execute sequentially, one after the other.

If a thread is blocked, like waiting for user input, the entire process is blocked until the thread is completed.

Asynchronous Threading

In asynchronous programming, threads can be executed concurrently, running multiple threads at the same time.

This way, the program can continue running multiple tasks simultaneously without making the main program unresponsive, even if one of the threads is blocked.

Synchronous: One task at a time, blocking

Asynchronous: Multiple tasks concurrently, non-blocking

Synchronous vs Asynchronous

Execution Sequential (line by line) Concurrent (multiple at once)
Blocking Blocks until complete Non-blocking

User Experience May freeze UI Keeps UI responsive
Examples Variable assignment, calculations API calls, file reading, timers

‚úì Primary Difference

Synchronous code executes sequentially, while asynchronous code executes concurrently.

Callbacks

The Traditional Way

Asynchronous programming often involves callbacks, promises, or async/await to handle non-blocking operations.

What is a Callback?

As you learned in earlier lessons, a callback is a function that you pass to another function to be called later.

Event handlers are a particular type of callback that you've worked with before.

Example

// Callback example with setTimeout console.log("Start"); setTimeout(() => { console.log("This runs after 2 seconds"); }, 2000); console.log("End"); // Output: // Start // End // This runs after 2 seconds

‚ö†Ô∏è Callback Hell

They used to be the most common way to implement asynchronous functions in JavaScript.

However, if the callback function also takes other callback functions, the code and logic can get quite complicated very quickly.

Promises

Modern Approach

Currently, the most commonly used technique for implementing asynchronous programming in JavaScript is the promise.

What is a Promise?

A Promise is an object that represents the eventual completion (or failure) of an asynchronous process and its value.

Unknown Value

The value of a promise is not known when the promise is created.

It's only known when the asynchronous process is completed.

Example

// Creating a promise const myPromise = new Promise((resolve, reject) => { setTimeout(() => { resolve("Data received!"); }, 2000); }); // Using the promise myPromise .then(data => { console.log(data); // "Data received!" }) .catch(error => { console.error(error); });

Real-World Promise Example

Fetching Data

For example, the process of fetching data from a remote API for your web application may take some time.

Meanwhile, you want your users to have a nice user experience, right?

‚úì Implementation

To implement this, you could create a promise to keep the user interface active and interactive while the asynchronous process is running.

When the process is completed, the promise will contain the data that was sent back by the API, so the application can handle it or render it appropriately when it's available.

Example: API Call

// Show loading indicator showLoader(); // Fetch data from API fetch('https://api.example.com/data') .then(response => response.json()) .then(data => { // Data is ready! Render it renderData(data); hideLoader(); }) .catch(error => { // Handle error showError(error); hideLoader(); }); // UI remains responsive during fetch!

Techniques for Asynchronous Programming

Three Main Approaches

1. Callbacks

When: Event handlers, simple async operations

setTimeout(() => { console.log("Done!"); }, 1000);

Issue: Callback hell with nested callbacks

2. Promises

When: API calls, file operations

fetch(url) .then(res => res.json()) .then(data => console.log(data));

Better: Chainable, cleaner than callbacks

3. Async/Await

When: Modern async code

async function getData() { const data = await fetch(url); console.log(data); }

Best: Looks synchronous, easiest to read

Event Listeners

When: User interactions

button.addEventListener('click', () => { console.log("Clicked!"); });

Special: Responds to user events

When to Use Synchronous vs Asynchronous

Use Synchronous When:

‚úì Operations are quick (< 1ms)

‚úì Order matters absolutely

‚úì Simple calculations

‚úì Variable assignments

‚úì Array operations (map, filter)

‚úì String manipulations

Example: Math calculations, data transformations

Use Asynchronous When:

‚úì Network requests (API calls)

‚úì File reading/writing

‚úì Database queries

‚úì Timers and delays

‚úì User input waiting

‚úì Any I/O operations

Example: Fetching user data from server

Rule of Thumb

If an operation might take more than a few milliseconds or depends on external factors (network, user, file system), use asynchronous programming!
