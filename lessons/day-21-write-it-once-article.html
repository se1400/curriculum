Write It Once

Functions, Loops, and Generating HTML

What You’ll Learn in This Lesson:

  • ►Why functions exist and how they make your code reusable
  • ►How to write function declarations with parameters and return values
  • ►Arrow function syntax — a shorter way to write the same thing
  • ►Default parameter values
  • ►The for loop and the for...of loop
  • ►How to generate entire sections of HTML using loops and template literals
  • ►The renderCards() pattern — passing data in, getting HTML out

By now you have written a significant amount of JavaScript. Events, conditionals, DOM manipulation, class toggles — all of it works. But if you look back at your code, you may notice a problem: the same logic appears in multiple places. A click handler that runs the same update logic as a different click handler. DOM queries repeated three times. The same validation check written out twice.

This lesson solves that problem with two tools: functions and loops. Functions give a block of code a name so you can call it anywhere without copying it. Loops repeat an action for every item in a collection without writing the action multiple times. Together, they unlock the most visually powerful pattern in this course: generating a full styled card grid from an array of data with a handful of lines.

Why Functions?

Consider the problem concretely. You have a dark mode toggle, an accordion button, and a preferences button — and all three need to run the same “update the UI state” logic after they fire. Without functions, you write that logic three times. When a bug appears, you fix it three times. When the logic changes, you update three places and probably miss one.

// Without a function — fragile, repetitive
document.querySelector('#dark-btn').addEventListener('click', function() {
  document.body.classList.toggle('dark-mode');
  document.querySelector('#theme-label').textContent =
    document.body.classList.contains('dark-mode') ? 'Light' : 'Dark';
});

document.querySelector('#pref-btn').addEventListener('click', function() {
  document.body.classList.toggle('dark-mode');
  document.querySelector('#theme-label').textContent =
    document.body.classList.contains('dark-mode') ? 'Light' : 'Dark';
  // ... same code, again
});
// With a function — write the logic once, call it anywhere
function toggleDarkMode() {
  document.body.classList.toggle('dark-mode');
  document.querySelector('#theme-label').textContent =
    document.body.classList.contains('dark-mode') ? 'Light' : 'Dark';
}

document.querySelector('#dark-btn').addEventListener('click', toggleDarkMode);
document.querySelector('#pref-btn').addEventListener('click', toggleDarkMode);

The version with a function is shorter, clearer, and has a single place to fix if anything changes. Functions are named recipes: define the steps once, call the name whenever you need them.

NOTE
You have already been calling functions: document.querySelector(), classList.toggle(), console.log(), Math.random() are all functions that someone else wrote. Now you write your own.

Function Declarations

A function declaration has four parts: the function keyword, a name, parentheses for parameters, and a body in curly braces:

function greet(name) {
  document.querySelector('#message').textContent = 'Hello, ' + name + '!';
}

greet('Ada');    // Calls the function — runs the body with name = 'Ada'
greet('Grace');  // Calls it again with a different argument

Parameters are the variables listed in the parentheses. They receive the values you pass when calling the function. A function can have zero parameters, one, or many:

function showWelcome() {                          // No parameters
  document.querySelector('h1').textContent = 'Welcome!';
}

function setTheme(hue, saturation) {             // Two parameters
  document.documentElement.style.setProperty('--hue', hue);
  document.documentElement.style.setProperty('--saturation', saturation);
}

Return Values

A function can compute a value and send it back to the caller using return:

function formatPrice(price) {
  return '$' + price.toFixed(2);
}

const label = formatPrice(9.99);  // label = '$9.99'
document.querySelector('.price').textContent = label;

Without return, the function returns undefined. Use return when the function produces a value someone else needs. Skip it when the function just performs an action (like updating the DOM).

Default Parameters

A parameter can have a default value used when no argument is passed:

function greet(name = 'friend') {
  return 'Hello, ' + name + '!';
}

greet('Maria');  // 'Hello, Maria!'
greet();         // 'Hello, friend!'  — default used

Arrow Functions

Arrow functions are a shorter syntax for writing the same thing. They use => instead of the function keyword:

// Function declaration
function formatPrice(price) {
  return '$' + price.toFixed(2);
}

// Arrow function — identical behavior
const formatPrice = (price) => {
  return '$' + price.toFixed(2);
}

// Arrow function — even shorter when the body is a single expression
const formatPrice = (price) => '$' + price.toFixed(2);

When the function body is a single expression, you can drop the curly braces and return — the value is returned automatically. This concise form is extremely common, especially inside array iteration methods:

const double = (n) => n * 2;
const isEven = (n) => n % 2 === 0;
const toCard = (name) => `<div class="card">${name}</div>`;

NOTE
Arrow functions and function declarations behave identically for everything you will do in this course. Use whichever feels more readable for the situation. Arrow functions are especially common in callbacks — functions passed to event listeners or array methods — because they are concise.

The for Loop

A for loop repeats a block of code a specific number of times. Here is the full syntax:

for (let i = 0; i < 5; i++) {
  console.log(i);  // 0, 1, 2, 3, 4
}

Three parts sit inside the parentheses, separated by semicolons:

  • ►Initialization: let i = 0 — create a counter starting at zero
  • ►Condition: i < 5 — keep running while this is true
  • ►Update: i++ — increment the counter after each iteration

To loop through an array by index:

const colors = ['red', 'blue', 'green'];

for (let i = 0; i < colors.length; i++) {
  console.log(colors[i]);  // 'red', 'blue', 'green'
}

The for…of Loop

The for...of loop is a simpler way to iterate over arrays when you do not need the index:

const colors = ['red', 'blue', 'green'];

for (const color of colors) {
  console.log(color);  // 'red', 'blue', 'green'
}

Cleaner and more readable. Use for...of as your default. Use the index-based for loop when you specifically need the position of each item.

Generating HTML with Loops

This is the pattern that makes card grids come alive. Instead of writing fifty <div class="card"> elements by hand, you store the data in an array and let a loop build the HTML:

const cities = ['Tokyo', 'Lisbon', 'Chicago', 'Nairobi', 'Sydney'];
let html = '';

for (const city of cities) {
  html += `<div class="card">${city}</div>`;
}

document.querySelector('.grid').innerHTML = html;

Each iteration concatenates one card’s HTML to the html string. After the loop, one innerHTML assignment injects the entire result. The CSS card styles you have been writing for weeks apply automatically — the elements are real HTML once they are in the DOM.

The renderCards() Pattern

Wrapping the loop in a function makes it reusable. Call the function once to render the initial list, call it again after a shuffle or filter, and the grid rebuilds automatically:

const container = document.querySelector('.grid');

function renderCards(items) {
  let html = '';
  for (const item of items) {
    html += `
      <div class="card">
        <h3>${item}</h3>
      </div>
    `;
  }
  container.innerHTML = html;
}

renderCards(cities);  // Initial render

An even more concise version uses the map() array method and join(''):

function renderCards(items) {
  container.innerHTML = items.map(item => `
    <div class="card">
      <h3>${item}</h3>
    </div>
  `).join('');
}

map() transforms each item in the array into a string of HTML. join('') combines the array of strings into one, removing the commas JavaScript would otherwise add between them. This two-method chain produces the same result as the loop, in two lines. You will use this pattern constantly as you build data-driven interfaces.

Putting It Together

Functions, loops, and template literals combine to create a clean separation between data and display. Your data lives in an array. Your function knows how to turn one item into HTML. Your loop (or map()) applies the function to every item. Changing what gets displayed means changing the array and calling the render function again — the CSS card layouts you have designed stay completely unchanged.

const projects = ['Portfolio Site', 'Weather App', 'Recipe Finder', 'Quiz Game'];

function renderProjects(list) {
  container.innerHTML = list.map(name => `
    <div class="card">
      <h3 class="card-title">${name}</h3>
      <p class="card-meta">Web Project</p>
    </div>
  `).join('');
}

// Render on load
renderProjects(projects);

// "Shuffle" button — reverses and re-renders
document.querySelector('#shuffle-btn').addEventListener('click', () => {
  renderProjects([...projects].reverse());
});

Best Practices

  • ►Name functions after what they do. renderCards(), formatPrice(), showError() — a reader should understand the function’s purpose from its name alone.
  • ►Keep functions focused. A function that renders cards should render cards, not also validate form input. When a function does two things, split it into two functions.
  • ►Default to for...of for arrays. It is shorter and more readable than index-based for loops when you do not need the position. Use index-based loops only when the index matters.
  • ►Use join('') when building HTML strings with map(). Without it, JavaScript adds a comma between each item in the resulting array, which will appear visibly in your rendered HTML.
  • ►Escape user content before inserting it with innerHTML. When the data comes from user input, use textContent on individual elements rather than injecting untrusted strings directly into HTML. When the data is under your control (your own array), innerHTML is fine.

Summary

Functions give code a name and let you call it multiple times without copying it. You learned to write function declarations with parameters and return values, default parameter values for optional arguments, and the shorter arrow function syntax you will see everywhere in modern JavaScript. Loops — both the index-based for loop and the simpler for...of loop — repeat an action for every item in an array. Combining loops with template literals and innerHTML produces the render pattern: pass an array of data to a function, get a full card grid back. The CSS layouts you designed in the flexbox and grid lessons are now populated dynamically by data instead of written by hand.

The next step is upgrading from arrays of strings to arrays of objects — structured data with multiple properties per item. This turns your card grid into a real product catalog, portfolio gallery, or recipe list, complete with names, prices, categories, and images.
