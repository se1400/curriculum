The Animation Constructor

Core of WAAPI

At the core of WAAPI is the Animation constructor, which provides several instance methods and properties that allow you to dynamically animate elements.

The animate() Method

A significant method in the Animation constructor is animate().

It allows you to create an animation by specifying keyframes and options like duration, direction, easing, and iterations.

Basic Syntax

element.animate(keyframes, options);

Parameters

keyframes: An array of objects defining the animation states

options: An object with animation settings (duration, easing, iterations, etc.)

Return Value

Returns an Animation object that you can control with methods like play(), pause(), reverse()

const square = document.getElementById("square"); square.animate( [ { transform: "translateX(0px)" }, { transform: "translateX(300px)" } ], { duration: 2000, iterations: Infinity, direction: "alternate" } );

Understanding Keyframes

What Are Keyframes?

Keyframes define the states of the animation at different points in time.

They're an array of objects, each representing a snapshot of CSS properties.

// Keyframes array [ { transform: "translateX(0px)" }, // Start: at position 0 { transform: "translateX(300px)" } // End: at position 300px ]

More Keyframe Examples

// Fade in [ { opacity: 0 }, { opacity: 1 } ] // Rotate [ { transform: "rotate(0deg)" }, { transform: "rotate(360deg)" } ] // Scale up [ { transform: "scale(1)" }, { transform: "scale(1.5)" } ]

Multiple Properties

// Combine effects [ { transform: "translateX(0px)", opacity: 0, backgroundColor: "blue" }, { transform: "translateX(300px)", opacity: 1, backgroundColor: "red" } ]

Understanding Options

Animation Options

The options object controls how the animation behaves.

{ duration: 2000, // How long animation lasts (ms) iterations: Infinity, // How many times to repeat direction: "alternate", // Play forward then backward easing: "ease-in-out", // Timing function delay: 0, // Delay before starting (ms) fill: "both" // How styles apply before/after }

duration

Animation length in milliseconds

Example: 2000 = 2 seconds

iterations

Number of times to repeat

1 = once, Infinity = forever

direction

Animation playback direction

"normal", "reverse", "alternate", "alternate-reverse"

easing

Timing function for acceleration

"linear", "ease", "ease-in-out", "cubic-bezier(...)"

Animation Instance Methods

Control Your Animations

The Animation constructor provides instance methods that allow you to control animations dynamically.

Instance Methods

  • play(): Starts or resumes the animation
  • pause(): Pauses the animation
  • reverse(): Reverses the animation direction
  • finish(): Jumps to the end of the animation
  • cancel(): Cancels the animation and resets to initial state

Example Usage

const animation = square.animate(keyframes, options); // Control the animation animation.play(); // Start/resume animation.pause(); // Pause animation.reverse(); // Play backwards animation.finish(); // Jump to end animation.cancel(); // Stop and reset

Animation Instance Properties

Read and Modify Animation State

The Animation object also has properties you can read or modify.

Instance Properties

  • playbackRate: Speed multiplier (1 = normal, 2 = double speed)
  • currentTime: Current position in milliseconds
  • startTime: When the animation started
  • effect: The KeyframeEffect object
  • timeline: The timeline object
  • playState: Current state ("idle", "running", "paused", "finished")
  • finished: Promise that resolves when animation completes
  • onfinish: Callback when animation finishes
  • oncancel: Callback when animation is cancelled

Example Usage

const animation = square.animate(keyframes, options); // Read properties console.log(animation.playState); // "running" // Modify speed animation.playbackRate = 2; // Play at 2x speed // Set callback animation.onfinish = () => { console.log("Animation completed!"); };

Example: Play and Pause Controls

Modified Example

const square = document.getElementById("square"); const animation = square.animate( [ { transform: "translateX(0px)" }, { transform: "translateX(300px)" } ], { duration: 2000, fill: "forwards" } ); animation.pause(); // Start paused document.getElementById("playBtn").onclick = () => animation.play(); document.getElementById("pauseBtn").onclick = () => animation.pause(); animation.onfinish = () => { console.log("Animation finished!"); };

WAAPI vs CSS Animations

How They Relate

WAAPI extends the capabilities of CSS animations by providing more dynamic control over animations right inside JavaScript.

CSS Animations

With CSS animations, you define animations declaratively using properties like:

‚Ä¢ animation-name

‚Ä¢ animation-duration

‚Ä¢ animation-timing-function

‚úì WAAPI Advantage

You also have the opportunity to do the same thing using the animate() method of WAAPI.

The difference is that you can control the animations you create with the animate() method more directly and dynamically.

But with CSS animations, you need to do way more by defining custom styles and triggering them inside your JavaScript file.

CSS vs WAAPI Comparison

Feature CSS Animations WAAPI
Where Defined In CSS files In JavaScript code
Control Method Declarative (automatic) Imperative (programmatic)
Dynamic Control Limited (add/remove classes) Full (methods & properties)
Pause/Resume Difficult Easy with pause()/play()
Reverse Complex workarounds Simple with reverse()
Speed Control Must redefine animation Easy with playbackRate
Best For Simple, automatic animations Interactive, complex animations

When to Use Each Approach

‚úì Use CSS Animations When:

CSS animation is ideal for simple and declarative animations that run automatically.

Those include:

‚Ä¢ Hover effects: Button hover states, link underlines

‚Ä¢ Transitions: Fade-ins, slide-ins on page load

‚Ä¢ Non-interactive animations: Animations that don't need much interaction once triggered

‚úì Use WAAPI When:

If your animation needs to respond to user interactions like clicks and scrolls, or you want the user to be able to pause, reverse, or change speed dynamically, WAAPI is the better choice.

Use cases:

‚Ä¢ Interactive controls: Play/pause buttons

‚Ä¢ User-driven animations: Drag-and-drop, scroll-based

‚Ä¢ Dynamic changes: Speed controls, direction changes

‚Ä¢ Complex sequences: Chained animations with logic

‚Ä¢ Game animations: Character movements, effects

üí° Best of Both Worlds

You can consider combining both WAAPI and CSS animations when you want the simplicity of CSS for basic setup but need to control animations at runtime.
