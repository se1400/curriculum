The minmax() Function

Flexible Grid Track Sizing

The minmax() function defines the range for the size of a grid track, specifying how much space a row or column can occupy.

Purpose: Instead of setting a fixed size, minmax() lets you set a minimum and maximum size, allowing the track to adapt flexibly within that range.

What You'll Learn:

  • How minmax() works
  • Setting minimum and maximum sizes
  • Creating flexible, responsive layouts
  • Advantages over fixed sizes

Track Sizing Options

Before minmax()

Remember that you can set the track size with units like px (pixels), rem, or even em, and with fractional units (fr).

/* Fixed sizes */ grid-template-columns: 200px 300px 100px; /* Flexible sizes */ grid-template-columns: 1fr 2fr 1fr; /* Mixed */ grid-template-columns: 200px 1fr 100px;

Limitation: Fixed sizes (px, rem, em) don't adapt to available space. Fractional units (fr) are flexible but have no minimum or maximum constraints.

Taking It Further with minmax()

The Best of Both Worlds

The minmax() function takes things a bit further by allowing you to set a minimum size and a maximum size for the grid track.

Advantage: Combines the stability of minimum sizes with the flexibility of maximum sizes, creating truly responsive layouts.

  • Flexibility ‚Äì Adapts to available space
  • Constraints ‚Äì Never too small or too large
  • Responsiveness ‚Äì Perfect for varying screen sizes

minmax() Syntax

Function Structure

Here's the syntax of the minmax() function:

minmax(min, max)

Parameters:

  • min ‚Äì The minimum size of the grid track
  • max ‚Äì The maximum size of the grid track
/* Example usage */ grid-template-columns: minmax(150px, 300px) 1fr; /* Multiple minmax() values */ grid-template-columns: minmax(100px, 200px) minmax(150px, 1fr);

The min Parameter

Setting the Minimum Size

min is the minimum size of the grid track, which can be set using pixels, percentage, or auto.

/* Different min values */ minmax(100px, 1fr) /* min = 100 pixels */ minmax(20%, 500px) /* min = 20% of container */ minmax(auto, 300px) /* min = auto (content-based) */ minmax(10rem, 1fr) /* min = 10 rem units */

The min value ensures the grid track will never shrink below a set size.

Common min Values:

  • pixels (px) ‚Äì Fixed minimum
  • percentage (%) ‚Äì Relative to container
  • auto ‚Äì Based on content

The max Parameter

Setting the Maximum Size

max is the maximum size of the grid track which you can set with the same units (pixels, percentage, auto), plus fractional units (fr).

/* Different max values */ minmax(100px, 300px) /* max = 300 pixels */ minmax(10%, 50%) /* max = 50% of container */ minmax(100px, 1fr) /* max = 1 fraction unit (flexible!) */ minmax(auto, auto) /* max = auto (content-based) */

The max value limits how large the grid track can grow.

Pro Tip: Using 1fr as the max value makes the track flexible while still respecting the minimum size!

What's Happening Here?

Breaking Down the Behavior

First Column: minmax(150px, 300px)

  • Will always be at least 150px wide
  • Will be at most 300px wide
  • Adjusts depending on available space
  • Grows when there's extra room (up to 300px)
  • Stays at 150px minimum when space is tight

Second Column: 1fr

  • Takes up any remaining space
  • Grows and shrinks flexibly
  • No other columns to share space with

Advantages of minmax()

Why Use minmax()?

The advantage of the minmax() function over fixed sizes and even fr units is that it is more flexible, making it ideal for adaptability and responsiveness.

Key Benefits:

  • Flexible ‚Äì Adapts to available space
  • Constrained ‚Äì Maintains usability with minimum size
  • Responsive ‚Äì Perfect for varying screen sizes
  • Predictable ‚Äì Never too small or too large
  • Content-aware ‚Äì Can use auto for content-based sizing

Best Practice: Use minmax() when you need flexibility with guardrails ‚Äì especially for responsive layouts that work across different devices!

Common minmax() Patterns

Real-World Use Cases

/* Sidebar with flexible content */ grid-template-columns: minmax(200px, 250px) 1fr; /* Responsive card grid */ grid-template-columns: repeat(auto-fit, minmax(250px, 1fr)); /* Flexible middle column */ grid-template-columns: 200px minmax(300px, 1fr) 200px; /* All flexible with minimums */ grid-template-columns: minmax(150px, 1fr) minmax(200px, 1fr) minmax(150px, 1fr);

Pro Tip: Combine minmax() with repeat() and auto-fit for truly responsive grid layouts!
