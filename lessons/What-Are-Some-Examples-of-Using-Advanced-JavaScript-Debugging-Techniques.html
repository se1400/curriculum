Breakpoints

Pause and Inspect

The first concept we will take a look at is working with breakpoints.

Breakpoints let you pause the execution of your code at a specific line of your choice.

What You Can Do After Pause

After the pause, you can:

‚Ä¢ Inspect variables - See current values

‚Ä¢ Evaluate expressions - Test code snippets

‚Ä¢ Examine the call stack - See how you got there

‚úì Why Use Breakpoints?

Instead of guessing where the problem is with console.log(), breakpoints let you:

‚Ä¢ Stop exactly where you need to investigate

‚Ä¢ See the state of your program at that moment

‚Ä¢ Step through code line by line

‚Ä¢ Understand the flow of execution

üí° Much Better Than console.log()

Breakpoints are more powerful because you can inspect everything at that moment in time, not just what you remembered to log!

Conditional Breakpoints

Break Only When Condition is True

You can also add a conditional breakpoint by right-clicking on a line and selecting "Add conditional breakpoint..."

Why Use Conditional Breakpoints?

Sometimes you only want to pause when a specific condition is met:

‚Ä¢ When a variable equals a certain value

‚Ä¢ When an error condition occurs

‚Ä¢ During a specific iteration of a loop

‚Ä¢ When a particular user action happens

Example Conditions

// Break when user ID is 123 userId === 123 // Break when array is empty arr.length === 0 // Break on 5th iteration i === 5 // Break when error occurs response.status >= 400 // Break when data is null or undefined data == null

‚úì Super Useful!

Conditional breakpoints save time by only pausing when you actually need to investigate, instead of pausing on every execution.

Watch Expressions

Monitor Variables

Now, let's move on to watching expressions.

Watch expressions lets you monitor the values of variables or expressions as the code runs even if they are out of the current scope.

How to Add Watch Expressions

1 Navigate to the Sources tab of the developer tools

2 Look for the Watch panel on the right

3 Click the plus (+) icon to add it

What You Can Watch

// Watch a simple variable userName // Watch an object property user.email // Watch an expression users.length > 10 // Watch a calculation totalPrice * 0.9 // Watch array element items[0].price

‚úì Key Advantage

Watch expressions update in real-time as you step through your code, showing you how values change!

You can watch variables even when they're not in the current scope, making it easy to track important values throughout your program.

Profiling

The next concept we will look at is profiling.

Profiling helps you identify performance bottlenecks by letting you capture screenshots and record CPU usage, function calls, and execution time.

How to Profile Your Code

1 Open the Performance tab in DevTools

2 Click record (red circle button)

3 Perform the action you want to profile

4 Stop the recording to analyze the results

‚úì What You Can Find

This could be useful if your application is lagging during certain operations.

With the profiler, you can see:

‚Ä¢ Which function consumes the most resources

‚Ä¢ How long each operation takes

‚Ä¢ CPU usage over time

‚Ä¢ Memory allocation patterns

‚Ä¢ Rendering performance

üí° Use Case

If your page is slow or unresponsive, profiling shows you exactly where the performance problems are!

console.table()

Display Data as a Table

For the last portion of this lesson, we will focus on console.table and console.dir.

console.table() displays tabular data as a table in the console.

Syntax

It takes one mandatory argument, which must be an array or an object, and one optional argument to specify which properties or columns to display.

Example with Array of Objects

const users = [ { id: 1, name: "Alice", age: 25, city: "NYC" }, { id: 2, name: "Bob", age: 30, city: "LA" }, { id: 3, name: "Charlie", age: 35, city: "Chicago" } ]; console.table(users);

Result in Console

(index) id name age city
0 1 Alice 25 NYC
1 2 Bob 30 LA
2 3 Charlie 35 Chicago

‚úì Much Easier to Read!

Instead of seeing a long array of objects in the console, you get a nice, organized table!

console.dir()

Interactive Object Inspector

console.dir() on the other hand lets you display an interactive list of the properties of a specified JavaScript object.

It outputs a hierarchical listing that can be expanded to see all nested properties.

Example

const user = { name: "Alice", age: 25, address: { street: "123 Main St", city: "NYC", zip: "10001" }, hobbies: ["reading", "coding", "hiking"] }; console.dir(user);

‚úì console.log() vs console.dir()

console.log(): Shows a string representation

console.dir(): Shows an interactive tree of properties you can expand/collapse

Best For

‚Ä¢ Exploring complex objects

‚Ä¢ Seeing all methods and properties

‚Ä¢ Inspecting DOM elements as objects

‚Ä¢ Understanding object structure
