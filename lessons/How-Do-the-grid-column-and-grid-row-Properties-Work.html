grid-column and grid-row

Precise Item Placement

The grid-column and grid-row properties let you specify the horizontal and vertical placement of grid items within a grid layout.

What You'll Learn:

  • How to position items precisely
  • Understanding grid lines
  • Using start and end lines
  • The powerful span keyword
  • Creating complex layouts

Key Concept: These properties give you complete control over where items are placed in your grid!

What Do They Control?

Positioning with Grid Lines

In other words, they both allow you to control where a grid item begins and ends by referencing grid lines.

Grid Lines: These grid lines are the boundaries that separate rows and columns you have already defined using the grid-template-rows and grid-template-columns properties.

The Process:

  • Step 1: Define grid structure with grid-template-*
  • Step 2: Grid lines are automatically created
  • Step 3: Reference those lines to position items

Let's Look at the Syntax

Two Properties, Same Pattern

Let's take a look at the syntaxes of both grid-row and grid-column, and also a practical example.

Good News: Both properties use the exact same syntax pattern!

  • grid-row controls vertical positioning
  • grid-column controls horizontal positioning
  • Both use the start / end format

grid-row Syntax

Vertical Positioning

Here's the syntax of the grid-row property:

grid-row: <start-line> / <end-line>;
/* Examples */ .item { grid-row: 1 / 2; /* Row 1 only */ grid-row: 1 / 3; /* Spans rows 1 and 2 */ grid-row: 2 / 5; /* Spans rows 2, 3, and 4 */ }

What it does: Controls which horizontal lines (rows) your item spans between.

grid-column Syntax

Horizontal Positioning

And of grid-column:

grid-column: <start-line> / <end-line>;
/* Examples */ .item { grid-column: 1 / 2; /* Column 1 only */ grid-column: 1 / 3; /* Spans columns 1 and 2 */ grid-column: 2 / 5; /* Spans columns 2, 3, and 4 */ }

What it does: Controls which vertical lines (columns) your item spans between.

Understanding the Parameters

start-line and end-line

<start-line> is the grid line where the item starts and <end-line> is the grid line where the item ends.

Simple Rule: Your item occupies the space between these two lines.

Important: 1-Indexed!

Where Grid Lines Come From

Automatic Generation

Remember, grid lines for rows are generated based on the number of rows specified in the grid-template-rows property. The same applies to columns with the grid-template-columns property.

/* 3 columns create 4 vertical lines */ grid-template-columns: 1fr 1fr 1fr; /* Lines: 1 | col 1 | 2 | col 2 | 3 | col 3 | 4 */ /* 2 rows create 3 horizontal lines */ grid-template-rows: 100px 100px; /* Lines: 1 — row 1 — 2 — row 2 — 3 */

Formula: If you have n tracks, you'll have n + 1 lines.

Example: 4 columns = 5 vertical lines (numbered 1, 2, 3, 4, 5)

Grid with Lines:

Targeting Grid Lines

Start Using the Properties

You can target these lines to start using the grid-row and grid-column properties to determine where you should place an item.

The Power: Reference any combination of lines to position items exactly where you want them!

/* Position an item */ .item { grid-column: 2 / 4; /* Columns 2-3 */ grid-row: 1 / 3; /* Rows 1-2 */ } /* This item now spans: - 2 columns (from line 2 to line 4) - 2 rows (from line 1 to line 3) */

Example: Spanning 2 Columns

Making Item 1 Wider

Here's how you can make the first grid item occupy the first two columns:

.item1 { grid-column: 1 / 3; }

Translation: With this, you're saying the first grid item should start at column 1 and end just before column 3.

Remember: From line 1 to line 3 = spans 2 columns (columns 1 and 2)

The span Keyword

Alternative Syntax

You can also use the span keyword to tell the grid item which row and column to span across.

For example, 1 / 3 is the same as 1 / span 2:

/* These are EQUIVALENT */ /* Method 1: End line number */ .item { grid-column: 1 / 3; } /* Method 2: span keyword */ .item { grid-column: 1 / span 2; }

Understanding span: span 2 means "span across 2 tracks" instead of specifying the ending line number.

span in Action

Practical Examples

/* Span 3 columns starting at line 1 */ .item { grid-column: 1 / span 3; /* Same as: grid-column: 1 / 4; */ } /* Span 2 rows starting at line 2 */ .item { grid-row: 2 / span 2; /* Same as: grid-row: 2 / 4; */ } /* Span both dimensions */ .item { grid-column: 1 / span 2; grid-row: 1 / span 2; }

Advantage: Using span is often more intuitive – you think "I want this to span 3 columns" rather than calculating the end line!

Complete Freedom

Position Any Item Anywhere

You can continue to apply this technique to any item on the grid and place them wherever you want.

/* Position multiple items precisely */ .item1 { grid-column: 1 / 3; grid-row: 1 / 2; } .item2 { grid-column: 3 / 5; grid-row: 1 / 3; } .item3 { grid-column: 1 / 2; grid-row: 2 / 4; } /* Items can overlap, be in any order, span any size! */

Power User Tip: Items don't have to follow source order – position them anywhere in the grid!

Best Practices

Tips for Success

Follow These Guidelines:

  • Use span when thinking about "how many"
  • Use line numbers for absolute positioning
  • Plan your grid structure first
  • Remember lines are 1-indexed
  • Use browser DevTools to visualize lines
  • Test responsively – adjust for different screens

Pro Tip: Most browsers' DevTools have a Grid Inspector that shows line numbers!

Common Patterns

Frequently Used Combinations

/* Span full width */ .full-width { grid-column: 1 / -1; /* -1 means last line! */ } /* Center item (in 5-column grid) */ .center { grid-column: 2 / 5; /* Skip first and last */ } /* Sidebar layout */ .sidebar { grid-column: 1 / 2; grid-row: 1 / -1; /* Full height */ } .main { grid-column: 2 / -1; /* Rest of width */ grid-row: 1 / -1; /* Full height */ }

Bonus Tip: Negative numbers count from the end! -1 is the last line.

Summary

Key Takeaways

Remember:

  • grid-column and grid-row control item placement
  • Syntax: start-line / end-line
  • Grid lines are numbered starting from 1
  • Lines are created by grid-template-* properties
  • Formula: n tracks = n + 1 lines
  • Use span keyword for intuitive sizing
  • Complete control over item positioning
  • Items can overlap and span multiple tracks

Next Step: Practice positioning items in different ways – experiment with overlapping, spanning, and creative layouts!
