preventDefault() Method

What is preventDefault()?

The preventDefault() method on event objects stops the default behavior of an event from happening.

Understanding Default Behaviors

Every event that triggers in the DOM has some sort of default behavior.

Common Default Behaviors

  • Clicking a checkbox toggles its state
  • Pressing space bar on a button activates it
  • Typing in an input displays characters
  • Submitting a form sends data and reloads the page
  • Clicking a link navigates to a URL

Why Use preventDefault()?

Taking Control

Sometimes you want to override the default behavior and implement your own custom handling.

Common Use Cases

  • Custom Form Validation: Validate before submitting
  • Ajax Form Submission: Submit without page reload
  • Custom Input Handling: Process keystrokes differently
  • Link Prevention: Trigger actions instead of navigation
  • Custom Interactions: Create unique user experiences

The Goal

Give yourself the ability to implement custom event handling instead of accepting default browser behavior.

Example: Custom Input Display

Our Goal

Instead of showing characters in an input field, display them in a separate element.

Step 1: Basic Input HTML

<label>Enter some characters:
  <input type="text">
</label>

Default Behavior

With this HTML, we can type in the input field as expected. Characters appear where we type them.

Step 2: Add Output Element

Display Characters Elsewhere

We want to show the character the user types in a separate element.

Updated HTML

<label>Enter some characters:
  <input type="text">
</label>
<p id="output"></p>

Next Step

Now we need JavaScript to listen for keystrokes and display them in the paragraph element.

Step 3: Listen to keydown Event

The keydown Event

We need to hook into the keydown event to listen for characters being typed on the keyboard.

JavaScript Setup

const input = document.querySelector("input");

input.addEventListener("keydown", (e) => {
  // We'll add code here
});

Why keydown?

We use keydown instead of change or input events because we need the keyboard information (which key was pressed).

keydown fires when you press down on a keyboard key.

Understanding e.key

Getting the Key Value

e.key gives you the value of the key pressed.

Examples of e.key Values

Letter Keys

  • Press 'a' → e.key = "a"
  • Press 'z' → e.key = "z"
  • Press 'A' → e.key = "A"

Special Keys

  • Enter → e.key = "Enter"
  • Space → e.key = " "
  • Escape → e.key = "Escape"

Useful Property

This property lets us know exactly which key the user pressed, enabling custom handling.

Step 4: Display the Character

Show in Output Element

Let's display the character in our paragraph element.

Complete Code

const input = document.querySelector("input");
const output = document.getElementById("output");

input.addEventListener("keydown", (e) => {
  output.textContent += e.key;
});

What Happens

When you type in the input, the character you type will be displayed in the paragraph element.

However, it will also still appear in the input field - that's the default behavior.

The Problem

Unwanted Behavior

Right now, characters appear in both places:

  • In the input field (default behavior)
  • In the output paragraph (our custom handling)

What We Want

We don't want to show the characters in the input field.

We only want them to appear in the output paragraph.

The Solution

This is where preventDefault() comes in!

Solution: preventDefault()

Stopping Default Behavior

The default behavior of a keydown event is to render the character in the input.

Let's prevent that by calling e.preventDefault().

Updated Code

const input = document.querySelector("input");
const output = document.getElementById("output");

input.addEventListener("keydown", (e) => {
  e.preventDefault(); // Stop default behavior!
  output.textContent += e.key;
});

Result

Now the characters will only appear in the output paragraph, not in the input field!

Custom Event Handling Achieved

Success!

You have prevented the default behavior to allow yourself to implement your own custom event handling.

Before preventDefault()

Input Field

✓ Characters appear

Output Element

✓ Characters appear

After preventDefault()

Input Field

✗ Characters DON'T appear

Output Element

✓ Characters appear

Common Example: Form Submission

Default Form Behavior

By default, submitting a form:

  • Sends data to the server
  • Reloads the page

Preventing Default Submission

const form = document.querySelector("form");

form.addEventListener("submit", (e) => {
  e.preventDefault(); // Stop page reload!
  // Your custom form handling here
  // e.g., validate, send via Ajax, etc.
});

Why This Is Useful

Allows you to handle form submission with JavaScript (validation, Ajax) without the page reloading.

Custom Form Handling

Complete Form Example

const form = document.querySelector("form");

form.addEventListener("submit", (e) => {
  e.preventDefault();

  // Get form data
  const formData = new FormData(form);

  // Validate
  if (isValid(formData)) {
    // Send via Ajax
    submitViaAjax(formData);
  } else {
    // Show errors
    showErrors();
  }
});

This approach gives you complete control over form submission.

Important: Accessibility

⚠️ Critical Consideration

Preventing default behavior is powerful, but it's important to keep accessibility in mind.

Best Practices

Maintain Features

Your custom behavior should provide the same features as the default behavior.

Examples

  • If you prevent form submission, ensure users can still submit via keyboard (Enter key)
  • If you prevent link navigation, provide alternative navigation methods
  • If you prevent checkbox toggling, ensure keyboard users can still interact
  • Maintain focus management and keyboard accessibility

When to Use preventDefault()

Good Use Cases

  • Form Validation: Check data before allowing submission
  • Ajax Submissions: Submit without page reload
  • Custom Interactions: Create unique user experiences
  • Single-Page Apps: Handle navigation without page changes
  • Special Input Handling: Implement custom keyboard behavior

When to Avoid

  • When default behavior is already what you want
  • When you can't maintain accessibility features
  • When it would confuse or frustrate users
  • When standard behavior is expected

Summary

Key Concepts

What is preventDefault()?

A method on event objects that stops the default behavior of an event from happening.

Common Uses

  • Forms: Prevent submission to handle with JavaScript
  • Inputs: Customize how input is processed
  • Links: Prevent navigation to run custom code
  • Buttons: Override default actions

Syntax

element.addEventListener("event", (e) => {
  e.preventDefault(); // Stop default behavior
  // Your custom code here
});

Remember

Always maintain accessibility when overriding default behaviors. Your custom implementation should provide the same features.
