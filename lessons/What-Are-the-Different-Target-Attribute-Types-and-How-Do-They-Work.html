The target Attribute

The target attribute is used on anchor elements (links) to tell the browser where to open the URL.

There are four important possible values for this attribute. Each value is preceded by an underscore.

Selecting the right target value controls where your users end up, which is an important consideration when creating a website.

Basic target Syntax
<a 
  href="https://freecodecamp.org"
  target="_blank"
>
  Visit freeCodeCamp
</a>

target="_self" (Default)

The _self value is the default behavior when no target is specified.

This opens the link in the current browsing context. In most cases, this will be the current tab or window.

If you don't set a target attribute, the browser automatically uses _self.

Explicit _self
<a 
  href="https://example.com"
  target="_self"
>
  Opens in same tab
</a>
No target (same as _self)
<a href="https://example.com">
  Opens in same tab
</a>

target="_blank"

The _blank value opens the link in a new browsing context.

Typically, this will open in a new tab. However, some users might configure their browsers to open a new window instead.

This is useful when you want users to keep your site open while viewing external content.

Opens in New Tab/Window
<a 
  href="https://example.com"
  target="_blank"
>
  Opens in new tab
</a>
Common Use Case
<!-- External links -->
<a 
  href="https://twitter.com"
  target="_blank"
>
  Follow us on Twitter
</a>

target="_parent"

The _parent value opens the link in the parent of the current context.

For example, if your website has an iframe, a _parent value in that iframe would open in your website's tab or window, not in the embedded frame.

This is particularly useful when working with embedded content.

Opens in Parent Context
<!-- Inside an iframe -->
<a 
  href="https://example.com"
  target="_parent"
>
  Opens in parent window
</a>
Context Example
<!-- Main page -->
<iframe src="page.html"></iframe>

<!-- Inside page.html -->
<!-- _parent opens in main page -->
<a target="_parent">...</a>

target="_top"

The _top value opens the link in the top-most browsing context—think "the parent of the parent".

This is similar to _parent, but the link will always open in the full browser tab or window, even for nested embedded frames.

Use this to break out of all iframe nesting levels.

Opens in Top-Most Context
<!-- Inside nested iframes -->
<a 
  href="https://example.com"
  target="_top"
>
  Opens in top window
</a>
Nested Context Example
<!-- Main page -->
<iframe src="frame1.html"></iframe>
  <!-- frame1.html -->
  <iframe src="frame2.html"></iframe>
    <!-- frame2.html -->
    <!-- _top breaks to main page -->
    <a target="_top">...</a>

Summary: Choosing the Right Target

  • _self (default): Opens in the current tab/window
  • _blank: Opens in a new tab/window
  • _parent: Opens in the parent context (useful for iframes)
  • _top: Opens in the top-most context (breaks all iframe nesting)

Note: There is also a fifth experimental value called _unfencedTop, currently used for the FencedFrame API, but it's rarely needed at this time.

Quick Reference
<!-- Same tab (default) -->
<a target="_self">...</a>

<!-- New tab -->
<a target="_blank">...</a>

<!-- Parent context -->
<a target="_parent">...</a>

<!-- Top-most context -->
<a target="_top">...</a>
