The Object() Constructor

What Is a Constructor?

In JavaScript, a constructor is a special type of function used to create and initialize objects.

  • Invoked with the new keyword
  • Creates a new object instance
  • Can initialize properties and methods

The Object() Constructor

The Object() constructor is a built-in constructor that creates objects. 

new Object()

When you call new Object(), it returns a new empty object that can be used to store values.

In This Lesson:

We'll explore how the Object() constructor works, when to use it, and why object literal syntax ({}) is usually preferred.

Using new Object()

Creating Empty Objects

The most basic use of new Object() is to create an empty object. 

const emptyObj = new Object();

console.log(emptyObj); // {}
{}

Equivalent to Object Literal

Using Constructor

const obj1 = new Object();

Using Object Literal

const obj2 = {};
Both create the same empty object: {}
Note:

There is no difference between new Object() and {} when creating empty objects. The object literal syntax {} is simpler and preferred.

Object() Without new

Different Behavior

The Object() constructor can be used with or without the new keyword.

When called without new, it behaves differently depending on the type of value passed to it.

Wrapping Primitives

const num = 42;
const numObj = Object(num); // Creates object wrapper

console.log(numObj);
console.log(typeof numObj); // "object"
Number { 42 }
"object"

What Happened?

Input: 42 (primitive number)
↓
Output: Number { 42 } (object wrapper)

Object() converted the primitive number into an object!

Object Wrapper:

Object(num) creates an object wrapper for the primitive value. The number is now wrapped in a Number object.

Handling null and undefined

What Happens with null or undefined?

When you pass null or undefined to Object(), the result is an empty object. 

const newObj = new Object(undefined);
console.log(newObj); // {}
{}

Visual Comparison

Input Code Result
null Object(null) {}
undefined Object(undefined) {}
42 Object(42) Number { 42 }
"hello" Object("hello") String { "hello" }
true Object(true) Boolean { true }
Pattern:

• null or undefined → Empty object {}
• Primitives (number, string, boolean) → Object wrapper

Practical Use Case: toObject()

Ensuring a Value Is an Object

One use case for Object() is when you're working with a value of unknown type and need to ensure it's an object. 

function toObject(value) {
    // Handle null or undefined
    if (value === null || value === undefined) {
        return {};
    }
    
    // Return value if already an object (includes arrays)
    if (typeof value === "object") {
        return value;
    }
    
    // Convert primitives to object wrappers
    return Object(value);
}

Testing the Function

console.log(toObject(null));       // {}
console.log(toObject(true));       // Boolean { true }
console.log(toObject([1, 2, 3])); // [1, 2, 3]
{}
Boolean { true }
[1, 2, 3]

How It Works

Step 1: Check if null/undefined → return {}
Step 2: Check if already object → return as-is
Step 3: Otherwise → wrap primitive with Object()
Note on Arrays:

The second condition checks for typeof value === "object", which is true for both objects AND arrays (since arrays are special types of objects in JavaScript).

Object Literal vs Constructor

Why Use Object Literal Syntax?

Most of the time, you will not be using the Object() constructor to create new objects.

Object() Constructor

const person = new Object();
person.name = "Beau";
person.age = 30;
Issues:
  • More verbose
  • Less readable
  • Requires multiple lines

Object Literal ✓

const person = {
    name: "Beau",
    age: 30
};
Benefits:
  • Concise
  • Readable
  • Preferred standard
Best Practice:

Always use object literal syntax ({}) to create objects. It's simpler, more readable, and the standard way to create objects in JavaScript.

Summary & When to Use Object()

Quick Reference

What Object() Does:
  • With new: Creates empty object
  • Without new: Wraps primitives as objects
  • With null/undefined: Returns empty object
  • With existing object: Returns the object unchanged
When to Use Object():
  • When you need to create an object wrapper for a primitive value
  • When ensuring a value of unknown type is an object
  • In utility functions that handle various data types

Comparison Table

Syntax Purpose Common?
{} Create empty object ✓ Yes
new Object() Create empty object ✗ Rare
Object(value) Wrap primitive / ensure object Sometimes
Key Takeaway:

While it's important to understand the basics of the Object() constructor, in practice you'll mostly use object literal syntax ({}) to create objects. The constructor is mainly useful in specialized scenarios where you need to wrap primitives or handle unknown data types.
