DOM Node Relationships

The DOM Tree Hierarchy

Just as a real tree has branches connected in a hierarchical structure, DOM nodes also have direct and indirect relationships with one another.

Types of Relationships

  • Parent & Child: Direct containment
  • Siblings: Same parent
  • Ancestor & Descendant: Indirect containment
  • Root: Top of the tree
Why This Matters:

Understanding these relationships is essential for manipulating and navigating the DOM tree using JavaScript!

Root, Parent, and Child Nodes

Root Node

The root of the DOM tree is the <html> element. It's the top-level container for all the content of an HTML document. All other nodes are descendants of this root node.

üå≥ html (ROOT)

All other nodes descend from here

Parent and Child Nodes

Parent Node:

An element that contains other elements

Child Node:

An element that is contained within another element

Sibling Nodes

What Are Siblings?

Sibling nodes are elements that share the same parent. They are at the same level in the DOM tree hierarchy.

Key Point:

Siblings are at the same level in the tree and have the same parent. This relationship is important when navigating between elements in JavaScript!

Descendant and Ancestor Nodes

Indirect Relationships

We also have indirect relationships across different levels in the hierarchy.

Descendant Nodes

Definition:

Parent ‚Üî Child (immediate connection, one level apart)

Indirect Relationships:
