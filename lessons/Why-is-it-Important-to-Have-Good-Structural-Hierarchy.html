Heading Elements

The most important aspect of creating a structural hierarchy is the proper use of heading elements.

Heading elements are numbered as h1, h2, h3, and so on. These numbers represent the heading level for that element.

Much like a text document, you do not want to use heading levels in the incorrect order.

Heading Elements
<h1>Top-level heading</h1>
<h2>Subheading</h2>
<h3>Sub-subheading</h3>
<h4>Lower level heading</h4>
<h5>Even lower level</h5>
<h6>Lowest level</h6>

h1: Top-Level Heading

Your h1 element is your top-level heading.

You will rarely have more than one of these on a page, and it should typically come before all of your content.

The h1 represents the main topic or title of the page.

Typical h1 Usage
<section>
  <h1>SE1400</h1>
  <p>
    Welcome to SE1400, where you 
    can learn to code.
  </p>
</section>

/* Typically only ONE h1 per page */

h2: Subheadings

Your h2 element is your subheading. It should always come after your h1 and might come after some introductory text.

Unlike an h1 element, you can have multiple h2 elements on your page. You'll often do this to delineate different sections of content.

Multiple h2 Elements
<section>
  <h1>freeCodeCamp</h1>
  
  <h2>Learn Front-End Development</h2>
  <p>Content about front-end...</p>
  
  <h2>Learn Back-End Development</h2>
  <p>Content about back-end...</p>
</section>

/* Multiple h2s are fine! */

Proper Heading Order

Following the pattern, your h3 element should always come after an h2 element.

You should never skip directly from h1 to h3. You can, however, have multiple heading elements at the same level.

Always maintain the hierarchical structure: h1 → h2 → h3 → h4, and so on.

Correct Order
<h1>Main Title</h1>
<h2>Section</h2>
<h3>Subsection</h3>
<h3>Another Subsection</h3>

✓ Correct hierarchy
Incorrect Order
<h1>Main Title</h1>
<h3>Subsection</h3>
<h2>Section</h2>

✗ Skipped h2, wrong order

Don't Use Headings for Styling

It may be tempting to use a specific heading element because of its styling, such as h1 for large text.

Instead, you should choose the appropriate element for your document structure and use CSS to achieve the styling you'd like.

Always prioritize semantic correctness over visual appearance when selecting heading elements.

Wrong Approach
<article>
  <p>
    Here is some
    <h1>Large Text</h1>
  </p>
</article>

✗ Using h1 just for size
Right Approach
<article>
  <p>
    Here is some
    <span class="large">Large Text</span>
  </p>
</article>

/* CSS */
.large { font-size: 2rem; }

✓ Use CSS for styling

Why Hierarchy Matters: Accessibility

Using the right hierarchy is important for accessibility. Assistive technologies, like screen readers, rely on the structure of a web page to determine how to parse and announce that web page to the user.

Using an h3 element after an h1 might cause a screen reader user to believe they have accidentally skipped over important content, due to the lack of an h2 element.

Screen Reader Experience
/* Good structure: */
<h1>Main Topic</h1>
<h2>Subtopic</h2>
<h3>Detail</h3>

Screen reader announces:
"Heading level 1: Main Topic"
"Heading level 2: Subtopic"
"Heading level 3: Detail"

User understands structure ✓
Poor Structure Impact
/* Bad structure: */
<h1>Main Topic</h1>
<h3>Detail</h3>

Screen reader announces:
"Heading level 1: Main Topic"
"Heading level 3: Detail"

User thinks they missed
content between h1 and h3 ✗

Why Hierarchy Matters: SEO and Validation

Proper structure is important for SEO. Search engines use automation to parse the content of your web page and determine when and where it should show up in results. If your structure is malformed, search engines may not be able to rank you very well.

Additionally, depending on how incorrect your structure is, your HTML may not even be technically valid. When this happens, the web browser has to effectively guess what you meant to do—and what it guesses might not be what you want at all.

SEO Benefits
/* Search engines understand: */
<h1>Best Pasta Recipes</h1>
  <h2>Italian Classics</h2>
    <h3>Carbonara</h3>
    <h3>Bolognese</h3>

✓ Clear topic hierarchy
✓ Better search rankings
✓ Proper content indexing
Summary
Proper heading hierarchy ensures:

1. Accessibility for all users
2. Better SEO rankings
3. Valid HTML code
4. Correct browser rendering

Keep this in mind as you 
build new projects!
