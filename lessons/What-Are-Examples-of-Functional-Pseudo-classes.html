What Are Functional Pseudo-classes?

Functional pseudo-classes allow you to select elements based on more complex conditions or relationships.

Key Characteristics:

  • Accept arguments within parentheses
  • More flexible than regular pseudo-classes
  • Enable complex selection patterns
  • Work with selectors as parameters
Unlike regular pseudo-classes (:hover, :focus), functional pseudo-classes use arguments to define selection criteria

Four Main Functional Pseudo-classes

:is()

Group selectors together

:where()

Group with zero specificity

:has()

Parent selector based on children

:not()

Exclude specific elements

:is() Pseudo-class

The :is() pseudo-class is useful when you want to style a group of elements that share some characteristics.

Purpose:

  • Simplify complex selectors
  • Reduce code repetition
  • Group multiple selectors
  • Improve readability

Example Scenario:

Style different types of buttons: button elements, links styled as buttons, and submit/reset inputs.

:is() - Without vs With

❌ Without :is()

button,
a.button,
input[type="submit"],
input[type="reset"] {
  background: #881518;
  color: white;
}

Long and repetitive selector list

✓ With :is()

:is(button, .button,
    input[type="submit"],
    input[type="reset"]) {
  background: #881518;
  color: white;
}

More compact and readable

:where() Pseudo-class

The :where() pseudo-class functions similarly to :is(), but with one crucial difference:

:where() doesn't increase the specificity of your selectors

When to Use :where():

  • Applying base styles without affecting specificity
  • Creating resets that won't interfere with other rules
  • Ensuring easy override capability
  • Building flexible, maintainable CSS

Specificity:

:where() has 0 specificity, making it ideal for base styles that should be easily overridden.

:where() Example

Using :where() to reset heading margins and padding:

:where(h1, h2, h3) {
  margin: 0;
  padding: 0;
}

:is() vs :where() Specificity

Understanding the specificity difference is crucial:

Pseudo-class Specificity Use Case
:is() Takes specificity of most specific selector in list When you want normal specificity behavior
:where() Always 0 (zero specificity) When you want easily overridable base styles

Example:

:is(#id, .class) → Specificity of #id (high)

:where(#id, .class) → Specificity of 0 (low)

:has() Pseudo-class

The :has() pseudo-class allows you to style a parent element based on its children.

Revolutionary Feature:

  • Style parents based on children's presence
  • Previously impossible without JavaScript
  • Also called the "parent selector"
  • Enables powerful conditional styling

CSS Example:

article:has(h2) {
  background-color: lightyellow;
  border-color: orange;
}

Only applies to article elements that contain an h2

:not() Pseudo-class

The :not() pseudo-class applies styles to elements except those matching the selector.

Purpose:

  • Apply styles to a group with exceptions
  • Exclude specific elements from styling
  • Invert selector logic
  • Simplify exclusion patterns

CSS Example:

button:not(.primary) {
  background-color: gray;
}

Any button that is NOT a primary button gets gray background

Functional Pseudo-classes Summary

Key Takeaways:

  • Functional pseudo-classes accept arguments for complex selections
  • :is() - Groups selectors, maintains specificity
  • :where() - Groups selectors, zero specificity
  • :has() - Styles parents based on children (parent selector)
  • :not() - Excludes specific elements from selection
  • Specificity matters: :is() vs :where()
  • Powerful tools: Reduce code, improve maintainability
Functional pseudo-classes enable complex, flexible selections that were previously difficult or impossible with CSS alone
