Bugs and Debugging

When you are building out your projects, you will frequently run into issues where your programs are not working as expected.

What are Bugs?

Programmers often refer to issues as bugs. These are errors, flaws, or unexpected behaviors in your code.

What is Debugging?

The process of finding and fixing these bugs is known as debugging.

To debug your code, you will need to use some tools provided by your browser.
Two Important Browser Tools:
  1. DOM Inspector - Inspect the HTML structure of the page
  2. Developer Tools - Inspect HTML, CSS, and JavaScript

DOM Inspector & Developer Tools

🔍 DOM Inspector

The DOM inspector allows you to inspect the HTML structure of the page you are on.

🛠️ Developer Tools

The developer tools allow you to inspect the HTML, CSS, and JavaScript of the page you are on.

What is the DOM?

Document Object Model (DOM)

The DOM stands for Document Object Model. It is a tree-like structure that represents the elements on a page.

You will learn more about the DOM in later modules.

Think of the DOM as a map of your webpage - every element (heading, paragraph, link, image) is represented as a node in this tree structure, making it easy to inspect and manipulate.

Example: Finding a Bug

Let's take a look at an HTML example that contains a small bug in the anchor element:
<a href="https://se1400.github.io/curriclum">se1400 curriculum</a>
Output:
se1400 curriculum

⚠️ The Problem

When you click on the link, it will lead to a 404 page. A 404 page is an error page that appears when a user tries to access a webpage that doesn't exist on the server.

The intent is for the link to lead to the se1400 curriculum.
Question: To see what the issue might be, what should we do?
Answer: Use the developer tools!

Opening Developer Tools

There are two ways to open developer tools in your browser:

Method 1: Right-Click

  1. Right-click anywhere on the page
  2. Select "Inspect" from the menu

Method 2: Keyboard Shortcuts

Windows/Linux:

Ctrl + Shift + I

Mac:

Command + Option + I
Pro Tip: Learning keyboard shortcuts will make you a more efficient developer. Try using the keyboard method until it becomes second nature!

DevTools Tabs & Finding the Bug

When you open developer tools in Google Chrome, you'll see a number of tabs. Here are the two main tabs:

📄 Elements Tab

This tab shows you the HTML structure of the page you are on. You can inspect and even temporarily modify elements to see how changes would look.

🖥️ Console Tab

This tab shows you any errors that might be occurring on the page. It's essential for debugging JavaScript and viewing error messages.

Solving Our Bug

In the situation where you have a broken link, you can check the Console tab to see the error messages for that broken link.

❌ 404 Error Found!

The common message that continues to display for the broken link is the 404 error. The 404 error indicates that the page is not found.

Console message: /larn returns 404

This lets us know that the issue is with the URL in the anchor element. When you inspect the href value you will see there is a typo.
❌ Wrong: href="https://se1400.github.io/curriclum"
✓ Correct: href="https://se1400.github.io/curriculum"
The Fix: The URL should be /curiculum not /curriclum. When the link is corrected, it will work as expected!
Looking Ahead: You will learn more about working with the developer tools throughout the certification, but this is just a short example of how it can help you debug your code.
