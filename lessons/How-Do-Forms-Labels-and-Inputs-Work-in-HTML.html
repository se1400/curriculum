The Form Element

The form element in HTML is used to gather user information, such as names and email addresses.

Basic Form Structure

<form action="url-goes-here">
  <!-- input elements go here -->
</form>
The action attribute specifies where the form data will be sent upon submission.
Forms are the primary way to collect user input on websites. They serve as containers for various input elements that allow users to enter and submit data.

The Input Element

To collect specific information like names and email addresses, you would use the input element.

Basic Text Input Example

  <input type="text" />
</form>

The Label Element

To add a label for the input, you would use a label element.

Implicit Association Example

<form action="">
  <label>
    Full Name:
    <input type="text" />
  </label>
</form>
Interactive Output (click the label text):
By nesting an input inside a label element, you create an implicit association between the label and the input field.
The term "implicit" refers to something that is understood or inferred without needing to be explicitly stated or defined with additional attributes or elements.

Explicit Association

To explicitly associate a label with an input, you can use the for attribute.

Email Input with Explicit Association

<form action="">
  <label for="email"> Email Address: </label>
  <input type="email" id="email" />
</form>
Interactive Output:
Critical: When using explicit association, the values for the for attribute and id need to be the same. In this example, both values are set to email.
The email type in the input provides basic validation for correctly formatted email addresses.

The Placeholder Attribute

If you want to show additional hints to users about the expected input, you can use the placeholder attribute.

Email Input with Placeholder

<form action="">
  <label for="email"> Email Address: </label>
  <input type="email" id="email" placeholder="example@email.com" />
</form>
Interactive Output (click and start typing):
The placeholder text will disappear once you start typing in the input field.
Best Practice: For placeholder text, provide helpful short text to show the format and type of data you expect from your users. In this case, example@email.com shows the user that they must enter a correctly formatted email address.
