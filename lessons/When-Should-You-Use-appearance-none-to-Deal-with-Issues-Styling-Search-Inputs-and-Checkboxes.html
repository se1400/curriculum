The Challenge: Default Browser Styling

Browsers apply default styling to many elements. For input elements, your ability to style them with CSS can feel rather restricted.

The Problem:

  • Checkboxes and radio buttons have limited styling options
  • Search inputs show browser-specific icons
  • Default styles vary across different browsers
  • Hard to match your brand's design system

The Solution:

appearance: none - Hide default styling and create your own custom designs!

What is appearance: none?

The appearance: none CSS property removes the default browser styling from elements, giving you complete control.

input[type="checkbox"] {
  appearance: none;
  -webkit-appearance: none; /* Safari/Chrome */
  -moz-appearance: none; /* Firefox */
}

What It Does:

  • Removes native checkbox/radio appearance
  • Hides browser-specific search icons
  • Allows full CSS customization
  • Creates consistent cross-browser designs

Search Inputs - The WebKit Problem

WebKit-based browsers (Safari, Chrome) show default search icons and cancel buttons.

What is WebKit?

WebKit is a software engine that helps web browsers display websites. Browsers like Safari use WebKit to make sure web pages look and work correctly.

Styling Search Inputs

CSS
input[type="search"] {
  appearance: none;
  -webkit-appearance: none;
  padding: 0.8rem 2.5rem 0.8rem 1rem;
  border: 2px solid #881518;
  border-radius: 25px;
}

/* Remove search cancel button */
input[type="search"]::-webkit-search-cancel-button {
  display: none;
}

This removes default icons and allows you to create indicators that appear consistently across all browsers.

Potential Drawbacks:

The appearance: none property gives you complete control over styling, but it comes with things to look out for:

What You Might Lose:

  • Focus indicators - Default keyboard focus styling
  • Error indicators - Native validation styling
  • Disabled states - Visual feedback for disabled inputs
  • Hover effects - Native hover feedback

You MUST ensure these features aren't lost when creating custom styles!

Preserving Focus Indicators

Always add custom focus styles when using appearance: none:

.custom-checkbox {
  appearance: none;
  /* ... other styles ... */
}

/* ‚úì REQUIRED: Custom focus indicator */
.custom-checkbox:focus {
  outline: none;
  box-shadow: 0 0 0 3px rgba(136, 21, 24, 0.2);
}

Complete Accessible Example

Here's a fully accessible custom checkbox with all states:

.custom-checkbox {
  appearance: none;
  width: 24px;
  height: 24px;
  border: 2px solid #881518;
  border-radius: 4px;
  cursor: pointer;
}

/* Checked state */
.custom-checkbox:checked {
  background: #881518;
}

/* Focus state - REQUIRED! */
.custom-checkbox:focus {
  outline: none;
  box-shadow: 0 0 0 3px rgba(136, 21, 24, 0.2);
}

/* Disabled state */
.custom-checkbox:disabled {
  opacity: 0.5;
  cursor: not-allowed;
}

/* Hover state */
.custom-checkbox:hover:not(:disabled) {
  border-color: #6d1013;
}
