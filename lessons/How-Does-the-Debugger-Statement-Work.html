The debugger Statement

What is the debugger Statement?

The debugger statement is a powerful JavaScript tool that lets you pause your code at a specific line to investigate what's going on in the program.

Why Use It?

When used correctly, the debugger statement can save you a lot of time trying to figure out why something is not working as it should.

What Can You Do?

  • Inspect variables at a specific point
  • Check function values
  • Understand the flow of code
  • See exactly what's going right or wrong

How JavaScript Executes Code

Normal Execution

JavaScript executes your code from top to bottom.

What Happens with debugger?

When JavaScript executes your code and hits a debugger statement, it immediately pauses execution at that line.

The Pause

This pause gives you the chance to:

  • Inspect variables at that exact moment
  • Check function values
  • Examine the flow of code in general
  • See exactly what is going right or wrong

Important Requirement

⚠️ Developer Tools Must Be Open

The debugger statement only works if your browser's developer tools are open.

Two Scenarios

Developer Tools Open

  • Code execution pauses
  • You can inspect variables
  • You can step through code
  • debugger is active

Developer Tools Closed

  • debugger is ignored
  • Code continues normally
  • No pause occurs
  • No debugging happens

Tip: Open developer tools with F12 (Windows/Linux) or Cmd+Option+I (Mac)

Modern Browser Behavior

What Modern Browsers Do

When you use the debugger statement, modern browsers will:

Browser Actions

  • Pause code execution at the specified line
  • Highlight the line where execution stopped
  • Show variable values in the scope
  • Provide a play button to resume execution
  • Keep the page active (no automatic reload)

Resuming Execution

You can resume execution by clicking the play button in the developer tools.

The page itself does not automatically reload unless manually triggered.

How to Use the debugger Statement

Simple Usage

To use the debugger statement, place it at the line where you want code execution to pause.

Syntax

debugger;

Placement Strategy

  • Place it before the code you want to inspect
  • Put it where you suspect the bug might be
  • Use it at the start of a function to check inputs
  • Add it before critical calculations

Simple Example

Basic Variable Inspection

Let's see a simple example of using the debugger statement.

Code

let firstNumber = 5;
let secondNumber = 10;
debugger; // Code execution pauses here
let sum = firstNumber + secondNumber;
console.log(sum);

What Happens?

When the code reaches the debugger line, execution pauses. At this point:

  • firstNumber is 5
  • secondNumber is 10
  • sum hasn't been calculated yet

Simple Example: Behavior

Console Closed Scenario

If you don't have the console open before the code runs and you eventually open it, you'll see 15 in the console.

The debugger statement was ignored because dev tools weren't open.

Console Open Scenario

If the console is open and you reload the page:

  • Execution pauses at the debugger line
  • You can inspect firstNumber and secondNumber
  • You can see that sum doesn't exist yet
  • Click play to continue and see 15 logged

More Complex Example

Function with debugger

Let's look at a more complex example with a function.

Code

function calculateTotalPrice(price, discountPercentage) {
  debugger; // Pauses when function is called
  let discountAmount = (price * discountPercentage) / 100;
  let totalPrice = price - discountAmount;
  console.log(`Original Price: ${price}`);
  console.log(`Discount Amount: ${discountAmount}`);
  console.log(`Total Price after Discount: ${totalPrice}`);
  return totalPrice;
}

Complex Example: Function Call

Calling the Function

let price = 100;
let discount = 15;
let finalPrice = calculateTotalPrice(price, discount);
console.log(`Final Price: ${finalPrice}`);

When Pause Occurs

The debugger statement is placed at the top of the calculateTotalPrice function.

When the function is called, execution will be paused immediately.

Complex Example: What You Can Inspect

When Paused

When execution pauses at the debugger statement, you can inspect:

Available Information

Function Parameters

  • price = 100
  • discountPercentage = 15

Variables Not Yet Created

  • discountAmount - not calculated yet
  • totalPrice - not calculated yet

You can then step through the code line by line to watch these variables get created and calculated.

Page Reloading Behavior

Continuous Execution

If you reload the page with the console opened, you'll notice something interesting.

What You'll See

The page keeps reloading, confirming that the execution of the code is only paused, not stopped.

To Continue Execution

Resume Button

If you want execution to continue, you can click the play button in the developer tools.

  • Usually labeled "Resume script execution"
  • Often has a play/triangle icon
  • Keyboard shortcut: F8 (in most browsers)

Typical Debugging Workflow

Step-by-Step Process

1. Add debugger Statement

Place debugger; where you want to pause

2. Open Developer Tools

Press F12 or Cmd+Option+I

3. Run or Reload Page

Execute the code that contains the debugger

4. Inspect Variables

Examine values in the scope panel

5. Step Through Code

Use step buttons to execute line by line

6. Resume or Stop

Click play to continue or stop debugging

Benefits of Using debugger

Why Use debugger Over console.log?

Advantages

✓ Better Control

  • Pause execution at exact points
  • Step through code line by line
  • Inspect the entire scope

✓ See Everything

  • View all variables in scope
  • Examine the call stack
  • Check function arguments

✓ Save Time

  • No need to add multiple console.log statements
  • Real-time variable inspection
  • Quickly identify issues

Best Practices

Do's

  • Use during development to understand code flow
  • Place strategically where you suspect issues
  • Combine with breakpoints in dev tools
  • Inspect variables to verify values
  • Step through code to trace execution

Don'ts

  • Don't leave in production code - remove before deployment
  • Don't add too many - can slow down debugging
  • Don't forget dev tools - must be open to work
  • Don't rely solely on it - use with other debugging tools

Summary

Key Concepts

What is debugger?

A statement that pauses code execution at a specific line to help you investigate what's happening in your program.

Requirements

  • Developer tools must be open for it to work
  • Otherwise, the statement is ignored

Usage

debugger; // Place where you want to pause

Benefits

  • Understand the flow of execution
  • Inspect variables in real-time
  • Quickly identify where things are going wrong
  • Save time debugging

Remember

Always remove debugger statements before deploying to production!
