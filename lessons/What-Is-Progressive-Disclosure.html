Progressive Disclosure

A design pattern that shows users only relevant content based on their current activity and hides the rest.

Purpose

  • Reduce cognitive load
  • Make the user experience more intuitive

Basic Example

Click the button to reveal additional information

Here are additional details that are revealed only when you want to see them. This keeps the interface clean and reduces cognitive load.

Example: Google Search

Most searches are simple, so Google provides a simple interface with very few options.

Advanced Search

For times when you need more features, Google provides an advanced search option that reveals more options.

Found in: Settings menu or google.com/advanced_search

Best Practice: Keep Important Info Visible

All important information should be visible at all times.

Avoid

Users should not have to struggle to find key information because it's hidden by extra advanced features.

Visibility Comparison

❌ Bad Practice

Essential information like price, product name, or "Add to Cart" hidden behind a button

✓ Good Practice

Essential information always visible. Extra details (specs, reviews) behind "More Info"

Best Practice: Single Access Point

Provide a single access point for users to access additional features or information.

Implementation

Use a button or link that is always visible on the page.

Avoid

Multiple access points can be confusing and lead to a poor user experience.

Access Point Comparison

❌ Multiple Access Points

Confusing - which one reveals what?

✓ Single Access Point

Clear - one obvious action

Impact on User Experience

Think about the overall user experience and how progressive disclosure can make it easier for users to find information.

When Done Correctly

Progressive disclosure can make a big difference to the overall user experience.

Best Practices Summary

  • Purpose - Show only relevant content, reduce cognitive load
  • Simple by default - Start with minimal options, reveal more when needed
  • Keep essentials visible - Important info always shown, not hidden
  • Single access point - One clear button/link to reveal more
  • Examples - Google advanced search, e-commerce "More Info" buttons
