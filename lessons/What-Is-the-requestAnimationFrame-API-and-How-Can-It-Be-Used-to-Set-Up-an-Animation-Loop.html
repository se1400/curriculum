The requestAnimationFrame() API

The Challenge

Creating smooth animations on a web page can be tricky, especially if you're not sure how to handle timing properly.

The great news is that the requestAnimationFrame() API makes it easier!

What is requestAnimationFrame()?

requestAnimationFrame() is a method that allows you to schedule the next step of your animation before the next screen repaint, resulting in a fluid and visually appealing experience.

Understanding Screen Repaint

What is "Next Screen Repaint"?

The next screen repaint refers to the moment when the browser refreshes the visual display of the web page.

This happens multiple times per second, typically around 60 times (or 60 frames per second) on most displays.

Frames Per Second (FPS)

Most displays refresh at 60 FPS, which means:

‚Ä¢ The screen repaints 60 times every second

‚Ä¢ Each frame lasts about 16.67 milliseconds

‚Ä¢ Smooth animation needs to update each frame

Animation Frame Timeline

Frame 1: Browser repaints (0ms)
‚Üì ~16.67ms
Frame 2: Browser repaints (16.67ms)
‚Üì ~16.67ms
Frame 3: Browser repaints (33.33ms)
‚Üì ~16.67ms
Frame 4: Browser repaints (50ms)

And so on... 60 times per second!

üí° Key Insight

requestAnimationFrame() schedules your animation code to run right before each repaint, ensuring smooth, synchronized updates!

Basic Syntax

Simple to Use

To use the requestAnimationFrame() method, all you need to do is call it and pass a callback function into it:

requestAnimationFrame(callback);

Parameter

callback: A function to call before the next repaint

This function will be executed when the browser is ready to update the screen.

Return Value

Returns a request ID (number)

You can use this ID to cancel the animation with cancelAnimationFrame().

Simple Example

// Schedule a function to run before next repaint requestAnimationFrame(() => { console.log("Frame updated!"); });

üìù Note

This only runs once. To create a continuous animation, you need to call requestAnimationFrame() repeatedly in a loop pattern.

Creating an Animation Loop

The Pattern

Calling requestAnimationFrame() must first occur inside a function that handles the animation, such as animate(), along with a function to update the animation, traditionally called update():

function animate() { // Update the animation... // for example, move an element, change a style, and more. update(); // Request the next frame requestAnimationFrame(animate); }

How It Works

1 animate() function is called

2 update() makes changes to the animation

3 requestAnimationFrame(animate) schedules the next frame

4 Browser repaints the screen

5 Loop repeats - animate() is called again!

‚úì The Magic of Recursion

The animate() function calls requestAnimationFrame(animate), which schedules animate() to run again!

This creates a continuous loop that runs ~60 times per second.

The update() Function

Where the Magic Happens

The update() function is where the magic happens. Inside it, you get to change whatever you want to animate.

For example, updating a style or changing the position of an element:

function update() { element.style.transform = `translateX(${position}px)`; position += 2; }

What This Does

Line 2: Moves the element horizontally by position pixels

Line 3: Increases position by 2 pixels for the next frame

Result: Element moves 2px to the right each frame (smooth animation!)

Common Updates

‚Ä¢ Change position (transform)

‚Ä¢ Modify opacity

‚Ä¢ Rotate elements

‚Ä¢ Scale size

‚Ä¢ Update colors

‚Ä¢ Draw on canvas

Example Updates

Kicking Off the Animation

Starting the Loop

What finally kicks off the animation is calling requestAnimationFrame() and passing in the animate function, this time outside the animate function:

// Define your animation function function animate() { update(); requestAnimationFrame(animate); } // Define your update function function update() { // Make changes here } // üöÄ START THE ANIMATION! requestAnimationFrame(animate);

üìå Important

The first call to requestAnimationFrame(animate) happens outside the function to start the loop.

After that, animate() calls it again from inside, creating the continuous loop.

The Loop Continues

The loop will continue until you stop it.

To stop an animation, you can use conditions or cancelAnimationFrame().

let animationId; function animate() { update(); // Keep looping animationId = requestAnimationFrame(animate); } // Start animation animationId = requestAnimationFrame(animate); // Stop animation later cancelAnimationFrame(animationId);
