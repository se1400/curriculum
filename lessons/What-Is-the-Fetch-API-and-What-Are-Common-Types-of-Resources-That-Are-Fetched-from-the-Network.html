The Fetch API

What is the Fetch API?

The Fetch API allows web apps to make network requests, typically to retrieve or send data to the server.

The fetch() Method

This API provides a fetch() method that you can use to make these requests.

✓ Primary Purpose

The primary purpose of the Fetch API in JavaScript is to make network requests and retrieve resources.

This allows your web applications to:

• Communicate with servers

• Retrieve data from APIs

• Send data to servers

• Load resources dynamically

Asynchronous Nature

The Fetch API is asynchronous - it returns a Promise that resolves to the response!

Basic fetch() Syntax

Simple Example

Let's look at a basic example of how to use fetch:

fetch('https://api.example.com/data')

What This Does

In this example, we're making a GET request to https://api.example.com/data.

This will then return us some data that we need to convert to JSON format and can use anywhere we want to.

Complete Example with JSON Conversion

fetch('https://api.example.com/data') .then(response => response.json()) .then(data => { console.log(data); // Use the data }) .catch(error => { console.error('Error:', error); });

Default Method

By default, the Fetch API uses the GET method to retrieve data.

This will be covered in the next lesson, along with other common HTTP request methods.

Understanding the Response

Two-Step Process

Working with fetch involves two main steps:

1 Get the Response

The first .then() receives the Response object.

.then(response => response.json())

2 Extract the Data

The second .then() receives the actual parsed data.

.then(data => console.log(data))

Complete Flow

fetch(url) .then(response => { // Step 1: Check if request was successful if (!response.ok) { throw new Error('Network response was not ok'); } // Step 2: Parse response as JSON return response.json(); }) .then(data => { // Step 3: Use the data console.log(data); }) .catch(error => { // Handle any errors console.error('Error:', error); });

Fetching JSON Data

Most Common Format

What type of data is commonly received from a fetch request to an API? JSON

Common JSON Data Types

In our web apps, we need some common data like:

• Weather data

• Professions list data

• Country names list

• Country code or country flag icons/images

Using these data we can make our app more informative and interactive.

Example: Fetching Weather Data

fetch('https://api.weather.com/current') .then(response => response.json()) .then(weather => { console.log(`Temperature: ${weather.temp}°F`); console.log(`Conditions: ${weather.conditions}`); });

✓ Why JSON?

JSON (JavaScript Object Notation) is:

• Easy to parse in JavaScript

• Lightweight and efficient

• Human-readable

• Universal standard for APIs

Fetching Images

Visual Resources

Images are some frequently fetched resources.

You might use fetch to load images statically or dynamically based on user actions, and display them on your web app.

Example: Fetching an Image

fetch('https://example.com/image.jpg') .then(response => response.blob()) .then(blob => { const imageUrl = URL.createObjectURL(blob); const img = document.createElement('img'); img.src = imageUrl; document.body.appendChild(img); });

Common Image Use Cases

• Profile pictures from user APIs

• Product images in e-commerce

• Country flags and icons

• Gallery images loaded on demand

• Thumbnails and previews

✓ Image Formats

Can fetch: JPG, PNG, GIF, WebP, SVG, and more!

Fetching Text Files

Plain Text Resources

Text files are another type of resource often fetched.

This could include configuration files, log files, or even entire documents that you want to display on your webpage.

Example: Fetching a Text File

fetch('https://example.com/config.txt') .then(response => response.text()) .then(text => { console.log(text); document.getElementById('output').textContent = text; });

Common Text File Use Cases

• Configuration files (.txt, .conf)

• Log files for debugging

• README documentation

• CSV data files

• Plain text documents

• Code files (.js, .css, .html)

Note

Use .text() instead of .json() for plain text files!

Fetching Audio and Video

Media Resources

In some cases, you might fetch audio or video files.

The Fetch API can handle these types of resources as well, allowing you to work with a wide variety of data types.

Example: Fetching Audio

fetch('https://example.com/audio.mp3') .then(response => response.blob()) .then(blob => { const audioUrl = URL.createObjectURL(blob); const audio = new Audio(audioUrl); audio.play(); });

Example: Fetching Video

fetch('https://example.com/video.mp4') .then(response => response.blob()) .then(blob => { const videoUrl = URL.createObjectURL(blob); const video = document.createElement('video'); video.src = videoUrl; video.controls = true; document.body.appendChild(video); });

Use Cases

• Music streaming services

• Podcast players

• Video streaming platforms

• Audio previews

• Video tutorials

Response Methods

Different Data Types

The Response object has different methods for different data types:

.json()

For JSON data (most common)

response.json()

Returns: JavaScript object/array

.text()

For plain text

response.text()

Returns: String

.blob()

For images, audio, video

response.blob()

Returns: Blob (binary data)

.arrayBuffer()

For raw binary data

response.arrayBuffer()

Returns: ArrayBuffer

.formData()

For form data

response.formData()

Returns: FormData object

✓ Choose the Right Method

Use the appropriate response method based on the type of data you're fetching!

Error Handling with Fetch

Always Handle Errors

Network requests can fail for many reasons. Always include error handling!

Complete Example with Error Handling

fetch('https://api.example.com/data') .then(response => { // Check if response is OK (status 200-299) if (!response.ok) { throw new Error(`HTTP error! status: ${response.status}`); } return response.json(); }) .then(data => { console.log('Success!', data); }) .catch(error => { // Handle network errors, HTTP errors, parse errors console.error('Fetch error:', error); });

⚠️ Common Errors

• Network connection failed

• Server returned error status (404, 500, etc.)

• Invalid JSON parsing

• CORS policy blocking

• Request timeout

Best Practices

✓ Always check response.ok

✓ Include .catch() block

✓ Provide user feedback on errors

✓ Log errors for debugging
