HTML is a Forgiving Language

HTML is a very forgiving language – elements still render even when you make mistakes, like forgetting to include a closing tag.

Example: Missing Closing Tag

<h1>Article Topic</h1>
<h2>Subheading 1</h2>
<h2>Subheading 2</h2>
<!-- This h2 does not have a closing tag -->
<h2>Subheading 3
Browser Output (still renders):

Article Topic

Subheading 1

Subheading 2

Subheading 3

The <h2> without a closing tag will still render fine. This happens because browsers use a parsing algorithm that handles common errors and tries to render HTML as closely as possible to the author's intention.

When Forgiveness Backfires

Problem: But this forgiving behavior could backfire sometimes. Let's see what happens when we add paragraphs under the headings.

Code with Paragraphs

<h1>Article Topic</h1>
<p>Lorem ipsum dolor sit amet consectetur adipisicing elit.</p>
<h2>Subheading 1</h2>
<p>Lorem, ipsum dolor sit amet consectetur adipisicing elit.</p>
<h2>Subheading 2</h2>
<p>Lorem ipsum dolor sit amet consectetur adipisicing elit.</p>
<!-- This h2 does not have a closing tag -->
<h2>Subheading 3
<p>Lorem ipsum dolor, sit amet consectetur adipisicing elit.</p>
Unexpected Browser Output:

Article Topic

Lorem ipsum dolor sit amet consectetur adipisicing elit. Maiores, nisi.

Subheading 1

Lorem, ipsum dolor sit amet consectetur adipisicing elit. At, doloremque.

Subheading 2

Lorem ipsum dolor sit amet consectetur adipisicing elit. Unde, placeat.

Subheading 3 Lorem ipsum dolor, sit amet consectetur adipisicing elit. Tempore, illum.

Result: The paragraph element under the <h2> without a closing tag renders as heading 2! The browser includes the paragraph text inside the heading element.
This is why you need an HTML validator.

What is an HTML Validator?

An HTML validator is a tool that checks the validity of your HTML code against the standard HTML specifications. It helps you identify errors and warnings in your HTML code, ensuring your web pages are correctly structured and compliant with web standards.

What Does It Do?

Checks code against HTML specifications
Identifies errors in your markup
Shows warnings for potential issues
Ensures correct structure
Verifies compliance with web standards
Using an HTML validator benefits:

You

Catch errors early and write cleaner code

Code Reviews

Make your code easier to review and maintain

Your Team

Help teammates understand your code better

Contributors

Support open-source contributors working on your project

W3C Markup Validation Service

There are several HTML validators available. The most widely accepted one is the W3C markup validation service.

🔗 W3C Validator

validator.w3.org

How to Use:

  1. Visit validator.w3.org
  2. Click on the "Validate by Direct Input" button
  3. Paste in your HTML code
  4. Click on the "Check" button
  5. Review the list of errors that need to be fixed
Industry Standard: The W3C validator is widely accepted as the authoritative HTML validation tool and is trusted by professional web developers worldwide.

Alternative HTML Validators

Another HTML validator that you can use is jsonformatter.org.

🔗 JSONFormatter HTML Validator

jsonformatter.org

How to Use:

  1. Visit jsonformatter.org
  2. Copy and paste your HTML code inside the first editor
  3. Click on the "Validate" button
  4. Review any errors displayed in your code
Best Practice: Make HTML validation a regular part of your development workflow. Catching errors early saves time and prevents bugs from reaching production.
