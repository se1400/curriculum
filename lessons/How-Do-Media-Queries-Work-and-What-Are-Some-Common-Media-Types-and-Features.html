Media Queries in CSS

Fundamental Component of Responsive Design

Media queries are a fundamental component of responsive web design, allowing developers to apply different styles based on the characteristics of the user's device or browser.

They provide a way to tailor the presentation of content to a variety of devices without changing the content itself.

What Media Queries Enable:

  • Adapt layouts to different screen sizes
  • Detect device capabilities (hover, resolution)
  • Respond to user preferences (dark mode)
  • Optimize for print vs. screen display

Key Concept: Media queries check for specific conditions. If these conditions are true, the corresponding styles are applied, creating responsive designs that adapt automatically.

Basic Syntax

How Media Queries Are Structured

A media query consists of a media type and one or more expressions that check for specific conditions.

@media mediatype and (feature: value) { /* CSS rules go here */ }
Syntax Breakdown:
@media
Keyword that begins the media query rule
mediatype
Specifies the type of media (screen, print, all, etc.)
and
Logical operator to combine conditions
(feature: value)
The condition that must be met for styles to apply (e.g., min-width: 768px)

Remember: If the conditions in the media query are true, the CSS rules inside the curly braces are applied to the page.

Media Features

Specific Device Characteristics

Media features describe specific characteristics of the user agent, output device, or environment. These are much more precise than media types.

Common Media Features:

  • width & height – Viewport dimensions
  • aspect-ratio – Width to height ratio
  • orientation – Landscape or portrait
  • resolution – Screen pixel density
  • hover – Hover capability detection
  • prefers-color-scheme – User's color preference

Power of Features: Media features allow you to target very specific device capabilities, making your designs more adaptable and user-friendly.

Width and Height Features

Targeting Viewport Dimensions

The width and height features refer to the viewport dimensions and are often used with min- or max- prefixes for range queries.

/* Minimum width query */ @media screen and (min-width: 768px) { /* Styles for screens at least 768px wide */ } /* Maximum width query */ @media screen and (max-width: 767px) { /* Styles for screens up to 767px wide */ } /* Range query */ @media screen and (min-width: 768px) and (max-width: 1024px) { /* Styles for screens between 768px and 1024px */ }
Common Breakpoints:
Mobile: max-width: 767px
Tablet: min-width: 768px and max-width: 1023px
Desktop: min-width: 1024px

Aspect Ratio Feature

Width to Height Ratio

The aspect-ratio feature describes the ratio between the width and height of the viewport. This is useful for targeting specific screen proportions.

/* Target 16:9 displays (widescreen) */ @media screen and (aspect-ratio: 16/9) { /* Styles for screens with a 16:9 aspect ratio */ } /* Target 4:3 displays (traditional) */ @media screen and (aspect-ratio: 4/3) { /* Styles for screens with a 4:3 aspect ratio */ } /* Minimum aspect ratio */ @media screen and (min-aspect-ratio: 16/9) { /* Styles for wider screens */ }

Orientation Feature

Landscape vs. Portrait

The orientation feature indicates whether the device viewport is in landscape (wider than tall) or portrait (taller than wide) orientation.

/* Landscape orientation */ @media screen and (orientation: landscape) { /* Styles for landscape orientation */ /* Viewport is wider than it is tall */ } /* Portrait orientation */ @media screen and (orientation: portrait) { /* Styles for portrait orientation */ /* Viewport is taller than it is wide */ }

Advanced Media Features

Resolution

The resolution feature describes the pixel density of the output device in dots per inch (dpi) or dots per centimeter (dpcm).

@media screen and (min-resolution: 300dpi) { /* Styles for high-resolution screens (Retina displays) */ }

Hover

The hover feature tests whether the primary input mechanism can hover over elements (mouse vs. touch).

@media (hover: hover) { /* Styles for devices that support hover (desktops) */ button:hover { background: #881518; } } @media (hover: none) { /* Styles for touch devices (no hover capability) */ button:active { background: #881518; } }

Prefers Color Scheme

The prefers-color-scheme feature detects if the user has requested a light or dark color theme.

@media (prefers-color-scheme: dark) { /* Styles for dark mode */ body { background: #1a1a1a; color: #ffffff; } } @media (prefers-color-scheme: light) { /* Styles for light mode */ body { background: #ffffff; color: #000000; } }

Logical Operators

Combining Multiple Conditions

Media queries can combine multiple conditions using logical operators to create more specific targeting rules.

and
Combines multiple media features. ALL conditions must be true.
@media screen and (min-width: 768px) and (orientation: landscape) { /* Both conditions must be true */ }
not
Negates the entire media query. Used to exclude certain conditions.
@media not screen and (color) { /* Applies when it's NOT a color screen */ }
only
Prevents older browsers that don't support media queries from applying the styles.
@media only screen and (min-width: 768px) { /* Modern browsers only */ }

Multiple Queries

Comma-Separated Lists

It's possible to target multiple queries in a comma-separated list, which functions like an "or" operator. If ANY condition is true, the styles apply.

/* "OR" logic - if EITHER condition is true, styles apply */ @media screen and (min-width: 768px), print { /* Applies to: - Screens at least 768px wide, OR - Print media */ .container { max-width: 1200px; } } /* Multiple size ranges */ @media (min-width: 768px) and (max-width: 1023px), (min-width: 1400px) { /* Applies to: - Tablets (768px - 1023px), OR - Large desktops (1400px+) */ }

Think of it as: Commas = OR, and keyword = AND. Use commas when you want styles to apply if any of several conditions are met.

The Cascade and Media Queries

Specificity and Order Matter

When working with media queries, it's important to consider the cascade. Media queries don't increase specificity – they just group conditional rules.

The normal rules of the CSS cascade still apply within each media query.

Order Matters:
/* Base styles */ .box { background: blue; width: 100%; } /* This will apply on tablets */ @media (min-width: 768px) { .box { background: red; width: 50%; } } /* This overrides the previous query on large screens */ @media (min-width: 1024px) { .box { background: green; width: 33.33%; } }

Best Practice: Order media queries from smallest to largest (mobile-first) or largest to smallest (desktop-first) to maintain predictable cascade behavior.

Mobile-First Responsive Design

Building Up from Mobile

A common pattern is to define base styles for mobile devices and then use media queries to enhance the layout for larger screens.

This approach is known as "mobile-first" responsive design.

/* Base styles (mobile) - no media query needed */ .container { width: 100%; padding: 10px; font-size: 14px; } /* Tablet and larger */ @media (min-width: 768px) { .container { width: 750px; margin: 0 auto; padding: 20px; font-size: 16px; } } /* Desktop and larger */ @media (min-width: 1024px) { .container { width: 1000px; padding: 30px; font-size: 18px; } }

Practical Example

Complete Responsive Layout

Let's see how all these concepts work together in a real-world example.

/* Mobile-first base styles */ .container { width: 100%; padding: 15px; } .grid { display: grid; grid-template-columns: 1fr; gap: 15px; } /* Tablet: 2 columns */ @media screen and (min-width: 768px) { .container { width: 90%; max-width: 750px; margin: 0 auto; padding: 25px; } .grid { grid-template-columns: repeat(2, 1fr); gap: 20px; } } /* Desktop: 3 columns */ @media screen and (min-width: 1024px) { .container { max-width: 1000px; padding: 40px; } .grid { grid-template-columns: repeat(3, 1fr); gap: 30px; } } /* Large desktop: 4 columns */ @media screen and (min-width: 1400px) { .container { max-width: 1300px; } .grid { grid-template-columns: repeat(4, 1fr); } } /* Print styles */ @media print { .container { width: 100%; padding: 0; } .grid { grid-template-columns: 1fr; } }

Summary

Powerful Tool for Responsive Design

Media queries are a powerful tool in CSS that allow for the creation of responsive, adaptable web designs.

Key Takeaways:

  • Media queries apply styles based on device characteristics
  • Media types (screen, print, all) define device categories
  • Media features (width, orientation, hover) provide precise targeting
  • Use logical operators (and, not) to combine conditions
  • Comma-separated queries act as "OR" operators
  • Mobile-first approach: base styles + progressive enhancement
  • Cascade rules still apply within media queries

Looking Forward: As web technologies continue to evolve, staying updated with new media features can help in creating more nuanced and user-friendly responsive designs.

By understanding how to use different media types and features, developers can create websites that provide optimal user experiences across a wide range of devices and preferences.
