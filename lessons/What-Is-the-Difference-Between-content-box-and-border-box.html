The box-sizing Property

The box-sizing property controls how the width and height of elements are calculated.

Two Possible Values:

box-sizing: content-box; /* Default */
box-sizing: border-box;

Understanding the difference between these two values is crucial for precise layout control in web design.

Setting box-sizing Globally

This property can be set on the universal selector (*) to apply to all elements in the document:

CSS
* {
  box-sizing: border-box;
}

Modern Best Practice:

Many developers set box-sizing: border-box globally at the start of their CSS. This makes sizing calculations much more intuitive!

Note: The default value is content-box, but you can choose border-box if you need to.

Quick Box Model Review

To understand how these models work, you need to be familiar with the four core concepts from the CSS box model:

Margin
Border
Padding
Content
  • Content: The space occupied by the element's content
  • Padding: Space between content and border
  • Border: Outline surrounding content and padding
  • Margin: Space outside the border

content-box (Default)

In the content-box model, the width and height determine the dimensions of the content area only.

‚ùå NOT Included:

Width and height do not include padding, border, or margin.

What This Means:

When you set width: 300px, you're only setting the width of the content area.

  • width = width of content
  • height = height of content

content-box: Calculating Total Width

To find the total width of the element (what you'll see on screen), you need to add everything together:

Total Width Formula:

content width
+ padding-left
+ padding-right
+ border-left
+ border-right
= Total Width

Remember: Margin is not included in the element's total width - it's space outside the element.

content-box: Calculating Total Height

The total height is calculated similarly:

Total Height Formula:

content height
+ padding-top
+ padding-bottom
+ border-top
+ border-bottom
= Total Height

content-box: Example

Let's look at a concrete example:

CSS
div {
  box-sizing: content-box;
  width: 300px;
  height: 200px;
  padding: 20px;
  border: 4px solid black;
}
Content: 300px √ó 200px

content-box: Calculation Breakdown

Total Width Calculation:

300px (content width)
+ 20px (padding-left)
+ 20px (padding-right)
+ 4px (border-left)
+ 4px (border-right)
= 348px Total Width

Total Height Calculation:

200px (content height)
+ 20px (padding-top)
+ 20px (padding-bottom)
+ 4px (border-top)
+ 4px (border-bottom)
= 248px Total Height

content-box: Key Takeaway

The Problem:

If content area is 300px wide, but you add 20px padding and 4px border on each side, the element actually takes up 348px on screen.

This can be confusing and lead to layout issues!

When to Use content-box:

Use content-box when you need precise control over the dimensions of the content area itself.

border-box

With border-box, the width and height include the content area, padding, AND border.

‚úì Included:

Width and height include content, padding, and border.

‚ùå NOT Included:

Margin is still not included.

What This Means:

The padding and border are inside the box. When you set width: 300px, the total element is 300px wide.

border-box: How It Works

The width value becomes:

Width = Everything Inside the Border:

content width
+ padding-left
+ padding-right
+ border-left
+ border-right
= Width Property Value

Important: The content area automatically shrinks to fit! The browser calculates: content width = width - padding - border

border-box: Example

Let's use the same CSS values as before:

CSS
div {
  box-sizing: border-box;
  width: 300px;
  height: 200px;
  padding: 20px;
  border: 4px solid black;
}
Total: 300px √ó 200px

border-box: Calculation Breakdown

Total Width:

300px (as specified!)

Content width is automatically calculated:

300px (specified width)
- 20px (padding-left)
- 20px (padding-right)
- 4px (border-left)
- 4px (border-right)
= 252px Content Width

Total Height:

200px (as specified!)

Content height is automatically calculated:

200px (specified height)
- 20px (padding-top)
- 20px (padding-bottom)
- 4px (border-top)
- 4px (border-bottom)
= 152px Content Height

border-box: Key Takeaway

The Advantage:

If you set width to 300px, the element is actually 300px wide on screen, regardless of padding and border!

This is much more intuitive and predictable.

When to Use border-box:

  • When you need to keep a fixed element size regardless of changes in padding or borders
  • For responsive web design - the content area adjusts automatically
  • When you want simpler calculations

Modern Best Practice

‚úì Recommended Approach:

border-box is becoming increasingly popular for its simplicity and flexibility.

Standard Setup:

/* Apply to all elements */
* {
  box-sizing: border-box;
}

/* Or with pseudo-elements */
*, *::before, *::after {
  box-sizing: border-box;
}

Most modern CSS frameworks and resets use border-box by default.

Why border-box Is Popular

Advantages of border-box:

  • Intuitive sizing: Width means total width
  • Easier calculations: No math needed for total size
  • Responsive-friendly: Elements stay within bounds
  • Flexible padding: Change padding without affecting layout
  • Percentage widths: Works better with fluid layouts
  • Form elements: Much easier to style consistently

Real-world benefit: If you set two elements to 50% width with border-box, they'll fit side-by-side perfectly, even with padding and borders!

Key Takeaways

content-box (Default):

  • width/height = content area only
  • Must add padding + border to get total size
  • Total size = content + padding + border
  • Use when you need precise content dimensions

border-box (Recommended):

  • width/height = content + padding + border
  • Total size equals specified width/height
  • Content area adjusts automatically
  • Better for responsive design and simpler layouts

Remember:

  • Margin is never included in either model
  • Set globally with * { box-sizing: border-box; }
  • Understanding both models is important for effective CSS

Making the Choice

Choosing between content-box and border-box really depends on the specific needs of your project.

Choose content-box When:
  • You need precise content area control
  • Working with existing content-box code
  • Content dimensions are critical
Choose border-box When:
  • Building new projects (recommended!)
  • Creating responsive layouts
  • You want simpler calculations
  • Working with percentage widths

Most developers prefer border-box for its simplicity and flexibility, but understanding both models is important for implementing effective CSS layouts!
