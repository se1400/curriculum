try...catch...finally

Graceful Error Handling

In the previous lesson, you learned how to throw exceptions in your programs.

In this lesson, we will take a look at how to gracefully handle these errors in a try...catch...finally block.

✓ Three Key Blocks

try: Wraps code that might throw an error (safe space)

catch: Captures and handles errors from try block

finally: Runs after try and catch, regardless of errors

try Block

Acts as a safe space to try something that could fail.

catch Block

Handles errors using the Error object to inspect what went wrong.

finally Block

Used for cleanup tasks like closing files or releasing resources.

Why Use try...catch...finally?

• Prevent program crashes from errors

• Handle errors gracefully

• Clean up resources properly

• Provide meaningful error messages

The try Block

Safe Space for Risky Code

The try block is used to wrap code that might throw an error.

It acts as a safe space to try something that could fail.

Basic Syntax

try { // Code that might throw an error // This is your "safe space" }

What Goes in try?

• Code that might fail (file operations, network requests)

• Function calls that could throw errors

• Operations on data that might not exist

• Parsing JSON or other data formats

• Any code where errors are expected possibilities

Example

try { const data = JSON.parse(userInput); console.log("Data parsed successfully!"); }

⚠️ Important

The try block alone doesn't handle errors - it just wraps risky code.

You need a catch block to actually handle any errors that occur!

The catch Block

Error Handler

The catch block captures and handles errors that occur in the try block.

You can use the Error object inside catch to inspect what went wrong.

Basic Syntax

try { // Code that might throw an error } catch (error) { // Code to handle the error console.error(error.message); }

The Error Object

The error passed into the catch block is an Error object which contains information about that error.

Common properties:

• message - Human readable error description

• name - Type of error (TypeError, ReferenceError, etc.)

• stack - Stack trace showing where error occurred

Example with Error Message

try { const data = JSON.parse(invalidJSON); } catch (error) { console.error("Error occurred:", error.message); // "Error occurred: Unexpected token..." }

✓ console.error

We are using console.error because it is designed specifically to log errors.

In many modern browsers, the output of console.error() appears in red in the console.

The finally Block

Always Executes

The finally block runs after the try and catch blocks, regardless of whether an error occurred.

It's commonly used for cleanup tasks, such as closing files or releasing resources.

Basic Syntax

try { // Code that might throw an error } catch (error) { // Code to handle the error } finally { // Code that runs regardless of whether an error occurred or not }

When to Use finally

The finally statement is executed regardless of if an exception was thrown or not.

Perfect for:

• Closing files

• Closing database connections

• Releasing resources

• Cleanup operations

• Logging completion

Example: File Operations

let file; try { file = openFile("data.txt"); writeToFile(file, data); } catch (error) { console.error("Error writing to file:", error.message); } finally { if (file) { closeFile(file); // Always close the file! } }

✓ Key Point

The finally block ensures cleanup happens whether the operation succeeds or fails!

Complete try...catch...finally Structure

try { // Code that might throw an error } catch (error) { // Code to handle the error } finally { // Code that runs regardless of whether an error occurred or not }

Execution Flow

1. Execute code in try block
↓
2a. If error → Jump to catch block
2b. If no error → Skip catch block
↓
3. Always execute finally block
↓
4. Continue with rest of program

With Error

1 try block starts

2 Error occurs

3 Jump to catch block

4 Execute finally block

Without Error

1 try block completes

2 Skip catch block

3 Execute finally block

Complete Example

processInput Function

Here is an example of using a try...catch block:

Function Definition

function processInput(input) { if (typeof input !== "string") { throw new TypeError("Input must be a string."); } return input.toUpperCase(); }

Using try...catch

try { console.log("Starting to process input..."); const result = processInput(9); // Number, not string! console.log("Processed result:", result); } catch (error) { console.error("Error occurred:", error.message); }

Console Output

"Starting to process input..."
"Error occurred:" "Input must be a string."

What Happened?

1 First console.log executes: "Starting to process input..."

2 processInput(9) is called with a number

3 Function throws TypeError

4 Execution jumps to catch block

5 Error message is logged in red

6 Second console.log never runs (skipped)

Error Object Properties

Inspecting Errors

The Error object contains useful information about what went wrong.

error.message

Human-readable description of the error

"Input must be a string."

error.name

Type of error that occurred

"TypeError"

error.stack

Stack trace showing where error occurred

"TypeError: Input must be a string. at processInput (script.js:3:11) at script.js:8:20"

Full Error Object

You can log the entire error object

console.error(error);

Example: Accessing Properties

try { throw new TypeError("Something went wrong"); } catch (error) { console.log("Name:", error.name); // "TypeError" console.log("Message:", error.message); // "Something went wrong" console.log("Stack:", error.stack); // Full stack trace }

Real-World Use Case: File Operations

Working with Files

A good use case for the finally statement is if you were working with files.

The Pattern

1. Open a file

2. Use try block to write data to the file

3. If there are any errors, use catch to catch those errors

4. Use finally statement to close the file

Example: File Operations

let fileHandle = null; try { // Open the file fileHandle = openFile("data.txt"); // Write data to the file (might throw error) writeToFile(fileHandle, userData); console.log("Data written successfully!"); } catch (error) { // Handle any errors that occurred console.error("Error writing to file:", error.message); } finally { // ALWAYS close the file, whether error occurred or not! if (fileHandle) { closeFile(fileHandle); console.log("File closed."); } }

✓ Why This Works Well

• File gets opened before try block

• try block handles potentially failing operations

• catch block handles any write errors

• finally ensures file is always closed, preventing resource leaks!

More Real-World Use Cases

1. Database Connections

try { connection = connectDB(); queryData(connection); } catch (error) { console.error(error.message); } finally { closeDB(connection); }

2. API Requests

try { const response = await fetch(url); const data = await response.json(); processData(data); } catch (error) { console.error(error.message); } finally { hideLoadingSpinner(); }

3. JSON Parsing

try { const obj = JSON.parse(jsonString); useData(obj); } catch (error) { console.error("Invalid JSON"); useDefaultData(); }

4. Cleanup Operations

try { startTimer(); performOperation(); } catch (error) { console.error(error.message); } finally { stopTimer(); logMetrics(); }

Best Practices

✓ Do

• Use try...catch for code that might throw errors

• Use console.error instead of console.log for errors

• Access error.message for user-friendly messages

• Use finally for cleanup operations

• Be specific about what errors you're catching

• Provide meaningful error messages to users

✗ Don't

• Don't use empty catch blocks - catch (error) {}

• Don't catch errors just to re-throw them unchanged

• Don't use try...catch for control flow

• Don't ignore error information

• Don't forget to clean up resources

⚠️ Remember

• try...catch only catches runtime errors

• Syntax errors can't be caught with try...catch

• finally always executes, even with return statements

• Async errors need special handling (try...catch with async/await)
