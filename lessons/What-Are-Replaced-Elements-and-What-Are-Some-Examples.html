What Are Replaced Elements?

A replaced element is an element whose content is determined by an external resource rather than by CSS itself.

CSS (Cascading Stylesheets) is used to add styles to a web page.

Common examples of replaced elements include the image, iframe, and video elements.

Key Characteristic

The content comes from an external source, not from the HTML or CSS code itself.

Common Examples

  • img (images)
  • iframe (embedded content)
  • video (video content)

CSS Limitations

With replaced elements, you can control the position or layout of an element.

But your CSS cannot directly modify the content of that element.

You can style the container, but not what's inside it.

What You CAN Style

✓ Position and layout
✓ Size (width/height)
✓ Borders and margins
✓ Filters and effects

What You CANNOT Style

✗ The actual content inside
✗ Elements within the resource

Image Element Example

Consider the image element, which embeds an image on your web page.

The element itself is replaced with the external object: the image.

Your CSS can control things like the positioning of the image or apply a filter to it, but you cannot actually modify the image itself.

Image Element
<img
  src="example-img-url"
  alt="Descriptive text goes here"
>

What Happens

The <img> tag is replaced by the actual image from the URL. You can position it, resize it, add borders, but you can't change the pixels of the image itself with CSS.

The iframe Element

A more robust example might be the <iframe> element, which embeds an external site on your web page.

The iframe allows you to display content from another website within your page.

iframe Purpose

Embeds external websites, videos, maps, or other content into your web page

Common Uses

  • YouTube videos
  • Maps (Google Maps, OpenStreetMap)
  • Social media feeds
  • External widgets

iframe Example - YouTube Video

Here is an example of using the iframe element for a YouTube video.

The src attribute points to the YouTube embed URL.

YouTube Video iframe
<iframe
  width="400"
  height="200"
  src="https://www.youtube.com/
  embed/VIDEO_ID"
></iframe>

Attributes

  • width/height: Size of the iframe
  • src: URL of content to embed

iframe Example - Embedded Map

Other common examples of using the iframe element would be to embed maps onto the page.

You can embed interactive maps from services like OpenStreetMap or Google Maps.

Map iframe
<iframe
  title="Map of Location"
  width="300"
  height="200"
  src="https://www.openstreetmap.org/
  export/embed.html?..."
></iframe>

Interactive Content

Embedded maps are fully interactive - users can zoom in/out and pan around.

CSS Limitations with iframe

The element itself is replaced with the external object: the site.

Your CSS can change the positioning of the embedded site, but you cannot modify the site's contents.

To dive a bit further, if the embedded site has an h1 element, your CSS would not be able to style that h1 element. You cannot change the size, font color, and so on.

What You Can Do

✓ Position the iframe
✓ Resize the iframe
✓ Add borders to iframe

What You Cannot Do

✗ Style elements inside the iframe
✗ Change text color in embedded content
✗ Modify fonts in embedded site

Other Replaced Elements

There are some other replaced elements, such as <video> and <embed>.

Some elements behave as replaced elements under specific circumstances.

Always Replaced

  • img
  • iframe
  • video
  • embed
  • object

Conditionally Replaced Elements

Here's an example of an input element with the type attribute set to image.

This type of input is considered to be a replaced element, but other input types like text or email are not replaced elements.

Input as Replaced Element
<input
  type="image"
  alt="Descriptive text goes here"
  src="example-img-url"
>

Input Types

Replaced: type="image"
Not Replaced: type="text", type="email"

Key Takeaways

Replaced elements are special because their content comes from external sources.

Understanding this distinction helps you know what you can and cannot style with CSS.

Remember

  • Content is from external resources
  • You can style the element's box (position, size, borders)
  • You cannot style the content inside
  • Common examples: img, iframe, video
  • Some elements are conditionally replaced
