CSS Custom Properties

CSS Variables Explained

CSS custom properties, also known as CSS variables, are entities defined by CSS authors that contain specific values to be reused throughout a document.

What Are They? Think of them as containers that store values you can use over and over again in your CSS!

What You'll Learn:

  • How to declare custom properties
  • Using the var() function
  • Working with the CSS cascade
  • Creating themeable designs
  • Using with media queries

Why Custom Properties?

Powerful Benefits

They are a powerful feature that allows for more efficient, maintainable, and flexible stylesheets.

Key Advantages:

  • Reusability – Define once, use everywhere
  • Maintainability – Change in one place
  • Flexibility – Dynamic runtime changes
  • Themeable – Easy theme switching
  • Cascading – Follows CSS cascade rules

Bottom Line: Custom properties make your CSS smarter and easier to manage!

Declaration Syntax

How to Declare

The syntax for declaring a custom property is straightforward. It begins with two dashes (--) followed by the property name:

:root { --main-color: #3498db; }
--
main-color
:
#3498db

Breaking it down: -- = prefix, main-color = name, #3498db = value

Understanding the Declaration

What's Happening?

In this example, we're declaring a custom property named --main-color with a value of #3498db.

:root { --main-color: #3498db; /* Blue color */ }

Key Points:

  • Property name: --main-color (you choose this)
  • Value: #3498db (a blue color)
  • Scope: Declared in :root (global)
  • Reusable: Can be used anywhere in your CSS

Using :root

Global Scope

The :root pseudo-class is commonly used to declare global custom properties, as it represents the highest-level parent in the DOM tree.

/* Global variables - available everywhere */ :root { --main-color: #3498db; --secondary-color: #2ecc71; --font-size: 16px; --spacing: 20px; }

Why :root? It's at the top of the DOM tree, so properties defined here are available to all elements in your document.

Think of it as: :root is like a global storage container for your CSS values!

Using var() Function

Accessing the Values

To use a custom property, you employ the var() function:

.button { background-color: var(--main-color); }

What this does: This sets the background color of elements with the class button to the value stored in --main-color.

Live Example:

This button uses var(--main-color) for its background!

Complete Example

Declaration + Usage

/* Step 1: Declare the variable */ :root { --main-color: #3498db; } /* Step 2: Use it anywhere! */ .button { background-color: var(--main-color); } .header { border-color: var(--main-color); } .link { color: var(--main-color); }

Powerful! Change --main-color in one place, and all three elements update automatically!

Following the Cascade

Redefining Properties

One of the key features of custom properties is that they follow the CSS cascade. This means that you can redefine them for specific elements or contexts:

:root { --main-color: #3498db; /* Global: Blue */ } .alert { --main-color: #e74c3c; /* Override: Red for alerts */ background-color: var(--main-color); }

In this case: Elements with the class alert will use a different --main-color value, overriding the global definition.

Cascade in Action

Different Contexts, Different Values

Same Property, Different Values:
Normal: Uses global --main-color (#3498db)
Alert: Uses overridden --main-color (#e74c3c)

Magic of Cascade: The same property name, but different values based on context!

Fallback Values

Safety Net

Custom properties also support fallback values. If a custom property is not defined or is invalid, you can provide a fallback value:

.text { color: var(--text-color, green); }

How it works: Here, if --text-color is not defined, the color will default to green.

Fallback in Action

Real-World Usage

/* Fallback examples */ .element { /* If --primary not defined, use #3498db */ background: var(--primary, #3498db); /* If --spacing not defined, use 20px */ padding: var(--spacing, 20px); /* Multiple fallbacks! */ color: var(--text-color, var(--default-color, black)); }

Smart Defaults: Fallbacks ensure your styles always have a value, even if a variable isn't defined!

Creating Themes

Theme Switching Made Easy

Custom properties are particularly useful in creating themeable designs. You can define a set of properties for different themes:

/* Light theme */ .light-theme { --bg-color: #ffffff; --text-color: #333333; } /* Dark theme */ .dark-theme { --bg-color: #2d2d2d; --text-color: #ffffff; } /* Use the theme variables */ body { background-color: var(--bg-color); color: var(--text-color); }

Simple Theme Switching

Just Add/Remove a Class

Switching themes becomes as simple as adding or removing a class from the body element.

/* HTML - Just change the class! */ <body class="light-theme"> <!-- Content --> </body> /* Switch to dark */ <body class="dark-theme"> <!-- Same content, different theme! --> </body>

That's It! One class change, entire site theme switches!

Custom Properties + Media Queries

Responsive Design

Custom properties can also be used with media queries to create responsive designs:

:root { --card-width: 300px; } @media (max-width: 768px) { :root { --card-width: 100%; } } .card { width: var(--card-width); }

Responsive Magic: Change the variable value at different breakpoints, and all elements using it adapt automatically!

Maintainable Responsive Layouts

Less Repetition

This approach allows for more maintainable responsive layouts, as you can change values in one place rather than throughout your stylesheet.

❌ Without Variables

.card { width: 300px; } .sidebar { width: 300px; } .modal { width: 300px; } @media (max-width: 768px) { .card { width: 100%; } .sidebar { width: 100%; } .modal { width: 100%; } }

✅ With Variables

:root { --width: 300px; } .card { width: var(--width); } .sidebar { width: var(--width); } .modal { width: var(--width); } @media (max-width: 768px) { :root { --width: 100%; } }

Reducing Repetition

DRY Principle

This can significantly reduce repetition in your stylesheets, especially for frequently used, complex values.

:root { --box-shadow: 0 4px 6px rgba(0, 0, 0, 0.1); --border-radius: 8px; --transition: all 0.3s ease; } .card, .button, .modal, .tooltip { box-shadow: var(--box-shadow); border-radius: var(--border-radius); transition: var(--transition); }

Smart Coding: Define complex values once, use them everywhere!

Referencing Other Properties

Building Relationships

Custom properties can also reference other custom properties:

:root { --primary-color: #3498db; --primary-light: #5dade2; --primary-dark: #2980b9; /* Reference other variables! */ --button-bg: var(--primary-color); --button-hover: var(--primary-dark); --link-color: var(--primary-light); } .button { background: var(--button-bg); } .button:hover { background: var(--button-hover); }

Creating Relationships

Consistent Themes

This allows for creating relationships between different style aspects, making it easier to maintain consistent themes across your site.

:root { /* Base colors */ --brand-color: #e74c3c; /* Related colors reference the base */ --accent: var(--brand-color); --border: var(--brand-color); --focus-ring: var(--brand-color); /* Spacing system */ --space-unit: 8px; --space-xs: var(--space-unit); --space-sm: calc(var(--space-unit) * 2); --space-md: calc(var(--space-unit) * 3); --space-lg: calc(var(--space-unit) * 4); }

Design System: Change one base value, all related values update automatically!

Browser Support

Compatibility Considerations

While custom properties offer many advantages, it's important to be aware of their browser support.

Good News: They are well-supported in modern browsers, but older browsers may not recognize them.

Important: Always have a fallback plan when using cutting-edge features.

/* Provide fallback for older browsers */ .button { background-color: #3498db; /* Fallback */ background-color: var(--main-color); /* Modern */ }

JavaScript Integration

Dynamic Changes

One powerful feature: Custom properties can be changed at runtime with JavaScript!

// Get the root element const root = document.documentElement; // Change a custom property root.style.setProperty('--main-color', '#e74c3c'); // Get a custom property value const color = getComputedStyle(root) .getPropertyValue('--main-color');

Dynamic Styling: Update themes, colors, and values on the fly based on user interaction!

Real-World Use Cases

Where to Use Them

Common Applications:

  • Theme Switching – Light/dark mode
  • Design Systems – Consistent spacing, colors
  • Responsive Design – Breakpoint-specific values
  • Component Libraries – Customizable components
  • Animation – Dynamic animation properties
  • User Preferences – Personalized styling

Best Practices

Tips for Success

Follow These Guidelines:

  • Use descriptive names (--primary-color, not --c1)
  • Declare global variables in :root
  • Group related variables together
  • Provide fallback values when appropriate
  • Consider browser support for your users
  • Use semantic names over specific values
/* Good: Semantic names */ --primary-color: #3498db; --heading-font: 'Arial', sans-serif; /* Avoid: Specific names */ --blue: #3498db; --arial: 'Arial', sans-serif;
