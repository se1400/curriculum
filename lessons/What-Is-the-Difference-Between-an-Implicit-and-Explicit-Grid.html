Implicit vs Explicit Grid

Two Types of Grid Tracks

In CSS Grid, there are two types of grid tracks (rows and columns): explicit and implicit.

Understanding Both Types:

  • Explicit Grid ‚Äì Tracks you define intentionally
  • Implicit Grid ‚Äì Tracks created automatically by the browser

Key Distinction: Explicit tracks are planned and defined by you, while implicit tracks are created automatically when needed.

What is Implicit Grid?

Automatically Created Tracks

Implicit grid refers to the rows and columns automatically created by the browser when placing items in a grid layout ‚Äì those not explicitly defined using grid-template-rows or grid-template-columns.

üî∑ Implicit Grid

  • Automatically created by the browser
  • Generated when items exceed explicit grid
  • Not manually defined in your CSS

Example: If you define 2 columns but have 6 items, the browser automatically creates additional rows to hold items 3-6. Those extra rows are implicit!

Controlling Implicit Tracks

Special Properties

The properties that control the columns and rows created implicitly by the browser are grid-auto-columns and grid-auto-rows.

.grid-container { display: grid; grid-template-columns: 1fr 1fr; /* Explicit: 2 columns */ grid-auto-rows: 100px; /* Implicit: auto rows at 100px */ }

Properties for Implicit Grid:

  • grid-auto-rows ‚Äì Sets size of implicit rows
  • grid-auto-columns ‚Äì Sets size of implicit columns

Default Behavior: If you don't set these properties, implicit tracks default to auto sizing (sized by their content).

Two Meanings of Implicit Grid

Dual Definition

Implicit grid also refers to the additional rows and columns the browser automatically generates when you place an item outside the explicitly defined rows and columns.

Implicit Grid Means:

  • Tracks created automatically when items overflow explicit grid
  • Additional tracks generated when items are placed outside explicit grid

In Practice: Both meanings describe the same concept ‚Äì the browser creating tracks you didn't explicitly define when they're needed.

Example: 2 Explicit Columns

Setting Up the Grid

For instance, let's say you define only two explicit columns in a grid layout this way:

.grid-container { display: grid; grid-template-columns: 1fr 1fr; /* Only 2 columns defined */ }
<div class="grid-container"> <div class="grid-item">Item 1</div> <div class="grid-item">Item 2</div> <div class="grid-item">Item 3</div> <div class="grid-item">Item 4</div> <div class="grid-item">Item 5</div> <div class="grid-item">Item 6</div> </div>

Question: We defined 2 columns but have 6 items. What happens to items 3-6?

The Result

Automatic Row Creation

Two items filled the first row using the two explicit columns ‚Äì Item 1 in the first column and Item 2 in the second column.

The next items start a new row with Item 3 going in the first column of the second row and Item 4 in the second column of the second row, and so on.

Understanding the Layout

Explicit vs Implicit

What We Defined (Explicit):

  • 2 columns using grid-template-columns: 1fr 1fr
  • No rows explicitly defined

What the Browser Created (Implicit):

  • 3 rows automatically to hold all 6 items
  • Row 1: Items 1-2 (using explicit columns)
  • Row 2: Items 3-4 (implicit row)
  • Row 3: Items 5-6 (implicit row)

Key Point: The columns were explicit (we defined them), but the rows were implicit (browser created them automatically).

What is Explicit Grid?

Intentionally Defined Tracks

As you've already seen, explicit grids are the areas of the grid you intentionally set up. That is, the rows and columns you explicitly define for a grid layout using the grid-template-rows and grid-template-columns properties.

üî∂ Explicit Grid

  • Manually defined by you in CSS
  • Uses grid-template-rows and grid-template-columns
  • Planned structure that you control
.grid-container { display: grid; grid-template-columns: 100px 200px 100px; /* 3 explicit columns */ grid-template-rows: 50px 100px; /* 2 explicit rows */ }

Properties for Explicit Grid

Defining Your Grid Structure

Properties for Explicit Grid:

  • grid-template-columns ‚Äì Defines column structure
  • grid-template-rows ‚Äì Defines row structure
/* Define 3 columns and 2 rows explicitly */ .grid-container { display: grid; grid-template-columns: 1fr 2fr 1fr; grid-template-rows: 100px 200px; } /* Use repeat() for repeated values */ .grid-container { display: grid; grid-template-columns: repeat(4, 1fr); grid-template-rows: 50px 100px; }

Full Control: With explicit grid, you have complete control over the size and number of tracks in your layout.

When to Use Each Type

Making the Right Choice

Use Explicit Grid When:

  • You know the exact layout structure in advance
  • You need precise control over track sizes
  • The layout is predictable and fixed
  • You want maximum performance

Use Implicit Grid When:

  • Content is dynamic or unknown
  • Number of items may vary
  • You want the browser to handle overflow automatically
  • Building flexible, adaptable layouts

Summary

Key Takeaways

Remember:

  • Explicit Grid ‚Äì Tracks you define with grid-template-*
  • Implicit Grid ‚Äì Tracks the browser creates automatically
  • Control implicit sizing with grid-auto-rows/columns
  • Explicit = planned, Implicit = automatic
  • Most layouts use both types together
/* Common pattern: Explicit columns, implicit rows */ .grid-container { display: grid; grid-template-columns: repeat(3, 1fr); /* Explicit: 3 columns */ grid-auto-rows: 100px; /* Implicit: rows at 100px */ gap: 20px; } /* Result: 3 columns defined, as many rows as needed */

Pro Tip: Understanding the difference helps you write more efficient grid layouts and predict how your grid will behave with varying content!
