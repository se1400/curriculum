Type Selectors

Type selectors, also known as element selectors, target elements based on their tag name.

Characteristics

  • Fundamental in CSS
  • Apply styles to all instances of a specific HTML element
  • Straightforward to use
  • Written simply as the tag name

Syntax

Type selectors are written as the tag name of the element you want to style.

Common Examples

  • p - paragraphs
  • div - divisions
  • h1 - headings
  • span - spans
  • a - anchors/links
CSS - Type Selector
p {
  color: blue;
}
What This Does

This targets all <p> (paragraph) elements on the page and sets their text color to blue.

Basic Example

Targeting all paragraph elements on the page with a type selector.

HTML
<p>Paragraph one</p>
<p>Paragraph two</p>
<p>Paragraph three</p>
CSS
p {
  color: blue;
}
Result

Paragraph one

Paragraph two

Paragraph three

All paragraph elements are styled blue

Type Selector Specificity

Type selectors have a relatively low specificity compared to other selectors.

Specificity Value: (0, 0, 0, 1)

What This Means

Type selectors will be overridden by class selectors, ID selectors, and inline styles, but can still apply styles unless those higher-specificity rules are present.

Specificity Hierarchy

Where type selectors fit in the specificity ladder:

Inline Styles (1, 0, 0, 0)
ID Selectors (0, 1, 0, 0)
Class, Attribute, Pseudo-class (0, 0, 1, 0)
Type Selectors ← You are here (0, 0, 0, 1)
Universal (*) (0, 0, 0, 0)

What Can Override Type Selectors?

Selector Type Specificity Can Override Type Selector?
Inline styles (1, 0, 0, 0) ✓ YES
ID selectors (0, 1, 0, 0) ✓ YES
Class selectors (0, 0, 1, 0) ✓ YES
Type selectors (0, 0, 0, 1) Same level
Universal (*) (0, 0, 0, 0) ✗ NO

Type Selector Examples

Styling Headings

h1 { font-size: 2rem; }

Targets all h1 elements

Styling Divs

div { padding: 1rem; }

Targets all div elements

Styling Links

a { color: blue; }

Targets all anchor elements

Styling Spans

span { font-weight: bold; }

Targets all span elements

Class Override Example

Let's see how a class selector overrides a type selector.

HTML
<p class="para">I am a paragraph</p>
<p class="para">Here is another paragraph</p>
CSS
p {
  color: blue;
}

.para {
  color: red;
}

Which Style Wins?

Both selectors target the same elements, but they have different specificity values.

Cascade Resolution

p { color: blue; } (0, 0, 0, 1) OVERRIDDEN
.para { color: red; } (0, 0, 1, 0) WINS!
Result

I am a paragraph

Here is another paragraph

All paragraphs with class="para" are red because class selectors have higher specificity than type selectors

Type vs Universal Selector

Type selectors DO override the universal selector because they have higher specificity.

CSS
* {
  color: blue;
}

p {
  color: red;
}
* { color: blue; } (0, 0, 0, 0) OVERRIDDEN
p { color: red; } (0, 0, 0, 1) WINS!
Result

This text is red

Key Takeaways

  • Type selectors target elements by their tag name (p, div, h1, etc.)
  • They are fundamental and straightforward to use
  • Specificity value: (0, 0, 0, 1)
  • Have relatively low specificity - can be overridden by class, ID, and inline styles
  • DO override universal selector (*)
  • Apply styles to ALL instances of that element type

Remember

Type selectors: Low specificity, but more specific than universal (*)
