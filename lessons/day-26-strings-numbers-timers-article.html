You have already learned to store data in objects and arrays — product catalogs, user profiles, shopping cart items. But raw data does not automatically look good to a user. A product name pulled from an object might have leading whitespace, inconsistent casing, or a length that blows out a card layout. A price stored as the number 19.9 needs to be displayed as $19.90. A date object needs a human-readable label. This lesson is about bridging the gap between stored data and displayed data: how to take what you have and format it beautifully for the screen.

Timers connect to this same thread. A live clock, a countdown, a search input that only fires after the user pauses — all of these make an interface feel responsive and alive. By the end of this lesson you will have the string and number tools to clean and format any value, and the timer tools to update that formatted output in real time.

Strings

Strings Are Immutable

The first and most important thing to understand about strings is that they are immutable. Once a string is created, it cannot be changed. Every string method you call — toUpperCase(), trim(), replace() — does not modify the original string. It returns a brand-new string with the transformation applied.

[IMG - String immutability methods return new string]

LISTING 26.1 — Strings Are Immutable 
let productName = "  wireless headphones  ";

// These do NOT modify productName
productName.trim();          // returns "wireless headphones"
productName.toUpperCase();   // returns "  WIRELESS HEADPHONES  "

console.log(productName);    // still "  wireless headphones  "

// To use the result, assign it to a variable (or the same variable)
let cleaned = productName.trim().toUpperCase();
console.log(cleaned);        // "WIRELESS HEADPHONES"

Immutability is actually a feature, not a limitation. Because methods always return a new string and never disturb the original, you can chain methods safely — each method receives the result of the one before it and returns a further-transformed string.

LISTING 26.2 — Method Chaining on Strings 
const raw = "  Wireless Headphones! On Sale  ";

const slug = raw
  .trim()
  .toLowerCase()
  .replaceAll(" ", "-")
  .replace(/[^a-z0-9-]/g, "");

console.log(slug); // "wireless-headphones-on-sale"

You will build that slug generator step by step later in this lesson. For now, just notice how each method feeds into the next without any intermediate variables.

String Basics

A few fundamentals before the methods:

  • string.length — the number of characters in the string, including spaces and punctuation
  • string[0] — bracket notation gives you the character at that index; strings are zero-indexed like arrays
  • string.at(-1) — the last character; negative indices count from the end, exactly as they do on arrays
  • string.at(-2) — the second-to-last character
LISTING 26.3 — String Indexing and Length 
const sku = "HDPH-2024-BLK";

console.log(sku.length);    // 13
console.log(sku[0]);        // "H"
console.log(sku.at(-1));    // "K"
console.log(sku.at(-3));    // "B"

Case Methods

toUpperCase() and toLowerCase() return a new string with all letters shifted to the requested case. They are most commonly used in two situations: normalizing input for comparison (so that “apple”, “Apple”, and “APPLE” all match the same way), and preparing text for display.

LISTING 26.4 — Case Normalization for Comparison 
function searchProducts(products, query) {
  const q = query.toLowerCase();
  return products.filter(p => p.name.toLowerCase().includes(q));
}

const catalog = [
  { name: "Wireless Headphones", price: 79.99 },
  { name: "USB-C Hub",           price: 34.99 },
  { name: "Notebook",            price: 12.99 },
];

console.log(searchProducts(catalog, "usb")); // finds "USB-C Hub"
console.log(searchProducts(catalog, "USB")); // also finds "USB-C Hub"

Note that toUpperCase() and toLowerCase() are locale-aware for most common cases, but if you need guaranteed locale-specific case handling (for languages like Turkish), use toLocaleLowerCase(locale) and toLocaleUpperCase(locale) instead.

Searching Within Strings

JavaScript provides several methods for locating text inside a string. Choosing the right one makes your intent clear at a glance.

  • includes(searchString) — returns true or false; the right choice whenever you just need a yes/no answer
  • startsWith(prefix) — returns true if the string begins with the given prefix; useful for checking URL protocols or file extensions
  • endsWith(suffix) — returns true if the string ends with the given suffix; useful for checking file types
  • indexOf(search) — returns the position (zero-based index) of the first match, or -1 if not found; use when you need the position
  • lastIndexOf(search) — same, but scans from right to left; returns the position of the last match
LISTING 26.5 — Search Methods in Practice 
const url = "https://example.com/products/headphones.jpg";

// Is it a secure URL?
console.log(url.startsWith("https")); // true

// Is it an image?
console.log(url.endsWith(".jpg"));    // true

// Does the path include "products"?
console.log(url.includes("products")); // true

// Where does the filename start?
const lastSlash = url.lastIndexOf("/");
const filename = url.slice(lastSlash + 1);
console.log(filename); // "headphones.jpg"

// Does the product name contain a keyword?
const name = "Wireless Noise-Cancelling Headphones";
if (name.toLowerCase().includes("wireless")) {
  console.log("Show wireless badge");
}

Use includes() when you want a boolean for control flow. Use indexOf() when you need the position to extract a substring.

Extracting Parts of a String

Two methods extract a substring from a larger string:

  • slice(start, end) — extracts characters from start up to (but not including) end; supports negative indices
  • substring(start, end) — similar to slice, but does not support negative indices; considered legacy

Prefer slice() in new code. Negative indices make it significantly more flexible.

LISTING 26.6 — slice() for Truncation and Extraction 
const description = "This wireless headphone set delivers studio-quality sound with 40-hour battery life.";

// Truncate to 50 characters with an ellipsis
function truncate(str, maxLength) {
  if (str.length <= maxLength) return str;
  return str.slice(0, maxLength) + "...";
}

console.log(truncate(description, 50));
// "This wireless headphone set delivers studio-qualit..."

// Extract a file extension
const filename = "product-photo.final.jpg";
const dotIndex = filename.lastIndexOf(".");
const extension = filename.slice(dotIndex + 1);
console.log(extension); // "jpg"

// Last 3 characters (negative index)
const code = "ORDER-2024-XF7";
console.log(code.slice(-3)); // "XF7"

Replacing Content

  • replace(search, replacement) — replaces the first match only; accepts a string or a regular expression
  • replaceAll(search, replacement) — replaces every match; also accepts a string or regex (regex must use the g flag)
LISTING 26.7 — replace() and replaceAll() 
// Template substitution
const template = "Hello, {name}! Your order {orderId} has shipped.";
const message = template
  .replace("{name}", "Priya")
  .replace("{orderId}", "#4821");
console.log(message);
// "Hello, Priya! Your order #4821 has shipped."

// Build a URL slug — spaces become hyphens
const title = "Top 10 Wireless Headphones in 2024";
const withHyphens = title.replaceAll(" ", "-");
console.log(withHyphens); // "Top-10-Wireless-Headphones-in-2024"

// Replace only the first occurrence
const csv = "name,price,name,quantity";
console.log(csv.replace("name", "product")); // "product,price,name,quantity"
console.log(csv.replaceAll("name", "product")); // "product,price,product,quantity"

Cleaning Strings

Form inputs frequently arrive with extra whitespace — a user might paste text with a leading space, or press the spacebar after the last word. Always clean input before storing or comparing it.

  • trim() — removes whitespace from both ends
  • trimStart() — removes whitespace from the beginning only
  • trimEnd() — removes whitespace from the end only
LISTING 26.8 — Cleaning Form Input 
function handleFormSubmit(formData) {
  const name = formData.name.trim();
  const email = formData.email.trim().toLowerCase();

  if (!name) {
    return { error: "Name is required" };
  }

  // Safe to store — no leading/trailing spaces, consistent case
  return { name, email };
}

const raw = { name: "  Priya Patel  ", email: "  Priya@Example.COM  " };
console.log(handleFormSubmit(raw));
// { name: "Priya Patel", email: "priya@example.com" }

Padding

Padding lets you reach a target string length by adding characters to the start or end. The most common use is zero-padding numbers for consistent display: "5" becomes "05", "9" becomes "09".

  • padStart(targetLength, padChar) — pads from the start until the string reaches targetLength
  • padEnd(targetLength, padChar) — pads from the end
LISTING 26.9 — Padding for Display 
// Zero-pad seconds and minutes in a timer display
function formatTime(totalSeconds) {
  const minutes = Math.floor(totalSeconds / 60);
  const seconds = totalSeconds % 60;
  return `${String(minutes).padStart(2, "0")}:${String(seconds).padStart(2, "0")}`;
}

console.log(formatTime(65));  // "01:05"
console.log(formatTime(9));   // "00:09"
console.log(formatTime(600)); // "10:00"

// Zero-pad an order ID to 6 digits
const orderId = 421;
console.log(String(orderId).padStart(6, "0")); // "000421"

// Right-align a product code in a fixed-width label
const code = "XF7";
console.log(code.padStart(8, " ")); // "     XF7"

Other Useful Methods

repeat(count) returns a new string with the original repeated count times. A natural fit for generating visual patterns from data:

LISTING 26.10 — repeat() for Star Ratings 
function renderStars(rating, maxStars = 5) {
  const filled = "★".repeat(rating);
  const empty  = "☆".repeat(maxStars - rating);
  return filled + empty;
}

const products = [
  { name: "Wireless Headphones", rating: 4 },
  { name: "USB-C Hub",           rating: 3 },
  { name: "Notebook",            rating: 5 },
];

products.forEach(p => {
  console.log(`${p.name}: ${renderStars(p.rating)}`);
});
// Wireless Headphones: ★★★★☆
// USB-C Hub:           ★★★☆☆
// Notebook:            ★★★★★

split(delimiter) breaks a string into an array at every occurrence of the delimiter. Passing an empty string "" splits into individual characters.

LISTING 26.11 — split() for Parsing 
// Parse a CSV row into an array
const row = "Wireless Headphones,79.99,Audio,In Stock";
const fields = row.split(",");
console.log(fields);
// ["Wireless Headphones", "79.99", "Audio", "In Stock"]

// Parse comma-separated tags
const tags = "bluetooth, wireless, noise-cancelling, audio";
const tagArray = tags.split(", ").map(tag => tag.trim());
console.log(tagArray);
// ["bluetooth", "wireless", "noise-cancelling", "audio"]

// Reverse a string using split + reverse + join
const word = "JavaScript";
const reversed = word.split("").reverse().join("");
console.log(reversed); // "tpircSavaJ"

Method Chaining: Building a URL Slug Generator

URL slugs are the human-readable parts of web addresses: /products/wireless-headphones-noise-cancelling. They are generated from text by removing unwanted characters, lowercasing everything, and replacing spaces with hyphens. String method chaining makes this a single expression.

LISTING 26.12 — URL Slug Generator 
function toSlug(title) {
  return title
    .trim()                            // remove leading/trailing spaces
    .toLowerCase()                     // normalize to lowercase
    .replaceAll(" ", "-")              // replace spaces with hyphens
    .replace(/[^a-z0-9-]/g, "");      // remove anything not a letter, digit, or hyphen
}

console.log(toSlug("  Wireless Headphones! On Sale  "));
// "wireless-headphones-on-sale"

console.log(toSlug("Top 10 Products for 2024 (Updated)"));
// "top-10-products-for-2024-updated"

console.log(toSlug("USB-C Hub & Docking Station"));
// "usb-c-hub-docking-station"

Each method receives the result of the previous one. trim() returns a string to toLowerCase(), which returns a string to replaceAll(), which returns a string to replace(). The regex pattern /[^a-z0-9-]/g is a regular expression — explained next.

A Brief Introduction to Regular Expressions

A regular expression (or regex) is a pattern language for matching text. Rather than looking for a fixed string like "jpg", a regex lets you describe a pattern: “one or more digits”, “any letter followed by an @ sign”, “anything that is not a letter”. Regexes are written between forward slashes: /pattern/.

A full regex course is its own subject. This section gives you just enough to read and write common patterns. Use regex101.com to build and test patterns interactively — it explains what each part of a pattern means in plain English as you type.

Three methods work with regexes directly:

  • /pattern/.test(str) — returns true or false: does this string match the pattern?
  • str.match(/pattern/) — returns an array of the matched text, or null if no match; add the g flag to find all matches
LISTING 26.13 — Regular Expressions for Validation 
// Email validation (basic — just checks the shape)
const emailPattern = /\S+@\S+\.\S+/;
console.log(emailPattern.test("priya@example.com")); // true
console.log(emailPattern.test("not-an-email"));       // false

// Check a US phone number format: (555) 123-4567
const phonePattern = /^\(\d{3}\) \d{3}-\d{4}$/;
console.log(phonePattern.test("(555) 123-4567")); // true
console.log(phonePattern.test("555-123-4567"));   // false

// Extract all hashtags from a post (match with g flag returns all matches)
const post = "Loving my new #headphones! #audio #tech";
const hashtags = post.match(/#\w+/g);
console.log(hashtags); // ["#headphones", "#audio", "#tech"]

// Remove all non-alphanumeric characters from a slug
const raw = "Top 10 Products! (2024)";
const cleaned = raw.toLowerCase().replace(/[^a-z0-9]+/g, "-");
console.log(cleaned); // "top-10-products-2024-"
NOTE: The pattern /[^a-z0-9-]/g means “any character that is NOT a lowercase letter, digit, or hyphen, anywhere in the string (global).” The ^ inside square brackets negates the character class. The g flag after the closing slash makes replace() replace all matches, not just the first. Bookmark regex101.com — it is the standard tool for building and understanding regex patterns.

Numbers

Introduction

JavaScript has a single number type: 64-bit floating-point (IEEE 754). This covers integers, decimals, and very large or very small values. For most purposes it works transparently — but it has one famous quirk you must know about, and a set of formatting tools you should always use for display.

Type Checking

Three static methods on the Number object let you check what kind of number you actually have. Prefer these over the older global functions — the global versions have surprising edge cases.

  • Number.isInteger(value) — true for whole numbers like 42 and -7, but not 42.0 (which is just 42 in JS) or 42.5
  • Number.isFinite(value) — true for normal numbers; false for Infinity, -Infinity, and NaN; safer than the global isFinite()
  • Number.isNaN(value) — true only for the value NaN; unlike the global isNaN(), it does not coerce its argument first
LISTING 26.14 — Number Type Checking 
console.log(Number.isInteger(42));      // true
console.log(Number.isInteger(42.5));    // false
console.log(Number.isInteger(42.0));    // true (same as 42 in JS)

console.log(Number.isFinite(100));      // true
console.log(Number.isFinite(Infinity)); // false
console.log(Number.isFinite(NaN));      // false

// Why Number.isNaN is safer than the global isNaN()
console.log(isNaN("hello"));        // true  — coerces "hello" to NaN first!
console.log(Number.isNaN("hello")); // false — "hello" is a string, not NaN

// Practical: validate a quantity field
function validateQuantity(input) {
  const qty = Number(input);
  if (Number.isNaN(qty) || !Number.isInteger(qty) || qty < 1) {
    return { valid: false, error: "Quantity must be a positive whole number" };
  }
  return { valid: true, value: qty };
}

console.log(validateQuantity("3"));    // { valid: true, value: 3 }
console.log(validateQuantity("3.5"));  // { valid: false, error: "..." }
console.log(validateQuantity("abc"));  // { valid: false, error: "..." }

Parsing Strings to Numbers

User input always arrives as a string. You need to convert it to a number before doing arithmetic. Three tools handle this:

  • Number.parseInt("42px") — reads digits until it hits a non-numeric character; returns 42; ignores the "px"
  • Number.parseFloat("3.14em") — same idea, but handles decimal points; returns 3.14
  • Number("42") — strict conversion; returns NaN for anything that isn’t a valid number string
LISTING 26.15 — Parsing Strings to Numbers 
// parseInt — good for CSS values, data attributes
console.log(Number.parseInt("42px"));      // 42
console.log(Number.parseInt("1.5rem"));    // 1
console.log(Number.parseInt("$99"));       // NaN — doesn't start with a digit

// parseFloat — good for CSS values with decimals
console.log(Number.parseFloat("3.14em")); // 3.14
console.log(Number.parseFloat("1.5rem")); // 1.5

// Number() — strict, best for validated form inputs
console.log(Number("42"));       // 42
console.log(Number("3.14"));     // 3.14
console.log(Number("42px"));     // NaN — strict, no trailing garbage
console.log(Number(""));         // 0  — empty string becomes 0

// Practical: read a duration from a form
const durationInput = "30"; // from <input type="number">
const duration = Number(durationInput);

if (Number.isNaN(duration)) {
  console.log("Invalid duration");
} else {
  console.log(`Running for ${duration} seconds`);
}

As a rule: use Number.parseInt() or Number.parseFloat() when reading values from CSS or DOM attributes that include units. Use Number() for form inputs you have already validated.

Formatting Numbers

.toFixed(digits) formats a number to a fixed number of decimal places and returns a string. It is often the first method developers reach for when displaying prices — but it has a caveat.

LISTING 26.16 — toFixed() 
const price = 19.9;
console.log(price.toFixed(2));  // "19.90"  — returns a STRING

const pi = 3.14159;
console.log(pi.toFixed(2));     // "3.14"

// toFixed uses banker's rounding in some environments —
// don't rely on it for critical financial rounding
console.log((1.005).toFixed(2)); // "1.00" in many engines, not "1.01"

The Floating-Point Problem

Here is the most famous quirk in JavaScript — and in almost every programming language that uses IEEE 754 floating-point arithmetic:

LISTING 26.18 — The Floating-Point Surprise 
console.log(0.1 + 0.2);           // 0.30000000000000004
console.log(0.1 + 0.2 === 0.3);   // false

[IMG - Floating point 0.1 plus 0.2 console result]

This is not a JavaScript bug — it is a consequence of how binary floating-point works. Computers represent numbers in binary (base 2). Just as the fraction 1/3 cannot be written exactly in decimal (it is 0.333333… forever), the fractions 0.1 and 0.2 cannot be written exactly in binary. The computer stores the closest representable approximation, and when you add two approximations, the tiny errors accumulate.

There are two reliable approaches:

  • Integer arithmetic for calculations: Store prices in cents (1999 instead of 19.99). Do all arithmetic in integers. Convert to dollars only at the moment of display. 1999 + 500 === 2499 is perfectly accurate.
  • Math.round for display rounding: Math.round((0.1 + 0.2) * 100) / 100 gives 0.3. Multiply by the precision factor, round to an integer, divide back.
LISTING 26.19 — Working Around Floating-Point Errors 
// Approach 1: round for display
const result = 0.1 + 0.2;
const rounded = Math.round(result * 100) / 100;
console.log(rounded);          // 0.3
console.log(rounded === 0.3);  // true

// Approach 2: integer cents (the professional approach for money)
const priceInCents  = 1999;  // $19.99
const taxInCents    = 160;   // $1.60
const totalInCents  = priceInCents + taxInCents; // 2159 — exact
const totalDisplay  = (totalInCents / 100).toFixed(2);
console.log(totalDisplay); // "21.59"

// Approach 3: let Intl.NumberFormat handle display (recommended)
const usd = new Intl.NumberFormat("en-US", { style: "currency", currency: "USD" });
console.log(usd.format(0.1 + 0.2)); // "$0.30" — correct for display

Intl.NumberFormat — Locale-Aware Number Formatting

Intl.NumberFormat is the correct, professional tool for formatting numbers for display. It handles currency symbols, thousand separators, decimal separators, and locale conventions automatically. Never hardcode "$" before a number in production code — use Intl.NumberFormat.

The API: create a formatter object with a locale string and an options object, then call .format(value).

LISTING 26.20 — Intl.NumberFormat for Currency 
const price = 1299.99;

// US Dollars
const usd = new Intl.NumberFormat("en-US", { style: "currency", currency: "USD" });
console.log(usd.format(price)); // "$1,299.99"

// Euros (German locale — comma as decimal separator)
const eur = new Intl.NumberFormat("de-DE", { style: "currency", currency: "EUR" });
console.log(eur.format(price)); // "1.299,99 €"

// British Pounds
const gbp = new Intl.NumberFormat("en-GB", { style: "currency", currency: "GBP" });
console.log(gbp.format(price)); // "£1,299.99"

// Japanese Yen (no decimal places — yen are whole units)
const jpy = new Intl.NumberFormat("ja-JP", { style: "currency", currency: "JPY" });
console.log(jpy.format(price)); // "￥1,300"
LISTING 26.21 — Intl.NumberFormat for Percentages and Compact Notation 
// Percentage
const pct = new Intl.NumberFormat("en-US", { style: "percent" });
console.log(pct.format(0.15));   // "15%"
console.log(pct.format(0.0825)); // "8%"

// Percentage with decimal places
const pctDetailed = new Intl.NumberFormat("en-US", {
  style: "percent",
  minimumFractionDigits: 1,
  maximumFractionDigits: 1
});
console.log(pctDetailed.format(0.0825)); // "8.3%"

// Compact notation for large numbers (follower counts, sales figures)
const compact = new Intl.NumberFormat("en-US", { notation: "compact" });
console.log(compact.format(1500));     // "1.5K"
console.log(compact.format(1500000)); // "1.5M"
console.log(compact.format(2300000000)); // "2.3B"

// Reusable formatter function
function formatPrice(amount, currency = "USD", locale = "en-US") {
  return new Intl.NumberFormat(locale, { style: "currency", currency }).format(amount);
}

const products = [
  { name: "Wireless Headphones", price: 79.99 },
  { name: "USB-C Hub",           price: 34.99 },
  { name: "Notebook",            price: 12.99 },
];

products.forEach(p => {
  console.log(`${p.name}: ${formatPrice(p.price)}`);
});
// Wireless Headphones: $79.99
// USB-C Hub: $34.99
// Notebook: $12.99
NOTE: Never use toFixed() for currency display in production. toFixed() does not add thousand separators, does not handle locale differences, and has edge-case rounding bugs. Intl.NumberFormat handles all of this correctly and adapts to the user’s locale automatically.

Intl.DateTimeFormat — A Brief Introduction

The Intl object also provides Intl.DateTimeFormat for formatting Date objects in locale-appropriate ways. Like NumberFormat, you create a formatter with locale and options, then call .format(date).

LISTING 26.22 — Intl.DateTimeFormat 
const now = new Date();

// Default: short date for the locale
const dateFormatter = new Intl.DateTimeFormat("en-US");
console.log(dateFormatter.format(now)); // e.g. "2/27/2026"

// Long date + short time
const fullFormatter = new Intl.DateTimeFormat("en-US", {
  dateStyle: "long",
  timeStyle: "short"
});
console.log(fullFormatter.format(now)); // e.g. "February 27, 2026 at 10:45 AM"

// Different locale
const deFormatter = new Intl.DateTimeFormat("de-DE", {
  dateStyle: "long"
});
console.log(deFormatter.format(now)); // e.g. "27. Februar 2026"

Date objects and the full Intl.DateTimeFormat API have many more options — weekday names, time zones, hour cycles, and more. Treat this as an entry point: you now know Intl.DateTimeFormat exists and how to reach for it when you need a formatted date string.

Timers

Introduction

JavaScript’s timer functions let you schedule code to run in the future — either once after a delay, or repeatedly at an interval. Timers are behind live clocks, countdown timers, toast notifications that dismiss themselves, autosave functionality, and the debounce pattern that prevents performance problems in search inputs. Every one of these connects to something visible on screen.

setTimeout: Run Code Once After a Delay

setTimeout(callback, delayMs) calls callback once after delayMs milliseconds. It returns a numeric ID that you can use to cancel the timer before it fires.

LISTING 26.23 — setTimeout 
// Show a message after 2 seconds
const timerId = setTimeout(() => {
  const toast = document.querySelector("#toast");
  toast.textContent = "Order saved!";
  toast.classList.add("visible");
}, 2000);

// Cancel before it fires (e.g., user navigates away)
function cancelToast() {
  clearTimeout(timerId);
}

// The delay is a minimum, not a guarantee
// If the browser is busy, the callback may fire slightly later than requested
console.log("This runs BEFORE the toast, even though it appears first in the file");
setTimeout(() => console.log("Runs after 0ms delay"), 0);
console.log("This also runs before the 0ms timeout");
NOTE: Even setTimeout(fn, 0) does not run immediately. The callback is placed in a queue and runs after the current synchronous code finishes. This is part of JavaScript’s event loop — timers never interrupt code that is already running.

setInterval: Run Code Repeatedly

setInterval(callback, intervalMs) calls callback every intervalMs milliseconds until you stop it. Like setTimeout, it returns an ID for cancellation.

[IMG - setTimeout versus setInterval timing diagram]

LISTING 26.24 — setInterval 
// Update a "time since last save" label every 30 seconds
const saveLabel = document.querySelector("#save-label");
let secondsSinceSave = 0;

const saveInterval = setInterval(() => {
  secondsSinceSave += 30;
  const minutes = Math.floor(secondsSinceSave / 60);
  saveLabel.textContent = minutes > 0
    ? `Last saved ${minutes}m ago`
    : `Last saved ${secondsSinceSave}s ago`;
}, 30000);

// Always store the ID so you can stop it
function stopTracking() {
  clearInterval(saveInterval);
}
CAUTION: Always store the return value of setInterval and call clearInterval when the interval is no longer needed — for example, when a component unmounts, when a user navigates away, or when the countdown reaches zero. A runaway setInterval with no way to stop it is a memory leak and will keep firing for the entire lifetime of the page.

Live Clock Example

A live clock is the canonical setInterval example. The pattern: call the update function once immediately so the clock shows the right time right away, then set up the interval to call it every second.

LISTING 26.25 — Live Clock with setInterval 
const clockEl = document.querySelector("#clock");

function updateClock() {
  const now = new Date();
  clockEl.textContent = now.toLocaleTimeString("en-US", {
    hour: "2-digit",
    minute: "2-digit",
    second: "2-digit",
  });
}

updateClock();                         // show immediately — no initial 1s blank
const clockInterval = setInterval(updateClock, 1000);

// To stop the clock:
// clearInterval(clockInterval);

Notice that updateClock() is called once before the interval starts. Without this, the clock would show a blank or stale value for the first second.

Countdown Timer Example

A countdown timer uses setInterval to decrement a counter every second, updates the display using padStart for consistent MM:SS formatting, and calls clearInterval when the count reaches zero.

LISTING 26.26 — Countdown Timer 
const displayEl  = document.querySelector("#countdown-display");
const startBtn   = document.querySelector("#start-btn");
const DURATION   = 60; // seconds

let remaining    = DURATION;
let intervalId   = null;

function formatTime(seconds) {
  const m = Math.floor(seconds / 60);
  const s = seconds % 60;
  return `${String(m).padStart(2, "0")}:${String(s).padStart(2, "0")}`;
}

function tick() {
  remaining -= 1;
  displayEl.textContent = formatTime(remaining);

  if (remaining <= 0) {
    clearInterval(intervalId);
    intervalId = null;
    displayEl.textContent = "Time's up!";
    displayEl.style.color = "red";
  }
}

startBtn.addEventListener("click", () => {
  if (intervalId) return;  // already running
  remaining   = DURATION;
  displayEl.textContent = formatTime(remaining);
  intervalId  = setInterval(tick, 1000);
});

The guard if (intervalId) return; prevents multiple overlapping intervals from starting if the button is clicked more than once. Always think about this edge case when writing interval-based code.

The Debounce Pattern

Events like input, resize, and scroll fire tens or hundreds of times per second as the user interacts. If your handler does something expensive — a network request, a complex DOM update, a heavy calculation — running it on every single event will cause the page to lag or hammer a server with unnecessary requests.

The debounce pattern solves this: instead of running immediately on every event, you wait until the user pauses. Each new event resets the timer. The handler only runs once the user stops triggering events for a specified duration.

LISTING 26.27 — A Simple Debounce Function 
function debounce(fn, delayMs) {
  let timeoutId = null;

  return function(...args) {
    // Cancel the previous timer if it exists
    clearTimeout(timeoutId);

    // Start a new timer; if no new call arrives before it fires, run fn
    timeoutId = setTimeout(() => {
      fn(...args);
      timeoutId = null;
    }, delayMs);
  };
}
LISTING 26.28 — Using debounce on a Search Input 
const searchInput = document.querySelector("#search");
const resultsEl   = document.querySelector("#results");

// The expensive function we want to limit
function fetchSearchResults(query) {
  console.log(`Fetching results for: "${query}"`);
  // In a real app: fetch(`/api/search?q=${encodeURIComponent(query)}`)
  //   .then(res => res.json())
  //   .then(data => renderResults(data));
  resultsEl.textContent = `Searching for "${query}"...`;
}

// Wrap it: only fires 300ms after the user stops typing
const debouncedSearch = debounce(fetchSearchResults, 300);

searchInput.addEventListener("input", (e) => {
  debouncedSearch(e.target.value);
});

// Without debounce: if user types "headphones" quickly (10 chars),
// fetchSearchResults fires 10 times.
// With debounce: it fires exactly once, when the user pauses.

The debounce function is one of the most commonly reused utility functions in JavaScript. You will encounter it in lodash, in React hooks, and in almost every production frontend codebase. Understanding how it works — a closure over a timeoutId that is reset on every call — makes you a significantly stronger JavaScript developer.

Summary

  • Strings are immutable: every string method returns a new string and leaves the original unchanged. This makes method chaining safe and natural.
  • Key string methods: toUpperCase(), toLowerCase(), trim(), includes(), startsWith(), endsWith(), indexOf(), slice(), replace(), replaceAll(), padStart(), padEnd(), repeat(), and split().
  • Method chaining — as in the URL slug generator — composes multiple transformations in a single readable expression.
  • Regular expressions describe text patterns; use /pattern/.test(str) for validation and str.match(/pattern/g) to extract all occurrences. regex101.com is your interactive reference.
  • JavaScript’s floating-point arithmetic means 0.1 + 0.2 !== 0.3. For money, use integer cents for calculations and format only at the display layer.
  • Intl.NumberFormat is the correct tool for formatting currency, percentages, and compact numbers in a locale-aware way. toFixed() is fine for quick rounding but not for production currency display.
  • setTimeout(fn, ms) runs a callback once after a delay and returns a cancellable ID. setInterval(fn, ms) runs a callback repeatedly at a fixed interval and must be stopped with clearInterval(id).
  • The debounce pattern uses setTimeout and clearTimeout to delay execution until the user pauses, preventing expensive handlers from firing on every keystroke or scroll event.

Q&A

What does it mean that strings are immutable?
It means that once a string is created, its characters cannot be changed in place. Every string method — toUpperCase(), trim(), replace(), and so on — returns a brand-new string with the transformation applied. The original string is untouched. This is why you must assign the result: const cleaned = raw.trim(). If you just call raw.trim() without assigning the result, the trimmed version is discarded.
What is the difference between slice() and substring()?
Both extract a portion of a string from a start index up to (but not including) an end index. The key difference is that slice() supports negative indices — str.slice(-3) gives you the last three characters — while substring() does not. Additionally, if you pass substring() arguments where start is greater than end, it silently swaps them. Prefer slice() in new code; it is more predictable and more flexible.
When should I use replace() vs. replaceAll()?
Use replace() when you only want to change the first occurrence of a pattern. Use replaceAll() when you want to change every occurrence. For a string search argument, replace() only replaces the first match and replaceAll() replaces all of them. When using a regular expression with replaceAll(), the regex must include the g (global) flag, or it will throw an error.
How do I format a number as currency?
Use Intl.NumberFormat: new Intl.NumberFormat('en-US', { style: 'currency', currency: 'USD' }).format(price). This handles the currency symbol, thousand separators, decimal separators, and locale conventions automatically. Do not use toFixed(2) and prepend a "$" — that approach breaks for non-US locales and has edge-case rounding bugs.
Why does 0.1 + 0.2 not equal 0.3?
JavaScript (like most programming languages) stores numbers in 64-bit binary floating-point (IEEE 754). The fractions 0.1 and 0.2 cannot be represented exactly in binary — the computer stores the closest approximation, similar to how 1/3 cannot be written exactly in decimal. When you add two approximations, the tiny errors accumulate, producing 0.30000000000000004 instead of 0.3. The fix for display is Intl.NumberFormat. The fix for calculations with money is to work in integer cents throughout and convert to dollars only at the display step.
What is the difference between setTimeout and setInterval?
setTimeout(fn, ms) runs fn exactly once, after ms milliseconds. setInterval(fn, ms) runs fn repeatedly, every ms milliseconds, until you stop it with clearInterval(). Both return a numeric ID you can pass to clearTimeout() or clearInterval() to cancel. Use setTimeout for one-time delayed actions like toast notifications. Use setInterval for repeating actions like a live clock or autosave.
How do I stop a setInterval from running?
Store the return value of setInterval in a variable, then pass that variable to clearInterval(). For example: const id = setInterval(fn, 1000); — later: clearInterval(id);. If you do not store the ID, there is no way to cancel the interval and it will run indefinitely for the life of the page. Always store the ID.
What is debouncing and when should I use it?
Debouncing is a technique that delays the execution of a function until a user has stopped triggering it for a specified amount of time. Each new trigger resets the delay timer. Use it when an event fires very rapidly — keyboard input, window resize, scroll — and your handler is expensive: a network fetch, a DOM-heavy re-render, or a complex calculation. A typical search input is debounced at 200–400ms so that the API call fires only when the user pauses, not on every individual keystroke.
