async and defer Attributes

Script Loading Attributes

The async and defer attributes in HTML script elements play a crucial role in how JavaScript files are loaded and executed in web pages.

Why They Matter

Understanding these attributes can improve your website's performance and user experience.

What You'll Learn

  • How normal script loading works (blocking)
  • How the async attribute works
  • How the defer attribute works
  • When to use each one

Normal Script Loading

Default Behavior

When you include a script in your HTML file, it looks like this:

HTML Code

<script src="example.js"></script>

What Happens?

When the browser finds this script tag:

  1. It stops parsing the HTML
  2. It downloads the script
  3. It executes the script
  4. Then it continues parsing the HTML

The Problem with Normal Loading

⚠️ Blocking Behavior

This blocking behavior can slow down the loading of your web page, especially if you have large scripts.

Visual Flow

1. Browser parsing HTML...
2. ❌ STOPS at <script> tag
3. Downloads script (network delay)
4. Executes script
5. ✓ Resumes parsing HTML

The page appears to freeze during script download and execution.

The Solution: async and defer

Better Script Loading

The async and defer attributes provide ways to load scripts more efficiently.

Two Attributes, Two Approaches

async

Downloads in parallel, executes immediately when ready

defer

Downloads in parallel, executes after HTML parsing completes

Key Benefit

Both allow the browser to continue parsing HTML while scripts download in the background.

The async Attribute

Asynchronous Script Loading

The async attribute makes scripts download and execute asynchronously.

Syntax

<script src="example.js" async></script>

What Happens?

By adding the async attribute to a script tag:

  • The browser continues parsing HTML while the script downloads
  • Once downloaded, the browser pauses HTML parsing
  • Executes the script
  • Resumes HTML parsing

async Loading Flow

Step-by-Step Process

1. Browser parsing HTML...
2. ✓ Encounters <script async> - starts download
3. ✓ Continues parsing HTML (non-blocking!)
4. Script finishes downloading
5. ❌ PAUSES HTML parsing
6. Executes script
7. ✓ Resumes HTML parsing

async Performance Benefit

Speed Improvement

Using async can significantly speed up page loading because:

Key Advantages

  • Parallel Download: Script downloads while HTML continues parsing
  • Non-Blocking: User sees content appear faster
  • Faster Perceived Load: Page becomes interactive sooner

Example

Instead of waiting 3 seconds for a script to download before showing content, the content appears immediately while the script downloads in the background.

Important: async Execution Order

⚠️ Order Not Guaranteed

It's important to note that async scripts are executed as soon as they're downloaded, which means they might not run in the correct order.

Example Problem

<script src="library.js" async></script>
<script src="app.js" async></script> <!-- Needs library.js -->

If app.js downloads faster than library.js, it will execute first and might cause errors!

The defer Attribute

Deferred Script Execution

The defer attribute provides a solution when script order matters.

Syntax

<script src="example.js" defer></script>

Key Difference

The defer attribute is similar to async. However, defer scripts are NOT executed immediately after they're downloaded.

How defer Works

Delayed Execution

defer scripts wait until HTML parsing is complete before executing.

Two Important Behaviors

1. Waits for HTML Parsing

Script executes after the entire HTML document has been parsed

2. Maintains Order

defer scripts execute in the order they appear in the HTML code

This makes defer perfect for scripts that depend on other scripts or need to interact with the DOM.

defer Loading Flow

Step-by-Step Process

1. Browser parsing HTML...
2. ✓ Encounters <script defer> - starts download
3. ✓ Continues parsing HTML (non-blocking!)
4. Script finishes downloading (but doesn't execute yet)
5. ✓ Continues parsing HTML
6. HTML parsing complete
7. Executes all defer scripts in order

async vs defer Comparison

Normal

  • Blocks HTML parsing
  • Downloads synchronously
  • Executes immediately
  • Order guaranteed

async

  • ✓ Non-blocking download
  • ❌ Blocks on execution
  • Executes ASAP
  • Order NOT guaranteed

defer

  • ✓ Non-blocking download
  • ✓ Non-blocking execution
  • Executes after parsing
  • ✓ Order guaranteed

When to Use Each Attribute

Choosing the Right Attribute

Use async When:

  • Script is independent (doesn't depend on other scripts)
  • Order of execution doesn't matter
  • Script doesn't need the DOM to be fully loaded
  • Examples: Analytics, ads, tracking scripts

Use defer When:

  • Script depends on other scripts
  • You need scripts to run in a specific order
  • Script needs to access the full DOM
  • Examples: Application code, libraries with dependencies

Real-World Example

Typical Website Setup

<!-- Analytics - independent, async -->
<script src="analytics.js" async></script>

<!-- jQuery library - needed by app, defer -->
<script src="jquery.js" defer></script>

<!-- App code - needs jQuery, defer -->
<script src="app.js" defer></script>

Why This Works

  • Analytics runs independently whenever it's ready
  • jQuery and app.js download in parallel
  • Both execute after HTML parsing, in order
  • app.js can safely use jQuery

Summary

Key Takeaways

Normal Script Loading

Blocks HTML parsing while downloading and executing - slowest option

async Attribute

  • Downloads in parallel with HTML parsing
  • Executes as soon as downloaded
  • Execution order not guaranteed
  • Use for independent scripts

defer Attribute

  • Downloads in parallel with HTML parsing
  • Executes after HTML parsing completes
  • Execution order guaranteed
  • Use when order matters

Performance Boost

Both attributes can significantly improve page load times by allowing the browser to continue parsing HTML while scripts download in the background.
