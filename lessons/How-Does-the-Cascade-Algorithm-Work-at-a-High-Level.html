The Cascade Algorithm

The Cascade Algorithm is the process the browser uses to decide which CSS rules to apply when there are multiple styles targeting the same element.

Purpose

It ensures that the most appropriate styles are used, based on a set of well-defined rules. This makes CSS behavior predictable and allows for complex, nuanced web pages.

The Cascade Flow

Step 1: Relevance

The process begins with relevance. The browser first filters all the CSS rules to find those that actually apply to the element in question.

What Gets Filtered

  • Matching selectors
  • Media queries that are in effect
  • Rules that target the specific element

ℹ️ Media Queries

A media query is a CSS technique used to apply styles based on the characteristics of the device or viewport, such as its width, height, or orientation.

Step 2: Origin and Importance

Next, the algorithm considers origin and importance. CSS can come from different sources.

Three Origins of CSS

  • User-Agent: Browser's default styles
  • User: Styles set by the user
  • Author: Styles written by you (the developer)

Importance

Rules marked with !important override other rules regardless of their source.

Step 3: Specificity

After filtering by origin and importance, the algorithm looks at specificity.

What Is Specificity?

Specificity is a measure of how targeted a selector is, with more specific selectors taking precedence over more general ones.

  • When two rules from the same origin and importance level apply
  • The rule with the higher specificity will be applied
  • More specific = more targeted selector

Step 4: Order of Appearance

Finally, if everything else is equal, the order of appearance comes into play.

Last Rule Wins

When two rules have the same specificity, the one that appears last in the CSS will be applied.

  • This is why the order you write styles matters
  • Later rules override earlier ones
  • Only applies when all other factors are equal

Example: Order of Appearance

Let's see how order of appearance works in practice.

The Setup

Two rules target the same element with the same specificity.

HTML
<p>example paragraph</p>
CSS
p {
  color: blue;
}

p {
  color: green;
}

Which Rule Wins?

Both rules have the same specificity (0,0,0,1), so order of appearance decides.

First rule: p { color: blue; } OVERRIDDEN
↓
Second rule: p { color: green; } WINS!
Result

example paragraph

The paragraph is green because the second rule appears last

Key Takeaways

  • The Cascade Algorithm determines which CSS rules to apply
  • It follows a specific order: Relevance → Origin/Importance → Specificity → Order
  • Relevance: Filters rules that match the element
  • Origin & Importance: Considers source (user-agent, user, author) and !important
  • Specificity: More specific selectors take precedence
  • Order: When all else is equal, last rule wins
  • This ensures predictable, consistent CSS behavior
