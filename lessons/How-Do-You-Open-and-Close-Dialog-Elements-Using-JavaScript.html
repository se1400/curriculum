Dialog Elements in JavaScript

What Are Dialogs?

Dialogs let you display important information or actions to users.

HTML's Built-in Dialog Element

With HTML's built-in <dialog> element, you can easily create dialogs in your web apps.

Two Types of Dialogs

Modal Dialogs

Force user interaction before accessing other content

Non-Modal Dialogs

Allow interaction with the rest of the page

In this lesson, you'll learn how to open and close dialogs using the showModal(), show(), and close() methods.

Modal Dialogs

What is a Modal Dialog?

A modal dialog is a type of dialog that forces the user to interact with it before they can access the rest of the application or webpage.

Modal Dialog Behavior

  • Blocks Interaction: Prevents access to other content
  • Adds Backdrop: Dims the rest of the page
  • Requires Action: User must close it or complete an action
  • Focus Trapped: User can't tab to elements outside the modal

Common Use Cases

  • Forms requiring user input
  • Confirmation dialogs
  • Critical information that requires acknowledgment
  • Actions that need user attention

Non-Modal Dialogs

What is a Non-Modal Dialog?

A non-modal dialog allows the user to continue interacting with other parts of the page or application even when the dialog is open.

Non-Modal Dialog Behavior

  • No Blocking: Doesn't prevent access to other content
  • No Backdrop: Doesn't dim the rest of the page
  • Optional Interaction: User can ignore it temporarily
  • Flexible Focus: User can tab to other elements

Common Use Cases

  • Tooltips or helper information
  • Notifications that don't require immediate action
  • Settings panels
  • Additional context while working

Modal vs Non-Modal

Modal Dialog

  • ✓ Blocks page interaction
  • ✓ Adds backdrop
  • ✓ Forces user focus
  • ✓ Requires action to dismiss

Non-Modal Dialog

  • ✓ Allows page interaction
  • ✓ No backdrop
  • ✓ Doesn't force focus
  • ✓ User can work around it

Choosing Between Them

Use Modal: When the action is critical and requires immediate attention

Use Non-Modal: When users should be able to reference the dialog while working with other content

Basic Dialog HTML Structure

The <dialog> Element

HTML provides a built-in <dialog> element for creating dialogs.

HTML Example

<dialog id="my-modal">
  <p>This is a modal dialog.</p>
</dialog>

Initial State

By default, the dialog is closed and hidden when the page loads. You won't see anything on the page until you open it with JavaScript.

You need JavaScript to control when the dialog appears and disappears.

showModal() Method

Opening a Modal Dialog

Use the showModal() method to open a modal dialog that blocks interaction with the rest of the page.

What showModal() Does

  • Opens the dialog and makes it visible
  • Adds a backdrop that dims other content
  • Disables other page elements from interaction
  • Traps keyboard focus within the dialog

Perfect For

Modals that display forms, confirmations, and critical information that requires user action.

Auto-Opening a Modal

Open on Page Load

You can automatically open a modal when the page loads.

HTML

<dialog id="modal">
  <p>This is a modal dialog.</p>
</dialog>

JavaScript

const modal = document.getElementById("modal");
modal.showModal(); // Opens immediately

Result

The browser will show a modal with the text "This is a modal dialog" as soon as the page loads.

User-Controlled Modal Opening

Best Practice

It's better to give control to the user by letting them open the modal with a button click.

HTML

<dialog id="modal">
  <p>This is a modal dialog.</p>
</dialog>
<button id="open-modal-btn">Open Modal Dialog</button>

JavaScript

const modal = document.getElementById("modal");
const openBtn = document.getElementById("open-modal-btn");

openBtn.addEventListener("click", () => {
  modal.showModal();
});

show() Method

Opening a Non-Modal Dialog

If you need to show a dialog while still allowing interaction with content outside of the dialog, use the show() method.

What show() Does

  • Opens the dialog and makes it visible
  • No backdrop is added
  • Page elements remain interactive
  • Focus is not trapped in the dialog

Example

const modal = document.getElementById("modal");
const openBtn = document.getElementById("open-modal-btn");

openBtn.addEventListener("click", () => {
  modal.show(); // Non-modal dialog
});

show() vs showModal()

showModal()

modal.showModal();
  • Creates modal dialog
  • Adds backdrop
  • Blocks page interaction
  • Traps keyboard focus

show()

modal.show();
  • Creates non-modal dialog
  • No backdrop
  • Allows page interaction
  • No focus trap

Choose Based on Need

Use showModal() when you need user attention. Use show() for supplementary information.

close() Method

Closing a Dialog

To close a dialog (modal or non-modal), use the close() method.

How to Close

Add a button inside the dialog element and attach a click event that calls close().

Works for Both Types

The close() method works for both modal and non-modal dialogs.

What Happens

  • The dialog becomes hidden
  • The backdrop is removed (if it was a modal)
  • Page elements become interactive again

Complete Example: HTML

Full Dialog with Open and Close

Let's create a complete example with both open and close functionality.

HTML Structure

<dialog id="modal">
  <p>This is a modal dialog.</p>
  <button id="close-modal-btn">Close Modal</button>
</dialog>

<button id="open-modal-btn">Open Modal Dialog</button>

Structure Breakdown

  • Dialog element with content and a close button inside
  • Open button outside the dialog

Complete Example: JavaScript

JavaScript Code

const modal = document.getElementById("modal");
const openBtn = document.getElementById("open-modal-btn");
const closeBtn = document.getElementById("close-modal-btn");

// Open the modal
openBtn.addEventListener("click", () => {
  modal.showModal();
});

// Close the modal
closeBtn.addEventListener("click", () => {
  modal.close();
});

How It Works

  • User clicks "Open Modal Dialog" button
  • Modal opens with backdrop
  • User clicks "Close Modal" button inside the dialog
  • Modal closes and backdrop disappears

Dialog Interaction Flow

Step-by-Step Process

1. Initial State

Dialog is closed and hidden when page loads

2. User Clicks Open Button

Event listener triggers showModal()

Dialog appears with backdrop

Rest of page becomes non-interactive

3. User Interacts with Dialog

User can read content, fill forms, etc.

Cannot interact with elements outside the dialog

4. User Clicks Close Button

Event listener triggers close()

Dialog disappears

Page becomes fully interactive again

Summary

Key Concepts

Dialog Types

  • Modal: Forces interaction, blocks page
  • Non-Modal: Allows interaction, doesn't block page

Opening Methods

  • showModal(): Opens as modal dialog with backdrop
  • show(): Opens as non-modal dialog without backdrop

Closing Method

  • close(): Closes any dialog (modal or non-modal)

Best Practices

  • Give users control with button clicks
  • Always provide a way to close the dialog
  • Use modal dialogs for critical actions
  • Use non-modal dialogs for supplementary information
