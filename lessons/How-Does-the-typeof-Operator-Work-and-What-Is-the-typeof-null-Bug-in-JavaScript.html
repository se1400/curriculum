The typeof Operator

typeof

The typeof operator in JavaScript is a simple yet powerful tool that lets you see the data type of a variable or value.

What It Returns

It always returns a string indicating the type.

Basic Syntax

typeof value

The operator tells you what type of data you're working with.

typeof with Numbers

Let's see how typeof works with a number:

let num = 42;
console.log(typeof num); // "number"

Breaking It Down

  1. We created a variable called num
  2. Assigned it the number 42
  3. When you use the typeof operator on num
  4. It returns the string "number"
Output: "number"

typeof with Booleans

Here's another example using the typeof operator on a boolean variable:

let isUserLoggedIn = true;
console.log(typeof isUserLoggedIn); // "boolean"

What Happens

When you use the typeof operator on the isUserLoggedIn variable, it returns the string "boolean" because the boolean true was assigned to the variable.

Output: "boolean"

Common typeof Results

Here's what typeof returns for different data types:

Value typeof Result
42 "number"
"Hello" "string"
true "boolean"
undefined "undefined"
{name: "Alice"} "object"
function() {} "function"

The typeof null Bug

However, there's a well-known quirk in JavaScript when it comes to null.

Let's take a look at an example:

let exampleVariable = null;
console.log(typeof exampleVariable); // "object"

üêõ The Bug

We have a variable assigned null, but when we use the typeof operator, it returns the string "object"!

Output: "object" ‚ùå (Expected: "null")

Expected: "null"

Actual: "object"

Why Does This Bug Exist?

‚ö†Ô∏è Historical Bug

This is widely considered a bug in JavaScript, dating back to its early days.

The Reason

The reason for this behavior is rooted in the way JavaScript was originally designed.

When the language was first implemented, values like null were represented as a special type of object, leading to this unexpected result.

The Reality

Unfortunately, this has become a part of the language, and while it's confusing, it's something you'll need to be aware of.

It cannot be fixed now without breaking existing code across the entire internet!

Working with null

Since typeof null returns "object", you need to check for null differently:

‚ùå This Won't Work

if (typeof value === "null") {
  // This will never be true!
}

‚úì Use This Instead

if (value === null) {
  // This works correctly
}

Checking for null or undefined

// Check specifically for null
if (value === null) {
  console.log("Value is null");
}

// Check for both null and undefined
if (value == null) {
  console.log("Value is null or undefined");
}
