Binary Logical Operators

What Are Binary Logical Operators?

Binary logical operators help you evaluate two expressions and return a result based on their truthiness.

The Three Main Operators

&& || ??
  • && - Logical AND
  • || - Logical OR
  • ?? - Nullish Coalescing

üí° Why "Binary"?

They're called "binary" because they work with two operands - a value on the left and a value on the right.

Logical AND (&&)

&&

What It Does

The logical AND operator is represented by a double ampersand (&&).

It checks if both operands are true and returns a result.

How It Returns Values

  • Both truthy: Returns the second value (right side)
  • Either falsy: Returns the falsy value
  • Both falsy: Returns the first falsy value

AND: Both Operands Truthy

If both operands are truthy, it returns the second value (the one on the right).

Example:

const result = true && 'hello';
console.log(result); // hello
hello

‚úì What Happened?

  1. true is truthy ‚úì
  2. 'hello' is truthy ‚úì
  3. Both are truthy, so return the second value
  4. Result: 'hello'

More Examples

console.log(5 && 'text');        // 'text'
console.log('hello' && 42);      // 42
console.log([1,2] && {a: 1});    // {a: 1}

In each case, both sides are truthy, so the second value is returned.

AND: One Operand Falsy

If either operand is falsy, it returns the falsy value.

Example:

const result = 0 && 3;
console.log(result); // 0
0

‚ö†Ô∏è What Happened?

  1. 0 is falsy ‚úó
  2. 3 is truthy ‚úì
  3. Since 0 is falsy, return it
  4. Result: 0

More Examples

console.log(null && 'hello');    // null
console.log('text' && 0);        // 0
console.log('' && true);         // ''
console.log(5 && undefined);     // undefined

In each case, the falsy value is returned.

AND: Both Operands Falsy

If both operands are falsy, it returns the first falsy value.

Example:

const result = false && 0;
console.log(result); // false
false

‚ö†Ô∏è What Happened?

  1. false is falsy ‚úó
  2. 0 is falsy ‚úó
  3. Both are falsy, so return the first one
  4. Result: false

More Examples

console.log(0 && '');            // 0
console.log(null && undefined);  // null
console.log('' && false);        // ''
console.log(NaN && 0);           // NaN

Always returns the first falsy value when both are falsy.

Using AND in Conditionals

The logical AND operator is useful when you want to check multiple conditions and ensure that all are true before proceeding.

Example:

if (2 < 3 && 3 < 4) {
  console.log('The if block runs');
} else {
  console.log('The else block runs');
}
The if block runs

‚úì Why It Runs

  1. Check: 2 < 3 ‚Üí true ‚úì
  2. Check: 3 < 4 ‚Üí true ‚úì
  3. Both conditions are true
  4. The if block executes

Real-World Example

const age = 25;
const hasLicense = true;

if (age >= 18 && hasLicense) {
  console.log("You can drive!");
} else {
  console.log("You cannot drive");
}
// Output: You can drive!

Both conditions must be true for permission.

Logical OR (||)

||

What It Does

The logical OR operator checks if at least one of the operands is truthy.

How It Returns Values

  • First operand truthy: Returns the first value
  • First falsy, second truthy: Returns the second value
  • Both falsy: Returns the second value (last falsy)

Key Point

OR returns the first truthy value it encounters, or the last value if none are truthy.

Logical OR Examples

Example 1: First Operand Truthy

const result = 'This is truthy' || false;
console.log(result); // This is truthy
This is truthy

Since the first operand is truthy, it's returned immediately.

Example 2: First Falsy, Second Truthy

const result = 0 || 'This is truthy';
console.log(result); // This is truthy
This is truthy

First operand is falsy, so check the second. It's truthy, so return it.

Example 3: Both Falsy

const result = 0 || '';
console.log(result); // ''
''

Both are falsy, so return the last value (empty string).

Using OR for Default Values

A common use of the OR operator is to provide default values.

Example:

let userName = '';
const displayName = userName || 'Guest';
console.log(displayName); // Guest
Guest

How It Works

  1. userName is '' (empty string) - falsy
  2. Since first is falsy, check second
  3. 'Guest' is truthy
  4. Return 'Guest'

With Non-Empty Name:

let userName = 'Alice';
const displayName = userName || 'Guest';
console.log(displayName); // Alice
Alice

Since 'Alice' is truthy, it's returned immediately.

Using OR in Conditionals

It's common to use the logical OR operator in if/else statements.

Example:

let userInput;

if (userInput || 'Guest') {
  console.log('A user is present');
} else {
  console.log('No user detected');
}
A user is present

What Happened?

  1. userInput is undefined (no value assigned)
  2. undefined is falsy
  3. Check second operand: 'Guest'
  4. 'Guest' is truthy
  5. Condition evaluates to true
  6. If block runs

Real-World Example

const isWeekend = false;
const isHoliday = true;

if (isWeekend || isHoliday) {
  console.log("No work today!");
}

Only one condition needs to be true.

Nullish Coalescing (??)

??

What Is It?

The nullish coalescing operator, represented by a double question mark (??), is more sophisticated than logical OR.

How It Works

It helps in scenarios where you want to return a value only if the first one is null or undefined.

Example:

const result = null ?? 'default';
console.log(result); // default
default

Since null is a nullish value, the string 'default' is returned.

Nullish Coalescing vs OR

|| (Logical OR)

Returns second value if first is any falsy value

console.log(0 || 'default');
// 'default'

console.log('' || 'default');
// 'default'

console.log(false || 'default');
// 'default'

Treats 0, '', and false as "missing"

?? (Nullish Coalescing)

Returns second value only if first is null or undefined

console.log(0 ?? 'default');
// 0

console.log('' ?? 'default');
// ''

console.log(false ?? 'default');
// false

Preserves 0, '', and false as valid values

üí° Key Difference

?? only treats null and undefined as "missing". Everything else is considered a valid value!

Nullish Coalescing with Settings

The nullish coalescing operator is incredibly useful when dealing with user settings or configurations where 0, false, or '' are valid values.

Example: User Preference Settings

const userSettings = {
  theme: null,
  volume: 0,
  notifications: false,
};

let theme = userSettings.theme ?? 'light';
console.log(theme); // light
light

‚úì Why This Works

Since userSettings.theme is null, the default 'light' is used.

Why Nullish Coalescing Matters

Problem with || for Settings:

const userSettings = {
  theme: 'dark',
  volume: 0,        // User wants volume at 0
  notifications: false,
};

// Using || (WRONG)
let volume = userSettings.volume || 50;
console.log(volume); // 50 (WRONG! User wanted 0)
50

‚ùå || treats 0 as falsy, so it uses the default instead!

Using ?? (CORRECT):

// Using ?? (CORRECT)
let volume = userSettings.volume ?? 50;
console.log(volume); // 0 (Correct! Respects user's choice)
0

‚úì ?? only replaces null/undefined, so 0 is preserved!

When to Use Which Operator

Use && (Logical AND) When:

  • You need all conditions to be true
  • Checking multiple requirements
  • Guard clauses (stop if something is falsy)
if (isLoggedIn && hasPermission) {
  // Both must be true
}

Use || (Logical OR) When:

  • You need at least one condition to be true
  • Providing fallback for any falsy value
  • Default values where empty strings, 0, false should be replaced
const name = inputName || 'Anonymous';
// Replace any falsy value

Use ?? (Nullish Coalescing) When:

  • You only want to replace null or undefined
  • Working with settings where 0, false, '' are valid
  • More precise control over defaults
const volume = settings.volume ?? 50;
// Only replace null/undefined, keep 0

Real-World Use Cases

Example 1: Form Validation with AND

const username = 'john123';
const password = 'secret';
const agreedToTerms = true;

if (username && password && agreedToTerms) {
  console.log("Registration successful!");
} else {
  console.log("Please complete all fields");
}

Example 2: Feature Access with OR

const isPremium = false;
const isTrialActive = true;

if (isPremium || isTrialActive) {
  console.log("Access granted to premium features");
} else {
  console.log("Upgrade to access premium features");
}

Example 3: Configuration with ??

const config = {
  timeout: 0,          // 0 is valid
  retries: undefined,  // Not set
  cache: false         // Disabled is valid
};

const timeout = config.timeout ?? 3000;     // 0
const retries = config.retries ?? 3;        // 3
const cache = config.cache ?? true;         // false

Binary Logical Operators Summary

Key Takeaways

The Three Operators

Operator Name When It Returns Second Value
&& Logical AND Both operands are truthy
|| Logical OR First operand is falsy
?? Nullish Coalescing First is null or undefined only

Logical AND (&&)

  • Both truthy: Returns second value
  • Either falsy: Returns the falsy value
  • Both falsy: Returns first falsy value
  • Use for: Checking multiple conditions (all must be true)

Logical OR (||)

  • First truthy: Returns first value
  • First falsy: Returns second value
  • Both falsy: Returns second (last) value
  • Use for: Default values, at-least-one conditions

Nullish Coalescing (??)

  • Only replaces: null and undefined
  • Preserves: 0, false, '', NaN
  • Use for: Settings where 0/false/'' are valid
  • More precise: Than || for default values

Quick Examples

// AND - All must be truthy
true && 'hello'        // 'hello'
0 && 3                 // 0
false && 0             // false

// OR - First truthy wins
'hello' || false       // 'hello'
0 || 'default'         // 'default'
false || null          // null

// Nullish Coalescing - Only null/undefined
0 ?? 'default'         // 0
null ?? 'default'      // 'default'
false ?? 'default'     // false
undefined ?? 'default' // 'default'

Decision Guide

Scenario Use
All conditions must be true &&
At least one condition must be true ||
Default value for any falsy ||
Default only for null/undefined ??
Settings where 0/false/'' are valid ??

Remember

These operators are essential for conditional logic and provide powerful ways to handle different scenarios in your code!
