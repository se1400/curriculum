What is Presentational HTML?

Presentational HTML focuses on the appearance and style of the content.

In the early days of HTML, developers would use elements like center, big, and font elements.

In modern web development, you shouldn't use these types of elements because of their limitations and negative impact on accessibility and maintainability.

Presentational Elements (Deprecated)
<font color="red">Red text</font>

<center>Centered content</center>

<big>Bigger text</big>

✗ Don't use these anymore!

Deprecated Elements

Many presentational HTML elements are deprecated, which means that they are outdated and no longer recommended.

There are better ways to get the same results using CSS.

However, it is helpful to know that they exist, so you can recognize them in older code and know to avoid them.

What "Deprecated" Means
Deprecated = Outdated

✗ No longer recommended
✗ May be removed in future
✗ Better alternatives exist
✗ Hurts accessibility
✗ Harder to maintain

→ Use CSS instead!

The font Element (Deprecated)

The font element is a deprecated element used to set the font size and color of the text.

The range for the size attribute is from 1 to 7, with 1 being the smallest and 7 being the largest. The default value is 3.

While this element still works, you should not use it because the font size and color should always be set in CSS, not in HTML.

Old Way (Don't Use)
<font size="5" color="blue">
  This text is blue and large.
</font>

✗ Deprecated presentational HTML
Modern Way (Use CSS)
<span class="large-blue">
  This text is blue and large.
</span>

/* CSS */
.large-blue {
  color: blue;
  font-size: 1.5rem;
}

✓ Semantic HTML with CSS styling

center and big Elements (Deprecated)

The center element is used to center content horizontally within its container.

The big element makes the enclosed text one level bigger than its surrounding text.

Remember that positioning and font size should always be set with CSS, so you should not use these elements in modern HTML.

Old Way (Don't Use)
<center>
  This text is centered.
  <p>HTML is awesome.</p>
</center>

<p>
  Normal text.
  <big>Bigger text.</big>
</p>

✗ Deprecated presentational HTML
Modern Way (Use CSS)
<div class="centered">
  This text is centered.
  <p>HTML is awesome.</p>
</div>

<p>
  Normal text.
  <span class="larger">Bigger text.</span>
</p>

/* CSS */
.centered { text-align: center; }
.larger { font-size: 1.2em; }

What is Semantic HTML?

Semantic HTML is now the recommended practice. It describes the content of the elements, so it's much easier to read, understand, and maintain.

Search engines can easily understand your website when you use semantic HTML.

It's also helpful for accessibility purposes, because screen readers need semantic information to describe what's on the web page.

Semantic HTML Benefits
Semantic HTML describes WHAT
content is, not HOW it looks

✓ Easier to read and understand
✓ Easier to maintain
✓ Better for SEO
✓ Better for accessibility
✓ Better for screen readers

Use elements that describe
the meaning of your content!

Examples of Semantic Elements

There are many semantic HTML elements available:

  • header - for defining the header of the document or section
  • nav - for sections with navigation links
  • section - for grouping related information
  • figure - for illustrations and diagrams

There are many different semantic HTML elements. You will definitely find one that fits the needs of your project.

Semantic Structure Example
<header>
  <nav>
    <a href="#">Home</a>
    <a href="#">About</a>
    <a href="#">Products</a>
    <a href="#">Contact</a>
  </nav>
</header>

✓ Clear semantic meaning
More Semantic Elements
<main>      Main content
<article>  Independent content
<aside>    Side content
<footer>   Footer information
<time>     Date/time
<mark>     Highlighted text

Summary: Key Differences

Presentational HTML focuses on appearance - how content looks. These elements are deprecated and should not be used.

Semantic HTML describes content - what the content means. This is the modern, recommended practice.

The semantic elements clearly show their purpose within the HTML structure, making your code more accessible, maintainable, and SEO-friendly.

Presentational (Avoid)
/* Focuses on APPEARANCE */

<font>, <center>, <big>

✗ Deprecated
✗ Hard to maintain
✗ Bad for accessibility
✗ Bad for SEO
Semantic (Use This)
/* Describes CONTENT */

<header>, <nav>, <section>,
<article>, <figure>

✓ Modern standard
✓ Easy to maintain
✓ Great for accessibility
✓ Great for SEO

Use CSS for styling!
