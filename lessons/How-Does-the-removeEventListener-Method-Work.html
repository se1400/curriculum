removeEventListener Method

Building on Previous Knowledge

You learned how to work with "click" and "input" events using addEventListener().

Now you'll learn how to remove event listeners using removeEventListener().

What Does It Do?

The removeEventListener() method is used to remove an event listener that was previously added to an element using addEventListener().

Why Is This Useful?

This is useful when you want to stop listening for a particular event on an element.

Basic Syntax

element.removeEventListener("event", listener);

Two Required Arguments

Just like addEventListener(), the removeEventListener() method takes two arguments:

1. Event Type

The event you want to remove

Example: "click", "input", "mouseover"

2. Listener Function

The listener function that was previously added

Must be the same function reference

Optional Third Argument

Additional Parameter

There is an optional third argument that can be passed to the removeEventListener() method.

Two Possible Values

options

An object that specifies options for the event listener

  • Whether it should be passive
  • Whether it should run once

useCapture

A boolean value

Specifies whether the event should be captured during the event propagation phase

In Practice

Most of the time, you will only need to pass the event and listener arguments to the removeEventListener() method.

Example: Toggle Background Color

Goal

Toggle the background color between grey and blue using an event listener.

HTML Structure

<h1>Event Listener examples</h1>
<button id="btn">Change background color</button>

We have a simple page with a heading and a button.

Step 1: Access Elements

Get References

First, we need to access the button element and the body element.

JavaScript Code

const bodyEl = document.querySelector("body");
const btn = document.getElementById("btn");

What's Happening?

  • bodyEl - Reference to the body element (to change its background)
  • btn - Reference to the button element (to add the click listener)

Step 2: Create Toggle Function

The Toggle Logic

Create a function responsible for toggling between grey and blue colors.

JavaScript Code

let isBgColorGrey = true;

function toggleBgColor() {
  bodyEl.style.backgroundColor = isBgColorGrey ? "blue" : "grey";
  isBgColorGrey = !isBgColorGrey;
}

How It Works

  • isBgColorGrey - A boolean variable to track the current background color
  • If background is grey → change to blue
  • If background is blue → change to grey
  • Toggle the boolean value after each change

Step 3: Add Event Listener

Attach the Function

Add an event listener to the button element to call the toggleBgColor function when clicked.

JavaScript Code

btn.addEventListener("click", toggleBgColor);

Result

Each time the button is clicked, the background color of the body will change between grey and blue.

Important Note

Notice we pass toggleBgColor without parentheses. We're passing the function reference, not calling it.

Adding Remove Functionality

New Requirement

Let's add a paragraph element that will remove the event listener when hovered over.

Updated HTML

<h1>Event Listener examples</h1>
<p id="para">MouseOver this text to remove the event listener</p>
<button id="btn">Change background color</button>

We've added a paragraph with the id "para" that will trigger the removal.

Using removeEventListener()

Remove on Hover

Add an event listener to the paragraph that removes the button's click listener.

JavaScript Code

const para = document.getElementById("para");

para.addEventListener("mouseover", () => {
  btn.removeEventListener("click", toggleBgColor);
});

What Happens?

When the paragraph is hovered over:

  • The event listener for the button click is removed
  • The background color will no longer change when the button is clicked
  • The button becomes non-functional

Complete Example with Remove

Full JavaScript Code

const bodyEl = document.querySelector("body");
const btn = document.getElementById("btn");
const para = document.getElementById("para");

let isBgColorGrey = true;

function toggleBgColor() {
  bodyEl.style.backgroundColor = isBgColorGrey ? "blue" : "grey";
  isBgColorGrey = !isBgColorGrey;
}

btn.addEventListener("click", toggleBgColor);

para.addEventListener("mouseover", () => {
  btn.removeEventListener("click", toggleBgColor);
});

Critical Concept: Function Reference

Why We Use Named Functions

To use removeEventListener(), you must pass the exact same function reference that was used with addEventListener().

✅ This Works

function myFunc() {
  // code
}

el.addEventListener("click", myFunc);
el.removeEventListener("click", myFunc);

❌ This Doesn't Work

el.addEventListener("click", () => {
  // code
});

el.removeEventListener("click", () => {
  // code
});

The anonymous functions are different references, so the listener won't be removed.

Real-World Use Cases

When to Use removeEventListener()

Real-world examples of when you might need to remove event listeners.

Common Scenarios

1. Closing Modals

Remove event listeners when a modal or dialog is closed to prevent memory leaks and unwanted behavior.

2. User Logout

When a user logs out of an application, remove event listeners that were specific to their session.

3. Component Cleanup

In modern frameworks, clean up event listeners when components are destroyed or unmounted.

4. Temporary Features

Remove listeners for features that are only available during certain states of your application.

Summary

Key Concepts

What is removeEventListener()?

A method to remove event listeners that were previously added with addEventListener().

Required Arguments

  • Event type: The event to remove (e.g., "click")
  • Listener function: The exact same function reference used when adding

Important Rules

  • Must use the same function reference
  • Anonymous functions cannot be removed unless stored in a variable
  • Use named functions for listeners you plan to remove

Real-World Applications

Essential for managing memory, preventing bugs, and cleaning up when features are no longer needed.
