Conditional Statements

What Are Conditional Statements?

Conditional statements let you make decisions in your JavaScript code.

They allow your program to flow in a particular way based on certain conditions.

What You'll Learn

  • Truthy and falsy values
  • if statements
  • else clauses
  • else if blocks
  • Ternary operator
  • When to use each

üí° Think of It Like...

Conditional statements are like decision points in a flowchart - your code can take different paths depending on the conditions you set!

Truthy Values

What Are Truthy Values?

Truthy values are any values that result in true when evaluated in a Boolean context like an if statement.

‚úì Examples of Truthy Values

  • "hello" - Non-empty strings
  • 4, -5 - Any number other than 0 and -0
  • [ ] - Arrays (even empty ones)
  • { } - Objects (even empty ones)
  • true - The boolean true

Examples in Code

if ("hello") {
  console.log("This runs!"); // Runs - non-empty string
}

if (42) {
  console.log("This runs too!"); // Runs - non-zero number
}

if ([]) {
  console.log("Arrays are truthy!"); // Runs - arrays are truthy
}

Falsy Values

What Are Falsy Values?

Falsy values are values that evaluate to false in a boolean context.

JavaScript has only a few falsy values, which makes them easy to remember!

‚ö†Ô∏è The Six Falsy Values

  • false - Boolean false
  • 0 - Zero (and -0)
  • "" - Empty string
  • null - Null
  • undefined - Undefined
  • NaN - Not a Number

Examples in Code

if (0) {
  console.log("Won't run"); // Doesn't run - 0 is falsy
}

if ("") {
  console.log("Won't run"); // Doesn't run - empty string is falsy
}

if (null) {
  console.log("Won't run"); // Doesn't run - null is falsy
}

The If Statement

How If Statements Work

An if statement takes a condition and runs a block of code if that condition is truthy.

Basic Syntax:

if (condition) {
  // Code runs if condition is truthy
}

Example with Truthy and Falsy Values:

if (null) {
  console.log("This will not run.");
}

if ("freeCodeCamp") {
  console.log("This will run.");
}
This will run.

Why?

  • null is a falsy value, so the first block doesn't run
  • "freeCodeCamp" is a truthy value (non-empty string), so the second block runs

If Statements with Comparisons

Let's look at how if statements work with comparison operators.

Example: Voting Eligibility

const age = 22;

if (age >= 18) {
  console.log("You're eligible to vote");
}
You're eligible to vote

‚úì What Happened?

  1. Check if age >= 18
  2. Since 22 >= 18 is true
  3. The code block runs
  4. Message is logged to console

Example with False Condition:

const age = 15;

if (age >= 18) {
  console.log("You're eligible to vote");
}
// Nothing happens - condition is false

Since 15 >= 18 is false, the code block doesn't run.

The Else Clause

What Does Else Do?

When a condition is false, you can use an else clause to run alternative code.

Syntax:

if (condition) {
  // Runs if condition is true
} else {
  // Runs if condition is false
}

Example:

const age = 15;

if (age >= 18) {
  console.log("You're eligible to vote");
} else {
  console.log("You're not eligible to vote");
}
You're not eligible to vote

How It Works

  1. Check if age >= 18
  2. Since 15 >= 18 is false
  3. Skip the if block
  4. Run the else block instead

Comparing If vs If-Else

Just If

const score = 75;

if (score >= 80) {
  console.log("Good job!");
}

console.log("Done checking");
Done checking

If condition is false, nothing happens, then continues to next code.

If-Else

const score = 75;

if (score >= 80) {
  console.log("Good job!");
} else {
  console.log("Keep trying!");
}

console.log("Done checking");
Keep trying! Done checking

If condition is false, else block runs instead.

Else If Blocks

Checking Multiple Conditions

If you want to check multiple conditions, you can use an else if block.

This allows your program to choose between more than two paths.

Syntax:

if (condition1) {
  // Runs if condition1 is true
} else if (condition2) {
  // Runs if condition1 is false AND condition2 is true
} else if (condition3) {
  // Runs if condition1 and condition2 are false AND condition3 is true
} else {
  // Runs if all conditions are false
}

Else If in Action

Example: Grade Calculator

const score = 87;

if (score >= 90) {
  console.log('You got an A');
} else if (score >= 80) {
  console.log('You got a B');
} else if (score >= 70) {
  console.log('You got a C');
} else {
  console.log('You failed! You need to study more!');
}
You got a B

How It Evaluates

  1. Check: 87 >= 90? false ‚úó
  2. Check: 87 >= 80? true ‚úì
  3. Run this block: console.log('You got a B')
  4. Skip all remaining blocks

üí° Important

Once a condition is true, that block runs and all other blocks are skipped, even if they would also be true!

Order Matters in Else If!

‚úì Correct Order

const score = 95;

if (score >= 90) {
  console.log('A'); // Runs
} else if (score >= 80) {
  console.log('B');
} else if (score >= 70) {
  console.log('C');
}
A

Checks highest first, gets correct grade

‚úó Wrong Order

const score = 95;

if (score >= 70) {
  console.log('C'); // Runs!
} else if (score >= 80) {
  console.log('B');
} else if (score >= 90) {
  console.log('A');
}
C

Checks lowest first, stops at first true condition!

‚ö†Ô∏è Best Practice

When using ranges, order your conditions from most specific to least specific (highest to lowest, or lowest to highest consistently).

The Ternary Operator

What Is It?

The ternary operator is a compact way to write simple if/else statements.

It has three parts:

  1. A condition
  2. A result if the condition is true
  3. A result if the condition is false

Basic Syntax:

condition ? expressionIfTrue : expressionIfFalse;

Breaking It Down

  • condition - What to check
  • ? - Separator (means "then")
  • expressionIfTrue - What to return if true
  • : - Separator (means "otherwise")
  • expressionIfFalse - What to return if false

Ternary Operator Examples

Example: Weather Temperature

const temperature = 30;
const weather = temperature > 25 ? 'sunny' : 'cool';
console.log(`It's a ${weather} day!`);
It's a sunny day!

How It Works

  1. Check: temperature > 25
  2. Since 30 > 25 is true
  3. Return 'sunny'
  4. Assign to weather

If Temperature Were 20:

const temperature = 20;
const weather = temperature > 25 ? 'sunny' : 'cool';
console.log(`It's a ${weather} day!`);
It's a cool day!

Since 20 > 25 is false, it returns 'cool' instead.

If-Else vs Ternary Comparison

If-Else Statement

const age = 20;
let status;

if (age >= 18) {
  status = 'adult';
} else {
  status = 'minor';
}

console.log(status);

6 lines of code

Ternary Operator

const age = 20;

const status = age >= 18 
  ? 'adult' 
  : 'minor';

console.log(status);

More compact! Same result.

adult

üí° Both Are Valid

Both approaches produce the same result. Choose based on readability and complexity!

When to Use If-Else vs Ternary

‚úì Use Ternary When:

  • Dealing with a single condition
  • Working with single expressions
  • You want compact syntax for simple logic
  • Assigning a value based on a condition
// Good use of ternary
const discount = isMember ? 0.10 : 0;
const message = isOnline ? 'Available' : 'Offline';
const icon = hasError ? '‚ùå' : '‚úì';

‚úì Use If-Else When:

  • Dealing with complex conditions
  • Working with multiple statements
  • Need multiple else if blocks
  • Code would be hard to read as a ternary
// Good use of if-else
if (score >= 90) {
  grade = 'A';
  message = 'Excellent!';
  sendCongrats();
} else if (score >= 80) {
  grade = 'B';
  message = 'Good job!';
}

Avoid Nested Ternaries

‚ö†Ô∏è Don't Do This!

// BAD - Hard to read!
const result = score >= 90 ? 'A' 
             : score >= 80 ? 'B' 
             : score >= 70 ? 'C' 
             : 'F';

This becomes unreadable very quickly!

‚úì Do This Instead!

// GOOD - Clear and readable
let result;

if (score >= 90) {
  result = 'A';
} else if (score >= 80) {
  result = 'B';
} else if (score >= 70) {
  result = 'C';
} else {
  result = 'F';
}

Much easier to understand!

üí° Remember

Things become unreadable if you nest ternaries. Keep it simple!

The Three Types of Conditionals

1. If Statement

if (condition) {
  // Runs if condition is truthy
}

2. If-Else Statement

if (condition) {
  // Runs if condition is true
} else {
  // Runs if condition is false
}

3. If-Else If-Else Statement

if (condition1) {
  // Runs if condition1 is true
} else if (condition2) {
  // Runs if condition1 is false and condition2 is true
} else {
  // Runs if all conditions are false
}

Ternary Operator

condition ? expressionIfTrue : expressionIfFalse;

Compact way to write simple if/else statements.

When to Use What

  • Ternary: Single conditions, simple logic, assigning values
  • If-Else: Complex conditions, multiple statements, better readability
  • Else If: Multiple conditions, choosing between many paths

Important Rules

  • Order matters in else if chains
  • Only ONE block runs (first true condition)
  • Avoid nested ternaries
  • Choose clarity over cleverness

Remember

Conditional statements are the foundation of decision-making in programming. They allow your code to react and adapt to different situations!
