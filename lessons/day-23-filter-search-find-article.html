Lesson 23: Filter, Search, Find

Array Iteration Methods and Live Search

What You’ll Learn in This Lesson:

  • ►forEach() — run a function for each item in an array
  • ►map() — transform each item into something new
  • ►filter() — keep only the items that pass a test
  • ►find() — get the first item that matches a condition
  • ►How to build category filter buttons that update the gallery in real time
  • ►How to build a live search input that filters as you type
  • ►How to chain filter() and map() together
  • ►Event delegation for dynamically generated content

You have a product gallery powered by an array of objects. It renders beautifully, sorts on command, and responds to clicks. Now you will add the two features that turn a static grid into something that feels like a real website: category filtering and live search.

The tools that make both possible are four array methods: forEach(), map(), filter(), and find(). Each one takes a function as an argument and applies it to every item in an array. They are called iteration methods because they iterate — go through each item — rather than requiring you to write a loop manually. The result is code that reads almost like plain English.

forEach — Do Something With Each Item

forEach() runs a function once for every item in an array. It is the functional equivalent of a for...of loop:

const products = ['Headphones', 'Shoes', 'Mug'];

products.forEach(function(product) {
  console.log(product);
});
// Logs: 'Headphones', 'Shoes', 'Mug'

Or with an arrow function:

products.forEach(product => console.log(product));

forEach() does not return anything — it just runs your function for each item. Use it when you want to perform a side effect (like updating the DOM or logging) for each item, not when you want to produce a new array.

A common use you have already seen: updating button states:

// Remove .active from all filter buttons
filterBtns.forEach(btn => btn.classList.remove('active'));

map — Transform Each Item

map() takes every item in an array, runs a function on it, and returns a new array with the transformed results. The original array is unchanged:

const prices = [79.99, 49.99, 12.99];
const formatted = prices.map(price => '$' + price.toFixed(2));
// ['$79.99', '$49.99', '$12.99']

You have been using map() since Lesson 21 for generating HTML:

container.innerHTML = products.map(product => `
  <div class="card">${product.name}</div>
`).join('');

map() transforms each object into a string of HTML. The resulting array of strings gets joined into one big string and assigned to innerHTML. The original products array is untouched.

filter — Keep Only What Passes

filter() tests each item with a function that returns true or false. Items that pass the test are kept. Items that fail are removed. The result is a new, shorter (or empty) array:

const products = [
  { name: 'Headphones', price: 79.99, category: 'Electronics' },
  { name: 'Shoes',      price: 49.99, category: 'Footwear'    },
  { name: 'Mug',        price: 12.99, category: 'Kitchen'     }
];

// Keep only Electronics
const electronics = products.filter(product => product.category === 'Electronics');
// [{ name: 'Headphones', price: 79.99, category: 'Electronics' }]

// Keep only items under $50
const affordable = products.filter(product => product.price < 50);
// Shoes and Mug

The original products array is not modified. filter() always returns a new array.

find — Get the First Match

find() works like filter() but returns the first matching item instead of an array of matches. If nothing matches, it returns undefined:

const product = products.find(p => p.id === 3);
// Returns the single object with id 3, not an array

This is the method you use in the click-to-detail pattern from Lesson 22: the user clicks a card, you read its data-id, and call find() to retrieve the matching object from the array:

container.addEventListener('click', function(event) {
  const card = event.target.closest('.card');
  if (!card) return;

  const id      = Number(card.dataset.id);
  const product = products.find(p => p.id === id);
  if (product) showDetail(product);
});

Category Filter Buttons

Connecting filter buttons to filter() + renderProducts() creates an instant, smooth gallery filter:

let activeCategory = 'all';

filterBtns.forEach(btn => {
  btn.addEventListener('click', function() {
    // Update active button style
    filterBtns.forEach(b => b.classList.remove('active'));
    btn.classList.add('active');

    // Update the active category and re-render
    activeCategory = btn.dataset.category;
    filterAndRender();
  });
});

function filterAndRender() {
  const results = activeCategory === 'all'
    ? products
    : products.filter(p => p.category === activeCategory);

  renderProducts(results);
}

The active category is stored in a variable outside the event listener. When a button is clicked, the variable updates and filterAndRender() is called. The entire rendering logic lives in one place.

Live Search

A live search input filters as the user types, using addEventListener('input') and filter() together:

let searchQuery = '';

searchInput.addEventListener('input', function() {
  searchQuery = searchInput.value.toLowerCase();
  filterAndRender();
});

Inside filterAndRender(), the search is applied as a second filter() condition:

function filterAndRender() {
  const results = products.filter(p => {
    const matchesCategory = activeCategory === 'all' || p.category === activeCategory;
    const matchesSearch   = p.name.toLowerCase().includes(searchQuery);
    return matchesCategory && matchesSearch;
  });

  renderProducts(results);
}

Two conditions, one filter(). Both must be true for a product to appear. Convert both sides to lowercase with .toLowerCase() so the search is case-insensitive. .includes() checks whether the product name contains the search query anywhere, not just at the start.

Chaining filter() and map()

Because filter() returns a new array, you can call .map() on its result. This is called chaining, and it produces compact, readable pipelines:

container.innerHTML = products
  .filter(p => p.category === activeCategory)
  .map(p => `<div class="card">${p.name}</div>`)
  .join('');

Read this as: “Take the products array, keep only those in the active category, transform each into an HTML string, join them, and set as innerHTML.” The rendering pattern from Lesson 21 now has a filtering step in front of it. Nothing else changes.

NOTE
Keep your chains to two methods maximum while learning. Chaining three or more methods produces code that is hard to debug when something goes wrong. If a chain is getting long, break it up: run filter(), store the result in a variable, then call map() on that variable. Clarity beats brevity.

Event Delegation for Dynamic Content

When cards are generated by JavaScript (not written in the HTML), you cannot attach event listeners to them at page load time — they do not exist yet. The solution is event delegation: attach one listener to the stable parent container, and let click events bubble up from any card to that parent.

// This works even after renderProducts() replaces all the cards
container.addEventListener('click', function(event) {
  const card = event.target.closest('.card');
  if (!card) return;  // Click was not on a card or its children

  const id      = Number(card.dataset.id);
  const product = products.find(p => p.id === id);
  if (product) openModal(product);
});

event.target.closest('.card') walks up the DOM tree from the element that was actually clicked (which might be a child element inside the card, like a title or badge) until it finds the nearest ancestor matching .card. If the click was outside any card entirely, closest() returns null and the early return exits the handler.

NOTE
Think of event delegation like a mailroom. Instead of putting a mailbox at every desk in the building, you put one mailroom at the entrance that sorts all incoming mail. One listener on the parent sorts all clicks from all cards, regardless of how many cards there are or when they were added to the page.

Best Practices

  • ►Keep filter logic and render logic separate. A filterAndRender() function that updates the filter state and calls renderProducts() is much easier to debug than mixing both in a single event handler.
  • ►Store filter state in variables outside event handlers. let activeCategory = 'all' and let searchQuery = '' at the top of your script, updated inside handlers. Both are available to filterAndRender() without being passed as arguments every time.
  • ►Always use .toLowerCase() for search comparisons. A user typing “shoes” should match “Running Shoes”. Normalize both the query and the data to lowercase before comparing.
  • ►Use event delegation for dynamic content. Any time cards are generated by JavaScript, attach the click listener to the container, not the cards.
  • ►Always handle the empty state. When filter() returns an empty array, show a helpful message, not a blank space.

Summary

forEach() runs a function for each item without returning anything — useful for side effects like clearing button states. map() transforms every item and returns a new array of the same length — your primary tool for generating HTML. filter() keeps only items that pass a test and returns a shorter array — the engine behind every gallery filter and search. find() returns the first matching item — essential for the click-to-detail pattern. Combining category filters and live search into one filterAndRender() function produces the polished, interactive gallery pattern that users expect from professional websites. Event delegation keeps all of this working correctly even as cards are dynamically added or removed.
