User Feedback in Web Design

User feedback is a crucial element of web design.

Why It Matters:

  • Users need visual cues when interacting with elements
  • Helps users understand the state of interactive elements
  • Indicates whether a link has been visited
  • Shows when buttons can be clicked
  • Improves overall user experience
Visual feedback helps users understand their interactions with your website

User Action Pseudo-classes

User action pseudo-classes in CSS are special keywords that allow you to provide feedback without JavaScript.

No JavaScript Needed

Pure CSS solution for interactivity

Better Performance

Faster than JavaScript handlers

Enhanced UX

Immediate visual feedback

Common User Action Pseudo-classes:

  • :hover - Mouse over element
  • :active - Element being clicked
  • :focus - Element has focus
  • :visited - Link previously visited
  • :checked - Checkbox/radio selected

:active Pseudo-class

The :active pseudo-class applies styles when an element is activated by the user.

When It Triggers:

  • User clicks a button
  • User clicks a link
  • Provides immediate visual feedback
  • Shows users their actions are recognized

CSS Example:

a:active {
  background-color: #881518;
  color: white;
}
Example link

Click and hold to see the :active state

:hover Pseudo-class

The :hover pseudo-class is triggered when a user hovers over an element with their mouse or pointing device.

Common Uses:

  • Create visual feedback for buttons
  • Highlight links
  • Show that elements are interactive
  • Respond to user attention

CSS Example:

.btn:hover {
  background-color: #a01a1e;
  transform: scale(1.05);
}

Move your mouse over the button to see the effect

:focus Pseudo-class

The :focus pseudo-class applies styles when an element gains focus.

How Elements Gain Focus:

  • Keyboard navigation (Tab key)
  • User clicks into a form input
  • Essential for accessibility
  • Helps keyboard users identify active elements

CSS Example:

input:focus {
  border-color: #881518;
  box-shadow: 0 0 0 3px rgba(136, 21, 24, 0.1);
}

Click the input or use Tab to navigate to it

Accessibility Importance:

The :focus pseudo-class is crucial for users who rely on keyboards for navigation. Never remove focus indicators without providing an alternative!

:visited Pseudo-class

The :visited pseudo-class targets links the user has already visited.

Benefits:

  • Helps users distinguish visited pages
  • Shows which content they've already seen
  • Improves navigation experience
  • Reduces confusion in link-heavy pages

CSS Example:

a:visited {
  color: cyan;
}
Visit Example.com

After clicking, the link will change to cyan

Privacy Limitation:

Due to privacy concerns, browsers limit which CSS properties can be modified with :visited (typically only color-related properties).

:checked Pseudo-class

The :checked pseudo-class styles form elements like checkboxes and radio buttons when they are selected.

Use Cases:

  • Customize checkbox appearance
  • Style radio button selections
  • Enhance user experience
  • Create custom form controls

CSS Example:

.checkbox:checked {
  background-color: green;
}

Click the checkbox to see the :checked state

Custom Checkbox with :checked

You can create fully custom checkboxes using appearance: none and :checked.

Complete CSS Example:

.checkbox {
  appearance: none;
  width: 24px;
  height: 24px;
  border: 2px solid #881518;
  border-radius: 4px;
}

.checkbox:checked {
  background-color: green;
  border-color: green;
}

Key Properties:

  • appearance: none - Removes default browser styling
  • :checked - Applies styles when checkbox is selected
  • Allows complete customization of checkbox appearance

Other User Action Pseudo-classes

:focus-within

Styles an element when it or any descendants have focus

:enabled

Targets form elements that are currently enabled

:disabled

Targets form elements that are disabled

:target

Styles an element that is the target of a URL fragment (after # symbol)

Quick Examples:

form:focus-within { border: 2px solid blue; }
button:enabled { cursor: pointer; }
button:disabled { opacity: 0.5; }
:target { background: yellow; }

User Action Pseudo-classes Summary

Key Takeaways:

  • User feedback is crucial for good web design
  • User action pseudo-classes provide feedback without JavaScript
  • :active - Element being clicked/activated
  • :hover - Mouse hovering over element
  • :focus - Element has keyboard/click focus (accessibility critical)
  • :visited - Link previously visited by user
  • :checked - Checkbox/radio button is selected
  • Other options: :focus-within, :enabled, :disabled, :target
