What is a Data Type?

In JavaScript, a data type refers to the kind of value a variable holds.

What is a Variable?

A variable is a named container that stores a value of a specific data type, allowing you to reference and manipulate it throughout your code.

Why Data Types Matter

Data types help the program understand the kind of data it's working with, whether it's:

  • A number (for calculations)
  • Text (for messages and labels)
  • True/false values (for decisions)
  • Or something else entirely

JavaScript has several basic data types that you'll use in your programs. Let's explore each one!

Number Data Type

Number

Represents both integers and floating-point values.

Integers

Whole numbers without decimal points

7
19
90

Floating-Point Numbers

Numbers with decimal points

3.14
5.2

Key Point: A floating-point number is a number with a decimal point.

String Data Type

String

A sequence of characters, or text, enclosed in quotes.

String Examples

Strings can use either single or double quotes:

"Hello, world"

'JavaScript'

When to Use Strings

  • User names and messages
  • Labels and titles
  • Any text-based information
  • HTML content in your code

Boolean Data Type

Boolean

Represents one of two possible values: true or false

Boolean Values

true

false

Common Boolean Uses

  • ✓ Check if a page is loading
  • ✓ Check if a user is logged in
  • ✓ Determine if a form is valid
  • ✓ Control program flow (if statements)

Remember: Booleans are perfect for yes/no, on/off, or true/false situations.

undefined and null

These two data types represent the absence of a value, but in different ways:

undefined

A variable has been declared but hasn't been given a value yet.

let myVariable;
// myVariable is undefined

This happens automatically when you create a variable without assigning it a value.

null

The variable has been intentionally set to "nothing".

let myVariable = null;
// Explicitly set to nothing

This is done intentionally by the programmer to indicate "no value".

Key Difference: undefined means "not yet assigned" while null means "intentionally empty".

Object Data Type

Object

A more complex data type that can hold collections of key-value pairs.

Understanding Objects

Key (Property Name)

Like a label for the data

Value

The actual data you want to store

Object Example

let book = {
  title: "The Great Gatsby",
  author: "F. Scott Fitzgerald",
  year: 1925
};

In this object:

  • title, author, and year are the keys (property names)
  • "The Great Gatsby", "F. Scott Fitzgerald", and 1925 are the values
  • Each key-value pair is called a property
  • This book object has three properties

Symbol and BigInt

These are two less commonly used data types, but they're still important to know about.

Symbol

A special type of value that is always unique and cannot be changed.

Often used to create unique labels or identifiers for properties.

Symbol('mySymbol');
BigInt

Used for very large numbers that exceed the limit of the Number type.

1234567890123456789012345678901234567890n;

Create a BigInt by adding n at the end of a very large number.

Note: Symbol and BigInt are less commonly used in everyday programming, but they solve specific problems when needed.
