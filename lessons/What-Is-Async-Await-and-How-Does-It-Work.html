async/await in JavaScript

What is async/await?

async/await is a modern way to handle asynchronous code that makes it easier to write and read.

Review: Asynchronous Programming

In previous lessons, you learned that asynchronous programming allows other code to run while we wait for time-consuming tasks to complete.

Common Async Tasks

  • Fetching data from a server
  • Reading data from a file
  • Waiting for timers
  • Database queries

Built on Top of Promises

The Foundation

async/await is built on top of promises, making writing and reading asynchronous code easier.

Why It Matters

Instead of chaining .then() and .catch() methods, async/await lets you write asynchronous code that looks and behaves like synchronous code.

Benefits

  • Cleaner syntax: More readable code
  • Better flow: Code reads top to bottom
  • Easier debugging: Simpler stack traces
  • Better error handling: Use try/catch blocks

The async Keyword

What async Does

When you put the async keyword before a function, it means that function will always return a Promise.

Syntax

async function myFunction() {
  // function code here
}

Automatic Promise Wrapping

Even if the function returns a regular value, it will be automatically wrapped in a resolved Promise.

Example

async function getValue() {
  return 42;
}

getValue().then(value => console.log(value)); // 42

The await Keyword

What await Does

The await keyword allows you to wait for a Promise to resolve before moving on to the next line of code.

Important Rules

⚠️ Critical Restriction

Only inside an async function can you use the await keyword.

What It Does

  • Pauses execution of the async function
  • Waits for the Promise to resolve
  • Returns the resolved value
  • Then continues to the next line

Complete Example

Delayed Greeting Function

Here's an example to illustrate how async/await works:

Code

async function delayedGreeting(name) {
  console.log("A Messenger entered the chat...");
  await new Promise(resolve => setTimeout(resolve, 2000));
  console.log(`Hello, ${name}!`);
}

delayedGreeting("Alice");
console.log("First Printed Message!");

Breaking Down the Example

Line-by-Line Analysis

1. Function Declaration

async function delayedGreeting(name)

Declares an async function that returns a Promise

2. Immediate Log

console.log("A Messenger entered the chat...");

Logs a message immediately

3. Wait for Promise

await new Promise(resolve => setTimeout(resolve, 2000));

Pauses execution for 2 seconds (2000ms)

Understanding the Output

Execution Order

"A Messenger entered the chat..."
"First Printed Message!"
// ... 2 seconds later ...
"Hello, Alice!"

Why This Order?

1. "A Messenger entered the chat..."

Logs immediately when function starts

2. "First Printed Message!"

The function is asynchronous - it doesn't block the rest of the code from running

3. "Hello, Alice!"

Appears after the 2-second delay completes

Asynchronous, Non-Blocking

Key Insight

When we call delayedGreeting("Alice"), you'll see "First Printed Message!" appear before the greeting.

Why?

Non-Blocking Behavior

The function is asynchronous - it doesn't block the rest of the code from running.

  • delayedGreeting() starts executing
  • Logs "A Messenger entered the chat..."
  • Hits await - pauses THAT function
  • Code continues outside the function
  • Logs "First Printed Message!"
  • After 2 seconds, greeting completes

Advantage: Better Error Handling

One of the Biggest Advantages

One of the biggest advantages of async/await is error handling.

The Improvement

With Promises

Use .catch() method

fetch(url)
  .then(r => r.json())
  .catch(err => {...})

With async/await

Use try/catch blocks

try {
  await fetch(url)
} catch(err) {...}

try/catch blocks are more intuitive and familiar to most developers.

Error Handling with try/catch

Fetching User Data

Here's an example using async/await with error handling:

Code

async function fetchUserData() {
  try {
    let response = await fetch(`https://api.example.com/users`);
    let userData = await response.json();
    console.log(userData);
  } catch (error) {
    console.log("Error fetching user data:", error);
  }
}

fetchUserData();

Breaking Down fetchUserData

What Happens?

Two await Statements

The await keyword is used twice in this example:

1. First await

let response = await fetch(`https://api.example.com/users`);

Wait for the fetch operation to complete

2. Second await

let userData = await response.json();

Wait for the JSON parsing to finish

How Errors are Caught

Automatic Error Handling

If any error occurs during this process, it will be caught in the catch block.

Possible Errors

Network Errors

  • Server is down
  • No internet connection
  • Request timeout

Parsing Errors

  • Response is not valid JSON
  • Unexpected data format

Handling

catch (error) {
  console.log("Error fetching user data:", error);
}

All errors are caught and handled gracefully

Comparison: Promises vs async/await

Same Functionality, Different Syntax

Using Promises

function fetchUserData() {
  fetch(url)
    .then(response => {
      return response.json();
    })
    .then(userData => {
      console.log(userData);
    })
    .catch(error => {
      console.log(error);
    });
}

Using async/await

async function fetchUserData() {
  try {
    let response = await fetch(url);
    let userData = await response.json();
    console.log(userData);
  } catch (error) {
    console.log(error);
  }
}

async/await version is more readable and easier to follow!

Best Practices

Tips for Using async/await

✓ Do This

  • Always use try/catch for error handling
  • Use async/await for cleaner, more readable code
  • Remember that async functions return Promises
  • Use await only inside async functions

✗ Avoid This

  • Using await outside of async functions
  • Forgetting to handle errors with try/catch
  • Using async when you don't need it
  • Overusing await when parallel execution would be faster

Parallel Execution

Use Promise.all() when you need multiple operations to run in parallel.
