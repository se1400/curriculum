Three Tools for Media Optimization

There are three tools to consider when using media, such as images, on your web pages.

Optimizing your media assets ensures faster page loading and better user experience.

Three Key Factors

  • Size - Resolution and dimensions
  • Format - File type (PNG, JPG, WEBP, etc.)
  • Compression - Reducing file size

Image Size and Resolution

When you are building a website, you'll often style images to display in a specific size.

For example, you might have an image display at a 640x480 resolution. 640 represents the width while 480 represents the height in pixels.

Understanding Dimensions

640 x 480

• Width: 640 pixels

• Height: 480 pixels

Resolution Basics

Images are measured in pixels (width x height). The resolution determines how detailed the image appears.

Matching Size to Styling

When preparing your image, you want to scale it to match the display size (e.g., 640x480) to match that styling.

If you serve an image that is 1920x1080 but you are styling it to be much smaller, you're requiring your users to download unnecessary data.

A smaller resolution results in a smaller file size.

Don't Oversend Data

Bad: Serve 1920x1080 image, display at 640x480

Good: Serve 640x480 image, display at 640x480

Benefits

  • Faster page load times
  • Less bandwidth usage
  • Better user experience
  • Smaller file sizes

Traditional File Formats

Two of the most common file formats are PNG and JPG, but these are no longer the most ideal formats for serving images.

These formats have been the standard for many years and are widely supported.

PNG (Portable Network Graphics)

Pros: Lossless compression, supports transparency
Cons: Larger file sizes

JPG (Joint Photographic Experts Group)

Pros: Good for photos, smaller file sizes
Cons: Lossy compression, no transparency

Modern File Formats

Unless you need support for older browsers, you should consider using a more optimized format, like WEBP or AVIF.

These modern formats provide better compression and quality compared to traditional formats.

WEBP

Features: Superior compression, supports transparency and animation
File Size: 25-35% smaller than JPG

AVIF

Features: Excellent compression, high quality
File Size: Even smaller than WEBP

Browser Support

Modern browsers support these formats. Check compatibility if supporting older browsers.

Compression Algorithms

Finally, you can run compression algorithms on your images.

A compression algorithm is used to reduce the size for files or data.

There are options like pngcrush to compress your images locally, or you can use online compression tools.

Compression Tools

  • Local tools: pngcrush, ImageOptim
  • Online tools: TinyPNG, Compressor.io
  • Build tools: webpack, gulp plugins

Lossless vs Lossy Compression

Lossless means that the original data can be perfectly reconstructed from the compressed data.

However, it's worth noting that specific file formats, such as JPG, are not lossless.

If you try to compress a JPG image, it will result in degraded quality.

Lossless Compression

Formats: PNG, WEBP (lossless mode)

Result: No quality loss, perfect reconstruction

Use for: Logos, icons, graphics with text

Lossy Compression

Formats: JPG, WEBP (lossy mode)

Result: Quality degradation, smaller files

Use for: Photographs where minor quality loss is acceptable

Compression Best Practices

You should keep all these things in mind when selecting images for your web pages.

Understanding compression types helps you make informed decisions about image optimization.

⚠️ Important: Don't Compress JPG

JPG is already a lossy format. Compressing it further will degrade quality without much file size benefit.

Compression Guidelines

  • Use lossless compression for PNG images
  • Avoid re-compressing JPG images
  • Consider converting JPG to WEBP instead
  • Test quality after compression
  • Balance file size with visual quality

Summary: Optimization Checklist

Effective media optimization involves carefully considering size, format, and compression.

Following these best practices ensures your images load quickly while maintaining quality.

Optimization Steps

  • Size: Match image resolution to display size
  • Format: Use WEBP or AVIF for modern browsers
  • Compression: Use lossless for PNG, avoid re-compressing JPG
  • Testing: Verify quality and load times
  • Fallbacks: Provide alternatives for older browsers
