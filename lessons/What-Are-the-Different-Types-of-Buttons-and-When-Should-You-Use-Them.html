The Button Element

The button element is used to perform a particular action when it is activated.

Basic Button Example

<button>Start Game</button>
Output:
Common Use Cases:
Submitting a form
Showing a modal
Toggling a side menu open and closed
The button element has a type attribute which controls the behavior of the button when it is activated.
1 / 5

Button Type: "button"

The first possible value for the type attribute would be the button type.

Example: Button with Alert Functionality

<button type="button">Show Alert</button>
Interactive Output (click to see alert):
By default, the button will not do anything when activated. However, you can add some JavaScript code to make the button interactive, such as showing an alert in this case.
Note: The interactive example above uses JavaScript to show the alert. You don't need to worry about understanding the JavaScript code right now - you will learn about JavaScript in future modules.

Button Type: "submit"

Another possible value for the type attribute is the submit value.

Example: Form with Submit Button

<form action="">
  <label for="email">Email address:</label>
  <input type="email" id="email" name="email" />
  <button type="submit">Submit form</button>
</form>
Interactive Output:
Inside this form element, there is a label and input element for the user's email address. When the user clicks on the submit button, their data will be sent to the server and will be processed.
3 / 5

Button Type: "reset"

The third possible value for the type attribute is the reset value.

Example: Form with Reset and Submit Buttons

<form action="">
  <label for="email">Email address:</label>
  <input type="email" id="email" name="email" />
  <button type="reset">Reset form</button>
  <button type="submit">Submit form</button>
</form>
Interactive Output (type email, then click Reset):
When the user clicks on the reset button, it will clear out all of their input data.
Important: Reset buttons are usually not the best idea because they could lead to users accidentally resetting their data. Also, multiple buttons in your form could clutter up the user interface.

Using Input Element as Buttons

Another way to create buttons in HTML is to use the input element. The input element also has a type attribute with the possible values of submit, reset, and button.

Example: Input Element as Button

<input type="button" value="Start Game" />
Output:
The value attribute is used to show the button text.

Key Differences

<input> Element

Void elements - cannot have child nodes such as text, and can only have a start tag.

Limited to text only via value attribute.

<button> Element

More flexible - you can nest text, images, and icons inside it.

Can contain rich content and child elements.
