The + Operator's Double Duty

The first thing you'll probably try is to add a number and a string.

Two Jobs for +

In JavaScript, the + operator does double duty:

  1. Addition - Adds numbers together
  2. String Concatenation - Joins strings together

What is String Concatenation?

String concatenation is a way to join two strings together.

Example: "Hello" + " " + "World" = "Hello World"

Adding a Number and String

When you use + with a number and a string, JavaScript decides to treat them both as strings and joins them together.

const result = 5 + '10';
console.log(result);        // "510"
console.log(typeof result); // "string"
510 string
5 + '10' = "510"

‚ö†Ô∏è Not What You Expected?

You might expect 15, but you get "510" instead!

If you check the type of the result with the typeof operator, you'd see it's indeed a string.

Order Doesn't Matter

What do you think will happen if you switch the order of 5 and '10'?

const result = '10' + 5;
console.log(result);        // "105"
console.log(typeof result); // "string"
105 string
'10' + 5 = "105"

‚úì Same Thing Happened!

You can see the same thing happened. JavaScript sees a string in '10' and a number in 5, so it converts the number to a string and concatenates them.

Type Coercion

What is Type Coercion?

Type coercion is when a value from one data type is converted into another.

Example of Type Coercion

In 5 + '10':

  1. JavaScript sees a number (5) and a string ('10')
  2. It converts the number 5 to the string "5"
  3. Then concatenates: "5" + "10" = "510"

üí° Important Note

JavaScript automatically performs type coercion - you don't have to do anything! But this can lead to unexpected results if you're not careful.

Other Operators: Different Behavior

Things get more interesting when you try to perform other arithmetic operations like subtraction, multiplication, or division with a string and number.

The Opposite Happens!

In these cases, JavaScript tries to convert the string into a number before doing the math ‚Äì another type coercion!

Number ‚Üí String

Concatenates as strings

With -, *, / Operators

Performs math operation

Subtraction with Strings

const subtractionResult = '10' - 5;
console.log(subtractionResult);        // 5
console.log(typeof subtractionResult); // "number"
5 number
'10' - 5 = 5

‚úì What Happened?

JavaScript successfully converts the string '10' to the number 10 and then performs the calculation.

As a result, '10' - 5 yields 5 (a number).

Multiplication and Division

Multiplication Example

const multiplicationResult = '10' * 2;
console.log(multiplicationResult);        // 20
console.log(typeof multiplicationResult); // "number"
20 number

Division Example

const divisionResult = '20' / 2;
console.log(divisionResult);        // 10
console.log(typeof divisionResult); // "number"
10 number

Pattern Detected!

In all these examples, JavaScript converts the string to a number before performing the operation.

What If String Isn't a Number?

But what if the string doesn't look like a number? Let's see what happens in that case:

const subtractionResult = 'abc' - 5;
console.log(subtractionResult);        // NaN
console.log(typeof subtractionResult); // "number"
NaN number

‚ö†Ô∏è NaN - Not a Number

The string 'abc' does not represent a valid numeric value, so JavaScript cannot convert it into a meaningful number.

When such conversion fails, JavaScript returns NaN, which stands for "Not a Number".

NaN is a special value of the Number type that represents an invalid or unrepresentable number.

More Invalid Conversion Examples

Multiplication with Invalid String

const multiplicationResult = 'abc' * 2;
console.log(multiplicationResult);        // NaN
console.log(typeof multiplicationResult); // "number"
NaN number

Division with Invalid String

const divisionResult = 'abc' / 2;
console.log(divisionResult);        // NaN
console.log(typeof divisionResult); // "number"
NaN number

üí° Important

Notice that typeof NaN is "number"! Even though it means "Not a Number", it's still classified as a number type.

Booleans in Arithmetic

What if you perform arithmetic operations with a boolean (true or false)?

Boolean Conversion Rules

JavaScript treats booleans as numbers in mathematical operations:

  • true becomes 1
  • false becomes 0

Example with true

const result1 = true + 1;
console.log(result1);        // 2
console.log(typeof result1); // "number"
2 number
true (becomes 1) + 1 = 2

More Boolean Examples

Example with false

const result2 = false + 1;
console.log(result2);        // 1
console.log(typeof result2); // "number"
1 number
false (becomes 0) + 1 = 1

Boolean with String Concatenation

const result3 = 'Hello' + true;
console.log(result3);        // "Hellotrue"
console.log(typeof result3); // "string"
Hellotrue string

In this example, JavaScript converted true to a string and concatenates it with 'Hello', resulting in 'Hellotrue'.

null and undefined in Arithmetic

Conversion Rules

  • null is treated as 0
  • undefined is treated as NaN

null Example

const result1 = null + 5;
console.log(result1);        // 5
console.log(typeof result1); // "number"
5 number
null (becomes 0) + 5 = 5

undefined Example

const result2 = undefined + 5;
console.log(result2);        // NaN
console.log(typeof result2); // "number"
NaN number
undefined (becomes NaN) + 5 = NaN
