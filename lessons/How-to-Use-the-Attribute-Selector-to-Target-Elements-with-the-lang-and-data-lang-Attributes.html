Multilingual Websites

When building multilingual websites or handling custom data attributes, you need to style elements based on language.

Common Scenarios:

  • Websites with content in multiple languages
  • Documents containing mixed-language text
  • International applications
  • Content management systems with translation
The lang and data-lang attributes help you apply language-specific styling

Language Attributes

Two main attributes are used for language specification:

lang Attribute

Standard HTML attribute

Specifies the language of content within an element

<p lang="en">English text</p>

data-lang Attribute

Custom data attribute

Stores additional language information in elements

<p data-lang="fr">French text</p>

The lang Attribute

The lang attribute is a standard HTML attribute used to specify the language of content within an element.

Purpose:

  • Helps screen readers pronounce text correctly
  • Assists search engines with content indexing
  • Enables language-specific styling
  • Improves accessibility

Why It Matters:

On a multilingual website, you might want to style elements differently based on the language they're written in.

Targeting lang="en"

Use attribute selectors to target elements with specific lang values.

CSS Example:

p[lang="en"] {
  font-style: italic;
  background-color: #e8f5e9;
}

HTML Example:

<p lang="en">This is an English paragraph.</p>
<p lang="fr">Ceci est un paragraphe en fran√ßais.</p>

What It Does:

Applies italic styling to any paragraph where lang is set to English ("en").

The data-lang Attribute

Custom data attributes like data-lang are commonly used to store additional language information.

Custom Data Attributes:

  • Prefix with data-
  • Store extra metadata in elements
  • Can be accessed with JavaScript
  • Stylable with CSS attribute selectors

data-lang vs lang:

data-lang is custom and can store any language information you define. lang is a standard HTML attribute with semantic meaning.

Targeting data-lang="fr"

Use attribute selectors to target elements with data-lang attributes.

CSS Example:

p[data-lang="fr"] {
  color: blue;
  font-weight: bold;
}

HTML Example:

<p data-lang="fr">Ceci est un paragraphe en fran√ßais.</p>
<p data-lang="en">This is a paragraph in English.</p>

What It Does:

Any paragraph with data-lang="fr" will have blue, bold text.

Visual Example: data-lang="fr"

French paragraphs are styled with blue, bold text:

This is a paragraph in English.

  • First paragraph: data-lang="fr" ‚Üí blue, bold, purple background

Benefits of Language Attributes

Conditional Styling:

  • Apply styles based on embedded metadata
  • Make web pages more dynamic
  • Create context-aware designs
  • Handle multilingual content effectively

Accessibility Benefits:

  • Screen readers pronounce text correctly
  • Better user experience for international audiences
  • Clear visual distinction between languages
  • Improved content navigation

Best Practices

Use lang for Semantics

  • Standard HTML attribute
  • Improves accessibility
  • Helps search engines
  • Better for screen readers

Use data-lang for Custom Needs

  • Store additional metadata
  • Custom language flags
  • JavaScript processing
  • Application-specific data

Recommendation:

Use the lang attribute for standard language specification. Use data-lang when you need custom language tracking or additional metadata.

lang and data-lang Summary

Key Takeaways:

  • lang attribute - Standard HTML attribute for language specification
  • data-lang attribute - Custom data attribute for additional language info
  • Syntax: p[lang="en"] or p[data-lang="fr"]
  • Use cases: Multilingual websites, mixed-language documents
  • Benefits: Improved accessibility, better SEO, visual distinction
  • Language codes: Follow ISO 639-1 standard (en, fr, es, de, etc.)
  • Styling: Apply unique styles to different languages
  • Best practice: Use lang for semantics, data-lang for custom needs
