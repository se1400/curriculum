Introduction

Arrays are the data structure that powers nearly every dynamic page you have ever visited. Every product grid in an online store, every dropdown menu populated from a database, every search result list, every autocomplete suggestion — all of it is powered by an array somewhere. When a web page feels alive and responsive, chances are a developer is reading, filtering, sorting, or modifying an array and re-rendering the result to the screen.

Throughout this lesson, the running example is an interactive to-do list. This is not an arbitrary choice: a to-do list is a pure, visible manifestation of array operations. When you push a new task, the list grows. When you pop the last task, the list shrinks. When you splice out the third item, it disappears from the middle. Every array method you learn here has an immediate, visible consequence. By the time you finish, you will be able to look at any method and picture exactly what happens to the list on screen.

What Is an Array?

An array is an ordered, indexed collection of values. “Ordered” means the items have a fixed sequence — item zero is always item zero until you explicitly move it. “Indexed” means every item is assigned a numeric position starting at zero. That starting-at-zero convention is called zero-based indexing, and it is the source of countless beginner mistakes, so let it sink in now.

[IMG - Zero-based index diagram for array]

An array with four items has indices 0, 1, 2, and 3. There is no index 4. The highest valid index is always array.length - 1. Arrays in JavaScript can hold any mix of data types: strings, numbers, booleans, objects, other arrays, even functions. You are not restricted to storing all the same type.

LISTING 24.1 — Arrays of Different Types 
// A string array
const tasks = ["Buy groceries", "Walk the dog", "Read a book"];

// A number array
const scores = [98, 74, 85, 91, 63];

// A mixed array (less common, but valid)
const mixed = ["hello", 42, true, null];

// An array of objects — the most common real-world shape
const products = [
  { name: "Notebook",  price: 12.99, inStock: true  },
  { name: "Pen Set",   price:  8.49, inStock: true  },
  { name: "Desk Lamp", price: 34.99, inStock: false },
];

// .length gives the number of items
console.log(tasks.length);    // 3
console.log(products.length); // 3

The .length property always reflects the current number of elements. It updates automatically when you add or remove items — you never have to maintain it yourself.

Creating Arrays and Reading Values

Array Literals

The standard way to create an array is with square brackets. This is called an array literal and it is what you will use 95% of the time.

LISTING 24.2 — Array Literal and Basic Reading 
const fruits = ["apple", "banana", "cherry", "date", "elderberry"];

// Read by index
console.log(fruits[0]); // "apple"   (first item)
console.log(fruits[2]); // "cherry"  (third item)

// Classic last-item pattern (error-prone)
console.log(fruits[fruits.length - 1]); // "elderberry"

// Modern last-item pattern using at()
console.log(fruits.at(-1)); // "elderberry"  ← clean!
console.log(fruits.at(-2)); // "date"        ← second-to-last
console.log(fruits.at(0));  // "apple"       ← same as fruits[0]

The at() Method: Negative Indexing

The at() method accepts both positive and negative integers. A positive argument works exactly like bracket notation: array.at(0) is the first item. A negative argument counts from the end: array.at(-1) is the last item, array.at(-2) is the second-to-last, and so on.

This matters because the old pattern — array[array.length - 1] — is verbose and easy to mis-type. The at(-1) pattern is cleaner, reads more naturally, and makes your intent obvious at a glance. Use it whenever you need to access items from the end of an array.

Array.from()

Array.from() converts array-like things into real arrays — most commonly the NodeList returned by querySelectorAll(). (Array.of() also exists to create arrays from a list of values, but you'll almost always use the literal syntax [] instead.)

LISTING 24.3 — Array.from() 
// From a NodeList (what querySelectorAll returns)
const divs = Array.from(document.querySelectorAll("div"));
divs.forEach(div => div.classList.add("processed")); // now you can use array methods

// From a string
const letters = Array.from("hello"); // ["h", "e", "l", "l", "o"]

// From a Set (removes duplicates)
const unique = Array.from(new Set([1, 2, 2, 3, 3, 3])); // [1, 2, 3]

// With a mapping function (second argument)
const doubled = Array.from([1, 2, 3, 4], n => n * 2); // [2, 4, 6, 8]

Array.from() is especially useful when working with the DOM. Methods like querySelectorAll() return a NodeList, which looks like an array but lacks most array methods. Converting it with Array.from() gives you the full array toolkit.

Mutating vs. Non-Mutating Methods

This is the most important concept in this lesson. JavaScript array methods fall into two camps: those that mutate (change) the original array, and those that return a new array while leaving the original untouched. Understanding which is which prevents some of the most confusing bugs in JavaScript.

[IMG - Mutating vs non-mutating method comparison]

NOTE: Before calling a method, ask yourself: will this change my original array? If you need to keep the original data intact — for example, to reset to the original state or to avoid corrupting shared data — use a non-mutating method or make a copy of the array first.

Here is a simple example that makes the distinction concrete:

LISTING 24.4 — Mutating vs. Non-Mutating Comparison 
const original = ["apple", "banana", "cherry"];

// MUTATING — reverse() changes original in place
const mutated = original.reverse();
console.log(original); // ["cherry", "banana", "apple"]  ← CHANGED!
console.log(mutated);  // ["cherry", "banana", "apple"]  ← same array!

// NON-MUTATING — toReversed() returns a new array
const fresh = ["apple", "banana", "cherry"];
const reversed = fresh.toReversed();
console.log(fresh);    // ["apple", "banana", "cherry"]  ← unchanged
console.log(reversed); // ["cherry", "banana", "apple"]  ← new array

With the mutating version, original and mutated point to the exact same array — reversing one reverses both. The non-mutating version creates a brand-new array, so fresh is untouched.

Mutating Methods — Adding and Removing

The four stack-and-queue methods are the most frequently used array methods in everyday code. They all modify the original array in place.

push() — Add to the End

push(...items) appends one or more items to the end of the array and returns the new length of the array. In a to-do list, this is the "Add task" button: the new item appears at the bottom of the list.

LISTING 24.5 — push() 
const tasks = ["Buy groceries", "Walk the dog"];

const newLength = tasks.push("Read a book");
console.log(tasks);     // ["Buy groceries", "Walk the dog", "Read a book"]
console.log(newLength); // 3

// Push multiple items at once
tasks.push("Call dentist", "Pay bills");
console.log(tasks.length); // 5

pop() — Remove from the End

pop() removes the last item and returns it. No arguments needed. In a to-do list, this is "Remove last task."

LISTING 24.6 — pop() 
const tasks = ["Buy groceries", "Walk the dog", "Read a book"];

const removed = tasks.pop();
console.log(removed); // "Read a book"
console.log(tasks);   // ["Buy groceries", "Walk the dog"]

// pop() on an empty array returns undefined
const empty = [];
console.log(empty.pop()); // undefined

unshift() — Add to the Beginning

unshift(...items) prepends one or more items to the front of the array and returns the new length. Every existing item shifts up one index position. In a to-do list, this adds an urgent item to the top.

LISTING 24.7 — unshift() 
const tasks = ["Walk the dog", "Read a book"];

tasks.unshift("URGENT: Call doctor");
console.log(tasks);
// ["URGENT: Call doctor", "Walk the dog", "Read a book"]
// index 0 is now the urgent task

shift() — Remove from the Beginning

shift() removes the first item and returns it. Every remaining item shifts down one index. In a to-do list, this "completes" or "dismisses" the top task.

LISTING 24.8 — shift() 
const tasks = ["URGENT: Call doctor", "Walk the dog", "Read a book"];

const completed = tasks.shift();
console.log(completed); // "URGENT: Call doctor"
console.log(tasks);     // ["Walk the dog", "Read a book"]
// "Walk the dog" is now at index 0

Mutating Methods — Modifying in Place

splice() — The Swiss Army Knife

splice(start, deleteCount, ...items) is the most powerful mutating method. It can delete items, insert items, or replace items — all at a specific position in the array. It modifies the original and returns an array of any deleted items.

LISTING 24.9 — splice() Three Ways 
// EXAMPLE 1: Delete items at a position
const tasks = ["Task A", "Task B", "Task C", "Task D", "Task E"];
const deleted = tasks.splice(1, 2); // start at index 1, delete 2 items
console.log(deleted); // ["Task B", "Task C"]
console.log(tasks);   // ["Task A", "Task D", "Task E"]

// EXAMPLE 2: Insert items without deleting
const colors = ["red", "green", "blue"];
colors.splice(1, 0, "orange", "yellow"); // at index 1, delete 0, insert two
console.log(colors); // ["red", "orange", "yellow", "green", "blue"]

// EXAMPLE 3: Replace items
const days = ["Mon", "Tue", "Wed", "Thu"];
days.splice(2, 1, "WEDNESDAY"); // at index 2, delete 1, insert "WEDNESDAY"
console.log(days); // ["Mon", "Tue", "WEDNESDAY", "Thu"]

In a to-do list, splice is used when the user clicks the × button on a specific item. You find the item's index and splice it out: tasks.splice(index, 1).

reverse() — Reverse in Place

reverse() reverses the order of an array's elements in place. The first element becomes the last, and the last becomes the first. It returns a reference to the same (now reversed) array — not a copy.

LISTING 24.10 — reverse() 
const tasks = ["First added", "Second added", "Third added"];
tasks.reverse();
console.log(tasks); // ["Third added", "Second added", "First added"]

// Common use: toggle chronological / reverse-chronological order
// in a rendered list

sort() — Sort in Place

sort(compareFn) sorts the array in place and returns the same array. The critical thing to understand: with no compare function, sort() converts every element to a string and sorts by Unicode character order. This works fine for alphabetical sorting of strings, but gives completely wrong results for numbers.

LISTING 24.11 — sort() — Alphabetical and Numeric 
// ALPHABETICAL — default sort works correctly for strings
const fruits = ["banana", "apple", "cherry", "date"];
fruits.sort();
console.log(fruits); // ["apple", "banana", "cherry", "date"]

// BUG: sorting numbers without a compare function
const scores = [100, 9, 21, 3, 40];
scores.sort();
console.log(scores); // [100, 21, 3, 40, 9]  ← WRONG! (sorted as strings)

// CORRECT: numeric sort with a compare function
scores.sort((a, b) => a - b); // ascending
console.log(scores); // [3, 9, 21, 40, 100]  ← correct

scores.sort((a, b) => b - a); // descending
console.log(scores); // [100, 40, 21, 9, 3]

// Sorting objects by a property
const products = [
  { name: "Notebook",  price: 12.99 },
  { name: "Pen Set",   price:  8.49 },
  { name: "Desk Lamp", price: 34.99 },
];

products.sort((a, b) => a.price - b.price); // sort by price ascending
// Now: Pen Set ($8.49), Notebook ($12.99), Desk Lamp ($34.99)

The compare function receives two items a and b. If it returns a negative number, a comes before b. If it returns a positive number, b comes before a. If it returns zero, their order stays the same. The subtraction shortcut (a, b) => a - b exploits this elegantly: if a is smaller, the result is negative (a first); if b is smaller, the result is positive (b first).

Non-Mutating Methods — Return a New Array

These methods leave the original array completely unchanged and return a new array with the result. Prefer them whenever you need to preserve the original data.

slice() — Extract a Portion

slice(start, end) extracts a section of an array and returns it as a new array. The start index is included; the end index is excluded. The original is untouched.

LISTING 24.12 — slice() 
const tasks = ["A", "B", "C", "D", "E"];

const middle = tasks.slice(1, 4); // indices 1, 2, 3 (not 4)
console.log(middle); // ["B", "C", "D"]
console.log(tasks);  // ["A", "B", "C", "D", "E"]  ← untouched

// Omit end to go to the end of the array
const fromThird = tasks.slice(2);    // ["C", "D", "E"]

// Negative indices count from the end
const lastTwo = tasks.slice(-2);     // ["D", "E"]

// Common trick: copy an entire array
const copy = tasks.slice();          // ["A", "B", "C", "D", "E"]  ← new array

concat() — Merge Arrays

concat(arr2, arr3, ...) merges two or more arrays into a new array. The originals are not modified.

LISTING 24.13 — concat() 
const morning = ["Stretch", "Breakfast", "Coffee"];
const afternoon = ["Lunch", "Walk", "Reading"];
const evening = ["Dinner", "Journal"];

const fullDay = morning.concat(afternoon, evening);
console.log(fullDay);
// ["Stretch", "Breakfast", "Coffee", "Lunch", "Walk", "Reading", "Dinner", "Journal"]

console.log(morning); // ["Stretch", "Breakfast", "Coffee"]  ← untouched

ES2023 Non-Mutating Alternatives

The ES2023 specification added non-mutating counterparts to the most commonly needed mutating methods. They follow a consistent naming pattern: the mutating name with a "to" prefix (or just with() for element replacement).

NOTE: The ES2023 non-mutating methods (toReversed, toSorted, toSpliced, with) are supported in all modern browsers as of 2023. They make working with state in frameworks like React much cleaner because you never accidentally mutate the state array.

LISTING 24.14 — ES2023 Non-Mutating Methods 
const tasks = ["Task A", "Task B", "Task C", "Task D"];

// toReversed() — like reverse() but returns a new array
const reversed = tasks.toReversed();
console.log(reversed); // ["Task D", "Task C", "Task B", "Task A"]
console.log(tasks);    // ["Task A", "Task B", "Task C", "Task D"]  ← untouched

// toSorted(compareFn) — like sort() but returns a new array
const scores = [40, 9, 100, 21];
const sorted = scores.toSorted((a, b) => a - b);
console.log(sorted); // [9, 21, 40, 100]
console.log(scores); // [40, 9, 100, 21]  ← untouched

// toSpliced(start, deleteCount, ...items) — like splice() but returns a new array
const updated = tasks.toSpliced(1, 1, "REPLACEMENT");
console.log(updated); // ["Task A", "REPLACEMENT", "Task C", "Task D"]
console.log(tasks);   // ["Task A", "Task B", "Task C", "Task D"]  ← untouched

// with(index, value) — returns new array with one element changed
const fixed = tasks.with(2, "Task C (fixed)");
console.log(fixed); // ["Task A", "Task B", "Task C (fixed)", "Task D"]
console.log(tasks); // ["Task A", "Task B", "Task C", "Task D"]  ← untouched

Finding Items

These methods help you locate elements inside an array. Each returns a slightly different thing, so choosing the right one matters.

indexOf() and lastIndexOf()

indexOf(value) searches from the beginning and returns the index of the first match, or -1 if not found. lastIndexOf(value) searches from the end. Both use strict equality (===).

LISTING 24.15 — indexOf() and lastIndexOf() 
const tasks = ["Buy milk", "Walk dog", "Buy milk", "Read book"];

console.log(tasks.indexOf("Buy milk"));     // 0  (first occurrence)
console.log(tasks.lastIndexOf("Buy milk")); // 2  (last occurrence)
console.log(tasks.indexOf("Exercise"));     // -1 (not found)

// Common pattern: check if item exists before adding it
const input = "Walk dog";
if (tasks.indexOf(input) === -1) {
  tasks.push(input); // only add if not already there
}

includes()

includes(value) returns true if the value is found, false otherwise. When you only need a yes/no answer, includes() is cleaner than checking if indexOf() equals -1.

LISTING 24.16 — includes() 
const tasks = ["Buy milk", "Walk dog", "Read book"];

console.log(tasks.includes("Walk dog")); // true
console.log(tasks.includes("Exercise")); // false

// Clean duplicate check
const input = "Walk dog";
if (!tasks.includes(input)) {
  tasks.push(input);
}

findLast() and findLastIndex() — ES2023

findLast(callback) searches from the end of the array and returns the last element for which the callback returns true. findLastIndex(callback) does the same but returns the index instead of the element. Both return undefined / -1 if nothing matches.

LISTING 24.17 — findLast() and findLastIndex() 
const tasks = [
  { text: "Buy milk",  done: true  },
  { text: "Walk dog",  done: false },
  { text: "Read book", done: false },
  { text: "Exercise",  done: true  },
];

// Find the last completed task
const lastDone = tasks.findLast(task => task.done);
console.log(lastDone); // { text: "Exercise", done: true }

// Find the index of the last incomplete task
const lastUndoneIndex = tasks.findLastIndex(task => !task.done);
console.log(lastUndoneIndex); // 2  (the "Read book" entry)

// DOM use: scroll to the last incomplete task
const items = document.querySelectorAll(".task-item");
if (lastUndoneIndex !== -1) {
  items[lastUndoneIndex].scrollIntoView({ behavior: "smooth" });
}

Joining Arrays into Strings

join(separator) converts all elements of an array into a single string, with each element separated by the separator you provide. If you omit the separator, it defaults to a comma.

LISTING 24.18 — join() 
const tasks = ["Buy milk", "Walk dog", "Read book"];

console.log(tasks.join(", "));  // "Buy milk, Walk dog, Read book"
console.log(tasks.join(" | ")); // "Buy milk | Walk dog | Read book"
console.log(tasks.join(" > ")); // "Buy milk > Walk dog > Read book"
console.log(tasks.join(""));    // "Buy milkWalk dogRead book"  (no separator)

// Real use: building a comma-separated display string
const ingredients = ["flour", "eggs", "butter", "sugar"];
const label = "Ingredients: " + ingredients.join(", ");
// "Ingredients: flour, eggs, butter, sugar"

// Generating HTML from an array of strings
const listItems = tasks.map(t => `<li>${t}</li>`).join("");
document.querySelector("ul").innerHTML = listItems;

The combination of map() and join("") is the standard pattern for generating HTML from arrays. You transform each item into an HTML string with map(), then glue them all together with join(""), and set the result as innerHTML.

The Spread Operator

The spread operator (...) expands an array into its individual elements. In an array literal context, it lets you copy and merge arrays without mutation.

LISTING 24.19 — Spread Operator with Arrays 
const morning = ["Stretch", "Breakfast"];
const afternoon = ["Lunch", "Walk"];

// Shallow copy of an array
const copy = [...morning];
copy.push("Coffee");
console.log(morning); // ["Stretch", "Breakfast"]  ← untouched
console.log(copy);    // ["Stretch", "Breakfast", "Coffee"]

// Merge two arrays
const allTasks = [...morning, ...afternoon];
console.log(allTasks); // ["Stretch", "Breakfast", "Lunch", "Walk"]

// Prepend an item (non-mutating)
const withUrgent = ["URGENT: Doctor call", ...morning];
console.log(withUrgent); // ["URGENT: Doctor call", "Stretch", "Breakfast"]
console.log(morning);    // ["Stretch", "Breakfast"]  ← untouched

// Append an item (non-mutating)
const withOptional = [...morning, "Optional: Read"];
console.log(withOptional); // ["Stretch", "Breakfast", "Optional: Read"]
console.log(morning);      // ["Stretch", "Breakfast"]  ← untouched

The spread-based non-mutating add pattern ([...array, newItem] or [newItem, ...array]) is how you update lists in React and other modern frameworks. Instead of pushing directly onto the state array, you spread the existing state into a new array with the new item included, then set the new array as state.

Array Destructuring

Destructuring lets you unpack array values into separate variables in a single statement. Instead of writing const first = array[0]; const second = array[1]; on separate lines, you write one compact expression.

LISTING 24.20 — Array Destructuring 
const scores = [98, 74, 85, 91];

// Basic destructuring
const [first, second, third] = scores;
console.log(first);  // 98
console.log(second); // 74
console.log(third);  // 85

// Skip elements with empty commas
const [gold, , bronze] = scores; // skip silver
console.log(gold);   // 98
console.log(bronze); // 85

// Default values for missing positions
const [a = 0, b = 0, c = 0, d = 0, e = 0] = [10, 20];
console.log(a); // 10
console.log(b); // 20
console.log(e); // 0  ← default, no value at index 4

// Rest syntax — collect remaining items
const [head, ...tail] = ["apple", "banana", "cherry", "date"];
console.log(head); // "apple"
console.log(tail); // ["banana", "cherry", "date"]
LISTING 24.21 — Real-World Destructuring Patterns 
// Destructuring coordinates
const point = [37.7749, -122.4194]; // lat, lng
const [lat, lng] = point;
console.log(`Latitude: ${lat}, Longitude: ${lng}`);

// Destructuring RGB values
const rgb = [255, 99, 71]; // tomato red
const [r, g, b] = rgb;
document.body.style.backgroundColor = `rgb(${r}, ${g}, ${b})`;

// Destructuring an API response
function getMinMax(numbers) {
  const sorted = [...numbers].sort((a, b) => a - b);
  return [sorted[0], sorted.at(-1)]; // return as array
}
const [min, max] = getMinMax([40, 9, 100, 21, 63]);
console.log(`Min: ${min}, Max: ${max}`); // "Min: 9, Max: 100"

// Swap two variables without a temp variable
let x = 1, y = 2;
[x, y] = [y, x];
console.log(x, y); // 2 1

Array.isArray()

One surprising JavaScript behavior: typeof [] returns "object", not "array". This is because arrays are objects in JavaScript. The typeof operator cannot distinguish between an array, a plain object, or null (all return "object"). Use Array.isArray() instead.

LISTING 24.22 — Array.isArray() 
// typeof is not helpful for arrays
console.log(typeof []);         // "object"  ← misleading
console.log(typeof {});         // "object"
console.log(typeof null);       // "object"

// Array.isArray() is the reliable check
console.log(Array.isArray([]));          // true
console.log(Array.isArray([1, 2, 3]));   // true
console.log(Array.isArray({}));          // false
console.log(Array.isArray("hello"));     // false
console.log(Array.isArray(null));        // false

// Real use: validating data from an API or form submission
function processTasks(data) {
  if (!Array.isArray(data)) {
    console.error("Expected an array, got:", typeof data);
    return;
  }
  data.forEach(task => renderTask(task));
}

The Mutation Trap

This is one of the most common sources of mysterious bugs in JavaScript, especially for beginners. It deserves its own section and careful attention.

[IMG - Array memory reference vs copy diagram]

When you assign an array to a variable, JavaScript does not copy the array. Both variables point to the same location in memory. This is called assignment by reference. Changing the array through either variable changes the one shared array.

LISTING 24.23 — The Mutation Trap Bug 
// THE BUG
const original = ["Task A", "Task B", "Task C"];
const copy = original; // This is NOT a copy — it's a second reference!

copy.push("Task D");
console.log(original); // ["Task A", "Task B", "Task C", "Task D"]
                       //  ↑ original changed too! Both point to the same array.
console.log(copy === original); // true — they ARE the same array
LISTING 24.24 — The Mutation Trap Fix 
// THE FIX — use spread or slice() to create a real copy
const original = ["Task A", "Task B", "Task C"];

const copy1 = [...original];    // spread — most modern, most common
const copy2 = original.slice(); // slice() with no arguments — classic approach

copy1.push("Task D");
console.log(original); // ["Task A", "Task B", "Task C"]  ← untouched!
console.log(copy1);    // ["Task A", "Task B", "Task C", "Task D"]

console.log(copy1 === original); // false — they are different arrays

CAUTION: Both [...arr] and arr.slice() create a shallow copy. If your array contains objects, the objects themselves are still shared by reference. To fully clone an array of objects, use structuredClone(array) (modern, supported in all current browsers) or JSON.parse(JSON.stringify(array)) (older technique with limitations).

LISTING 24.25 — Shallow Copy vs. Deep Copy Caveat 
const tasks = [
  { text: "Buy milk", done: false },
  { text: "Walk dog", done: false },
];

// Shallow copy — the array is new, but the objects inside are shared
const shallow = [...tasks];
shallow[0].done = true; // modifying the object inside shallow...
console.log(tasks[0].done); // true  ← tasks[0] changed too! Same object.

// Deep copy — every object is also cloned
const deep = structuredClone(tasks);
deep[0].done = false;
console.log(tasks[0].done); // true  ← unchanged — deep[0] is a new object

Summary

  • Arrays are ordered, zero-indexed collections. The last valid index is always array.length - 1. Use array.at(-1) for the last element instead of array[array.length - 1].
  • Create arrays with the literal syntax []. Use Array.from() to convert array-like values — especially NodeLists from querySelectorAll() — into real arrays with the full method toolkit.
  • Mutating methods modify the original array: push / pop add and remove from the end; unshift / shift add and remove from the beginning; splice inserts, removes, or replaces at any position; reverse and sort reorder in place.
  • Non-mutating methods return a new array and leave the original untouched: slice, concat, and the ES2023 methods toReversed(), toSorted(), toSpliced(), and with().
  • Always use a compare function with sort() for numbers: (a, b) => a - b ascending, (a, b) => b - a descending. Without it, numbers are sorted as strings and the result is wrong.
  • Use includes() for boolean presence checks, findLast() to find the last matching element, and join() to build display strings and HTML from arrays.
  • The spread operator ([...arr]) creates a shallow copy and is the standard way to merge and non-mutably update arrays. Array destructuring unpacks values into named variables and pairs with rest syntax (...rest) to collect remainders.
  • The mutation trap (const arr2 = arr1) makes both variables point to the same array. Fix it with [...arr1] or arr1.slice() for a shallow copy, or structuredClone(arr1) for a full deep copy of objects inside the array.

Q & A

What is the difference between push() and unshift()?
push() adds one or more items to the end of the array and returns the new length. unshift() adds one or more items to the beginning of the array, shifting all existing items up by one index, and also returns the new length. Use push() when order doesn't require the new item to be first (most common); use unshift() when you need to prepend, such as adding an urgent item to the top of a task list. Note that unshift() is slower on large arrays because every existing element must be re-indexed.
When should I use a non-mutating method instead of a mutating one?
Use a non-mutating method whenever you need to preserve the original array. Common scenarios include: working with React state (you must never mutate state directly), keeping an original dataset so the user can reset a filtered or sorted view, and passing an array to a function you don't control and not wanting that function to modify your data. The ES2023 methods (toReversed, toSorted, toSpliced, with) plus slice(), concat(), and spread all return new arrays without touching the original.
How do I sort numbers correctly?
Always pass a compare function: array.sort((a, b) => a - b) for ascending order and array.sort((a, b) => b - a) for descending. Without a compare function, sort() converts numbers to strings and sorts by Unicode character code, which puts 100 before 9 because "1" comes before "9" alphabetically. The subtraction trick works because the compare function needs to return a negative number (a before b), zero (same), or positive (b before a) — and a - b naturally produces those results.
What does Array.isArray() do that typeof can't?
typeof [] returns "object" because arrays are objects in JavaScript. This makes typeof useless for distinguishing arrays from plain objects, null, or other object-type values. Array.isArray(value) performs a proper check and returns true only for actual arrays. Always use it when you receive data from an external source (API, form submission, function argument) and need to confirm the value is an array before calling array methods on it.
What is the mutation trap and how do I avoid it?
The mutation trap occurs when you write const arr2 = arr1. This does not copy the array — it creates a second variable pointing to the exact same array in memory. Any change made through either variable affects the one shared array. Avoid it by making an explicit copy: const arr2 = [...arr1] (spread) or const arr2 = arr1.slice() are the two standard approaches for a shallow copy. If your array contains objects and you need those objects to also be independent copies, use structuredClone(arr1) for a deep clone.
What does at(-1) do?
array.at(-1) returns the last element of the array. Negative arguments to at() count from the end: at(-1) is the last item, at(-2) is the second-to-last, and so on. It is a cleaner alternative to the older array[array.length - 1] pattern. at() also accepts positive indices, where it behaves identically to bracket notation.
How is splice() different from slice()?
Despite the similar names, they serve opposite purposes. splice(start, deleteCount, ...items) is a mutating method that modifies the original array — it can delete, insert, or replace elements at any position. slice(start, end) is a non-mutating method that extracts a portion of the array into a new array without touching the original. A memory aid: splice modifies in place; slice creates a subset copy.
When would I use findLast() instead of find()?
Use findLast() when you need the last element matching a condition rather than the first. For example, if you have a list of log entries and want the most recent error, findLast(entry => entry.type === "error") returns the last error entry without reversing the array first. find() searches from index 0 forward and returns the first match. If duplicates exist in your data and the most recent or last occurrence is the meaningful one (as is common with time-sorted data), findLast() is the right choice.
