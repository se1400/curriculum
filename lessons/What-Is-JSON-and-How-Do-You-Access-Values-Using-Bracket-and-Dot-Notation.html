What Is JSON?

JSON = JavaScript Object Notation

JSON is a lightweight, text-based data format commonly used to exchange data between a server and a web application.

Why JSON Is Popular

  • Machine-parseable: Easy for computers to read and generate
  • Human-readable: Easy for humans to read and write
  • Language-independent: Works across different programming languages
  • Lightweight: Minimal syntax, fast to transmit

Language Independence

Cross-Platform Data Exchange:

You can easily send JSON data:
• From a Java application to a Python application
• From a JavaScript application to a C# application
• Between any programming languages

JSON Supported Data Types

What Can JSON Store?

JSON supports six data types:

Objects

{"name": "Alice"}

Arrays

[1, 2, 3, 4]

Strings

"Hello World"

Numbers

42, 3.14, -5

Booleans

true, false

null

null
Versatile Format:

JSON can store many data types, including arrays and nested objects, making it perfect for complex data structures.

JSON Syntax

JSON Object Example

{
    "name": "Alice",
    "age": 30,
    "isStudent": false,
    "list of courses": ["Mathematics", "Physics", "Computer Science"]
}

Key Syntax Rules

1. Key-Value Pairs

JSON uses key-value pairs to store information: "key": value

2. Double Quotes Required

Each key must be wrapped in double quotes, otherwise you'll get an error

3. Comma Separation

Each pair is separated by a comma (no trailing comma after last item)

❌ Common Error:
{name: "Alice"} ← ERROR! Keys need double quotes
{"name": "Alice"} ← CORRECT!

Accessing with Dot Notation

Using Dot Notation

To access data from a JSON object, you can use dot notation. 

import data from "./example.json" with { type: "json" };

console.log(data.age); // 30
30

Visual Breakdown

JSON Object:
{
  "name": "Alice",
  "age": 30,
  "isStudent": false
}
↓
data.age → Returns 30
Note on Import Statement:

The import statement imports the JSON object into your file so you have access to it. You will learn more about imports in future lessons.

Accessing with Bracket Notation

Using Bracket Notation

You can also use bracket notation to access information from JSON objects. 

import data from "./example.json" with { type: "json" };

console.log(data["list of courses"]);
["Mathematics", "Physics", "Computer Science"]

Why Bracket Notation Here?

⚠️ Spaces in Keys:

Using bracket notation is particularly useful when the key contains multiple words separated by spaces.

data.list of courses ← ERROR!
data["list of courses"] ← CORRECT!

Both Work for Simple Keys

Dot Notation
data.name
data.age
Bracket Notation
data["name"]
data["age"]
Both produce the same result for keys without spaces!

Dot vs Bracket Notation

Decision Guide

Use Dot Notation

When:
  • Key has no spaces
  • Key is a valid identifier
  • You want cleaner syntax
data.name
data.age
data.isStudent

Use Bracket Notation

When:
  • Key has spaces
  • Key has special characters
  • Using variables
data["list of courses"]
data["first-name"]
data[keyVariable]

Examples Table

Key Dot Notation Bracket Notation
"age" ✓ data.age ✓ data["age"]
"first name" ✗ Error ✓ data["first name"]
"user-id" ✗ Error ✓ data["user-id"]

Summary: JSON & Accessing Values

What We Learned

JSON Basics:
  • Lightweight, text-based data format
  • Machine-parseable and human-readable
  • Language-independent
  • Supports: objects, arrays, strings, numbers, booleans, null
JSON Syntax:
  • Key-value pairs: "key": value
  • Keys must use double quotes
  • Pairs separated by commas
Accessing Values:
  • Dot notation: data.key (simple keys)
  • Bracket notation: data["key"] (keys with spaces/special chars)

Quick Reference

JSON Example:
{
  "name": "Alice",
  "age": 30,
  "list of courses": ["Math", "Physics"]
}
✓ Correct:
data.name → "Alice"
data.age → 30
data["list of courses"] → ["Math", "Physics"]
✗ Error:
data.list of courses → Syntax Error!
Key Takeaway:

JSON is a versatile format for storing and exchanging data. By using dot notation or bracket notation, you can easily access the values stored within a JSON object. Choose bracket notation when keys contain spaces or special characters!
