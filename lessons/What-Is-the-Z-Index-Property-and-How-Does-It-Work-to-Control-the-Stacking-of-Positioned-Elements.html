The Z-Index Property

The z-index property in CSS is used to control the vertical stacking order of positioned elements that overlap on the page.

Think of it as Layers:

When multiple elements are stacked on top of each other, the z-index value determines which element appears on top.

How It Works:

  • Higher z-index = Closer to the viewer (on top)
  • Lower z-index = Farther back in the stack (behind)
  • Think of the z-axis coming out of the screen toward you

The Z-Axis Concept

Imagine a 3D coordinate system:

X X-axis: Left to right (horizontal)
Y Y-axis: Top to bottom (vertical)
Z Z-axis: Front to back (depth) - This is what z-index controls!

Visual metaphor: Imagine stacking sheets of paper. The z-index determines which sheet is on top of the pile!

Critical Requirement

Here's the most important rule about z-index:

⚠️ z-index ONLY Works on Positioned Elements!

The element must have a position value other than static, such as:

  • position: relative ✓
  • position: absolute ✓
  • position: fixed ✓
  • position: sticky ✓
  • position: static ❌ (z-index has no effect!)

Default value: The default z-index value is auto, which places the element in the default stacking order.

Basic Example

Let's create three overlapping boxes with different z-index values:

HTML
<div class="container">
  <div class="box1">Box 1</div>
  <div class="box2">Box 2</div>
  <div class="box3">Box 3</div>
</div>
CSS
.container {
  position: relative;
}

.box1 {
  position: absolute;
  z-index: 1; /* Back */
}

.box2 {
  position: absolute;
  z-index: 2; /* Middle */
}

.box3 {
  position: absolute;
  z-index: 3; /* Front */
}

Negative Z-Index Values

Z-index can be negative! This places elements behind normally positioned content:

CSS
.background-element {
  position: absolute;
  z-index: -1; /* Behind everything */
}

.normal-element {
  position: relative;
  z-index: 0; /* Default stacking level */
}

.top-element {
  position: absolute;
  z-index: 10; /* On top */
}

Z-Index Range:

  • Can be any integer: -999, 0, 1, 999, 9999
  • Negative values go behind
  • Positive values come forward
  • auto is the default (equivalent to 0)

Default Stacking Order

When elements have the same z-index (or no z-index), the default stacking order is:

1 Background of the parent element (bottom)
2 Positioned elements with negative z-index
3 Non-positioned elements in order they appear in HTML
4 Positioned elements with z-index: auto or z-index: 0
5 Positioned elements with positive z-index (top)

Important: Elements that appear later in the HTML will stack on top of earlier elements if they have the same z-index!

Stacking Contexts

This is one of the most confusing aspects of z-index:

⚠️ Stacking Context:

A stacking context is a three-dimensional conceptualization of HTML elements. Certain CSS properties create new stacking contexts.

What Creates a Stacking Context:

  • The root element (<html>)
  • Elements with position: absolute or relative and z-index other than auto
  • Elements with position: fixed or sticky
  • Elements with opacity less than 1
  • Elements with transform, filter, etc.

Stacking Context Example

This is why z-index doesn't always work as expected:

HTML
<div class="parent1"> <!-- z-index: 1 -->
  <div class="child1"></div> <!-- z-index: 9999 -->
</div>

<div class="parent2"> <!-- z-index: 2 -->
  <div class="child2"></div> <!-- z-index: 1 -->
</div>

⚠️ Surprising Result:

Even though child1 has z-index: 9999, it will appear behind child2 because parent1 has a lower z-index than parent2!

Rule: Child elements cannot break out of their parent's stacking context. The parent's z-index determines the stacking order of the entire group!

Common Use Cases

Modal Overlays
z-index: 9999;

Needs to appear above everything else on the page

Fixed Headers
z-index: 1000;

Above content but below modals

Dropdown Menus
z-index: 100;

Above normal content

Tooltips
z-index: 500;

Above most content

Background Elements
z-index: -1;

Behind all normal content

Sticky Sidebars
z-index: 10;

Slightly above content

Z-Index Scale Best Practice

Use a consistent scale across your site:

Example Scale
/* Recommended z-index scale */

/* Background layers */
.background { z-index: -1; }

/* Normal content */
.content { z-index: 1; }

/* Positioned elements */
.positioned { z-index: 10; }

/* Dropdowns, tooltips */
.dropdown { z-index: 100; }

/* Sticky elements */
.sticky-header { z-index: 1000; }

/* Modals, overlays */
.modal { z-index: 9999; }

Tip: Use spacing (1, 10, 100, 1000) to leave room for intermediate values!

Practical Example: Modal

HTML
<div class="modal-overlay">
  <div class="modal-content">
    <h2>Modal Title</h2>
    <p>Modal content...</p>
  </div>
</div>
CSS
.modal-overlay {
  position: fixed;
  top: 0;
  left: 0;
  width: 100%;
  height: 100%;
  background: rgba(0, 0, 0, 0.5);
  z-index: 9999; /* Above everything */
}

.modal-content {
  position: relative;
  z-index: 10000; /* Above overlay */
}

Practical Example: Fixed Header

HTML
<header class="site-header">
  <nav>Navigation...</nav>
</header>

<main class="content">
  <!-- Page content -->
</main>
CSS
.site-header {
  position: fixed;
  top: 0;
  left: 0;
  width: 100%;
  z-index: 1000; /* Above content, below modals */
  background: white;
  box-shadow: 0 2px 5px rgba(0,0,0,0.1);
}

.content {
  position: relative;
  z-index: 1; /* Normal stacking level */
}

Common Pitfalls

❌ Mistake 1: Using on Static Elements

/* This won't work! */
.element {
  position: static; /* or no position set */
  z-index: 999; /* Has no effect! */
}

Fix: Add a position property other than static!

❌ Mistake 2: Z-Index Wars

Using increasingly large numbers: 999999999

Fix: Use a consistent scale and understand stacking contexts!

❌ Mistake 3: Ignoring Stacking Contexts

Not realizing that a parent's stacking context limits child elements

Fix: Understand that children can't escape their parent's stacking level!

Debugging Z-Index Issues

✓ Debugging Steps:

  • Check position: Is the element positioned? (not static)
  • Check stacking contexts: Look at parent elements' z-index
  • Use DevTools: Inspect computed styles and stacking order
  • Temporarily use extreme values: Try z-index: 999999 to test
  • Check for transforms/opacity: These create stacking contexts

Browser DevTools Tip:

Most modern browsers show the stacking order and stacking contexts in their DevTools. Use the Elements panel to inspect!

Z-Index vs Flexbox/Grid

Z-index works differently with modern layout methods:

Flexbox and Grid:

  • Flex and grid items can use z-index without needing position
  • Flex/grid containers create stacking contexts
  • Order property affects stacking in addition to z-index
Example with Flexbox
.flex-container {
  display: flex;
}

.flex-item {
  /* No position needed! */
  z-index: 10; /* This works! */
}

Best Practices

✓ Do These:

  • Use a consistent z-index scale across your site
  • Document your z-index values (comment your CSS)
  • Use spacing (1, 10, 100, 1000) for flexibility
  • Understand stacking contexts before using z-index
  • Test overlapping elements thoroughly
  • Use CSS variables for z-index values

❌ Avoid These:

  • Using z-index on static elements
  • Arbitrary large numbers (99999999)
  • Not considering stacking contexts
  • Using z-index as a quick fix without understanding the problem
  • Forgetting to position elements

CSS Variables for Z-Index

A modern approach using CSS custom properties:

CSS
:root {
  /* Define z-index scale */
  --z-background: -1;
  --z-default: 1;
  --z-elevated: 10;
  --z-dropdown: 100;
  --z-sticky: 1000;
  --z-modal: 9999;
}

/* Use throughout your CSS */
.header {
  z-index: var(--z-sticky);
}

.modal {
  z-index: var(--z-modal);
}

.dropdown {
  z-index: var(--z-dropdown);
}

Benefits: Centralized management, easy to update, self-documenting!

Key Takeaways

What Z-Index Does:

  • Controls vertical stacking order (depth)
  • Higher values = closer to viewer = on top
  • Lower values = farther back = behind
  • Can be negative, zero, or positive

Critical Requirements:

  • MUST have position other than static!
  • Works with: relative, absolute, fixed, sticky
  • Does NOT work with: static (default)
  • Also works with flex/grid items without position

Stacking Contexts:

  • Parent stacking context contains child elements
  • Children can't escape parent's stacking level
  • Created by positioned elements with z-index, opacity, transforms, etc.
  • Understanding contexts prevents most z-index problems

Best Practices:

  • Use consistent scale (1, 10, 100, 1000, 9999)
  • Document your z-index values
  • Consider CSS variables for centralized management
  • Understand stacking contexts before debugging
