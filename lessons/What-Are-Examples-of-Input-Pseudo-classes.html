Input Elements Matter

The appearance and behavior of input elements matter when building web forms.

Why It Matters:

  • Forms should respond to user actions
  • Improves user experience
  • Should not cause confusion or frustration
  • Provides visual feedback and guidance
CSS provides several input pseudo-classes that make forms more intuitive and user-friendly

:focus Pseudo-class

The :focus pseudo-class targets an input element when a user selects or clicks on it.

Purpose:

  • Draws attention to the active input field
  • Helps users identify where they're currently typing
  • Provides clear visual feedback
  • Essential for accessibility

CSS Example:

input:focus {
  border-color: #881518;
  box-shadow: 0 0 0 3px rgba(136, 21, 24, 0.1);
}

:hover Pseudo-class

The :hover pseudo-class is triggered when the user places the cursor over an element.

Benefits:

  • Provides visual feedback before interaction
  • Alerts users that the input is ready for action
  • Improves discoverability of interactive elements
  • Enhances user confidence

CSS Example:

input:hover {
  background-color: orange;
}

The background changes to orange when you hover

:checked Pseudo-class

The :checked pseudo-class styles radio buttons or checkboxes differently when selected.

Purpose:

  • Users can easily see which option they've chosen
  • Provides clear selection feedback
  • Enables custom checkbox/radio designs
  • Improves form usability

CSS Example:

.checkbox:checked {
  background-color: green;
  border-color: green;
}

:required Pseudo-class

The :required pseudo-class targets input elements that have the required attribute.

Purpose:

  • Signals to users that a field must be filled out
  • Indicates mandatory fields before submission
  • Prevents form submission with empty required fields
  • Improves form completion rates

HTML Example:

<input type="text" required />

CSS Example:

input:required {
  border-color: #881518;
  background-color: #fff5f5;
}

Required fields have a red border and pink background

:valid Pseudo-class

The :valid pseudo-class styles input fields that meet validation criteria.

When It Applies:

  • Input value matches the expected format
  • Required fields are filled
  • Email addresses are properly formatted
  • Numbers are within acceptable ranges

CSS Example:

input:valid {
  border-color: green;
  background-color: #f0fff0;
}

Try typing a valid email address to see the green styling

:invalid Pseudo-class

The :invalid pseudo-class styles input fields that do not meet validation criteria.

When It Applies:

  • Input value doesn't match expected format
  • Required fields are empty
  • Email addresses are improperly formatted
  • Numbers are outside acceptable ranges

CSS Example:

input:invalid {
  border-color: red;
  background-color: #fff0f0;
}

Invalid emails show a red border and pink background

:valid vs :invalid

When validating forms, use :valid and :invalid to provide instant feedback.

:valid - Green Styling

Indicates the input meets all validation requirements

border-color: green;

:invalid - Red Styling

Indicates the input does not meet validation requirements

border-color: red;
Typically use green for valid input and red for invalid input to provide clear visual feedback

:disabled Pseudo-class

The :disabled pseudo-class selects and styles input elements that are disabled.

Characteristics:

  • Elements have the disabled attribute
  • Prevents users from interacting with them
  • Cannot be clicked, focused, or edited
  • Often styled with grayed-out appearance

CSS Example:

.text-input:disabled {
  background-color: lightgray;
  cursor: not-allowed;
}

Try clicking the input - it won't respond

cursor: not-allowed

The cursor: not-allowed CSS property changes the cursor appearance to indicate an action is not permitted.

How It Works:

When applied to an element, it shows a cursor with a circle-slash symbol (⊘) to signal the element is not interactive.

Hover over this box to see cursor: not-allowed

Common Uses:

  • Disabled buttons
  • Disabled input fields
  • Non-interactive elements that look interactive
  • Elements that are temporarily unavailable

Other Input Pseudo-classes

:autofill

Applies styles to input fields that the browser automatically fills with saved data

:optional

Styles input elements that are not required and can be left empty

:in-range

Styles elements when their values are within specified range constraints

:out-of-range

Styles elements when their values are outside specified range constraints

Examples:

input:autofill { background-color: #e8f4f8; }
input:optional { border-style: dashed; }
input:in-range { border-color: green; }
input:out-of-range { border-color: red; }

Input Pseudo-classes Summary

Key Takeaways:

  • Input appearance matters for good form design
  • :focus - Highlights the active input field
  • :hover - Provides feedback before interaction
  • :checked - Styles selected checkboxes/radio buttons
  • :required - Indicates mandatory fields
  • :valid - Shows inputs that meet validation (green)
  • :invalid - Shows inputs that fail validation (red)
  • :disabled - Styles non-interactive inputs
  • Other options: :autofill, :optional, :in-range, :out-of-range
Input pseudo-classes make forms more intuitive and user-friendly without requiring JavaScript
