CSS Floats

Floats in CSS are a technique originally designed to allow text to wrap around an element, such as an image.

A Brief History:

Over time, developers found new ways to use floats, applying them to layout design in creative ways. While modern layout methods like Flexbox and Grid are now more commonly used, understanding floats is still important!

Why Learn Floats Today?

  • Working with older code or legacy projects
  • Achieving specific layout effects
  • Understanding CSS history and evolution
  • Text wrapping around images (still commonly used)

How Floats Work

When an element is floated, two important things happen:

Normal Flow
(Element in document)
→
Float Applied
(Removed from flow)
→
Pushed Left/Right
(Content wraps)

Key Behavior:

  • The element is taken out of the normal document flow
  • It's pushed to the left or right of its container
  • Content that follows will wrap around the floated element
  • Fills the remaining space naturally

Float Values

The float property accepts several values:

CSS
element {
  float: left; /* Float to the left */
  float: right; /* Float to the right */
  float: none; /* Don't float (default) */
}

What Each Value Does:

  • left - Element floats to the left side, content wraps on the right
  • right - Element floats to the right side, content wraps on the left
  • none - Element stays in normal flow (removes float)

Classic Use Case: Text Wrapping

One classic use is wrapping text around images, where an image is floated to one side and text wraps around it.

This technique is still widely used, especially in articles and blogs where images need to be placed alongside text.

HTML
<div class="container">
  <img src="image.jpg" alt="Image">
  <p>This is text flowing around a floated image.</p>
</div>
CSS
img {
  float: left;
  margin-right: 15px;
  margin-bottom: 15px;
}

The Collapsing Parent Problem

When you float elements, a serious problem can occur:

⚠️ The Problem:

The container doesn't wrap around the floated element. The image is out of the normal document flow, and the container collapses to zero height because it doesn't "see" the floated children elements.

HTML
<div class="container">
  <img src="image.jpg" alt="Image">
</div>
CSS
.container {
  border: 3px solid red;
}

img {
  float: left;
}

Visual: Collapsed Container

Without clearfix, the container collapses:

Image

Notice: The red dashed border shows the collapsed container. It has basically zero height because it can't "see" the floated image!

Why this happens: Floated elements are removed from the normal document flow, so the parent element doesn't account for their height.

The clear Property

The clear property is used to control the behavior of elements after floated elements.

CSS
element {
  clear: left; /* Clear left floats */
  clear: right; /* Clear right floats */
  clear: both; /* Clear both left and right floats */
  clear: none; /* Don't clear (default) */
}

What clear Does:

  • clear: both - Element moves below all floated elements
  • clear: left - Element moves below left-floated elements
  • clear: right - Element moves below right-floated elements

The Clearfix Technique

The clearfix technique is the solution to the collapsing parent problem!

CSS - The Clearfix
.clearfix::after {
  content: "";
  display: block;
  clear: both;
}

✓ How to Use:

Simply add the clearfix class to any container with floated children:

HTML
<div class="container clearfix">
  <img src="image.jpg" alt="Image">
</div>

Clearfix: Step-by-Step

Let's break down how the clearfix technique works:

1 ::after is a pseudo-element that adds an invisible block after the content of the container
2 content: "" ensures the pseudo-element is present but doesn't display any content
3 display: block makes the pseudo-element a block-level element
4 clear: both ensures the pseudo-element clears both sides of any floated elements above it

Result: The parent element now "sees" the floated children and wraps around them properly!

Visual: Fixed with Clearfix

With clearfix applied, the container wraps properly:

Image

Success! The green border shows the container now properly wraps around the floated image. The clearfix technique forces the parent container to "see" the floated child elements.

The Fix Applied
.container {
  border: 3px solid green;
}

.clearfix::after {
  content: "";
  display: block;
  clear: both;
}

Historical Use: Multi-Column Layouts

Floats were popular for creating multi-column layouts before Flexbox and Grid became widespread.

How It Worked:

By floating elements next to each other, developers could create columns that aligned horizontally.

CSS - Three Column Layout
.column {
  float: left;
  width: 33.33%;
  padding: 15px;
}

.container {
  /* Apply clearfix */
}

.clearfix::after {
  content: "";
  display: block;
  clear: both;
}

Modern Alternatives

While floats are no longer the go-to method for complex layouts, we now have better options:

Past: Floats

Used for both text wrapping AND layout

Problems: Collapsing parents, clearfix needed, inflexible

Modern: Flexbox

One-dimensional layouts (rows or columns)

Benefits: Alignment, distribution, no clearfix needed

Modern: CSS Grid

Two-dimensional layouts (rows AND columns)

Benefits: Complex layouts, precise control, responsive

Today: Floats Still Used

Primarily for text wrapping around images

Best use: Articles, blogs, magazine-style layouts

When to Use Floats Today

✓ Good Use Cases for Floats:

  • Text wrapping around images - Still the best solution!
  • Legacy browser support - Works in very old browsers
  • Magazine-style layouts - Where text flows around content
  • Maintaining old codebases - Understanding existing code

⚠️ Don't Use Floats For:

  • Page layouts - Use Flexbox or Grid instead
  • Navigation bars - Flexbox is much better
  • Card grids - Use CSS Grid
  • Centering elements - Use Flexbox
  • Responsive layouts - Grid and Flexbox are more flexible

Common Pitfalls and Solutions

Problem 1: Collapsed Parent

Solution: Use the clearfix technique!

Problem 2: Text Not Wrapping

Solution: Make sure you have enough text and the floated element has appropriate margins.

Problem 3: Elements Breaking Layout

Solution: Use clear: both on elements that should appear below floated elements.

Problem 4: Width Issues

Solution: Use box-sizing: border-box to include padding in width calculations.

Best Practices

✓ Do These:

  • Always use clearfix on containers with floated children
  • Add margins to floated elements for spacing
  • Use floats primarily for text wrapping
  • Test in different screen sizes
  • Consider using Flexbox/Grid for layouts
  • Document when and why you're using floats

❌ Avoid These:

  • Floating elements without clearfix
  • Using floats for complex layouts
  • Forgetting margins on floated elements
  • Nesting too many floats
  • Using floats when Flexbox/Grid would be better

Key Takeaways

What Floats Do:

  • Remove elements from normal document flow
  • Push elements to the left or right
  • Allow content to wrap around them
  • Originally designed for text wrapping around images

The Collapsing Parent Problem:

  • Floated elements don't contribute to parent height
  • Parent containers collapse to zero height
  • Solution: Use the clearfix technique

The Clearfix Technique:

  • Uses ::after pseudo-element
  • Adds invisible block with clear: both
  • Forces parent to wrap around floated children
  • Essential when using floats!

Modern Context:

  • Still useful for text wrapping around images
  • Use Flexbox/Grid for layouts instead
  • Important for legacy code and understanding CSS
