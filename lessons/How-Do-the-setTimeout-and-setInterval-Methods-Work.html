setTimeout and setInterval

Timing in JavaScript

When building programs, you'll often want something to happen after a delay, or repeat at regular intervals.

Two Essential Methods

setTimeout()

Runs code once after a delay

setInterval()

Runs code repeatedly at intervals

Common Parameters

Both methods accept two parameters: a function and a delay (in milliseconds)

setTimeout() Method

What Does It Do?

setTimeout() lets you delay an action for a specified time.

Use Cases

  • Display Messages: Show a notification after a delay
  • Delayed Actions: Wait before triggering an operation
  • Timeouts: Give users time to complete an action
  • Animations: Start an effect after a pause

One-Time Execution

The function runs exactly once after the specified delay.

setTimeout() Syntax

setTimeout(functionToRun, delay);

Parameters

functionToRun

The code you want to execute after the delay

delay

The time in milliseconds to wait before running the function

Time Conversion

  • 1000 milliseconds = 1 second
  • 3000 milliseconds = 3 seconds
  • 60000 milliseconds = 1 minute

setTimeout() Example

Using a Regular Function

setTimeout(function() {
  console.log("This runs after 3 seconds");
}, 3000);

Result

// Wait 3 seconds...
This runs after 3 seconds

What Happens?

  • JavaScript waits for 3000 milliseconds (3 seconds)
  • After the delay, the function executes
  • The message is logged to the console
  • The setTimeout completes and doesn't run again

setTimeout() with Arrow Function

Modern Syntax

You can use an arrow function for cleaner, more concise code.

Arrow Function Syntax

setTimeout(() => {
  console.log("This runs after 3 seconds");
}, 3000);

Result

// Wait 3 seconds...
This runs after 3 seconds

Both regular functions and arrow functions work the same way. Arrow functions are just a more modern, concise syntax.

setInterval() Method

What Does It Do?

setInterval() keeps repeating a function at a set interval.

It's like having an alarm that goes off every few seconds.

Use Cases

  • Refreshing Data: Update information periodically
  • Animations: Create continuous movement
  • Clocks/Timers: Update time displays
  • Polling: Check for updates regularly
  • Auto-Save: Save user data at intervals

Continuous Execution

The function runs over and over at the specified interval until stopped.

setInterval() Syntax

setInterval(functionToRun, delay);

Parameters

functionToRun

The code you want to run repeatedly

delay

The time in milliseconds between each execution

Important Difference

Unlike setTimeout() which runs once, setInterval() keeps running until you explicitly stop it.

setInterval() Example

Code

setInterval(() => {
  console.log("This runs every 2 seconds");
}, 2000);

Result

This runs every 2 seconds
This runs every 2 seconds
This runs every 2 seconds
This runs every 2 seconds
...

What Happens?

  • The function executes immediately, then waits 2 seconds
  • After 2 seconds, it runs again
  • This continues indefinitely
  • It will keep running until you stop it with clearInterval()

Stopping setInterval with clearInterval()

Why Stop It?

Since setInterval() keeps executing forever, you'll often want to stop it at some point.

The clearInterval() Method

To stop an interval, use clearInterval() with the interval ID.

Getting the ID

setInterval() returns an ID that you can store in a variable:

const intervalID = setInterval(() => {
  console.log("Running...");
}, 1000);

// Later, stop it:
clearInterval(intervalID);

Auto-Stop with setTimeout

Combining Methods

You can use setTimeout() to automatically stop a setInterval() after a certain time.

Example Code

const intervalID = setInterval(() => {
  console.log("This will stop after 5 seconds");
}, 1000);

setTimeout(() => {
  clearInterval(intervalID);
}, 5000);

What Happens?

  • The interval runs every 1 second (1000ms)
  • After 5 seconds (5000ms), setTimeout triggers
  • clearInterval stops the repeating function
  • No more messages appear

Stopping setTimeout with clearTimeout()

Cancel Before It Runs

You can also stop a timeout before it executes using clearTimeout().

Example: Cancel Immediately

let timeoutID = setTimeout(() => {
  console.log("This will not run");
}, 5000);

clearTimeout(timeoutID); // Cancel it immediately

Result

// Nothing is logged - the timeout was cancelled

The setTimeout() will not run at all because it was cleared before the delay expired.

Practical clearTimeout Example

User-Controlled Cancellation

A more practical approach: let users cancel a timeout by clicking a button.

HTML

<h1>Cancel Timeout Example</h1>
<button id="cancelButton">Cancel Timeout</button>

JavaScript

let timeoutID = setTimeout(() => {
  console.log("This runs after 5 seconds");
}, 5000);

const cancelBtn = document.getElementById("cancelButton");
cancelBtn.addEventListener("click", () => {
  clearTimeout(timeoutID);
  console.log("Timeout cancelled!");
});

setTimeout vs setInterval

setTimeout()

  • Runs once
  • After a delay
  • Then stops automatically
  • Use for single delayed actions
setTimeout(() => {
  // Runs once
}, 3000);

setInterval()

  • Runs repeatedly
  • At regular intervals
  • Continues until stopped
  • Use for repeating actions
setInterval(() => {
  // Runs forever
}, 3000);

Clearing Timers

clearTimeout()

Cancels a setTimeout() before it executes

let id = setTimeout(...);
clearTimeout(id);

clearInterval()

Stops a setInterval() from continuing

let id = setInterval(...);
clearInterval(id);

Remember the ID

Both timer methods return an ID. Store this ID in a variable so you can clear the timer later.

Summary

Key Concepts

setTimeout()

  • Delays an action for a specified time
  • Runs the function once
  • Cancel with clearTimeout()

setInterval()

  • Repeats a function at a set interval
  • Runs continuously until stopped
  • Stop with clearInterval()

Both Methods Share

  • Accept two parameters: function and delay (in milliseconds)
  • Return an ID that can be used to cancel them
  • Delay time is in milliseconds (1000ms = 1 second)

Use Them Wisely

These methods are essential for creating dynamic, time-based functionality in your web applications.
