What Are Link States?

You may have seen a link on a web page become purple after you click it. This is because the state of the link has changed, so different styling gets applied.

There are five different states a link can be in. Each state represents a different way users interact with links, and you can style each one differently to provide visual feedback.

Five Link States
a:link { }     /* Default state */
a:visited { }  /* Already visited */
a:hover { }    /* Mouse hovering */
a:focus { }    /* Focused element */
a:active { }   /* Being clicked */

:link (Default State)

The :link state is the default state. It represents a link which the user has not visited, clicked, or interacted with yet.

You can think of this state as providing the base styles for all links on your page. The other states build on top of it.

By default, links in this state are typically blue.

HTML
<a href="https://se1400.github.io"
   target="_blank">
  Visit SE1400
</a>
CSS for Default State
a:link {
  color: blue;
  text-decoration: underline;
}

:visited State

The :visited state applies when a user has already visited the page being linked to.

By default, this turns the link purple—but you can use CSS to provide a different visual indication to the user.

This is helpful to let someone know they have already read a portion of your documentation, or that the site is one they can trust because they have used it before.

CSS for Visited State
a:visited {
  color: purple;
}

/* Custom visited color */
a:visited {
  color: brown;
}
Visual Feedback
/* Before clicking */
Visit SE1400

/* After clicking and visiting */
Visit SE1400

:hover State

The :hover state applies when a user is hovering their cursor over a link.

This state is helpful for providing extra attention to a link, to ensure a user actually intends to click it.

Hover effects give immediate visual feedback that an element is interactive.

CSS for Hover State
a:hover {
  color: red;
  text-decoration: none;
}

/* Other hover effects */
a:hover {
  background-color: yellow;
  cursor: pointer;
}
User Experience
/* When mouse hovers over link */
The link changes appearance
to indicate interactivity

Common hover effects:
- Color change
- Underline removal
- Background highlight
- Cursor change

:focus State

The :focus state applies when we focus on a link.

This typically happens when using keyboard navigation (like pressing the Tab key) to move through interactive elements on a page.

The focus state is important for accessibility, especially for users who navigate with keyboards instead of mice.

CSS for Focus State
a:focus {
  color: green;
  outline: 2px solid green;
}

/* Accessibility-friendly focus */
a:focus {
  background-color: lightyellow;
  outline: 3px solid orange;
}
Keyboard Navigation
/* User presses Tab key */
→ Focus moves to the link
→ Link styling changes
→ Visual indicator appears

Important for:
- Keyboard users
- Screen reader users
- Accessibility compliance

:active State

The :active state applies to links that are being activated by the user.

This typically means clicking on the link with the primary mouse button (left clicking in most cases).

This state can be helpful for showing a user that the element they clicked on is interactive. This happens very quickly, so it's a brief visual moment.

CSS for Active State
a:active {
  color: black;
}

/* Other active effects */
a:active {
  transform: scale(0.98);
  background-color: darkred;
  color: white;
}
Click Moment
/* During the click action */
Mouse button pressed down
     ↓
Link enters :active state
     ↓
Visual feedback appears
     ↓
Mouse button released
     ↓
Navigation begins

CSS Styling Order

When you use these states to style your links, there is a specific order you need to write your CSS in:

link → visited → hover → focus → active

This order ensures that styles cascade properly and that each state displays correctly. Writing them in the wrong order can cause some states to not work as expected.

You should now know how to give links on your page your own personal flair, while still providing important information a user needs about the state of each link.

Correct Order
/* 1. Default state */
a:link {
  color: blue;
}

/* 2. Visited state */
a:visited {
  color: purple;
}

/* 3. Hover state */
a:hover {
  color: red;
}

/* 4. Focus state */
a:focus {
  color: green;
}

/* 5. Active state */
a:active {
  color: black;
}
