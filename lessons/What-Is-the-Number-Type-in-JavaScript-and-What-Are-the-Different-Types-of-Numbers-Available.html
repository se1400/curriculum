The Number Type in JavaScript

The Number type is one of the most used data types in JavaScript and other programming languages.

Number

What It Represents

In JavaScript, the Number data type represents a numeric value.

Numbers Might Seem Simple, But...

There's a lot to explore when it comes to numbers in JavaScript!

Let's take a deeper look at all the different types of numbers available.

JavaScript's Unified Number Type

Unlike many other programming languages that separate integers and floating-point numbers into different types, JavaScript uses one unified Number type.

What This Means

You can work with whole numbers, decimals, and even special numeric values all under the same Number data type umbrella.

Other Languages

  • int for integers
  • float for decimals
  • double for larger decimals
  • Multiple separate types

JavaScript

This unified approach makes JavaScript simpler to work with - you don't need to worry about choosing the right number type!

All Numbers Have Type "number"

Here's a basic example showing you that integers, floating point numbers, and negative numbers are all of type number:

const wholeNumber = 50;
const decimalNumber = 4.5;
const negativeNumber = -7;

console.log(typeof wholeNumber);    // "number"
console.log(typeof decimalNumber);  // "number"

Types of Numbers in JavaScript

JavaScript's Number type includes various kinds of numeric values, ranging from simple integers to special cases.

Main Types You'll Encounter

  • Integers - Whole numbers (100, -25, 0)
  • Floating-Point Numbers - Decimals (4.5, 89.56)
  • Infinity - Beyond maximum limit
  • NaN - "Not a Number"

Let's break down each of these main types!

Integers

What Are Integers?

Integers are whole numbers without any fractional or decimal part.

They can be positive, negative, or zero.

Examples:

const positiveInteger = 100;
const negativeInteger = -25;
const zero = 0;

console.log(typeof positiveInteger);  // "number"
console.log(typeof negativeInteger);  // "number"
console.log(typeof zero);             // "number"

Floating-Point Numbers

What Are Floating-Point Numbers?

Floating point numbers are numbers with decimal points.

They're often referred to as just "floats" by JavaScript developers.

When to Use Floats

Floats are useful when you need more precision, such as when you're dealing with:

  • Measurements
  • Currencies
  • Scientific calculations
  • Percentages

Examples:

const floatingPointNumber = 4.5;
const anotherFloat = 89.56;
const oneMoreFloat = 16.462;

console.log(typeof floatingPointNumber); // "number"
console.log(typeof anotherFloat);        // "number"
console.log(typeof oneMoreFloat);        // "number"
number number number

Infinity

JavaScript can represent numbers that are beyond the maximum limit with Infinity.

When You'll Encounter Infinity

  • When you try to divide a number by zero
  • On rare occasions, exceed the upper boundary of the Number type

Example:

const infiniteNumber = 1 / 0;
console.log(infiniteNumber);        // Infinity
console.log(typeof infiniteNumber); // "number"
Infinity number

More Infinity Examples

console.log(10 / 0);           // Infinity
console.log(-10 / 0);          // -Infinity
console.log(Infinity + 1);     // Infinity
console.log(Infinity * 2);     // Infinity

NaN - "Not a Number"

Sometimes in JavaScript, some mathematical operations don't result in a valid number.

When You Get NaN

If you try to perform a mathematical operation on something that isn't a number, you'll get NaN, which stands for "Not a Number".

Example:

const notANumber = 'hello world' / 2;
console.log(notANumber); // NaN
NaN

‚ö†Ô∏è Surprisingly...

The type of NaN is also "number"!

const notANumber = 'hello world' / 2;
console.log(typeof notANumber); // "number"
number

This might seem confusing, but NaN is a special numeric value indicating an invalid number.

More NaN Examples

Here are more situations where you'll encounter NaN:

Example 1: Invalid Mathematical Operations

console.log('text' * 5);           // NaN
console.log('hello' - 10);         // NaN
console.log(undefined + 5);        // NaN

Example 2: Invalid Conversions

console.log(parseInt('abc'));      // NaN
console.log(Number('xyz'));        // NaN
console.log(parseFloat('hello'));  // NaN

Example 3: Math Operations with NaN

console.log(NaN + 5);              // NaN
console.log(NaN * 2);              // NaN
console.log(Math.sqrt(-1));        // NaN (can't take square root of negative)

üí° Important Note

NaN is "contagious" - any mathematical operation with NaN results in NaN!

Numbers in Different Bases

Apart from the standard decimal system (base 10), JavaScript also supports numbers in different bases.

Hexadecimal Example

Hexadecimal uses digits 0 to 9 and letters a to f, like you see in CSS hex colors:

const hexColor = 0xFF5733;  // Same as decimal 16733491
console.log(hexColor);      // 16733491

Number Type Summary

Key Takeaways

JavaScript's Unified Number Type

Unlike many other programming languages, JavaScript uses one Number type for all numeric values - integers, decimals, and special values.

Types of Numbers

  • Integers: Whole numbers (100, -25, 0)
  • Floating-Point Numbers: Decimals (4.5, 89.56)
  • Infinity: Result of dividing by zero or exceeding limits
  • NaN: "Not a Number" - result of invalid operations

All Have Type "number"

console.log(typeof 42);           // "number"
console.log(typeof 3.14);         // "number"
console.log(typeof -7);           // "number"
console.log(typeof Infinity);     // "number"
console.log(typeof NaN);          // "number"

When to Use Different Types

  • Integers: Counting, indexing, whole quantities
  • Floats: Measurements, currencies, precise calculations
  • Infinity: Representing unlimited values
  • NaN: Detecting invalid mathematical operations

Other Number Bases

  • Binary (Base 2): 0b1010
  • Octal (Base 8): 0o12
  • Hexadecimal (Base 16): 0xff

Remember

JavaScript's Number type is simple yet powerful - one type handles all your numeric needs!
