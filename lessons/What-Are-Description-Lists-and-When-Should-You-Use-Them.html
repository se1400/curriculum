What Are Description Lists?

Description lists are perfect for presenting terms and definitions in an organized and easy-to-read format.

They're commonly used in glossaries or real dictionaries, where you can find words with their corresponding definitions.

The details could be definitions or other information related to the terms.

Basic Structure
<dl>
  <dt>Term</dt>
  <dd>Definition or details</dd>
  
  <dt>Another Term</dt>
  <dd>Another definition</dd>
</dl>

Three Required Elements

You will need three HTML elements to define a description list:

  • dl - Description list element (container for the entire list)
  • dt - Description term element (for each term)
  • dd - Description details element (for descriptions/details associated with each term)
Element Breakdown
<!-- dl = Description List (container) -->
<dl>

  <!-- dt = Description Term -->
  <dt>HTML</dt>
  
  <!-- dd = Description Details -->
  <dd>HyperText Markup Language</dd>

</dl>

Example: Glossary

Here's an example of a description list with two terms (HTML and CSS) and their corresponding details (their expansions).

In the browser, you would see each term followed by its corresponding description. By default, the descriptions are slightly more indented towards the right to distinguish them visually.

Beyond Terms and Definitions

Description lists are not limited to only terms and definitions. They are much more versatile than that.

Essentially, when you have two related pieces of information in a key-value pair format, where one acts as a label (the key) and the other acts as additional related information (the value), you can use a description list.

Key-Value Pair Concept
/* Key-Value Format */

Key (dt) → Label, term, question
Value (dd) → Details, answer, info

Perfect for any paired data!

Example: Recipe Ingredients

Here we have a recipe with ingredients as an example. The entire description list is within a dl element.

Each ingredient (Flour, Sugar) is within a dt element. The measurements (2 cups, 1/2 cup) are within dd elements directly after their corresponding ingredients.

In the browser, you would see the ingredients aligned to the left, and the measurements indented to separate them visually.

Recipe Format
<dl>
  <dt>Flour</dt>
  <dd>2 cups</dd>
  
  <dt>Sugar</dt>
  <dd>1/2 cup</dd>
  
  <dt>Vegetable Oil</dt>
  <dd>2 tablespoons</dd>
</dl>
Browser Display
Flour
  2 cups
Sugar
  1/2 cup
Vegetable Oil
  2 tablespoons

Other Use Cases

Description lists have many practical applications:

  • Product specifications
  • Frequently asked questions (FAQs)
  • Contact information
  • Metadata
  • Any key-value pair data

Whenever you need to present related pairs of information where one item describes or provides details about the other, consider using a description list.

Product Specifications
<dl>
  <dt>Screen Size</dt>
  <dd>15.6 inches</dd>
  
  <dt>RAM</dt>
  <dd>16 GB</dd>
  
  <dt>Storage</dt>
  <dd>512 GB SSD</dd>
</dl>
FAQ Format
<dl>
  <dt>What are your hours?</dt>
  <dd>We're open 9 AM - 5 PM</dd>
  
  <dt>Do you ship internationally?</dt>
  <dd>Yes, to most countries</dd>
</dl>
