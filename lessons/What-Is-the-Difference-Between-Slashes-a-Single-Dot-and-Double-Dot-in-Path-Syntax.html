File Path Syntax

File paths use special syntax to navigate directories and locate files. There are three key syntaxes to know:

  • Slash - The path separator (forward slash / or backslash \)
  • Single dot (.) - Refers to the current directory
  • Double dot (..) - Refers to the parent directory
Path Syntax Examples
<!-- Using slash separator -->
/public/logo.png

<!-- Using single dot (current) -->
./script.js

<!-- Using double dot (parent) -->
../styles.css

The Slash (Path Separator)

The slash is known as the "path separator". It can be a backslash (\) or forward slash (/) depending on your operating system.

The slash is used to indicate a break in the text between folder or file names. This is how your computer knows the difference between directory levels.

Understanding Path Separators
/* Single directory with file */
naomis-files/
→ Points to "naomis-files" directory

/* Nested directories */
naomis/files/
→ Points to "files" directory 
  inside "naomis" directory

Single Dot (Current Directory)

A single dot (.) points to the current directory.

You will typically see a single dot used to ensure that the path is recognized as a relative path.

This makes it explicit that you're referencing a file in the same directory as the current file.

Current Directory Reference
<!-- File in same directory -->
<script src="./script.js"></script>

<img src="./logo.png" alt="Logo">

<link rel="stylesheet" href="./styles.css">
What It Means
./favicon.ico
↓
Look in the CURRENT directory
for "favicon.ico"

Double Dot (Parent Directory)

Two dots (..) point to the parent directory.

Double dots are much more common and are used to access files outside of the current working directory.

You can chain multiple double dots to navigate up multiple directory levels.

Parent Directory Reference
<!-- Go up one level -->
<link rel="stylesheet" href="../styles.css">

<!-- Go up, then into another folder -->
<script src="../src/index.js"></script>

<!-- Go up two levels -->
<img src="../../images/logo.png">
What It Means
../src/index.css
↓
1. Go UP to parent directory
2. Go INTO "src" folder
3. Find "index.css"

Practical Example

Consider a typical project structure. If public/index.html needs to load files, it uses different path syntax depending on the file location.

For files in the same directory (like favicon.ico), use a single dot: ./favicon.ico

For files in a different directory (like src/index.css), use double dots to go up first: ../src/index.css

File Tree Structure
my-app/
├─ public/
│  ├─ favicon.ico
│  ├─ index.html
├─ src/
   ├─ index.css
   ├─ index.js
From public/index.html
<!-- Same folder: use ./ -->
<link rel="icon" href="./favicon.ico">

<!-- Different folder: use ../ -->
<link rel="stylesheet" 
      href="../src/index.css">

<script src="../src/index.js"></script>
