What Are Functions?

Reusable Pieces of Code

Functions are reusable pieces of code that perform a specific task or calculate a value.

Think of functions as a machine: It takes some input, does some operations on it, and then produces an output.

Declaring a Function

Basic Function Declaration

To declare a function, use the function keyword, followed by the function name, parentheses, and curly braces containing the code to execute.

function greet() {
    console.log("Hello, Jessica!");
}

Function Components

  • function keyword - declares a function
  • greet - the function name (identifier)
  • () - parentheses for parameters (empty in this case)
  • {} - curly braces containing the function body
Just declaring a function doesn't execute it! You need to call (invoke) the function for the code inside to run.

Calling (Invoking) a Function

Function Execution

A function call or invocation is when you actually use or execute the function. To call a function, reference the function name followed by parentheses:

function greet() {
    console.log("Hello, Jessica!");
}

greet(); // "Hello, Jessica!"

Parameters and Arguments

Making Functions Reusable

What if we wanted the message to say "Hello, Nick!" or "Hello, Anna!"? We don't want to write a new function each time. Instead, we can create a reusable function using parameters and arguments.

Definitions

  • Parameters - Placeholders for values in the function definition
  • Arguments - Actual values passed to the function when it's called
function greet(name) {  // 'name' is a parameter
    console.log("Hello, " + name + "!");
}

greet("Alice");  // "Alice" is an argument
greet("Nick");   // "Nick" is an argument

// Output:
// Hello, Alice!
// Hello, Nick!
Now we have a reusable function that can be used dozens of times throughout our code with different arguments!

Multiple Parameters

Functions Can Accept Multiple Inputs

Functions can have multiple parameters separated by commas, allowing them to work with multiple pieces of data.

function introduce(name, age, city) {
    console.log("Hi, I'm " + name + 
                    ". I'm " + age + 
                    " years old and I live in " + city + ".");
}

introduce("Alice", 25, "New York");
// Hi, I'm Alice. I'm 25 years old and I live in New York.

introduce("Bob", 30, "San Francisco");
// Hi, I'm Bob. I'm 30 years old and I live in San Francisco.

Return Values - The Default

Functions Always Return Something

When a function finishes its execution, it will always return a value. By default, if no return statement is specified, the return value will be undefined.

function doSomething() {
    console.log("Doing something...");
}

let result = doSomething();
console.log(result); // undefined

// Output:
// Doing something...
// undefined
Without a return statement, the function performs its operations but doesn't give back any usable value. The result is undefined.

Using Return Statements

Returning Specific Values

If you need your function to return a specific value, use the return statement. This allows you to use the function's output later in your code.

function calculateSum(num1, num2) {
    return num1 + num2;
}

console.log(calculateSum(3, 4)); // 7

// You can also store the returned value
let total = calculateSum(10, 20);
console.log(total); // 30
Often times you will be using the return statement, because you can use that value that was output from the function later on in your code.

Return Stops Function Execution

Important Behavior

When a return statement is executed, the function immediately stops and returns the value. Any code after the return statement will not run.

function checkNumber(num) {
    if (num > 10) {
        return "Number is greater than 10";
        console.log("This will never execute"); // Won't run!
    }
    return "Number is 10 or less";
}

console.log(checkNumber(15)); // "Number is greater than 10"
console.log(checkNumber(5));  // "Number is 10 or less"
Code written after a return statement is called "unreachable code" and will never execute.

Anonymous Functions

Functions Without Names

So far, we've been working with named functions, but you can also create what's called an anonymous function. An anonymous function is a function without a name that can be assigned to a variable.

// Anonymous function assigned to a variable
const sum = function (num1, num2) {
    return num1 + num2;
};

console.log(sum(3, 4)); // 7

How It Works

  • We have a const variable called sum
  • We assign it an anonymous function that returns the sum of two numbers
  • We call sum just like any other function
Anonymous functions are useful when you want to pass a function as an argument to another function, or when the function is only used in one place.

Default Parameters

Setting Default Values

Functions support default parameters, allowing you to set default values for parameters. These default values are used if the function is called without an argument for that parameter.

function greetings(name = "Guest") {
    console.log("Hello, " + name + "!");
}

greetings();         // Hello, Guest!
greetings("Anna");  // Hello, Anna!

How Default Parameters Work

  • If no argument is provided for name, it defaults to "Guest"
  • If an argument is provided, it overrides the default value
  • Useful for optional parameters with sensible defaults

Multiple Default Parameters

Combining Regular and Default Parameters

You can have multiple parameters, some with defaults and some without. Default parameters typically come after required parameters.

function createProfile(name, age = 18, country = "USA") {
    return `${name} is ${age} years old and lives in ${country}.`;
}

console.log(createProfile("Alice"));
// Alice is 18 years old and lives in USA.

console.log(createProfile("Bob", 25));
// Bob is 25 years old and lives in USA.

console.log(createProfile("Charlie", 30, "Canada"));
// Charlie is 30 years old and lives in Canada.

Summary

Key Takeaways

  • Functions are reusable pieces of code that perform specific tasks
  • Declaration - Define a function with the function keyword
  • Invocation - Call a function to execute it with functionName()
  • Parameters - Placeholders in the function definition
  • Arguments - Actual values passed when calling the function
  • Return values - Functions return undefined by default, or a specific value with return
  • Anonymous functions - Functions without names, assigned to variables
  • Default parameters - Set default values for optional parameters
Functions allow you to write reusable and organized code. They can take inputs (parameters), perform actions, and return outputs, making your code more maintainable and easier to understand.
