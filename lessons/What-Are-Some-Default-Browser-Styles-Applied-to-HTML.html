Default Browser Styles

Browsers apply basic formatting to HTML elements before any CSS is written. These are called "default browser styles" or "user-agent styles".

Purpose

These styles ensure HTML elements display in a readable way across all browsers, though they can vary slightly between browsers.

Why This Matters

Understanding default styles is important for designing consistent websites. Even without writing CSS, your HTML will have basic formatting applied automatically.

Heading Elements

Headings (h1-h6) have varying sizes and weights by default. h1 is the largest and boldest, decreasing in size through h6.

Hierarchy

These default styles help define the structure and hierarchy of your content, making it easier to scan and understand.

Horizontal Rule

The <hr> element creates a horizontal line used to visually separate sections of content.

Default Appearance

Browsers apply a thin line style with spacing above and below to distinguish between content sections.

HTML
<p>Paragraph 1</p>
<p>Paragraph 2</p>
<hr>
<p>Paragraph 3</p>
<p>Paragraph 4</p>

Blockquote Element

Blockquotes indicate text that is a quotation from another source. Browsers typically add indentation to set them apart.

Visual Distinction

The indentation makes it clear that the content is quoted from another source, differentiating it from regular text.

HTML
<p>Paragraph 1</p>
<p>Paragraph 2</p>
<blockquote>
  I think, therefore I am. 
  (Rene Descartes)
</blockquote>
<p>Paragraph 3</p>
<p>Paragraph 4</p>

Anchor Tags

The <a> element (anchor tag) is styled with blue color and underlining by default to make it recognizable as a link.

Link Recognition

The default blue and underlined styling signals to users that the text is clickable and will navigate to another page.

HTML
<p>Paragraph 1</p>
<p>Paragraph 2</p>
<a href="https://freecodecamp.org/">
  Visit the freeCodeCamp learn page
</a>
<p>Paragraph 3</p>
<p>Paragraph 4</p>

List Elements

Browsers add basic formatting to lists including indentation and either bullets (unordered lists) or numbers (ordered lists).

Two Types

  • <ul> - Unordered lists use bullets
  • <ol> - Ordered lists use numbers
  • Both are indented to the right by default
HTML
<p>Sample Paragraph</p>
<ol>
  <li>item 1</li>
  <li>item 2</li>
  <li>item 3</li>
</ol>
<ul>
  <li>item</li>
  <li>another item</li>
  <li>yet another item</li>
</ul>
<p>Ending Paragraph</p>
Browser Output

Sample Paragraph

  1. item 1
  2. item 2
  3. item 3
  • item
  • another item
  • yet another item

Ending Paragraph

Key Takeaways

Browsers apply default spacing, varying font sizes, and weights to HTML elements automatically.

CSS Resets

In future lessons, you'll learn how to reset these default styles using a CSS reset to create more consistent designs across different browsers.

Summary

  • Default browser styles ensure basic readability
  • Styles can vary slightly between browsers
  • Understanding defaults helps create consistent designs
  • CSS resets can override these defaults when needed
