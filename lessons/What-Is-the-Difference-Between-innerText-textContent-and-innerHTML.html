innerText, textContent, and innerHTML

What Are These Properties?

Properties you can access in JavaScript to get or change the content of an HTML element

They May Look Similar, But...

These three properties have key differences that are important to understand.

innerText

Visible text only

textContent

All text content

innerHTML

HTML markup

Choosing the right one depends on your specific use case. Let's explore each property in detail.

innerText Property

What is innerText?

innerText represents the visible text content of the HTML element and its descendants.

Key Characteristics

  • Visible Text Only: Does not include hidden text
  • No HTML Tags: Only rendered text is returned
  • Includes Descendants: Gets text from all child elements

Think of it this way:

innerText returns exactly what a user would see on the screen when looking at the element.

innerText Example

HTML Structure

<div id="container">
  <p>Hello, World!</p>
  <p>I'm learning JavaScript</p>
</div>

JavaScript

const container = document.getElementById("container");
console.log(container.innerText);

Output

Hello, World!
I'm learning JavaScript

The property returns a string with the text contained within the element, including text from its descendants.

innerText and Visibility

Important Behavior

innerText only returns text that is visible at the moment the string is requested.

If a child element is hidden, its text won't be included.

Example with Hidden Element

<div id="container">
  <p>Hello, World!</p>
  <p hidden>I'm learning JavaScript</p>
</div>

const container = document.getElementById("container");
console.log(container.innerText);

Output

Hello, World!

Notice the second paragraph's text is not included because it's hidden.

Setting innerText

Modifying Content

You can set the innerText of an HTML element, but this will:

  • Replace all existing content
  • Add a <br> element for every line break

Example

const container = document.getElementById("container");
container.innerText = "New text\nOn multiple lines";

⚠️ Performance Consideration

Since innerText takes visibility into account, getting its value triggers a process called "reflow".

Reflow recalculates the position of elements on the website and can be computationally intensive.

Best Practice: Avoid triggering reflow when possible.

textContent Property

What is textContent?

textContent returns the plain text content of an element, including all text within its descendants.

Key Difference from innerText

Most Important Distinction

textContent always returns the full text content of an HTML element and its descendants, regardless of whether it's visible or hidden.

Additional Characteristics

  • Preserves Formatting: Keeps indentation and spacing
  • Includes Hidden Text: Returns text from hidden elements
  • No Reflow: More performant than innerText

textContent Example

HTML Structure

<div id="container">
  <p>Hello, World!</p>
  <p>I'm learning JavaScript</p>
</div>

JavaScript

const container = document.getElementById("container");
console.log(container.textContent);

Output

  Hello, World!
  I'm learning JavaScript

Notice that indentation and spacing are preserved in the output.

textContent and Hidden Elements

Critical Difference

Even if an HTML element is not visible, its text will still be included in the textContent property.

Example with Hidden Element

<div id="container">
  <p>Hello, World!</p>
  <p hidden>I'm learning JavaScript</p>
</div>

const container = document.getElementById("container");
console.log(container.textContent);

Output

  Hello, World!
  I'm learning JavaScript

The hidden paragraph's text is still included, unlike with innerText.

Additional textContent Behavior

Includes Script and Style Content

textContent will also include the content of <script> and <style> elements.

Setting textContent

Replacing Content

When you set textContent on a node:

  • It will remove all child nodes
  • Replace them with a single text node containing the new string
const container = document.getElementById("container");
container.textContent = "New plain text content";

All existing HTML structure inside the container is replaced with plain text.

innerHTML Property

What is innerHTML?

innerHTML allows you to get or set the HTML content of an element.

Key Capability

innerHTML is helpful for injecting new HTML into the DOM dynamically.

const container = document.getElementById("container");
container.innerHTML = "<h2>New Title</h2><p>New content</p>";

This will create actual HTML elements, not just text.

innerHTML Security Considerations

⚠️ Security Risk

Important: innerHTML poses a security risk if you don't have control over the string being inserted.

The Problem

  • User Input: If strings contain data entered by users
  • Malicious Code: Users could inject harmful scripts
  • XSS Attacks: Can lead to serious security issues

Best Practice

To avoid security vulnerabilities, it's recommended to use the textContent property to insert plain text instead of innerHTML.

Safe Alternative

Use textContent when inserting user-generated content or any untrusted data.

Quick Comparison

Feature innerText textContent innerHTML
Returns Visible text only All text content HTML markup
Hidden Elements Excluded Included Included
Formatting As rendered Preserved HTML tags
Performance Slower (reflow) Faster Varies
Security Safe Safe Risk with user input
Use Case User-visible text Plain text manipulation Dynamic HTML injection

Summary and Best Practices

Choosing the Right Property

Use innerText when:

  • You need only the visible text
  • Working with user-facing content
  • Replicating what a user sees

Use textContent when:

  • You need all text, visible or hidden
  • Performance matters (no reflow)
  • Inserting plain text safely

Use innerHTML when:

  • You need to insert HTML markup
  • You control the content source
  • Building dynamic HTML structures

Key Takeaway

Understanding these differences is essential for working with HTML content in JavaScript effectively.
