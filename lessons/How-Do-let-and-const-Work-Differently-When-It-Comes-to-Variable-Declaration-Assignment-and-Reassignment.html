Variable Declaration in JavaScript

When working with JavaScript, you'll often declare variables to store data that you plan to use throughout your program.

Modern JavaScript Standards

In modern JavaScript, let and const are the preferred ways to declare variables.

Key Difference

They differ in how they handle value assignment and reassignment.

In this lesson, we'll explore how let and const differ in variable declaration, assignment, and reassignment.

The let Keyword

The let keyword allows you to declare variables that can be updated or reassigned later.

üí° Think of let as a Flexible Container

Once you've stored a value in it, you can change that value as needed throughout your program.

Declaring and Assigning with let

let score = 10;

In this case, the variable score is declared and assigned the value 10.

Reassigning let Variables

If you want to update the value later, you can easily do that:

let score = 10;
console.log(score); // 10

score = 20;
console.log(score); // 20

‚úì Success!

Now, score holds the value 20.

This makes let particularly useful when you know the value of a variable will change as your program runs.

Common Use Cases for let

  • Counters and loop variables
  • Game scores that change
  • User input values
  • Any value that needs to be updated

The const Keyword

On the other hand, const is used to declare variables that are constant.

Constant Variables

Once you assign a value to a variable declared with const, you cannot reassign it.

This makes const ideal for values that you don't want to change accidentally during the execution of your program.

Declaring and Assigning with const

const maxScore = 100;
console.log(maxScore); // 100

Once maxScore is assigned the value 100, it cannot be changed.

Cannot Reassign const Variables

Once a const variable is assigned, attempting to change it will cause an error:

const maxScore = 100;
console.log(maxScore); // 100

maxScore = 200; // This will result in an error

‚ùå Error!

Trying to reassign a value to a const variable will throw an error in your JavaScript console.

const variables are immutable once they are assigned.

Immutable: Cannot be changed after initial assignment.

Declaring Without Assignment

The way let and const handle declaration without assignment is very different.

let

You can declare a let variable without immediately assigning it a value, and assign it later:

let age;
console.log(age); // undefined

age = 25;
console.log(age); // 25

‚úì This works perfectly fine!

const

Variables declared with const must be assigned a value at the time of declaration:

const age; 
// Error: Missing initializer 
// in const declaration

‚ùå This causes an error!

When to Use let vs const

Use let when:

You need to declare variables that will be reassigned later.

  • Tracking a changing score
  • Updating a value over time in your program
  • Loop counters and iterators
  • Any value that needs to change

Use const when:

You want to declare variables that should remain constant.

  • Configuration values
  • Settings that shouldn't be changed accidentally
  • Mathematical constants (like PI)
  • Fixed values that never change

What About var?

You can also use the var keyword, but it's not as recommended anymore.

Why Avoid var?

The var is kind of like let, except it has a wider scope, which is more likely to cause problems in your program.

Modern Best Practice

Stick with let and const in modern JavaScript development.

You'll learn more about scope and the issues with var in future lessons.
