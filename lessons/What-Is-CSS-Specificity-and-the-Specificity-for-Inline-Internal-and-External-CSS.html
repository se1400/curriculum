CSS Specificity

CSS specificity is a fundamental concept that determines which styles are applied to an element when multiple rules could apply.

Why This Matters

Understanding specificity helps developers resolve conflicts between different CSS rules and ensures that the desired styles are consistently applied.

Key Concept

CSS specificity is calculated based on the type of selectors used. Different selectors have different weights in determining which style wins.

Specificity Hierarchy

From highest to lowest specificity. For example, 1000 takes precedent over 100, 10, 1, or 0.

1. Inline Styles (1,0,0,0)
2. ID Selectors (0,1,0,0)
3. Class, Attribute, Pseudo-class (0,0,1,0)
4. Type & Pseudo-elements (0,0,0,1)
5. Universal Selector (*) (0,0,0,0)

Inline Styles

The highest specificity is attributed to inline styles, applied directly to an element through the style attribute.

Specificity Value

(1, 0, 0, 0)

HTML
<p style="color: red;">Red paragraph</p>
<p>Other paragraph</p>
CSS (styles.css)
p {
  color: blue;
}
Result

Red paragraph WINS

Other paragraph

Inline style overrides external CSS

ID Selectors

ID selectors have a high level of specificity, second only to inline styles.

Specificity Value

(0, 1, 0, 0)

HTML
<p id="para">Red paragraph</p>
<p>Other paragraph</p>
CSS
#para {
  color: red;
}

p {
  color: blue;
}
Result

Red paragraph ID WINS

Other paragraph

Class Selectors

Class selectors, attribute selectors, and pseudo-classes have moderate specificity.

Specificity Value

(0, 0, 1, 0)

Examples

  • .container - class
  • [type="text"] - attribute
  • :hover - pseudo-class
HTML
<p id="para">Example paragraph</p>
<p class="example-para">Other paragraph</p>
<p>Yet another paragraph</p>
CSS
#para { color: red; }
.example-para { color: green; }
p { color: blue; }
Result

Example paragraph ID

Other paragraph CLASS

Yet another paragraph TYPE

Type Selectors

Type selectors and pseudo-elements have lower specificity compared to classes and IDs.

Specificity Value

(0, 0, 0, 1)

Examples

  • div - type selector
  • h1 - type selector
  • ::before - pseudo-element
  • ::after - pseudo-element
CSS
p {
  color: blue;
}

div {
  background: lightgray;
}

Priority Order

Higher: ID selectors

Higher: Class selectors

→ Type selectors

Lower: Universal selector

Universal Selector

The universal selector * has the lowest specificity of all selectors.

Specificity Value

(0, 0, 0, 0)

Important Note

The universal selector contributes 0 to the specificity calculation and does not affect specificity.

CSS
* {
  color: red;
}

#para {
  color: blue;
}

p {
  color: green;
}
Result

You won't see any red elements because ID and type selectors override the universal selector, highlighting its low specificity.

Specificity Calculation

Specificity values are calculated as a four-part value: (a, b, c, d)

a: Inline styles 1 or 0
b: Number of ID selectors Count IDs
c: Number of class, attribute, pseudo-class selectors Count classes
d: Number of type selectors and pseudo-elements Count types

Weight Distribution

Each part of the specificity value carries different weight:

Selector Type Weight Contribution
Inline styles (a) Highest (1, 0, 0, 0)
ID selectors (b) Very High (0, 1, 0, 0) per ID
Class, attribute, pseudo-class (c) Moderate (0, 0, 1, 0) per selector
Type & pseudo-elements (d) Low (0, 0, 0, 1) per selector
Universal selector (*) None (0, 0, 0, 0)

Inline CSS Specificity

Inline CSS has the highest specificity because it is applied directly to the element. It overrides any internal or external CSS.

Specificity Value

(1, 0, 0, 0)

HTML
<p style="color: red;">
  This text is red.
</p>

Why Inline Has Highest Specificity

  • Applied directly to the element
  • Most specific targeting possible
  • Overrides all other CSS sources
  • Cannot be overridden except by !important

Internal CSS Specificity

Internal CSS is defined within a <style> element in the <head> section. It has lower specificity than inline styles but can override external styles.

Specificity Value

Determined by the selectors used. Example: ID selector = (0, 1, 0, 0)

HTML
<head>
  <style>
    .text {
      color: blue;
    }
  </style>
</head>
<body>
  <div class="text">This text is blue.</div>
</body>
Result

This text is blue.

Text will be blue unless an inline style or more specific external style is applied

External CSS Specificity

External CSS is linked via a <link> element and written in separate .css files. It has the lowest specificity but provides the best maintainability.

Specificity Value

Determined by the selectors used. Example: class selector = (0, 0, 1, 0)

HTML
<head>
  <link rel="stylesheet" href="styles.css">
</head>
<body>
  <div class="text">
    Text color defined in external CSS
  </div>
</body>
styles.css
.text {
  color: purple;
}
Result

Text color defined in external CSS

Will be applied unless overridden by internal or inline styles

Inline vs Internal vs External

CSS Type Specificity Override Priority
Inline CSS (1, 0, 0, 0) Highest - overrides all
Internal CSS Based on selector Medium - can override external
External CSS Based on selector Lowest - best for maintainability

Important Note

When comparing internal and external CSS, specificity is determined by the selectors used, not the location. The location only matters when comparing inline styles to the others.

Key Takeaways

  • CSS specificity determines which styles are applied when conflicts arise
  • Specificity is calculated as a four-part value: (a, b, c, d)
  • Inline styles have the highest specificity
  • ID selectors have very high specificity
  • Classes, attributes, pseudo-classes have moderate specificity
  • Type selectors and pseudo-elements have low specificity
  • Universal selector (*) has no specificity weight

Remember the Hierarchy

Inline > ID > Class > Type > Universal
