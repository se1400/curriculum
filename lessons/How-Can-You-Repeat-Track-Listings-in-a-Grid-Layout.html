Repeating Track Listings

A Better Way to Write Repeated Values

In previous lessons, we have been working with the grid-template-columns property and setting the value to a few fractional units.

While the following code is completely valid, there is an easier way to repeat a section or all of your track listings.

.grid-container { display: grid; grid-template-columns: 1fr 1fr 1fr 1fr; column-gap: 10px; }

The Problem: Writing the same value multiple times is repetitive and harder to maintain. What if you need 10 or 20 columns?

Issues: Repetitive typing, easy to make mistakes, hard to update (imagine changing 20 columns!), and cluttered code.

The repeat() Function

A More Efficient Solution

The repeat() function is used to repeat a section or all of the tracks for columns or rows.

This function takes in a repeat count and the tracks you wish to repeat.

repeat(count, tracks)

Function Parameters:

  • count – How many times to repeat
  • tracks – The pattern to repeat

Purpose: Simplifies repetitive column or row declarations by letting you specify a pattern once and repeat it.

Using repeat() - Basic Example

Refactored Code

Here is a revised version of the earlier example using the repeat function:

.grid-container { display: grid; grid-template-columns: repeat(4, 1fr); column-gap: 10px; }

How repeat() Expands

Behind the Scenes

Understanding how the browser interprets repeat(4, 1fr):

The Expansion Process:
repeat(4, 1fr)
↓
1fr, 1fr, 1fr, 1fr
↓
Creates 4 columns, each at 1fr

Remember: The browser automatically expands repeat() into the full list of values. It's just a convenient shorthand!

Repeating Patterns

Beyond Simple Repetition

The repeat() function will accept any valid pattern that you can use for rows or columns.

Key Feature: You're not limited to repeating single values. You can repeat entire patterns!

/* Repeat a single value */ grid-template-columns: repeat(4, 1fr); /* Repeat a pattern of multiple values */ grid-template-columns: repeat(2, 20px 1fr); /* Mix repeat() with other values */ grid-template-columns: 100px repeat(3, 1fr) 100px;

Flexibility: The pattern inside repeat() can be as simple or complex as needed.

Repeating a Pattern

Advanced Usage

Here is an example using the repeat() function to set the first and third columns to 20 pixels and the second and fourth columns to one fractional unit.

.grid-container { display: grid; grid-template-columns: repeat(2, 20px 1fr); column-gap: 10px; }
Pattern Expansion:
repeat(2, 20px 1fr)
↓

Combining repeat() with Other Values

Maximum Flexibility

You can mix repeat() with other column values in the same declaration.

/* Sidebar layout: fixed, repeated, fixed */ grid-template-columns: 200px repeat(3, 1fr) 200px; /* Result: 200px 1fr 1fr 1fr 200px */ /* Featured layout */ grid-template-columns: 2fr repeat(2, 1fr); /* Result: 2fr 1fr 1fr */ /* Complex mix */ grid-template-columns: 100px repeat(2, 1fr 2fr) 100px; /* Result: 100px 1fr 2fr 1fr 2fr 100px */

When to Use repeat()

Making the Right Choice

Sometimes, you might opt to write out each individual value instead of using the repeat() function. But there are times when this function comes in handy, especially when you want to repeat a particular pattern for a track listing.

Use repeat() when:

  • You have 3 or more identical track sizes
  • You need to repeat a pattern multiple times
  • You want easy maintenance – change count, not code
  • Creating many columns (5, 10, 20, etc.)

Write out values when:

  • You have unique column sizes
  • Only 2-3 columns with different sizes
  • The pattern is irregular and doesn't repeat

Summary

Key Takeaways

Remember:

  • repeat() simplifies repetitive track listings
  • Syntax: repeat(count, tracks)
  • Works with single values or patterns
  • Can mix with other column values
  • More maintainable than writing values repeatedly
  • Especially useful for many columns or repeating patterns

Without repeat()

grid-template-columns: 1fr 1fr 1fr 1fr 1fr;

With repeat()

grid-template-columns: repeat(5, 1fr);

Best Practice: Use repeat() whenever you're writing the same track size 3+ times. Your future self will thank you!
