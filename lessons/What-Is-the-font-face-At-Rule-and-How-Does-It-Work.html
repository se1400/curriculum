What Are At-Rules in CSS?

At-rules are statements that provide instructions to the browser.

Uses of At-Rules:

  • Define various aspects of the stylesheet
  • Media queries
  • Keyframes for animations
  • Font faces
  • And more
This lesson focuses on the @font-face at-rule

What is @font-face?

The @font-face at-rule allows you to define a custom font.

What You Can Define:

  • Font file location
  • Font format
  • Font properties (weight, style, etc.)

Basic Syntax:

@font-face {
  /* Descriptors */
}

Structure:

The @font-face keyword is followed by a block enclosed in curly brackets. Within the brackets, you include descriptors to customize your font face.

font-family Descriptor

The font-family descriptor specifies the name you will use throughout the stylesheet to refer to that font.

Example @font-face Rule:

@font-face {
  font-family: "MyCustomFont";
}

This defines a custom font named "MyCustomFont"

Important:

The name you assign here is what you'll use to reference this font throughout your stylesheet.

Using Your Custom Font

Once defined, use the custom font name as the value of font-family wherever you want to apply it.

Example in CSS:

body {
  font-family: "MyCustomFont";
}

How It Works:

The custom font name matches your @font-face definition, allowing CSS selectors to apply that font to elements.

src Descriptor

The src descriptor is required for a valid @font-face rule.

REQUIRED

What src Contains:

  • References to font resources
  • List of external references or locally-installed font names
  • Separated by commas
  • Can include format and technology hints

Example:

@font-face {
  font-family: "MyCustomFont";
  src: url("path/to/font.woff2"),
       url("path/to/font.woff"),
       url("path/to/font.otf");
}

url() Function

Use the url() function to include font files in your stylesheet.

Syntax:

url("path/to/font-file.extension")

Requirements:

  • Write file path within parentheses and quotation marks
  • Include the file extension
  • Each resource can be on a different line for readability
  • Last resource should end with a semicolon

Complete Example:

@font-face {
  font-family: "MyCustomFont";
  src: url("path/to/font.woff2"),
       url("path/to/font.woff"),
       url("path/to/font.otf");
}

format() - Font Format Hints

You can specify the format for each font resource.

OPTIONAL

Why Use format()?

  • It's a hint for the browser about the font format
  • If omitted: Resource is downloaded and format detected after download
  • If invalid: Resource will not be downloaded

Syntax:

@font-face {
  font-family: "MyCustomFont";
  src: url("path/to/font.woff2") format("woff2"),
       url("path/to/font.otf") format("opentype"),
       url("path/to/font.woff") format("woff");
}

Understanding Font Formats

WOFF & WOFF2

WOFF stands for "Web Open Font Format"

The difference between WOFF and WOFF2 is the compression algorithm used. WOFF2 offers better compression.

OpenType

Format for scalable computer fonts developed by Microsoft and Adobe. Allows users to access additional features in a font. Widely used across major operating systems.

Example with Multiple Formats:

@font-face {
  font-family: "MyCustomFont";
  src: url("path/to/font.woff2") format("woff2"),
       url("path/to/font.otf") format("opentype"),
       url("path/to/font.woff") format("woff");
}

tech() - Technology Hints

You can also specify the technology of the font resource.

OPTIONAL

Example:

@font-face {
  font-family: "MyCustomFont";
  src: url("path/to/font.woff2") format("woff2"),
       url("path/to/font.otf") format("opentype") tech(color-COLRv1),
       url("path/to/font.woff") format("woff");
}

Note:

In this example, the second font resource specifies the color-COLRv1 technology, which is a specific color font technology.

@font-face Fundamentals

Key Takeaways:

  • At-rules provide instructions to the browser
  • @font-face defines custom fonts
  • font-family descriptor names your custom font
  • src descriptor is required - specifies font file locations
  • url() function includes font files
  • format() is optional - hints at font format
  • tech() is optional - specifies font technology
  • Common formats: WOFF, WOFF2, OpenType
