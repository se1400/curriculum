File and Folder Organization

When building out web applications, it is important to think about the organization of your files and folders.

This will help keep your code organized and easier to maintain.

Good organization makes it simple to find what you need and helps others contribute to your project.

Benefits of Organization

  • Keeps code organized
  • Easier to maintain
  • Quick file location
  • Enables team collaboration
  • Professional project structure

Example Folder Structure

Let's look at an example folder structure for an HTML and CSS project.

This organized structure separates assets, stylesheets, and HTML files into logical groupings.

.
├── /assets
│   ├── /images
│   │   ├── logo.png
│   │   ├── banner.jpg
│   │   └── icons.svg
│   ├── /fonts
│   │   ├── custom-font.woff
│   │   └── custom-font.woff2
├── /css
│   ├── main.css
│   ├── about.css
│   └── contact.css
├── index.html
├── about.html
├── contact.html
└── README.md

Root Directory

At the top of the example, there is a single dot (.), which represents the root directory.

The root directory is the top-level directory in a file system, serving as the starting point for all file paths and containing all other directories and files.

A directory is another name for a folder.

Root Directory

The starting point of your project's file structure. All other folders and files are contained within or descend from the root directory.

.

Assets Directory

Inside the root directory, the first directory shown is the assets directory.

The term assets refers to any files that are used in the project, such as images, fonts, or other resources.

In this example, there are two directories inside the assets directory called images and fonts.

/assets

Stores project resources:

  • /images - Logo, banners, icons, photos
  • /fonts - Custom font files
  • Other resources like videos, documents, etc.

CSS Directory

The next directory would be the css directory.

Sometimes this is also referred to as a stylesheets directory.

This is where you would store all of your CSS files.

/css

Contains all stylesheet files:

  • main.css - Main styles
  • about.css - About page styles
  • contact.css - Contact page styles

Can also be named /stylesheets

HTML Files

Below the css directory are the HTML files.

The index.html file represents the homepage, while about.html and contact.html represent the About and Contact pages, respectively.

HTML files are typically stored in the root directory for easy access.

HTML Page Files

  • index.html - Homepage (main entry point)
  • about.html - About page
  • contact.html - Contact page
  • Other page files as needed

README.md File

Lastly, there is a README.md file.

README files are commonly used to provide useful information about the project, such as what it does, how to use it, and any other relevant details.

The .md extension stands for Markdown, which is a lightweight markup language that is often used to write documentation.

README.md

Project documentation file that typically includes:

  • Project description
  • How to use the project
  • Installation instructions
  • Features and functionality
  • Author and license information

Written in Markdown (.md) format

Benefits and Flexibility

By organizing your project into assets, CSS, and HTML files, it makes it easy to find what you are looking for.

This also makes it easy for other developers to contribute to your project.

It is important to note that this is just one example of how you can organize your files and folders. There are many different ways to structure a project.

Key Takeaways

  • Logical grouping improves findability
  • Clear structure aids collaboration
  • Multiple valid organizational approaches exist
  • Choose what works best for your project
  • Stay consistent throughout the project
