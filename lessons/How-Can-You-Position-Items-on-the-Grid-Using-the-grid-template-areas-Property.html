grid-template-areas Property

Visual Grid Layout Design

The grid-template-areas property lets you design a visual grid layout by using named labels.

Key Idea: Instead of positioning items with line numbers, you create a visual "map" of your layout using descriptive names!

What You'll Learn:

  • How to use named grid areas
  • Creating visual layouts with strings
  • Connecting items with grid-area
  • Building the Holy Grail layout

Named Labels (Grid Area Names)

Two-Part System

You then assign the labels to specific grid items using the grid-area property. So, in other words, those named labels are also called "grid area names".

Two Steps:

  • Step 1: Define the layout with grid-template-areas
  • Step 2: Assign items to areas with grid-area

Think of it like: Drawing a blueprint (grid-template-areas) and then placing furniture (grid-area) according to that blueprint!

Basic Syntax

Creating the Visual Map

Here's the basic syntax of the grid-template-areas property:

grid-template-areas: 'header header header' 'left-sidebar main right-sidebar' 'footer footer footer';

Look at that! You can actually "see" the layout structure just by reading the CSS!

Breaking Down the Syntax

Key Points

Here are the points to take away from the basic syntax:

1 Values like header and main are the names of the grid areas.

You can name them anything meaningful: header, sidebar, content, etc.

2 Each space-separated value within a string corresponds to a column.

'header header header' = 3 columns

3 Each string represents a row in the grid.

3 strings = 3 rows

Step 2: Assign with grid-area

Connecting Items to Areas

After defining the template, you then have to use the grid-area property and the named labels (or areas) as values to tell CSS that a specific element belongs in that area of the grid.

/* Grid Container: Define the layout */ .grid-container { display: grid; grid-template-areas: 'header header header' 'left-sidebar main right-sidebar' 'footer footer footer'; } /* Grid Items: Assign to areas */ .header { grid-area: header; } .sidebar-left { grid-area: left-sidebar; } .main { grid-area: main; } .sidebar-right { grid-area: right-sidebar; } .footer { grid-area: footer; }

How grid-area Connects

The Bridge Between Template and Item

The grid-area property connects the grid item to the named region you define in the grid-template-areas.

grid-template-areas: 'header'
⬇
grid-area: header
The connection is made!

Simple Rule: The value you use in grid-area must match a name defined in grid-template-areas.

The Holy Grail Layout

Classic Web Design Pattern

A popular way to demonstrate the capabilities of the grid-template-areas property is by creating the classic Holy Grail layout.

What is the Holy Grail layout? If you don't know what it is, the Holy Grail layout is a web design pattern with a header, footer, two sidebars, and a main content area.

Components:

  • Header – Top of the page (full width)
  • Left Sidebar – Navigation or secondary content
  • Main Content – Primary content area
  • Right Sidebar – Ads or additional info
  • Footer – Bottom of the page (full width)

Holy Grail Characteristics

Design Principles

It ensures the main content takes priority, while sidebars and other sections adjust responsively within the layout.

Why Use grid-template-areas?

The Best Solution

Many solutions exist to implement the holy grail layout, but using grid-template-areas and the grid-area property is the most straightforward way to create it.

Advantages:

  • Visual Code – You can literally "see" the layout
  • Intuitive – Named areas are self-documenting
  • Maintainable – Easy to modify and update
  • Flexible – Rearrange with minimal changes
  • Readable – Anyone can understand the structure

Bottom Line: grid-template-areas makes complex layouts simple and obvious!

Holy Grail: Complete Code

Full Implementation

.grid-container { display: grid; grid-template-areas: 'header header header' 'left-sidebar main right-sidebar' 'footer footer footer'; grid-template-columns: 200px 1fr 200px; grid-template-rows: 100px 1fr 80px; gap: 10px; min-height: 100vh; } .header { grid-area: header; } .sidebar-left { grid-area: left-sidebar; } .main { grid-area: main; } .sidebar-right { grid-area: right-sidebar; } .footer { grid-area: footer; }

Notice: We also set column widths (200px 1fr 200px) and row heights to make the layout perfect!

Properties Can Be Independent

Important Note

Please note that both the grid-template-areas and grid-area properties can be used independently of each other.

grid-template-areas

Purpose: Specifically used to define a visual layout by mapping out named grid areas within the grid container.

grid-template-areas: 'a a b' 'c d d';

grid-area

Purpose: Used to position individual grid items, either by specifying row/column positions OR by referencing named areas.

/* With names */ grid-area: header; /* With lines */ grid-area: 1/2/3/4;

grid-area: Two Uses

Versatile Property

On the other hand, the grid-area property is used to position individual grid items, either by specifying their row and column positions or by referencing the named areas defined with the grid-template-areas property.

/* METHOD 1: Reference named area (requires grid-template-areas) */ .header { grid-area: header; } /* METHOD 2: Specify positions with line numbers (no grid-template-areas needed) */ .item { grid-area: 1 / 2 / 3 / 4; /* row-start / col-start / row-end / col-end */ }

Flexibility: You can use grid-area either way, depending on what makes sense for your layout!

Best Practices

Tips for Using grid-template-areas

Follow These Guidelines:

  • Use descriptive names (header, sidebar, main)
  • Keep area names consistent across your site
  • Use hyphens for multi-word names (left-sidebar)
  • Align your strings in code for better visualization
  • Consider responsiveness – you can change areas with media queries
/* Good: Descriptive and aligned */ grid-template-areas: 'header header header' 'sidebar main aside' 'footer footer footer'; /* Avoid: Unclear names */ grid-template-areas: 'a a a' 'b c d' 'e e e';

Responsive with Media Queries

Adapt to Different Screens

One of the best features: You can completely rearrange your layout for different screen sizes!

/* Desktop layout */ .grid-container { grid-template-areas: 'header header header' 'sidebar main aside' 'footer footer footer'; grid-template-columns: 200px 1fr 200px; } /* Mobile layout - stack everything */ @media (max-width: 768px) { .grid-container { grid-template-areas: 'header' 'main' 'sidebar' 'aside' 'footer'; grid-template-columns: 1fr; } }

Powerful! Change the entire layout structure with just a few lines of CSS!

Summary

Key Takeaways

Remember:

  • grid-template-areas creates a visual map of your layout
  • Use strings for rows, space-separated names for columns
  • Named areas make code intuitive and self-documenting
  • grid-area assigns items to named areas
  • Perfect for Holy Grail and other complex layouts
  • Both properties can work independently
  • Most straightforward way to build visual layouts
  • Easy to make responsive with media queries

Final Thought: When your CSS visually represents your layout, everyone wins – you, your team, and future maintainers!
