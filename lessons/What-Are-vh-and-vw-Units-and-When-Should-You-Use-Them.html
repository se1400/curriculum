Viewport-Relative Units: vh and vw

What Are They?

In CSS, vh and vw are viewport-relative units that allow you to size elements based on the dimensions of the browser window.

Purpose: These units are particularly useful for creating responsive designs that adapt to different screen sizes.

Unlike pixels or ems, these units respond directly to the viewport dimensions, making them powerful tools for modern responsive design.

vh: Viewport Height

vh stands for "viewport height"

1vh = 1% of the viewport's height

Example Calculations

If viewport height = 800px:

1vh = 8px (1% of 800px)
50vh = 400px (50% of 800px)
100vh = 800px (100% of 800px)
An element with height: 100vh will occupy the full height of the viewport, regardless of the actual pixel dimensions of the device.

vw: Viewport Width

vw stands for "viewport width"

1vw = 1% of the viewport's width

Example Calculations

If viewport width = 1200px:

1vw = 12px (1% of 1200px)
50vw = 600px (50% of 1200px)
100vw = 1200px (100% of 1200px)
Key Insight: The actual pixel size changes based on the viewport dimensions, but the proportion remains constant.

Creating Full-Screen Layouts

Use Case: Hero Section

These units are especially handy when you want to create full-screen layouts or elements that maintain a specific proportion of the screen.

100vh / 100vw Example

This section fills the entire browser window.

This CSS ensures that the hero section will always be exactly the size of the viewport, regardless of the device's screen size.

HTML

<section class="hero">
  <h1>100vh / 100vw Example</h1>
  <p>This section fills the 
     entire browser window.</p>
</section>

CSS

.hero {
  width: 100vw;
  height: 100vh;
  background: linear-gradient(
    135deg, 
    #667eea 0%, 
    #764ba2 100%
  );
  display: flex;
  flex-direction: column;
  align-items: center;
  justify-content: center;
}

Responsive Typography with vw

Scaling Text with Viewport

vh and vw units can also be used for typography to create responsive text sizes.

Responsive Heading

The h1 heading scales with the viewport width.

(Try resizing your browser to see it scale)

This sets the font size of h1 elements to 5% of the viewport width, allowing text to scale smoothly with the browser window size.

HTML

<h1>Responsive Heading</h1>
<p>
  The h1 heading scales with 
  the viewport width.
</p>

CSS

h1 {
  font-size: 5vw;
}

Advantages of vh and vw Units

Real-Time Response

Dynamic Adjustment: One of the advantages of vh and vw units is that they respond to changes in the viewport size in real-time.

This means that if a user resizes their browser window, elements sized with these units will adjust accordingly without needing to reload the page.

Common Use Cases

  • Full-screen hero sections
  • Maintaining aspect ratios
  • Creating smoothly scaling designs
  • Responsive typography (with caution)

Use vh and vw Judiciously

Typography Caution

Font Size Warning

It's important to use these units judiciously. Setting font sizes solely with vw units can lead to text becoming:

  • Too small on narrow screens
  • Too large on wide screens
Best Practice: Often combine vw with other units (like rem or em) or use CSS functions like clamp() to set minimum and maximum sizes.

Mobile Device Considerations

The Address Bar Problem

Mobile Viewport Changes

On mobile devices, the viewport height can change when the browser's address bar appears or disappears.

Potential Issue

This can cause unexpected layout shifts if you're using vh units extensively for critical layout elements.

Address Bar Visible

Viewport height is smaller, 100vh is reduced

Address Bar Hidden

Viewport height is larger, 100vh increases

Solution: Consider testing thoroughly on mobile devices and potentially using alternative approaches for critical mobile layouts.

Summary: vh and vw Best Practices

Key Takeaways

vh and vw units are powerful tools for creating responsive layouts and elements that adapt to the viewport size.

Best Use Cases

  • Full-screen sections and hero areas
  • Maintaining aspect ratios
  • Creating smoothly scaling designs
  • Elements that need to relate to viewport dimensions

Important Guidelines

  • Use thoughtfully, not for everything
  • Combine with other CSS techniques for best results
  • Be cautious with font sizes (can become too small/large)
  • Test on mobile devices (address bar changes viewport height)
  • Ensure good user experience across all devices
Remember: 1vh = 1% of viewport height, 1vw = 1% of viewport width. Use them to create fluid, responsive designs that adapt seamlessly to any screen size.
