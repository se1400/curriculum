Font Size and Color Contrast

Text inputs need adequate font sizing and color contrast for accessibility.

‚ùå Poor Practice
  • Small font size (13.6px)
  • Thin border (1px)
  • Light placeholder color
‚úì Good Practice
  • Readable font size (16px)
  • Clear border (2px)
  • Visible placeholder color

Don't Forget the Placeholder!

The placeholder is often forgotten, but it's also text and needs proper styling to ensure readability.

CSS
input::placeholder {
  color: #666; /* Readable, but not too dark */
  opacity: 1; /* Ensure consistent rendering */
}

‚ö†Ô∏è Common Mistake:

Using placeholder colors that are too light (#ccc, #ddd) makes them hard to read, especially for users with visual impairments.

‚úì Best Practice:

Use a medium gray (#666, #767676) that provides sufficient contrast while still differentiating placeholder text from actual input.

Allow User Control: Resizing

You should allow the user to modify the input. For example, if it's a textarea you should not remove the ability to resize it.

‚ùå Poor Practice
textarea {
  resize: none; /* ‚ùå Bad */
}
‚úì Good Practice
textarea {
  resize: vertical; /* ‚úì Good */
}

Responsive to Zoom

The input should scale accordingly when the user zooms the page.

Best Practices:

  • Use relative units (rem, em) for font sizes
  • Avoid fixed pixel widths when possible
  • Test your forms at 200% zoom
  • Ensure inputs don't break layout when zoomed
input {
  font-size: 1rem; /* Scales with zoom */
  padding: 0.8rem; /* Scales with zoom */
  width: 100%; /* Responsive to container */
  max-width: 500px; /* Prevents too wide */
}

Focus Indicators

Input elements are focusable. When styling, you should preserve a noticeable indicator when the element has focus, such as a bold border.

Notice the border color and shadow when focused

input:focus {
  outline: none;
  border-color: #881518;
  box-shadow: 0 0 0 3px rgba(136, 21, 24, 0.1);
}

Why Focus Indicators Matter

Focus indicators are essential for:

  • Keyboard users - Shows which element is currently active
  • Motor impairments - Helps users track where they are
  • Screen reader users - Visual confirmation of focus location
  • Everyone - Makes navigation clearer

‚ö†Ô∏è Never Remove Focus Indicators:

/* ‚ùå NEVER do this */
input:focus {
  outline: none; /* Without a replacement */
}

If you remove the default outline, you MUST provide an alternative focus indicator.

Error States

There's also the error state to consider. When the user's text does not pass validation, there should be a visual indicator.

‚ö†Ô∏è Please enter a valid email address.
CSS
.error-input {
  border-color: #dc3545;
}

.error-message {
  color: #dc3545;
  font-size: 0.9rem;
  margin-top: 0.5rem;
}

Error State Best Practices

Error states should be:

  • Noticeable - Clear visual indication of the error
  • Distinct from focus - Different from the focus indicator
  • Color + Icon/Text - Don't rely on color alone
  • Properly labeled - Use aria-describedby for screen readers
HTML with ARIA
<input
  type="email"
  id="email"
  name="email"
  aria-describedby="email-error"
/>
<p id="email-error" class="error-message">
  Please enter a valid email address.
</p>

Focus vs Error States

The error state should be distinct from the focus indicator.

Focus State
  • Brand color border
  • Subtle shadow
  • Indicates active element
Error State
‚ö†Ô∏è This field is required
  • Red/error color border
  • Error message with icon
  • Indicates validation issue

Accessibility Checklist

‚úì Text Input Accessibility Checklist:

  • ‚òê Font size is at least 16px (1rem)
  • ‚òê Sufficient color contrast (text and placeholder)
  • ‚òê Placeholder text is readable
  • ‚òê Textareas can be resized (don't use resize: none)
  • ‚òê Inputs scale properly when page is zoomed
  • ‚òê Clear focus indicator is present
  • ‚òê Error states are noticeable and distinct from focus
  • ‚òê Error messages use aria-describedby
  • ‚òê Labels are properly associated with inputs
  • ‚òê Tested with keyboard navigation

CSS Best Practices Template

/* Base input styles */
input,
textarea {
  font-size: 1rem; /* At least 16px */
  padding: 0.8rem;
  border: 2px solid #ddd;
  border-radius: 4px;
  font-family: inherit;
}

/* Placeholder styling */
input::placeholder,
textarea::placeholder {
  color: #666; /* Readable gray */
  opacity: 1;
}

/* Focus state */
input:focus,
textarea:focus {
  outline: none;
  border-color: #881518;
  box-shadow: 0 0 0 3px rgba(136, 21, 24, 0.1);
}

/* Error state */
input.error {
  border-color: #dc3545;
}

.error-message {
  color: #dc3545;
  font-size: 0.9rem;
  margin-top: 0.5rem;
}

/* Textarea resizing */
textarea {
  resize: vertical; /* Allow vertical resize */
  min-height: 100px;
}

Key Takeaways

Essential Points:

  • Font size: Use at least 16px (1rem) for inputs
  • Color contrast: Ensure text and placeholders are readable
  • Placeholder styling: Don't forget to style placeholders with readable colors
  • User control: Allow textarea resizing and proper zoom behavior
  • Focus indicators: Always provide clear, noticeable focus states
  • Error states: Make errors distinct, noticeable, and properly labeled

Remember: With all this in mind, you can now create captivating user forms that are accessible to everyone!
