Styling Different Link States

Links have different states that represent various user interactions and help users understand their browsing experience.

:link

Unvisited

:visited

Already clicked

:hover

Mouse over

:focus

Keyboard navigation

:active

Being clicked

Why Styling Link States Is Important

Styling different link states is crucial for usability and accessibility.

Provides Visual Cues About:

  • The current state of the link
  • Which links users have already visited
  • Which link they are currently interacting with
  • What will happen when they click

Accessibility & User Experience Benefits

For Users with Visual Impairments or Cognitive Disabilities

Distinct link styles make navigation much easier and more intuitive.

For All Users

Clear link states enhance overall user experience by:

  • Providing immediate feedback on interactions
  • Reducing confusion
  • Improving the site's navigability

CSS Pseudo-Classes

Link states can be styled using pseudo-classes in CSS.

What is a Pseudo-Class?

A keyword added to a selector that specifies a special state of the selected element.

For example:

  • :hover can change a button's color when the user's pointer hovers over it
  • :visited can change the color of a link that has already been visited

Why Use Pseudo-Classes?

Pseudo-classes allow you to style elements based on their state or the user's interaction with them.

Key Advantage

No additional markup needed in your HTML.

You can create dynamic, interactive styles purely with CSS, without touching your HTML structure.

Pseudo-Class Syntax

The syntax of a pseudo-class follows this pattern:

A:B {
  property: value;
}

Where:

  • A is the selector (element you're targeting)
  • :B is the pseudo-class (special state)

Example: a:hover { color: red; }

The :link Pseudo-Class

The :link pseudo-class styles unvisited links, indicating that they are clickable.

CSS Code

a:link {
  color: red;
}

HTML

<a href="/">Example link</a>

Result

Example link

This changes the link's default blue color to red when it is unvisited.

The :visited Pseudo-Class

The :visited pseudo-class styles links that have already been visited or clicked.

This helps users track which links they have clicked before.

CSS Code

a:visited {
  color: green;
}

Result

After clicking, the link turns green.

This provides visual history of user navigation.

The :hover Pseudo-Class

The :hover pseudo-class changes the link's style when the user hovers over it.

This provides a visual cue that the link is interactive.

CSS Code

a:hover {
  color: green;
}

The :focus Pseudo-Class

The :focus pseudo-class adds styles when a link is focused, such as when navigating with a keyboard.

This enhances accessibility for keyboard users.

CSS Code

a:focus {
  outline: 2px solid orange;
}

The :active Pseudo-Class

The :active pseudo-class changes the link's styles while the link is being clicked.

This provides immediate feedback to the user that their action is being registered.

CSS Code

a:active {
  color: pink;
}

Putting It All Together

/* Unvisited links */
a:link {
  color: blue;
}

/* Visited links */
a:visited {
  color: purple;
}

/* Mouse hover */
a:hover {
  color: green;
  background-color: yellow;
}

/* Keyboard focus */
a:focus {
  outline: 2px solid orange;
}

/* Active (being clicked) */
a:active {
  color: red;
}

Important: Declaration Order

When styling link pseudo-classes, order matters due to CSS specificity rules.

Recommended Order (LoVe HAte)

  1. Link - a:link
  2. Visited - a:visited
  3. Hover - a:hover
  4. Active - a:active

Following this order ensures that all states work correctly without conflicts.
