sessionStorage

What is sessionStorage?

In the previous lesson, we learned about working with localStorage and were briefly introduced to sessionStorage.

Key Characteristic

Recall, that sessionStorage is when the data is cleared as soon as the user closes the tab or window in which the web application is running.

✓ When to Use sessionStorage

It's ideal for situations where data only needs to persist for the length of a single session, such as:

• Maintaining form data during navigation

• Storing temporary state information during a checkout process

• Keeping track of user actions within one session

⚠️ Important Difference

sessionStorage: Cleared when tab closes

localStorage: Persists indefinitely

Key-Value Storage

How sessionStorage Works

Much like localStorage, sessionStorage uses key-value pairs to store and retrieve data.

Same Methods as localStorage

The methods used with sessionStorage are also the same as localStorage, with the only real difference being how long the data is stored.

Key-Value Pair Structure

// Key: a string identifier // Value: the data to store (always stored as a string) sessionStorage.setItem('key', 'value');

Example Keys

• 'username'

• 'cart'

• 'currentPage'

• 'formData'

Example Values

• 'JohnDoe'

• '3 items'

• '2'

• '{"name":"Alice"}'

✓ Remember

All values are stored as strings. If you need to store objects or arrays, use JSON.stringify() when storing and JSON.parse() when retrieving!

setItem() Method

Storing Data

sessionStorage.setItem() stores a key-value pair in sessionStorage.

Syntax

sessionStorage.setItem(key, value);

Example

sessionStorage.setItem('cart', '3 items');

Parameters

key (string): The name of the key to create/update

value (string): The value to store

More Examples

// Store a username sessionStorage.setItem('currentUser', 'JohnDoe'); // Store a number (converted to string) sessionStorage.setItem('pageNumber', '5'); // Store JSON data const user = { name: 'Alice', age: 25 }; sessionStorage.setItem('userData', JSON.stringify(user));

✓ Key Point

If a key already exists, setItem() will update its value. If it doesn't exist, it will create it.

getItem() Method

Retrieving Data

sessionStorage.getItem() retrieves the value of a given key from sessionStorage.

Syntax

const value = sessionStorage.getItem(key);

Example

let cart = sessionStorage.getItem('cart'); console.log(cart); // Outputs: '3 items'

Return Value

Returns the value associated with the key, or null if the key doesn't exist.

More Examples

// Retrieve a username const user = sessionStorage.getItem('currentUser'); console.log(user); // 'JohnDoe' // Retrieve and parse JSON data const userDataString = sessionStorage.getItem('userData'); const userData = JSON.parse(userDataString); console.log(userData.name); // 'Alice' // Check if key exists const value = sessionStorage.getItem('nonexistent'); console.log(value); // null

⚠️ Remember

Always check if the returned value is null before using it!

removeItem() Method

Removing Specific Data

sessionStorage.removeItem() removes a specific item from sessionStorage using its key.

Syntax

sessionStorage.removeItem(key);

Example

sessionStorage.removeItem('cart');

What It Does

Deletes the key-value pair with the specified key from sessionStorage.

If the key doesn't exist, nothing happens (no error is thrown).

Complete Example

// Store data sessionStorage.setItem('tempData', 'temporary value'); // Check it exists console.log(sessionStorage.getItem('tempData')); // 'temporary value' // Remove it sessionStorage.removeItem('tempData'); // Try to get it again console.log(sessionStorage.getItem('tempData')); // null

✓ Use Case

Use removeItem() when you want to delete one specific piece of data but keep everything else.

clear() Method

Removing All Data

sessionStorage.clear() clears all data stored in sessionStorage.

Syntax

sessionStorage.clear();

Example

sessionStorage.clear();

⚠️ Important

This removes ALL items from sessionStorage at once!

Use this carefully - it cannot be undone.

Complete Example

// Store multiple items sessionStorage.setItem('user', 'JohnDoe'); sessionStorage.setItem('cart', '3 items'); sessionStorage.setItem('page', '5'); // Clear everything sessionStorage.clear(); // All items are now gone console.log(sessionStorage.getItem('user')); // null console.log(sessionStorage.getItem('cart')); // null console.log(sessionStorage.getItem('page')); // null

✓ Use Case

Use clear() when you want to reset the session completely, such as when a user logs out or completes a multi-step process.

Complete Example

All Methods Together

Let's look at an example where we store data in sessionStorage which only lasts as long as the browser tab or window is open:

// Store data in sessionStorage sessionStorage.setItem('currentUser', 'JohnDoe'); // Retrieve the stored data const user = sessionStorage.getItem('currentUser'); console.log(user); // 'JohnDoe' // Remove a specific key from sessionStorage sessionStorage.removeItem('currentUser'); // Clear all sessionStorage data sessionStorage.clear();

In this example, we:

1 Store the current user's name (JohnDoe) in sessionStorage.

2 Retrieve and display it.

3 Remove the item associated with the key currentUser.

4 Clear all sessionStorage data.

⚠️ Key Difference

The key difference from localStorage is that as soon as the user closes the tab, all stored session data will be lost.

When to Use sessionStorage

Ideal Scenarios

sessionStorage is particularly useful in scenarios like:

1. Multi-Page Forms

Storing temporary data such as form entries during a multi-page form process.

Example: A checkout process spanning multiple pages

2. Temporary Preferences

Storing temporary selections or preferences that don't need to persist across sessions.

Example: Filter selections, sort order

3. Single-Page Apps

Maintaining state on a single-page application that doesn't need to be remembered once the tab is closed.

Example: Current view, temporary settings

4. Session-Based Data

Any data that should only exist for one browsing session.

Example: Shopping cart, current game state

✓ Key Benefit

sessionStorage ensures that once the user leaves the page, the session data is cleared, which is great for scenarios where you don't want to hold onto information beyond the current session.

localStorage vs sessionStorage

Key Differences

Both use the same methods, but differ in data persistence:

Feature sessionStorage localStorage
Data Persistence Until tab/window closes Indefinitely (until manually cleared)
Scope Per tab/window Shared across all tabs/windows
Use Case Temporary session data Long-term user preferences
Storage Limit ~5-10MB ~5-10MB
Methods setItem, getItem, removeItem, clear setItem, getItem, removeItem, clear
Shared Between Tabs No Yes

Choose Based on Your Needs

Use sessionStorage when: Data should be temporary and session-specific

Use localStorage when: Data should persist across sessions and browser restarts

Real-World Examples

1. Multi-Step Form

// Save form data on each step function saveFormStep(step, data) { sessionStorage.setItem(`formStep${step}`, JSON.stringify(data)); } // Retrieve form data function getFormStep(step) { const data = sessionStorage.getItem(`formStep${step}`); return data ? JSON.parse(data) : null; }

2. Shopping Cart

// Add item to cart function addToCart(item) { let cart = JSON.parse(sessionStorage.getItem('cart')) || []; cart.push(item); sessionStorage.setItem('cart', JSON.stringify(cart)); } // Get cart items function getCart() { return JSON.parse(sessionStorage.getItem('cart')) || []; }

3. Current Page State

// Save current state function saveState(state) { sessionStorage.setItem('appState', JSON.stringify(state)); } // Restore state on page load window.addEventListener('load', () => { const savedState = sessionStorage.getItem('appState'); if (savedState) { const state = JSON.parse(savedState); restoreApp(state); } });

Best Practices

✓ Do

• Use sessionStorage for temporary, session-specific data

• Always check if data exists before using it

• Use JSON.stringify() for objects and arrays

• Use JSON.parse() when retrieving objects

• Clear sensitive data when no longer needed

• Validate data after retrieving from storage

⚠️ Don't

• Don't store sensitive information (passwords, credit cards)

• Don't rely on sessionStorage for critical data persistence

• Don't store very large amounts of data

• Don't forget to handle the case when storage is full

• Don't assume data will persist across tabs

Security Considerations

• sessionStorage is accessible via JavaScript

• Data is not encrypted

• Vulnerable to XSS attacks

• Never store passwords or tokens

Common Patterns

Helper Function for Safe Storage

// Safe set with error handling function safeSetItem(key, value) { try { sessionStorage.setItem(key, JSON.stringify(value)); return true; } catch (error) { console.error('Storage error:', error); return false; } } // Safe get with default value function safeGetItem(key, defaultValue = null) { try { const item = sessionStorage.getItem(key); return item ? JSON.parse(item) : defaultValue; } catch (error) { console.error('Parse error:', error); return defaultValue; } }

Checking Storage Availability
