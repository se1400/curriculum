The action Attribute

Where to Send Data

The first attribute that we need to look at is the action attribute.

The action attribute should contain either a URL or a relative path for the current domain.

What It Does

This value determines where the form attempts to send data.

If you do not set an action attribute, the form will send data to the current page's URL.

Example 1: Absolute URL

<form action="https://freecodecamp.org"> <input type="number" placeholder="Enter a number" name="number" /> <button type="submit">Submit</button> </form>

When this form is submitted, it will send data to freeCodeCamp's homepage.

Example 2: Relative Path

<form action="/data"> <input type="number" placeholder="Enter a number" name="number" /> <button type="submit">Submit</button> </form>

This one submits to the /data path on the current domain.

http://127.0.0.1:5500/data

The method Attribute

How to Send Data

The second attribute to control how a submission behaves is the method attribute.

This attribute accepts a standard HTTP method, such as GET or POST, and uses that method when making the request to the action URL.

What is HTTP?

HTTP stands for Hypertext Transfer Protocol and it is used to transfer data over the web.

HTTP methods are used to define the actions that can be performed on resources, such as GET, POST, PUT, DELETE, and so on.

You will learn more about these methods in future lessons.

Default Method

When a method is not set, the form will default to a GET request.

The GET Method

Retrieving Data

A GET request is used to retrieve data from a specified resource without making any changes to it.

The parameters are typically appended to the URL in the form of a query string.

How GET Works

The data in the form will be URL encoded as name=value pairs and appended to the action URL as query parameters.

Example

<form action="/data"> <input type="number" name="number" /> <button type="submit">Submit</button> </form>

What Happens

For instance, by submitting the form with the value of 3342 for the input field, the form would try to GET the URL:

http://127.0.0.1:5500/data?number=3342

Notice the ?number=3342 part - this is the query string containing your form data!

‚úì Great For

This is great for something like a search form, where your user is querying data.

The POST Method

Sending New Data

But if you want your user to submit new data, such as in a registration form, the idiomatic method to use would be the POST method.

The POST method is used to send data to the server to create or update a resource.

Example with POST

<form action="/data" method="POST"> <input type="number" name="number" /> <button type="submit">Submit</button> </form>

Key Difference

When you send a POST request, a body can be included which contains the data for your request.

So unlike a GET, the data are not appended to the URL.

What Happens

Our form sends the POST request to:

http://127.0.0.1:5500/data

Notice: No query string! The data, instead, can be found in the body of the request, still as URL-encoded form data.

‚úì Great For

‚Ä¢ Registration forms

‚Ä¢ Login forms

‚Ä¢ Creating new records

‚Ä¢ Updating existing data

URL-Encoded Form Data

What is URL Encoding?

URL-encoded form data is when form data is converted into a string of key-value pairs.

Format

Data is formatted as name=value pairs, separated by & symbols.

Special characters are replaced with encoded versions to safely send the data over the web.

Example

name=John+Doe&email=john%40example.com

What Happened Here?

‚Ä¢ Each field is a name=value pair

‚Ä¢ Pairs are joined with &

The enctype Attribute

Encoding Type

But maybe you don't want to send the data as a URL-encoded form payload?

The form element accepts an enctype attribute, which represents the encoding type to use for the data.

Three Accepted Values

This attribute only accepts three values:

1. application/x-www-form-urlencoded

Default value

Sends the data as a URL-encoded form body

Example: name=John+Doe&email=john%40example.com

2. text/plain

Sends the data in plaintext form

In name=value pairs separated by new lines

No encoding of special characters

3. multipart/form-data

Specifically for handling forms with a file upload

Each value is sent as a separate part of the message

Required when uploading files!

enctype in Action

1. URL-Encoded (Default)

<form action="/submit" method="POST"> <input name="username" /> <input name="email" /> <button type="submit">Submit</button> </form>
username=john&email=john%40example.com

2. Text/Plain

<form action="/submit" method="POST" enctype="text/plain"> <input name="username" /> <input name="email" /> <button type="submit">Submit</button> </form>
username=john email=john@example.com

3. Multipart/Form-Data (For File Uploads)

<form action="/upload" method="POST" enctype="multipart/form-data"> <input type="file" name="document" /> <button type="submit">Upload</button> </form>

Data is sent as separate parts with boundaries between each field.

‚ö†Ô∏è Important

Always use enctype="multipart/form-data" when your form includes file uploads!
