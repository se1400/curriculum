Width and Height Properties

In CSS, the width and height properties control the dimensions of elements on a webpage.

Available Units

These properties can be defined in different units:

  • Pixels (px) - Fixed-size units
  • Percentages (%) - Relative to parent container
  • Viewport units (vw, vh) - Relative to viewport
  • And more...

Width Property

The width property specifies the width of an element. Default is auto, which lets the browser determine width based on content, parent, and display type.

Height Property

The height property specifies the height of an element. Default is auto, which adjusts to the content inside.

Pixels (px)

Pixels are a fixed-size unit of measurement in CSS, providing precise control over dimensions.

CSS
.box {
  width: 100px;
  height: 100px;
  background-color: skyblue;
}
Visual Result
100px × 100px

min-width Property

Specifies the minimum width an element can be. Even if content is smaller, the element won't shrink below this value.

min-height Property

Specifies the minimum height an element can be. Ensures the element doesn't become shorter than the set value.

Override Behavior

If min-width or min-height are larger than width or height, they will override the smaller values.

CSS
.box {
  width: 50px;
  min-width: 100px;
  height: 50px;
  min-height: 100px;
  background-color: lightcoral;
}
Visual Result

Set to 50px but displays as 100px

100px × 100px (min overrides)

Minimum Properties Use Case

This ensures that elements don't become too small, which is important for maintaining a consistent and usable design.

Remember: When min-width or min-height are larger than width or height, the minimum values take precedence.
Comparison

Without min-width/height

50px × 50px

With min-width/height

100px × 100px

max-width Property

Specifies the maximum width an element can grow to, even if there is enough space for it to be wider.

max-height Property

Specifies the maximum height an element can grow to, regardless of the content size.

Override Behavior

When max-width or max-height are smaller than width or height, they take precedence.

CSS
.box {
  width: 200px;
  max-width: 150px;
  height: 200px;
  max-height: 150px;
  background-color: lightgreen;
}
Visual Result

Set to 200px but displays as 150px

150px × 150px (max restricts)

Maximum Properties Use Case

This is important for controlling the maximum size of elements in your layouts, preventing them from becoming too large.

Remember: When max-width or max-height are smaller than width or height, the maximum values limit the size.
Comparison

Without max-width/height

200px × 200px

With max-width/height

150px × 150px

CSS Priority Rules

CSS prioritizes min-width and min-height over width and height. The max-width and max-height restrict dimensions if values exceed their limits.

Scenario Properties Set Final Result
Max restricts width width: 600px; max-width: 500px; Element will be 500px wide
Min overrides height height: 150px; min-height: 200px; Element will be 200px tall
Normal width width: 300px; (no min/max) Element will be 300px wide

Key Takeaways

  • width and height set the basic dimensions of elements
  • Default value is auto for both properties
  • min-width and min-height prevent elements from becoming too small
  • max-width and max-height prevent elements from becoming too large
  • Min/max properties always override standard width/height when there's a conflict

Why This Matters

These properties ensure elements stay within desired size ranges, regardless of standard width and height values. This is essential for creating responsive, consistent, and usable designs.
