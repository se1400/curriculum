Array includes() Method

Checking Array Contents

In JavaScript, the includes() method is a simple and efficient way to check if an array contains a specific value.

What It Returns

This method returns a boolean value:

• true if the array contains the specified element

• false otherwise

✓ When to Use

The includes() method is particularly useful when you need to quickly verify the presence of an element in an array without needing to know its exact position.

Basic Syntax

array.includes(searchElement) array.includes(searchElement, fromIndex)

Basic Usage

Simple Example

Let's start with an example of how to use the includes() method:

let fruits = ["apple", "banana", "orange", "mango"]; console.log(fruits.includes("banana")); // true console.log(fruits.includes("grape")); // false

Explanation

In this example, we have an array of fruits.

• We use the includes() method to check if "banana" is in the array. It returns true because banana is indeed present.

• We then check for "grape", which returns false because it's not in the array.

Case Sensitivity

⚠️ Important

The includes() method is case-sensitive when dealing with strings.

This means that "Banana" with a capital B and "banana" with all lowercase letters are considered different values.

Example

let fruits = ["apple", "banana", "orange"]; console.log(fruits.includes("banana")); // true console.log(fruits.includes("Banana")); // false

Explanation

In this case, "banana" (all in lowercase) is found in the array, but "Banana" (with the first letter capitalized) is not, so the second includes() call returns false.

Starting Position Parameter

Optional Second Parameter

The includes() method can also accept an optional second parameter that specifies the position in the array to start the search.

This is useful if you want to check for an element's presence in a specific part of the array.

Syntax

array.includes(searchElement, fromIndex)

Example

let numbers = [10, 20, 30, 40, 50, 30, 60]; console.log(numbers.includes(30, 3)); // true console.log(numbers.includes(30, 4)); // true

Explanation

For the first console.log, we are looking for the number 30 starting at index 3. In this case, there is a number 30 that appears after index 3, so the includes() method returns true.

The same is true for the second console.log. We are looking for the number 30 starting at index 4. Since the number 30 does appear after that index, then it will return true.

Strict Equality (Type Checking)

⚠️ Important

It's worth noting that includes() uses the strict equality comparison (===), which means it can distinguish between different types.

Example

let mixedArray = [1, "2", 3, "4", 5]; console.log(mixedArray.includes(2)); // false console.log(mixedArray.includes("2")); // true

Explanation

In this case, the number 2 and the string "2" are considered different data types.

So, the first console.log will return false, while the second console.log will return true.

Common Use Cases

1. Validation

const allowedRoles = ['admin', 'editor']; if (allowedRoles.includes(userRole)) { // Grant access }

Check if user has permission

2. Filtering

const exclude = ['spam', 'deleted']; const filtered = emails.filter( email => !exclude.includes(email.status) );

Filter out unwanted items

3. Membership Testing

const premium = ['gold', 'platinum']; if (premium.includes(subscription)) { showPremiumFeatures(); }

Check membership in a group

4. Avoiding Duplicates

if (!cart.includes(item)) { cart.push(item); }

Add only if not already present

Why Use includes()?

✓ Advantages

Simple & Readable

Clear and concise syntax

arr.includes('value')

vs arr.indexOf('value') !== -1

Efficient

Fast performance

• Optimized by JavaScript engines

• Stops when item found

• No unnecessary iterations

Flexible

Works with any type

• Strings

• Numbers

• Booleans

• Objects (by reference)

No Complex Logic

Saves you from writing loops

// No need for: for (let item of arr) { if (item === value) ... }

Best Practices

✓ Do

• Use includes() for simple existence checks

• Remember case sensitivity with strings

• Be aware of type differences (number vs string)

• Use the second parameter to search from a specific position

• Prefer includes() over indexOf() !== -1

⚠️ Be Careful

• Case sensitivity: "Apple" ≠ "apple"

• Type checking: 2 ≠ "2"

• Object references: Checks reference equality, not value equality

• NaN handling: includes() can find NaN, indexOf() cannot

When Not to Use

• If you need the index position → use indexOf()

• If you need to find an object by property → use find()

• For complex conditions → use some()

Key Takeaways

The includes() Method

The includes() method is a powerful tool for checking the presence of elements in arrays.

✓ Essential Facts

• Returns true or false

• Case-sensitive for strings

• Uses strict equality (===)

• Optional starting position parameter

• Simple, efficient, and readable

Syntax

array.includes(searchElement) array.includes(searchElement, fromIndex)

Remember

It's simple to use, efficient, and can save you from writing more complex loops or conditions to search through arrays.

Whether you're working with strings, numbers, or mixed data types, includes() provides a straightforward way to verify if a value exists in your array.
