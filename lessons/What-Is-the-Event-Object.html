What is the Event Object?

Definition

The Event object is a payload that triggers when a user interacts with your web page in some way.

Types of User Interactions

Clicking a button
Typing on keyboard
Moving the mouse
Focusing an input
Scrolling the page
Shaking device
Touching screen
Submitting a form

Any interaction from clicking a button to shaking a mobile device can trigger an Event object.

Event Object Properties

Like All JavaScript Objects

The Event object has a number of properties that might be helpful.

Important Note

The properties available depend on the event that triggered the payload.

Different types of events provide different information through their properties.

What Properties Provide

  • Information: Details about the interaction
  • Context: What element was involved
  • Data: Specific values related to the event

The type Property

Universal Property

All Event objects will have the type property.

What Does type Tell Us?

The type property reveals the type of event that triggered the payload.

Common Event Types

User Actions

  • "click"
  • "dblclick"
  • "mousedown"
  • "mouseup"

Keyboard Events

  • "keydown"
  • "keyup"
  • "keypress"

These values correspond to the same values you pass to addEventListener().

Capturing the Event Object

Using addEventListener()

You can capture and utilize the Event object in event listeners.

Example Code

const button = document.querySelector("button");

button.addEventListener("click", function(event) {
  console.log(event.type); // Output: "click"
});

Breaking it Down

  • We add an event listener for the "click" event
  • The callback function receives the event object as a parameter
  • We can access event.type to see what type of event occurred

The target Property

Another Universal Property

Event objects will always have the target property.

What is target?

The target property is a reference to whatever object triggered the event.

Common Target Types

  • HTMLElement: Most commonly, like a button, div, or input
  • Document: The document object
  • Window: The window object
  • Specific Objects: Like an AudioContext or other specialized objects

The target tells you what element or object the user interacted with.

Using the target Property

Example Code

const button = document.querySelector("button");

button.addEventListener("click", function(event) {
  console.log(event.target); // The button element
  event.target.style.backgroundColor = "blue";
});

What's Happening?

  • We access event.target to get the element that was clicked
  • We can then modify that element directly
  • In this case, we change its background color to blue

Useful for: Identifying which element triggered an event, especially when multiple elements share the same event listener.

Event Methods

What Are Event Methods?

Events have methods, which are functions exposed as properties on the object.

Two Commonly Used Methods

preventDefault()

Prevents the default behavior of the event

stopPropagation()

Prevents event bubbling to parent elements

These methods give you control over how events behave in your application.

preventDefault() Method

What Does It Do?

preventDefault() prevents the default behavior of the event when called.

Common Use Case: Form Submission

If you want to handle form submissions yourself, call preventDefault() to keep the browser from trying to submit the form data as a POST request.

const form = document.querySelector("form");

form.addEventListener("submit", function(event) {
  event.preventDefault(); // Stop default form submission
  // Now handle form data with JavaScript
  console.log("Form submitted!");
});

Note: You'll learn more about POST requests in future lessons.

More preventDefault() Use Cases

Preventing Link Navigation

const link = document.querySelector("a");

link.addEventListener("click", function(event) {
  event.preventDefault(); // Don't navigate to URL
  // Handle click in a custom way
});

Other Common Uses

  • Links: Prevent navigation to create custom click behavior
  • Forms: Validate input before submission
  • Context Menus: Prevent right-click menu from appearing
  • Keyboard Shortcuts: Override browser shortcuts

stopPropagation() Method

What Does It Do?

stopPropagation() prevents the event from bubbling up or propagating to parent elements.

Event Bubbling

When an event occurs on an element, it also triggers on all parent elements (bubbles up the DOM tree).

You'll learn more about event bubbling in a future lesson.

Example Scenario

const button = document.querySelector("button");

button.addEventListener("click", function(event) {
  event.stopPropagation(); // Stop bubbling
  console.log("Button clicked");
});

This prevents the click event from triggering event listeners on parent elements.

Event-Specific Properties

Beyond the Basics

There are a large number of properties that are specific to certain implementations of the Event object.

Examples of Specific Properties

FetchEvent

Has a request property containing the request that triggered the event

KeyboardEvent

Has properties like key, keyCode, and shiftKey

MouseEvent

Has properties like clientX, clientY, and button

TouchEvent

Has properties like touches and changedTouches

Discovering Event Properties

Unsure What's Available?

If you are ever unsure of what properties are available, you have two options:

Method 1: Log the Event Object

button.addEventListener("click", function(event) {
  console.log(event); // See all properties
});

Method 2: Check Documentation

  • MDN Web Docs (developer.mozilla.org)
  • Official JavaScript documentation
  • Browser developer tools

Best Practice: Console.log the event object during development to see what's available, then refer to documentation for detailed information.

Summary

Key Concepts

What is the Event Object?

A payload containing information about user interactions with your web page.

Universal Properties

  • type: The type of event ("click", "keydown", etc.)
  • target: The element or object that triggered the event

Common Methods

  • preventDefault(): Stop default behavior
  • stopPropagation(): Prevent event bubbling

Remember

Different events have different properties. Always explore the Event object or check documentation to discover what's available for your specific use case.
