Break and Continue Statements

What Are These Statements?

JavaScript provides two special statements to control the flow of loops:

Break Statement

Purpose:

Used to exit a loop early, completely terminating the loop before it would naturally end.

break;

Continue Statement

Purpose:

Used to skip the current iteration and move to the next one without exiting the loop.

continue;
Key Difference:

• break → Stops the entire loop
• continue → Skips only the current iteration

The Break Statement

Exiting a Loop Early

for (let i = 0; i < 10; i++) {
    if (i === 5) {
        break; // Exit the loop when i equals 5
    }
    console.log(i);
}
Output:
0
1
2
3
4

Execution Flow

i = 0 → 0 !== 5 → Print 0 ✓
i = 1 → 1 !== 5 → Print 1 ✓
i = 2 → 2 !== 5 → Print 2 ✓
i = 3 → 3 !== 5 → Print 3 ✓
i = 4 → 4 !== 5 → Print 4 ✓
i = 5 → 5 === 5 → BREAK! Loop exits immediately ❌
Loop stops here. i = 6, 7, 8, 9 never execute.
When to Use Break:

The break statement is useful when you want to exit a loop early based on a condition. For example, when searching for a specific value in an array, you can use break to exit once you find it.

The Continue Statement

Skipping an Iteration

for (let i = 0; i < 10; i++) {
    if (i === 5) {
        continue; // Skip when i equals 5
    }
    console.log(i);
}
Output:
0
1
2
3
4
6
7
8
9

Execution Flow

i = 0 → 0 !== 5 → Print 0 ✓
i = 1 → 1 !== 5 → Print 1 ✓
i = 2 → 2 !== 5 → Print 2 ✓
i = 3 → 3 !== 5 → Print 3 ✓
i = 4 → 4 !== 5 → Print 4 ✓
i = 5 → 5 === 5 → CONTINUE! Skip this iteration ⏭️
i = 6 → 6 !== 5 → Print 6 ✓
i = 7 → 7 !== 5 → Print 7 ✓
i = 8 → 8 !== 5 → Print 8 ✓
i = 9 → 9 !== 5 → Print 9 ✓
Notice:

The number 5 is skipped in the output, but the loop continues with 6, 7, 8, and 9. The loop doesn't exit—it just skips that one iteration.

Break vs Continue: Side by Side

Same Loop, Different Results

With Break

for (let i = 0; i < 10; i++) {
    if (i === 5) {
        break;
    }
    console.log(i);
}
0, 1, 2, 3, 4
Loop exits completely

Stops at i = 5. Numbers 5-9 never print.

With Continue

for (let i = 0; i < 10; i++) {
    if (i === 5) {
        continue;
    }
    console.log(i);
}
0, 1, 2, 3, 4, 6, 7, 8, 9
Loop continues running

Skips i = 5 only. Continues with 6-9.

Key Takeaway:

Use break when you want to stop the loop entirely. Use continue when you want to skip one iteration but keep the loop running.

Using Labels with Break and Continue

What Are Labels?

You can use labels to specify which loop you want to break or continue when working with nested loops.

Label Syntax

// Syntax for labels
labelName: for (...) {
    break labelName; // Break the labeled loop
    continue labelName; // Continue the labeled loop
}
Why Use Labels?

Labels are useful when you have nested loops (loops inside loops) and you want to control the flow of the outer loop from within the inner loop.

Note:

Most of the time you will not find the need to use labels, but it's good to know they exist for complex nested loop scenarios.

Labeled Break in Nested Loops

Breaking Out of Nested Loops

outerLoop: for (let i = 0; i < 3; i++) {
    innerLoop: for (let j = 0; j < 3; j++) {
        if (i === 1 && j === 1) {
            break outerLoop; // Exit the outer loop
        }
        console.log(`i: ${i}, j: ${j}`);
    }
}
Output:
i: 0, j: 0
i: 0, j: 1
i: 0, j: 2
i: 1, j: 0

What Happens

i = 0, j = 0 → Print "i: 0, j: 0" ✓
i = 0, j = 1 → Print "i: 0, j: 1" ✓
i = 0, j = 2 → Print "i: 0, j: 2" ✓
Outer loop: i becomes 1, inner loop starts
i = 1, j = 0 → Print "i: 1, j: 0" ✓
i = 1, j = 1 → break outerLoop! Exit BOTH loops ❌
Both inner and outer loops exit immediately.
i = 1 (j = 2), i = 2 never execute.
Key Point:

Using break outerLoop; exits the outer loop from inside the inner loop. Without the label, only the inner loop would exit.

Summary & Best Practices

Quick Reference

Break Statement

  • Exits the loop completely
  • Use when you've found what you're looking for
  • Can use with labels to exit outer loops

Continue Statement

  • Skips current iteration only
  • Loop continues with next iteration
  • Use to skip unwanted values

Common Use Cases

Break Examples

  • Searching for an item in an array
  • Stopping when a condition is met
  • Early termination for efficiency
  • Exiting nested loops

Continue Examples

  • Skipping invalid values
  • Filtering data in a loop
  • Ignoring certain conditions
  • Processing only specific items

Labels

When to Use Labels:

• Primarily used with nested loops
• Control outer loop from inner loop
• Uncommon in everyday code
• Good to know, but rarely needed

Remember:

Choose break to stop completely, choose continue to skip and keep going. Both are powerful tools for controlling loop execution and making your code more efficient!
