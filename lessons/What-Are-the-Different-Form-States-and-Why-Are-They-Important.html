Form States in HTML

In HTML, form controls, like inputs, can be in different stages or conditions such as a focused state, readonly state, or disabled state.

Default State

The initial appearance when first rendered on the page (typically blank).

Focused State

When the user clicks or tabs to the input.

Disabled State

Input cannot be focused or activated.

Readonly State

Input is not editable but can be focused.

Why Form States Matter: Understanding the different form states is important because they ensure a smooth user experience by providing clear feedback and guidance while handling errors.

Default State

The first state would be considered the default state. The default state of an email address input is a blank input. That is what the email input looks like when it is first rendered on the page.

Example: Default Email Input

<input type="email" name="email" id="email" />
Output (default state):
This is the natural state of the input before any user interaction. It appears blank and ready to accept input.

Focused State

When the user clicks on a form control or selects it with the keyboard's tab key, then that means it is in the focused state.
When an input is in the focused state, most browsers will show a blue highlighted border around the input. But you can choose to add additional styles in CSS.

Example: Try Clicking or Tabbing

<input type="email" name="email" id="email" />
Interactive Output (click or press Tab to focus):
Try It: Click on the input above or press the Tab key to see the focused state with the highlighted border.

Disabled State

Another form state is the disabled state. This state shows users that an input cannot be focused or activated.

Example: Disabled Email Input

<input disabled type="email" name="email" id="email" />
Interactive Output (try clicking on it):
When you try clicking on the email input above, you will notice that it will not focus. The disabled state prevents all interaction.
Similar to the focused state, you can choose to add additional styles for the disabled state using CSS to make it more visually apparent to users.

Readonly State

Another type of form state is the readonly state. This is when a form control, like an input, is not editable by the user.

Example: Readonly Email Input

<input
  readonly
  type="email"
  name="email"
  id="email"
  value="example@email.com"
/>
Interactive Output (try editing the value):
The value attribute is used to set the value shown inside the input field. Try editing the value above, and you will notice that is not possible.

Disabled State

Cannot be focused - completely inactive and prevents all user interaction.

Readonly State

Can be focused - the value can be selected and copied, but not edited.

Key Difference: A readonly input can be focused while the disabled state cannot. This makes readonly useful for displaying information that users need to see or copy but not change.
