What are Template Literals?

In JavaScript, template literals are a powerful and flexible way to work with strings.

The Key Difference

Unlike regular strings, which use single (') or double (") quotes, template literals are defined with backticks (`).

`

What They Enable

  • Easier string manipulation
  • Embedding variables directly inside strings
  • Multi-line strings without escape characters
  • Embedding JavaScript expressions

Backticks vs Quotes

Regular Strings

Use single or double quotes:

let message1 = 'Hello';
let message2 = "World";

Limited functionality

Template Literals

Use backticks:

let message = `Hello, World!`;

Enhanced functionality with interpolation!

Template literals make it easier to create strings that span multiple lines or include expressions (like variables or even JavaScript code) directly within the string.

Your First Template Literal

Here's an example of a template literal:

const name = "Alice";
const greeting = `Hello, ${name}!`;
console.log(greeting);
Hello, Alice!

Notice:

  • The use of backticks (`) instead of quotes
  • The ${name} syntax inserts the variable value
  • Much cleaner than concatenation!

What is String Interpolation?

Definition

String interpolation allows you to embed variables and expressions directly inside a string.

The Syntax

${ variable }

Place this anywhere inside your template literal!

✓ Significant Improvement

This is a significant improvement over the older method, where you would concatenate strings and variables using the + operator.

Old Way vs New Way

The Old Way: String Concatenation with +

const name = "Alice";
const age = 25;
const message = "My name is " + name + " and I am " + age + " years old.";
console.log(message);

❌ Hard to read, lots of quotes and + operators

The New Way: Template Literals with Interpolation

const name = "Alice";
const age = 25;
const message = `My name is ${name} and I am ${age} years old.`;
console.log(message);

✓ Much cleaner and easier to read!

My name is Alice and I am 25 years old.

Why Template Literals are Better

As you can see, string interpolation with template literals is much cleaner and easier to read, especially when working with multiple variables.

Concatenation Problems

  • Hard to read with many variables
  • Easy to miss spaces
  • Lots of typing (quotes, +)
  • Error-prone

Template Literal Benefits

  • Easy to read
  • Clear variable placement
  • Less typing
  • Fewer errors

Compare These Two:

// Concatenation - confusing!
let msg = "Hello " + name + ", you have " + count + " new messages.";

// Template literal - clear!
let msg = `Hello ${name}, you have ${count} new messages.`;

Multiline Strings

Another great feature of template literals is that they support multiline strings.

Old Way: Escape Characters

let poem = "Roses are red,\nViolets are blue,\nJavaScript is fun,\nAnd so are you.";

Need \n for each line break

New Way: Natural Line Breaks

let poem = `Roses are red,
Violets are blue,
JavaScript is fun,
And so are you.`;

Just write naturally!

With template literals, you can simply write the string across multiple lines, and the formatting is preserved!

Roses are red, Violets are blue, JavaScript is fun, And so are you.

Embedding JavaScript Expressions

Template literals allow you to embed JavaScript expressions directly within the string, not just variables!

Example: Calculations Inside Template Literals

const song = "Bohemian Rhapsody";
const score = 9.5;
const highestScore = 10;
const output = `One of my favorite songs is "${song}". I rated it ${
  (score / highestScore) * 100
}%.`;
console.log(output);
One of my favorite songs is "Bohemian Rhapsody". I rated it 95%.

What Happened?

Inside the ${}, we performed a calculation:

  • (score / highestScore) * 100 was evaluated
  • Result: (9.5 / 10) * 100 = 95
  • The number 95 was inserted into the string

More Expression Examples

Example 1: Math Operations

const price = 20;
const quantity = 3;
const message = `Total cost: $${price * quantity}`;
console.log(message);
Total cost: $60

Example 2: String Methods

const name = "alice";
const greeting = `Hello, ${name.toUpperCase()}!`;
console.log(greeting);
Hello, ALICE!

Example 3: Conditional (Ternary) Expression

const age = 17;
const status = `You are ${age >= 18 ? 'an adult' : 'a minor'}.`;
console.log(status);
You are a minor.

When to Use Template Literals

Template literals are particularly useful when you need to:

  • Include variables or expressions in strings
  • Format complex strings with multiple values
  • Work with multiline text
  • Perform calculations within strings
  • Call methods on variables within strings

✓ The Bottom Line

Template literals are more concise and readable compared to traditional string concatenation.

Template Literals Summary

Key Takeaways

What are Template Literals?

A powerful way to work with strings using backticks (`) instead of quotes.

String Interpolation

Use ${variable} or ${expression} to embed values directly in strings.

Advantages

  • Cleaner syntax: No more messy concatenation with +
  • Multiline strings: Natural line breaks without \n
  • Expressions: Can evaluate JavaScript code inside strings
  • More readable: Especially with multiple variables

Basic Syntax

// Basic interpolation
`Hello, ${name}!`

// With expressions
`Total: $${price * quantity}`

// Multiline
`Line 1
Line 2
Line 3`

Remember

Use backticks for template literals, and ${} for interpolation!
