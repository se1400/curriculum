Accessing Array Elements

Zero-Based Indexing

Arrays in JavaScript use zero-based indexing, meaning:

  • The first element is at index 0
  • The second element is at index 1
  • The third element is at index 2
  • And so on...

Accessing Elements by Index

Use square brackets with the index number to access an element: 

const fruits = ["apple", "banana", "cherry"];

// Access the second element (index 1)
console.log(fruits[1]);
Output: "banana"
Index 0: "apple"
Index 1: "banana"
Index 2: "cherry"

Accessing Non-Existent Indices

What Happens When Index Doesn't Exist?

When you try to access an index that doesn't exist in the array, JavaScript returns undefined. 

let fruits = ["apple", "banana", "cherry"];

// Array has indices 0, 1, 2 (length is 3)
// Trying to access index 3 (doesn't exist)
console.log(fruits[3]);
Output: undefined

Why Undefined?

JavaScript doesn't throw an error when accessing a non-existent index. Instead, it returns undefined to indicate that no value exists at that position.

This is important to remember when working with loops or dynamic array access!

Updating Array Elements

Changing Existing Values

You can update an element by assigning a new value to a specific index using the assignment operator = 

let fruits = ["apple", "banana", "cherry"];

// Replace "banana" at index 1 with "blueberry"
fruits[1] = "blueberry";

console.log(fruits);
Output: ["apple", "blueberry", "cherry"]

Key Points

  • You can change any element in the array as long as you know its index
  • The array must be declared with let or var (not const) to modify elements
  • The original value is completely replaced with the new value

Adding New Elements

Extending an Array

You can add new elements to an array by assigning a value to an index that doesn't yet exist: 

let fruits = ["apple", "banana", "cherry"];

// Add a new element at index 3
fruits[3] = "date";

console.log(fruits);
Output: ["apple", "banana", "cherry", "date"]

How It Works

When you assign to an index beyond the current array length, JavaScript automatically:

  • Extends the array to accommodate the new index
  • Places the new value at that position
  • Updates the array's length property

⚠️ Caution: Creating Gaps in Arrays

The Danger of Skipping Indices

Exercise caution when assigning to indices much larger than the current array length! 

let fruits = ["apple", "banana"]; // length: 2, indices: 0, 1

// Skipping ahead to index 5
fruits[5] = "mango";

console.log(fruits);
console.log(fruits.length);
Output: ["apple", "banana", undefined, undefined, undefined, "mango"]
Length: 6
⚠️ Warning:

Creating gaps like this will fill the in-between indices with undefined elements, which can lead to unexpected behavior in your code. This is generally not recommended!

Best Practice

Instead of assigning to arbitrary indices, use array methods like push() to add elements safely to the end of an array.

Practical Applications

Real-World Usage

These fundamental array operations are essential for many programming tasks:

  • Building a todo list: Access tasks by index, update status, add new items
  • Processing data: Loop through arrays to read and modify values
  • Managing collections: Store and manipulate groups of related items
  • Complex data structures: Arrays are building blocks for more advanced patterns

Key Takeaways

  • Arrays use zero-based indexing (first element is at index 0)
  • Accessing non-existent indices returns undefined
  • Update elements with: array[index] = newValue
  • Add elements by assigning to the next available index
  • Avoid creating gaps by skipping indices
