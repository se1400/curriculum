Tables and Accessibility

The Visual Association Challenge
When we see a table, we immediately start making visual associations between the data and the headers.
While a sighted person may be able to understand the relationships in a table, making the connections between the values and the headers is much harder for people who use screen readers to navigate the table.
Your Role as a Developer
You are in charge of creating these associations and structuring your HTML markup in a way that is also easy for screen reader users to interpret.

Best Practice: Table Captions

Using the Caption Element
With the caption element, you can write the caption (or title) of a table, so users, especially those who use assistive technologies, can quickly understand the table's purpose and content.
Placement
You should place the caption element immediately after the opening tag of the table element. This way, screen readers and other assistive technologies can provide more context by announcing the caption before reading the content.
Example:
<table> <caption>Our Pets</caption> <!-- Table Rows and Columns --> </table>

Row and Column Headers

What Are Headers?
Headers are special cells, typically found at the start of a row or column, that describe the type of data stored in the row or column.
Defining Headers
You can define a row or column header with the table header element, th.
Example:
<table> <caption>Our Pets</caption> <thead> <tr> <!-- Column Headers --> <th>Name</th> <th>Age</th> <th>Type</th> </tr> </thead> <tbody> <tr> <th>Nora</th> <!-- Row Header --> <td>5</td> <td>Dog</td> </tr> </tbody> </table>

The Scope Attribute

Why Scope Matters
Associating the data cells with their corresponding headers is very important for screen readers. The scope attribute determines if a header is a row header or a column header.
Screen readers may guess this correctly from the table's structure, but it's usually recommended to explicitly indicate the scope to ensure clarity.
Common Values
The scope attribute has four possible values. The two you will use most often are:
  • col for column headers
  • row for row headers

Using Scope in Practice

Adding scope to headers:
<table> <caption>Our Pets</caption> <thead> <tr> <th scope="col">Name</th> <th scope="col">Age</th> <th scope="col">Type</th> </tr> </thead> <tbody> <tr> <th scope="row">Nora</th> <td>5</td> <td>Dog</td> </tr> <tr> <th scope="row">Gino</th> <td>2</td> <td>Cat</td> </tr> </tbody> </table>

Complex Tables with Spanning Headers

Headers Spanning Multiple Cells
If a column or row header spans across multiple cells, the scope will also be applied to each one of the cells individually.
Important Consideration
Some screen readers may not be able to interpret tables with complex structures, so you should try to flatten the table as much as possible to avoid row and column headers that span across multiple cells.
Your goal should always be to make sure users can access this information, even if their screen readers can handle complex table structures.

Cell Sizing Best Practices

Cell Width: Avoid using fixed values. Use relative values instead, like percentages.
Cell Height: Try to avoid defining cell height. This will allow users to adjust the text size to fit their needs.
Table Width: Let the browser determine the table width whenever possible, to reduce the need for horizontal scrolling.
Remember
HTML tables are essential for presenting structured data in an accessible and understandable format. By following these accessibility guidelines, you can create tables that are easy to understand for everyone.
