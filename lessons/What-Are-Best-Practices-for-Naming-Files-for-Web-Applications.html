File Naming Importance

As you start to build out your own web applications, you will need to be mindful of what you name your files.

You will want to name your files in a way that is easy to understand and maintain.

Good file naming practices help you and your team quickly identify what each file contains.

Why File Naming Matters

  • Makes files easy to understand
  • Improves maintainability
  • Helps team collaboration
  • Saves time searching for files
  • Creates organized projects

Bad File Name Example

By the name alone, it is not clear what the file is about.

This file could be for a products page, blog page, or any other type of page, but we don't know that from the file name.

The .html extension tells us this is an HTML file, but the name itself provides no useful information.

❌ Bad Example

index1234.html

This name is not descriptive. It uses random numbers that don't convey any meaning about the file's purpose or content.

Good File Name Example

By using a more descriptive name like about-us.html, it is clear what the file is about.

You will also notice in this file name that there are no spaces. Instead, you use a hyphen (-) to separate the words.

Descriptive names make it immediately obvious what content the file contains.

✓ Good Example

about-us.html

This name is descriptive and tells us exactly what the file contains. It uses hyphens to separate words instead of spaces, which is a web best practice.

The Special index.html File

Sometimes you might see a file name called index.html. This is a special file name that is used to represent the main page of a website.

When you visit a website, the index.html file is the first file that is loaded.

This is a universally recognized convention for the home page of a website.

index.html

index.html

Special file name for the main page of a website. Web servers automatically look for and load this file when someone visits your site.

Avoiding Special Characters

Another important consideration when naming files is the use of special characters.

Using a mixture of special characters, numbers, and letters can make it difficult to understand what the file is about.

This name looks unnecessarily complicated and does not give us any information about what the file is about.

❌ Bad Example

file-1!@.html

Special characters like ! and @ make the filename confusing and provide no useful information. Stick to letters and dashes only.

Best Practices for File Naming

It is best to stick with only using letters and dashes in your file names.

Keep names simple, descriptive, and consistent across your project.

Recommended Guidelines

  • Use descriptive names
  • Use hyphens (-) to separate words
  • Avoid spaces in file names
  • Avoid special characters (!@#$%)
  • Use only letters and dashes
  • Keep names concise but meaningful

Language and Team Conventions

As you start to learn more languages, you will see that there are common conventions for naming files in those languages as well.

In these situations, it is always best to consult with official documentation or other resources to understand the best practices for naming files in that language.

Following Conventions

  • Consult official documentation for language-specific conventions
  • Research best practices for the technologies you're using
  • Be consistent with your naming patterns

Working on a Team

You will be in situations where you are working on a team and you will need to follow the conventions that the team has established for naming files.

Always consult with the team's style guide to make sure your file names are consistent with the rest of the team.

Consistency across a codebase is essential for collaboration and maintainability.

Team Collaboration

  • Follow the team's established conventions
  • Consult the team's style guide
  • Maintain consistency with existing files
  • Ask questions if conventions are unclear
  • Document naming patterns for new team members
