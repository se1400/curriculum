What Are Booleans?

Definition

Booleans are a data type with only two possible values:

true false

‚úì Why Are They Useful?

Booleans allow you to do something based on conditions.

They're essential when you want to evaluate whether something should happen or not.

Real-World Example

Deciding if someone can access a certain feature in your app:

  • Is the user logged in? ‚Üí true or false
  • Does the user have permission? ‚Üí true or false
  • Is the form valid? ‚Üí true or false

Creating Boolean Variables

Here is an example of setting the value true to a variable called isOldEnoughToDrive:

let isOldEnoughToDrive = true;
console.log(isOldEnoughToDrive); // true
true

Boolean true Example

let isLoggedIn = true;
let hasPermission = true;
let isValid = true;

Boolean false Example

let isLoggedIn = false;
let hasPermission = false;
let isValid = false;

üí° Naming Convention

Boolean variables often start with is, has, or can to make their purpose clear:

  • isActive
  • hasAccess
  • canEdit

Using Booleans in Conditionals

You can use boolean variables inside conditionals to make decisions in your code.

let isOldEnoughToDrive = true;

if (isOldEnoughToDrive) {
  console.log("You're old enough to drive");
} else {
  console.log("Sorry, you are not old enough to drive");
}
You're old enough to drive

A conditional helps you make decisions in your code based on a condition.

This example uses what is called an if/else statement.

Comparison Operators

To compare two values, you can use equality and inequality operators.

Result of Comparisons

The result of any comparison will be a boolean - either true or false.

Four Main Comparison Operators

== === != !==
  • == - Equality (with type coercion)
  • === - Strict Equality (no type coercion)
  • != - Inequality (with type coercion)
  • !== - Strict Inequality (no type coercion)

Equality Operator (==)

==

The equality operator is represented by a double equals sign (==).

Example:

console.log(5 == "5"); // true
true

‚ö†Ô∏è Type Coercion

In this example, JavaScript converts the string "5" into the number 5 and then checks if they are equal.

Since both values are now the same, the result is true.

The equality operator uses type coercion before checking if each value is equal.

More Examples

console.log(5 == 5);      // true (both numbers)
console.log("5" == "5");  // true (both strings)
console.log(0 == false);  // true (false converts to 0)
console.log(1 == true);   // true (true converts to 1)

Strict Equality Operator (===)

===

The strict equality operator is represented by a triple equals sign (===).

How It's Different

The strict equality operator does NOT perform type coercion.

It checks if both the type AND the value are the same.

Example:

console.log(5 === '5'); // false
false

‚úì Why false?

The comparison is false because a string data type is not the same as a number data type.

  • 5 is a number
  • '5' is a string
  • Different types = false

Comparing == vs ===

== (Equality)

With Type Coercion

Converts types before comparing

console.log(5 == "5");
// true

console.log(0 == false);
// true

console.log("" == false);
// true

Values are converted to same type first

=== (Strict Equality)

No Type Coercion

Checks type AND value

console.log(5 === "5");
// false

console.log(0 === false);
// false

console.log("" === false);
// false

‚úì More predictable!

üí° Remember

== compares values (with type conversion)

=== compares values AND types (no conversion)

Inequality Operator (!=)

!=

If you need to check if something is not equal to another value, you can use the inequality operator.

Example:

console.log(5 != "5"); // false
false

Why false?

The result is false because:

  1. The inequality operator converts the string to a number (type coercion)
  2. "5" becomes 5
  3. 5 is equal to 5
  4. Since values are the same, "not equal" is false

‚ö†Ô∏è Type Coercion Again

Just like ==, the != operator performs type coercion before comparing.

Strict Inequality Operator (!==)

!==

The strict inequality operator is represented by an exclamation mark followed by two equal signs (!==).

Example:

console.log(5 !== "5"); // true
true

‚úì Why true?

The result is true because:

  1. The strict inequality operator does NOT perform type coercion
  2. 5 (number) is NOT equal to "5" (string)
  3. Different types means they're not equal
  4. Therefore, "not equal" is true

Comparing != vs !==

!= (Inequality)

With Type Coercion

Converts types before comparing

console.log(5 != "5");
// false (converted, then equal)

console.log(0 != false);
// false (both become 0)

console.log("" != 0);
// false (empty string is 0)

Can be unpredictable

!== (Strict Inequality)

No Type Coercion

Checks type AND value

console.log(5 !== "5");
// true (different types)

console.log(0 !== false);
// true (different types)

console.log("" !== 0);
// true (different types)

‚úì More predictable!

Best Practices

‚úì Use Strict Operators

It is considered best practice to use === and !== whenever possible.

Why Use Strict Operators?

  • They do not perform type coercion
  • More predictable results
  • Fewer unexpected bugs
  • Clearer intent in code

üíº In Professional Projects

Most of the time in professional projects, you will see codebases that usually prefer strict operators (=== and !==) over the regular equality and inequality operators (== and !=).

Recommended Pattern

// ‚úì GOOD - Use strict operators
if (age === 18) {
  console.log("Exactly 18!");
}

if (name !== "") {
  console.log("Name is not empty");
}

// ‚úó AVOID - Regular operators
if (age == "18") {  // Could cause issues
  console.log("Might be unexpected");
}

Booleans and Equality Summary

Key Takeaways

What Are Booleans?

A data type with only two possible values: true or false

Used to make decisions in code based on conditions.

Four Comparison Operators

Operator Name Type Coercion? Example
== Equality ‚úì Yes 5 == "5" ‚Üí true
=== Strict Equality ‚úó No 5 === "5" ‚Üí false
!= Inequality ‚úì Yes 5 != "5" ‚Üí false
!== Strict Inequality ‚úó No 5 !== "5" ‚Üí true

Type Coercion

  • == and != perform type coercion
  • Converts values to same type before comparing
  • Can lead to unexpected results

Strict Comparison (No Coercion)

  • === and !== do NOT perform type coercion
  • Check both value AND type
  • More predictable and safer

Quick Examples

// Booleans
let isActive = true;
let isDisabled = false;

// Equality (with coercion)
console.log(5 == "5");    // true
console.log(0 == false);  // true

// Strict Equality (no coercion)
console.log(5 === "5");   // false
console.log(0 === false); // false

// Inequality (with coercion)
console.log(5 != "5");    // false

// Strict Inequality (no coercion)
console.log(5 !== "5");   // true

Best Practice

Always use strict operators (=== and !==) unless you have a specific reason to use type coercion. This leads to more predictable and maintainable code!
