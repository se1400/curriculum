Three Display Behaviors

When working with CSS, you encounter three different types of display behaviors: inline, block, and inline-block.

Focus of This Lesson

We will focus on how the inline-block property works and how it differs from both inline and block-level elements.

Why This Matters

Each of these properties affects how elements are positioned and how they interact with other elements on the page.

Quick Review: Block Elements

Block-level elements behave like large containers or "blocks" that take up the full width of their parent container.

  • Always start on a new line
  • Height and width can be adjusted

Quick Review: Inline Elements

Inline elements only take up as much space as they need.

  • Flow within surrounding content
  • Do not break onto a new line

Inline-Block: A Hybrid

The inline-block property is a hybrid of inline and block behaviors.

Like Inline Elements

inline-block elements remain in the text flow without starting on a new line.

Like Block Elements

You can adjust the width and height of an inline-block element, just as you would with block-level elements.

Key Difference

The key difference between inline and inline-block:

inline elements cannot have their size controlled
inline-block elements allow for full control over dimensions while still staying inline with other content

In Short

  • inline - No size control, flows with text
  • inline-block - Size control + flows with text
  • block - Size control, new line

Inline-Block Example

Elements with display: inline-block appear side by side (inline behavior) but have specified width and height (block-like properties).

HTML
<div class="container">
  <span class="inline-block-element element1">
    Inline-Block Element 1
  </span>
  <span class="inline-block-element element2">
    Inline-Block Element 2
  </span>
</div>
CSS
.inline-block-element {
  display: inline-block;
  width: 150px;
  height: 80px;
}

.element1 { background: #9C27B0; }
.element2 { background: #673AB7; }

Without Inline-Block

If you remove display: inline-block, the height and width will not be applied even though they're defined.

What Happens

The span elements revert to their default inline behavior. The specified width and height are ignored, and elements only take up space needed for their content.

CSS (without inline-block)
.inline-block-element {
  width: 150px;
  height: 80px;
}

.element1 { background: #9C27B0; }
.element2 { background: #673AB7; }

Complete Comparison

Block

  • display: block;
  • Full width
  • New line
  • Width/height control
  • Examples: div, p

Inline-Block

  • display: inline-block;
  • Needed width
  • Same line
  • Width/height control
  • Hybrid behavior

Inline

  • display: inline;
  • Needed width
  • Same line
  • No width/height control
  • Examples: span, a

When to Use Inline-Block

Understanding how inline-block works is useful for creating layouts that require both alignment and dimension control within a continuous text flow.

Best Use Cases

  • Creating navigation menus with specific button sizes
  • Building grid-like layouts that flow horizontally
  • Styling elements that need to sit side-by-side with controlled dimensions
  • Creating responsive layouts with elements that wrap naturally
  • Any scenario where you need inline flow WITH size control

Key Takeaways

  • inline-block combines the best of both inline and block behaviors
  • Elements stay in text flow (like inline) but accept width/height (like block)
  • Without display: inline-block, inline elements ignore width and height
  • Use inline-block when you need side-by-side elements with specific dimensions
  • Perfect for layouts requiring alignment and dimension control in continuous flow

Remember

Inline: No size control, flows with text
Inline-Block: Size control + flows with text
Block: Size control, starts on new line
