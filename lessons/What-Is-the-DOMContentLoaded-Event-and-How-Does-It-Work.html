The DOMContentLoaded Event

What is DOMContentLoaded?

The DOMContentLoaded event is fired when everything in the HTML document has been loaded and parsed.

‚úì What It Waits For

‚Ä¢ HTML document to be loaded

‚Ä¢ HTML to be fully parsed

‚Ä¢ DOM tree to be constructed

‚úó What It Doesn't Wait For

‚Ä¢ External stylesheets

‚Ä¢ Images

‚Ä¢ Iframes

‚Ä¢ Other external resources

Key Point

If you have external stylesheets, or images, the DOMContentLoaded event will not wait for those to be loaded.

It will only wait for the HTML to be loaded.

DOMContentLoaded vs load Event

Key Difference

This differs from the load event, which waits for everything to be loaded, including external stylesheets, images, and so on.

Basic Syntax

How to Use DOMContentLoaded

Here is the example syntax for using the DOMContentLoaded event:

document.addEventListener("DOMContentLoaded", function () { console.log("DOM is loaded."); });

What Happens

Once the DOM is loaded, the function will be executed and the message "DOM is loaded." will be logged to the console.

Step-by-Step

1. Browser starts loading HTML

2. Browser parses HTML and builds DOM tree

3. DOMContentLoaded event fires

4. Event listener function executes

5. Console logs "DOM is loaded."

Example: Changing an Image

Scenario

Let's take a look at another example using the DOMContentLoaded event.

In this example, we have an image inside the HTML. To update the image, we can create a function that changes the src and alt attributes.

HTML

<h1>Image Change on DOM Loaded</h1> <img id="example-img" src="relaxing-cat.jpg" alt="Cat relaxing" /> <script src="index.js"></script>

JavaScript Function

function changeImg() { const img = document.getElementById("example-img"); img.src = "different-cat.jpg"; img.alt = "Different cat"; console.log("image was just changed"); }

Checking document.readyState

The Smart Way

We can check if the DOM is still loading and add an event listener for the DOMContentLoaded event.

If the DOMContentLoaded event has already fired, we can call the function directly.

if (document.readyState === "loading") { // DOM is still loading, wait for DOMContentLoaded document.addEventListener("DOMContentLoaded", changeImg); } else { // DOM already loaded, run function immediately console.log("DOMContentLoaded has already fired"); changeImg(); }

document.readyState Values

"loading"

‚Üí Document is still loading

"interactive"

‚Üí DOM is ready, DOMContentLoaded has fired

"complete"

‚Üí All resources loaded, load event fired

Why Check readyState?

Sometimes your script might load after the DOM is already loaded.

This check ensures your code runs either:

‚Ä¢ When DOMContentLoaded fires (if still loading)

‚Ä¢ Immediately (if already loaded)

Complete Example Flow

Putting It All Together

Scenario 1: Script Runs While Page Loading

1. Script executes

2. Check: document.readyState === "loading" ‚Üí TRUE

3. Add event listener for DOMContentLoaded

4. Wait for DOM to finish loading...

5. DOMContentLoaded fires

6. changeImg() function runs

7. Image changes!

Scenario 2: Script Runs After DOM Already Loaded

1. DOM already finished loading

2. Script executes (late)

3. Check: document.readyState === "loading" ‚Üí FALSE

4. Go to else branch

5. Log "DOMContentLoaded has already fired"

6. Call changeImg() immediately

7. Image changes!

‚úì Result

Either way, the image changes! This pattern ensures your code works regardless of when the script loads.

When to Use DOMContentLoaded

Perfect Use Cases

The DOMContentLoaded event is useful when you want to run some JavaScript code as soon as the DOM is loaded, but before all the external resources like images and stylesheets are loaded.

‚úì Good Use Cases

1. Attaching Event Listeners

Set up click handlers, form listeners, etc.

2. Manipulating the DOM

Add, remove, or modify elements

3. Reading Element Properties

Get dimensions, positions, text content

4. Initializing JavaScript Frameworks

Start React, Vue, or other frameworks

üí° Why Not Just Use load?

DOMContentLoaded is faster!

If you don't need images or stylesheets to be loaded, why wait?

Better user experience:

Your page becomes interactive sooner

Example:

A page with many large images - DOMContentLoaded fires seconds before load!

// Common pattern: Set up page once DOM is ready document.addEventListener("DOMContentLoaded", () => { // All these work because DOM is ready const button = document.querySelector("button"); button.addEventListener("click", handleClick); const form = document.querySelector("form"); form.addEventListener("submit", handleSubmit); initializeApp(); });

Common Patterns

Pattern 1: Simple Listener

Most common pattern - just wait for DOM to be ready

document.addEventListener("DOMContentLoaded", () => { // Your initialization code here initApp(); });

Pattern 2: With readyState Check

Safest pattern - handles both early and late script execution

function init() { // Your initialization code } if (document.readyState === "loading") { document.addEventListener("DOMContentLoaded", init); } else { init(); }

Pattern 3: Arrow Function Inline

Quick setup for short code blocks

document.addEventListener("DOMContentLoaded", () => { document.querySelector("#btn").addEventListener("click", () => { alert("Button clicked!"); }); });
