  • How to layer multiple animations on a single element
  • Why transform and opacity are the right properties to animate
  • How to respect user motion preferences with prefers-reduced-motion

  • How to style placeholder text, selected text, list markers, and drop caps

CSS transitions animate between two states — from the default style to a hover style, for example. Transitions work beautifully for simple A-to-B changes, but they cannot loop, run automatically on page load, or pass through multiple intermediate states. That is where CSS animations come in.

CSS animations, introduced in the CSS Animations Level 1 specification and now universally supported, let you define a named sequence of style snapshots called keyframes. The browser smoothly interpolates between those snapshots, running the sequence automatically, as many times as you choose, in any direction. Combined with CSS pseudo-elements — special selectors that generate decorative content without touching your HTML — animations open up a rich world of expressive, performant motion directly in your stylesheet.

From Transitions to Animations

Transitions are reactive: they respond to state changes. Animations are proactive: they run on their own schedule. This distinction makes animations the right tool whenever you need motion that:

  • Starts automatically when the page loads or an element appears
  • Loops indefinitely (a spinner, a pulsing badge, a marquee)
  • Passes through three or more distinct visual states
  • Needs to reverse direction, hold its end state, or pause and resume

Before CSS animations existed, developers relied on JavaScript setInterval loops or Flash to produce this kind of motion. CSS animations move that logic into the stylesheet, where it belongs, leaving JavaScript free to focus on logic rather than rendering.

Defining Animations with @keyframes

Every CSS animation begins with a @keyframes rule. This rule gives your animation a name and lists the style values the element should have at specific points in the animation cycle.

The from / to Shorthand

The simplest form uses from (the start) and to (the end), which are aliases for 0% and 100%:

LISTING 15.1 — A Simple Fade-In with from / to

@keyframes fadeIn {
  from {
    opacity: 0;
  }
  to {
    opacity: 1;
  }
}

.hero {
  animation: fadeIn 0.6s ease-out;
}

When the browser encounters .hero, it immediately begins interpolating opacity from 0 to 1 over 0.6 seconds.

Percentage-Based Keyframes

For animations with more than two states, use percentage values. You can include as many stops as you need:

LISTING 15.2 — A Bounce Effect Using Percentage Keyframes

@keyframes bounce {
  0%   { transform: translateY(0); }
  30%  { transform: translateY(-40px); }
  60%  { transform: translateY(-20px); }
  80%  { transform: translateY(-10px); }
  100% { transform: translateY(0); }
}

The percentages represent progress through the animation cycle, not clock time. If your animation runs for 2 seconds, the 50% keyframe occurs at exactly 1 second.

NOTE: You can mix from/to with percentage keyframes in the same rule, and you can share properties across multiple keyframe stops by comma-separating them: 0%, 100% { opacity: 1; }

Keyframe Names

Choose descriptive, intention-revealing names for your animations. Prefer names like slideInLeft, pulseRed, or spin over generic names like anim1. Names can use camelCase or kebab-case — both are valid and widely used.

The Eight Animation Properties

Once you have a @keyframes rule, you attach it to an element using animation properties. There are eight individual properties; together they give you precise control over every aspect of playback.

animation-name

References the name of the @keyframes rule you want to play. If no keyframes rule with that name exists, the animation silently does nothing.

animation-name: bounce;
animation-name: fadeIn, pulse;  /* multiple animations */

animation-duration

Sets how long one full cycle of the animation takes. Accepts seconds (s) or milliseconds (ms). There is no default — if you omit this property, the animation plays with a duration of 0s and appears invisible.

animation-duration: 1s;
animation-duration: 250ms;

animation-timing-function

Controls the speed curve within each keyframe interval — the same values as transition-timing-function: ease, linear, ease-in, ease-out, ease-in-out, cubic-bezier(), and steps(). You can also apply different timing functions at individual keyframe stops.

@keyframes expand {
  0%   { transform: scaleX(0); animation-timing-function: ease-out; }
  80%  { transform: scaleX(1.05); animation-timing-function: ease-in; }
  100% { transform: scaleX(1); }
}

animation-delay

Waits before starting the animation. Positive values delay the start; negative values are particularly useful — a delay of -0.5s on a 2-second animation begins playback as if it already started half a second ago, showing the animation mid-cycle on page load. This technique staggers items in a list so they each appear to start slightly later than the previous one.

.card:nth-child(1) { animation-delay: 0s; }
.card:nth-child(2) { animation-delay: 0.1s; }
.card:nth-child(3) { animation-delay: 0.2s; }

animation-iteration-count

Controls how many times the animation runs. Accepts any positive number, including fractions, or the keyword infinite for a looping animation.

animation-iteration-count: 1;        /* default */
animation-iteration-count: 3;        /* runs three times */
animation-iteration-count: infinite; /* loops forever */
animation-iteration-count: 2.5;      /* two and a half cycles */

animation-direction

Determines whether each cycle plays forward, backward, or alternates:

  • normal — plays from 0% to 100% every cycle (default)
  • reverse — plays from 100% to 0% every cycle
  • alternate — first cycle forward, second cycle backward, alternating (great for breathing effects)
  • alternate-reverse — first cycle backward, then forward, alternating
/* A pulsing glow that breathes in and out */
@keyframes glow {
  from { box-shadow: 0 0 4px rgba(79, 109, 245, 0.3); }
  to   { box-shadow: 0 0 20px rgba(79, 109, 245, 0.9); }
}

.badge {
  animation: glow 1.2s ease-in-out infinite alternate;
}

animation-fill-mode

This property answers the question: what styles apply before the animation starts and after it ends? By default, none — the element snaps back to its original styles as soon as the animation finishes. Often this causes an unwanted visual "snap."

  • none — the element uses its normal styles before and after (default)
  • forwards — after the animation ends, the element holds the styles from the 100% keyframe
  • backwards — during the delay period, the element immediately applies the styles from the 0% keyframe
  • both — combines forwards and backwards; the most commonly needed value

[IMG - Animation fill-mode forwards versus no fill-mode visual comparison]

LISTING 15.3 — Slide-In with fill-mode both

@keyframes slideInUp {
  from {
    opacity: 0;
    transform: translateY(30px);
  }
  to {
    opacity: 1;
    transform: translateY(0);
  }
}

.notification {
  animation: slideInUp 0.4s ease-out both;
}
/*
  "both" means:
  - During the 0s delay (if any): opacity 0, translateY(30px)
  - After the animation ends: holds opacity 1, translateY(0)
*/

animation-play-state

Switches a running animation between running (default) and paused. You can toggle this property with JavaScript to let users pause and resume animations — useful for accessibility and interactive media.

/* CSS: pause on hover */
.spinner:hover {
  animation-play-state: paused;
}

/* JavaScript: toggle pause */
const el = document.querySelector('.spinner');
el.style.animationPlayState = 'paused';  // pause
el.style.animationPlayState = 'running'; // resume

NOTE: When you change animation-play-state from paused back to running, the animation resumes from exactly where it stopped — it does not restart from the beginning.

The animation Shorthand

Like the transition shorthand, animation combines all eight sub-properties into a single declaration. The order of values mirrors the sub-property order, though only animation-name and animation-duration are required. The browser distinguishes between the two time values by position: the first time value is always duration, and the second is always delay.

LISTING 15.4 — The animation Shorthand

/* Full form for clarity */
animation-name: slideInUp;
animation-duration: 0.5s;
animation-timing-function: ease-out;
animation-delay: 0.2s;
animation-iteration-count: 1;
animation-direction: normal;
animation-fill-mode: both;
animation-play-state: running;

/* Shorthand equivalent */
animation: slideInUp 0.5s ease-out 0.2s 1 normal both running;

/* Simplified (most common form) */
animation: slideInUp 0.5s ease-out 0.2s both;

CAUTION: If you include two time values in the shorthand, the browser always interprets the first as duration and the second as delay. Writing animation: spin 1s 0.5s means "duration 1s, delay 0.5s" — not the other way around.

Combining Animations with Transforms

Animations become most powerful when they manipulate transform and opacity. Since transforms do not trigger layout reflow, these properties animate smoothly even on lower-powered devices. You can build complex motion — movement, rotation, scaling — all in a single @keyframes rule.

LISTING 15.5 — A Rocket Launch Effect

@keyframes launch {
  0% {
    transform: translateY(0) scale(1) rotate(0deg);
    opacity: 1;
  }
  50% {
    transform: translateY(-60px) scale(1.1) rotate(-5deg);
    opacity: 1;
  }
  100% {
    transform: translateY(-200px) scale(0.8) rotate(10deg);
    opacity: 0;
  }
}

.rocket-icon {
  animation: launch 1.2s ease-in forwards;
}

Notice that all three transform functions are combined inside a single transform property at each keyframe stop. Because transforms are applied in order, the sequence translateY(-60px) scale(1.1) rotate(-5deg) first moves, then scales, then rotates — the order matters and changes the visual result.

Layering Multiple Animations

You can apply more than one animation to the same element by separating @keyframes names and their corresponding values with commas. Each animation runs independently on its own timeline.

LISTING 15.6 — Applying Two Animations Simultaneously

@keyframes floatUp {
  0%, 100% { transform: translateY(0); }
  50%       { transform: translateY(-12px); }
}

@keyframes colorShift {
  0%   { color: #4f6df5; }
  50%  { color: #ec4899; }
  100% { color: #4f6df5; }
}

.floating-title {
  /* duration, timing, and count must be listed per animation */
  animation:
    floatUp   3s ease-in-out infinite,
    colorShift 6s linear     infinite;
}

The floatUp animation completes two full cycles in the same time colorShift completes one, because their durations differ. Each property list maps position-by-position: the first duration corresponds to the first animation name, the second duration to the second name, and so on.

Animation Performance

Not all CSS properties are equally efficient to animate. When a browser renders a page, it works through a pipeline: Style → Layout → Paint → Composite. Animating properties that trigger layout (like width, height, margin, or top/left) forces the browser to recalculate the position of other elements on every frame — an expensive operation that can cause dropped frames and janky motion.

Animating transform and opacity bypasses both layout and paint entirely, handing the work off to the GPU compositor. This is why they are the gold standard for CSS animation.

[IMG - Layout-triggering versus GPU-composited animation pipeline comparison]

LISTING 15.7 — Moving Elements: The Wrong Way and the Right Way

/* AVOID — triggers layout recalculation on every frame */
@keyframes moveLeft-bad {
  from { left: 0; }
  to   { left: 300px; }
}

/* PREFER — handled entirely by the GPU compositor */
@keyframes moveLeft-good {
  from { transform: translateX(0); }
  to   { transform: translateX(300px); }
}

The will-change Property

The will-change property is a hint to the browser that a particular element is about to be animated, giving it permission to promote the element to its own GPU layer in advance. This can eliminate the small startup cost of the first animation frame.

LISTING 15.8 — Using will-change

.animated-card {
  will-change: transform, opacity;
}

/* Remove the hint once the animation is over (JavaScript) */
card.addEventListener('animationend', () => {
  card.style.willChange = 'auto';
});

CAUTION: will-change is frequently misused. Do not apply it to every element as a blanket performance optimization — each promoted layer consumes GPU memory. Only use it on elements where you have measured a real performance problem, and remove it after the animation completes.

Respecting User Preferences with prefers-reduced-motion

Many people experience dizziness, nausea, or epileptic episodes triggered by motion on screen. Operating systems provide a setting — "Reduce Motion" on macOS/iOS, "Remove Animations" on Android, "Show animations in Windows" on Windows — that users can enable to request less motion. CSS exposes this preference through the prefers-reduced-motion media query.

[IMG - macOS Reduce Motion accessibility setting in System Preferences]

The recommended pattern is to define your full animation for those who want it, then override it inside the media query for users who don't:

LISTING 15.9 — The prefers-reduced-motion Pattern

/* Default: full animation */
.hero-text {
  animation: slideInUp 0.6s ease-out both;
}

/* Reduced motion: instant appearance, no sliding */
@media (prefers-reduced-motion: reduce) {
  .hero-text {
    animation: fadeIn 0.2s ease both;
  }
}

/* Or, disable animation entirely */
@media (prefers-reduced-motion: reduce) {
  * {
    animation-duration: 0.01ms !important;
    animation-iteration-count: 1 !important;
  }
}

NOTE: Never skip the prefers-reduced-motion check entirely when building looping animations. A continuously spinning element is one of the most commonly reported accessibility barriers on the web.

Modern: Scroll-Driven Animations

Traditionally, animating elements as the user scrolls required JavaScript — attaching a scroll event listener, computing scroll position, and updating styles on every frame. CSS scroll-driven animations, introduced in Chrome 115 (2023) and continuing to gain browser support, bring this capability natively to CSS using the animation-timeline property.

NOTE: As of early 2025, scroll-driven animations are supported in Chrome and Edge but not yet in Firefox or Safari stable. Always check caniuse.com and provide a static fallback for unsupported browsers. Wrap scroll-driven animation styles in @supports (animation-timeline: scroll()).

The scroll() Timeline

The scroll() function ties an animation to the scroll progress of a scrollable container. The animation plays from its 0% keyframe at the start of the scroll range and reaches 100% at the end.

LISTING 15.10 — A Reading Progress Bar with scroll()

@keyframes growWidth {
  from { transform: scaleX(0); }
  to   { transform: scaleX(1); }
}

.progress-bar {
  position: fixed;
  top: 0;
  left: 0;
  width: 100%;
  height: 4px;
  background: #4f6df5;
  transform-origin: left;

  animation: growWidth linear;
  animation-timeline: scroll(root block);
  /* root = the page scroll container; block = the block (vertical) axis */
}

Notice that animation-duration is omitted — with a scroll timeline, duration is irrelevant because position drives progress, not time.

The view() Timeline

The view() function triggers an animation based on how much of the element itself is visible in the viewport — perfect for entrance animations when elements scroll into view.

LISTING 15.11 — Fade-In-On-Scroll with view()

@keyframes revealCard {
  from {
    opacity: 0;
    transform: translateY(40px);
  }
  to {
    opacity: 1;
    transform: translateY(0);
  }
}

.card {
  animation: revealCard linear both;
  animation-timeline: view();
  animation-range: entry 0% entry 40%;
  /* Plays as the card enters the viewport from 0% to 40% of its entry range */
}

@supports not (animation-timeline: view()) {
  /* Fallback: just show the card normally */
  .card {
    opacity: 1;
    transform: none;
  }
}

Pseudo-elements

Pseudo-elements are special CSS selectors that let you style a part of an element — or even generate entirely new content — without modifying your HTML. They use a double-colon syntax (::before, ::after) to distinguish them from pseudo-classes (:hover, :focus). Older browsers supported a single-colon syntax (:before, :after), and most browsers still accept it, but the double-colon form is correct in modern CSS.

::before and ::after

These two pseudo-elements insert a generated child at the very beginning or very end of an element's content. They are rendered by the browser but are not part of the DOM, which means they are invisible to screen readers by default — ideal for purely decorative additions.

The key rule: a pseudo-element must have a content property to appear at all. Without content, the browser does not render it, even if other properties are set. Use an empty string (content: "") when you want a visual shape with no text.

LISTING 15.12 — Adding Decorative Quotation Marks

<blockquote class="styled-quote">
  Design is not just what it looks like and feels like.
  Design is how it works.
</blockquote>

.styled-quote {
  position: relative;
  padding: 1.5rem 1.5rem 1.5rem 3rem;
  font-style: italic;
}

.styled-quote::before {
  content: "\201C"; /* Unicode opening double quotation mark */
  position: absolute;
  left: 0.25rem;
  top: -0.25rem;
  font-size: 4rem;
  color: #4f6df5;
  line-height: 1;
}

Positioning pseudo-elements requires position: relative on the parent element, so that position: absolute on the pseudo-element is positioned relative to the parent rather than a distant ancestor.

LISTING 15.13 — An Animated Underline with ::after

.nav-link {
  position: relative;
  text-decoration: none;
}

.nav-link::after {
  content: "";
  position: absolute;
  bottom: -2px;
  left: 0;
  width: 100%;
  height: 2px;
  background: #4f6df5;
  transform: scaleX(0);
  transform-origin: right;
  transition: transform 0.3s ease;
}

.nav-link:hover::after,
.nav-link:focus::after {
  transform: scaleX(1);
  transform-origin: left;
}

This pattern creates an underline that grows from right to left on hover and shrinks back from left to right when you move away — without any extra HTML elements. The underline is produced entirely by CSS.

The content Property

The content property accepts several types of values:

  • content: "text" — a string literal, including Unicode escapes like "\2713" (✓)
  • content: attr(data-label) — pulls the value from an HTML attribute dynamically
  • content: counter(section) — displays an incrementing CSS counter value
  • content: "" — an empty string (required when you want a visible shape with no text)
  • content: none — suppresses a pseudo-element inherited from another rule

LISTING 15.14 — Using attr() to Build Tooltips

<button data-tooltip="Opens your saved items">Cart</button>

[data-tooltip] {
  position: relative;
}

[data-tooltip]::before {
  content: attr(data-tooltip);
  position: absolute;
  bottom: calc(100% + 6px);
  left: 50%;
  transform: translateX(-50%);
  background: #1e293b;
  color: white;
  font-size: 0.8rem;
  padding: 0.35rem 0.6rem;
  border-radius: 4px;
  white-space: nowrap;
  opacity: 0;
  transition: opacity 0.2s ease;
}

[data-tooltip]:hover::before,
[data-tooltip]:focus::before {
  opacity: 1;
}

Because content: attr(data-tooltip) reads the attribute value at render time, you can change the tooltip text by updating the attribute in HTML without touching the CSS.

::placeholder

The ::placeholder pseudo-element styles the placeholder hint text shown inside form inputs before the user types. This is commonly used to improve contrast or match brand colors, since browser defaults are often a grey that fails WCAG contrast requirements.

LISTING 15.15 — Styling Placeholder Text

input::placeholder,
textarea::placeholder {
  color: #94a3b8;
  font-style: italic;
  font-size: 0.9em;
}

CAUTION: Placeholder text is not a substitute for a <label> element. Once the user starts typing, the placeholder disappears and the field becomes unlabeled. Always pair every input with a visible <label>.

::selection

Styles the background and text color of content the user has highlighted by clicking and dragging. You can use this to reinforce brand colors in a subtle, delightful way.

LISTING 15.16 — Custom Selection Colors

::selection {
  background-color: #4f6df5;
  color: white;
}

/* You can also scope it to specific elements */
.article-body::selection {
  background-color: #fde68a;
  color: #1e293b;
}

Only color, background-color, cursor, caret-color, outline, text-decoration, text-emphasis-color, and text-shadow are allowed on ::selection.

::marker

Targets the list marker — the bullet point or number — of <li> elements. Before ::marker, customizing bullets required either removing the default marker entirely and using ::before as a replacement, or using list-style-image. ::marker makes this far simpler.

LISTING 15.17 — Styling List Markers

ul.styled-list li::marker {
  color: #4f6df5;
  font-size: 1.2em;
}

ol.styled-numbers li::marker {
  color: #ec4899;
  font-weight: 700;
  content: counter(list-item) ". ";
}

::first-line and ::first-letter

::first-line styles only the first rendered line of a block of text — regardless of how wide the container is. As the user resizes the browser, the browser automatically adjusts which words constitute the first line.

::first-letter is the foundation of the classic typographic drop cap — a large, decorated initial letter that draws the eye to the start of an article or section.

LISTING 15.18 — Drop Cap with ::first-letter

.article p:first-of-type::first-letter {
  float: left;
  font-size: 3.5rem;
  font-weight: 700;
  line-height: 0.85;
  margin-right: 0.1em;
  color: #4f6df5;
}

Practical Animation Patterns

Now that you understand the building blocks, here are patterns you will reach for repeatedly in real projects.

CSS Spinner / Loading Indicator

A pure-CSS spinner uses a border and rotate animation — no images, no JavaScript, just CSS.

LISTING 15.19 — A CSS Spinner

<div class="spinner" aria-label="Loading" role="status"></div>

.spinner {
  width: 40px;
  height: 40px;
  border: 4px solid #e2e8f0;        /* light track */
  border-top-color: #4f6df5;        /* colored segment */
  border-radius: 50%;
  animation: spin 0.8s linear infinite;
}

@keyframes spin {
  to { transform: rotate(360deg); }
}

@media (prefers-reduced-motion: reduce) {
  .spinner {
    animation: none;
    border-top-color: #4f6df5;
    opacity: 0.7;
  }
}

Note the role="status" and aria-label attributes on the spinner element — screen readers need these to announce that loading is in progress.

Staggered List Entrance

A staggered entrance gives each list item a slightly later animation-delay, creating a cascading reveal effect.

LISTING 15.20 — Staggered List Entrance

@keyframes itemIn {
  from {
    opacity: 0;
    transform: translateX(-20px);
  }
  to {
    opacity: 1;
    transform: translateX(0);
  }
}

.menu-item {
  animation: itemIn 0.4s ease-out both;
}

.menu-item:nth-child(1) { animation-delay: 0s; }
.menu-item:nth-child(2) { animation-delay: 0.05s; }
.menu-item:nth-child(3) { animation-delay: 0.1s; }
.menu-item:nth-child(4) { animation-delay: 0.15s; }
.menu-item:nth-child(5) { animation-delay: 0.2s; }

Tooltip Arrow with ::before

A tooltip bubble often has a small triangular arrow pointing toward the element it describes. This arrow is drawn entirely with CSS using the border trick: setting only one border to a color and the adjacent borders to transparent creates a triangle shape.

LISTING 15.21 — Tooltip Arrow Using Border Trick

.tooltip::after {
  content: "";
  position: absolute;
  top: 100%;          /* position below the tooltip */
  left: 50%;
  transform: translateX(-50%);
  border-width: 6px;
  border-style: solid;
  border-color: #1e293b transparent transparent transparent;
  /* Top border is visible (dark), other three are transparent = triangle */
}

Summary

In this lesson you learned how CSS @keyframes rules define named animation sequences that the browser plays automatically, independent of user interaction. You explored each of the eight animation sub-properties — animation-name, animation-duration, animation-timing-function, animation-delay, animation-iteration-count, animation-direction, animation-fill-mode, and animation-play-state — and how to combine them into the animation shorthand.

You learned why animating transform and opacity is the high-performance choice, how to use will-change judiciously, and how the prefers-reduced-motion media query lets you respect users who are sensitive to motion. You also got an introduction to scroll-driven animations, one of the most exciting recent additions to CSS.

In the second half of the lesson, you explored CSS pseudo-elements: ::before and ::after for inserting generated content, ::placeholder for styling form hints, ::selection for highlighted text, ::marker for list bullets, and ::first-letter for typographic drop caps. Pseudo-elements keep your HTML clean while giving CSS full creative control over visual decoration.

Q&A

Q. What is the difference between a CSS transition and a CSS animation?

A. A transition reacts to a state change — it only plays when a property's value changes (for example, on hover). An animation is proactive: it starts automatically, runs for as many iterations as you specify, can loop, reverse, and pass through any number of keyframe stops. Transitions are best for simple interactive feedback; animations are best for motion that needs to play on its own.

Q. My animation runs once and the element snaps back to its original style. How do I make it hold the final state?

A. Add animation-fill-mode: forwards (or both if you also have a delay and want the starting styles applied immediately). forwards tells the browser to hold the styles from the last keyframe after the animation completes.

Q. Can I animate a pseudo-element independently from the element itself?

A. Yes. You can attach an animation directly to ::before or ::after. Just write the animation property on the pseudo-element selector itself: .card::before { animation: spin 1s linear infinite; }. The pseudo-element's animation runs entirely separately from any animation on .card.

Q. Do screen readers read content generated by ::before and ::after?

A. Browser behavior is inconsistent, but most modern screen readers do read generated content from content. However, generated content should only be used for purely decorative purposes. Any meaningful text should be in your HTML. If a pseudo-element's content would cause confusion for screen reader users, hide it with content: "" / "" — the second empty string is an accessibility alternative text value supported in modern browsers.

Q. How does a negative animation-delay work?

A. A negative delay starts the animation as if it already started that many seconds in the past. For example, animation-delay: -1s on a 3-second animation means the browser begins playback at the 1-second mark. This technique is used to stagger looping animations so they do not all start at 0% simultaneously — useful for a wave of dots in a loading indicator.

Q. What does animation-timeline: view() actually measure?

A. It measures the element's progress through the viewport — specifically, how much of the element has entered the scrollport (the visible area). The animation's 0% keyframe corresponds to the moment the element begins entering the viewport from below, and 100% corresponds to the moment it has fully entered (or fully exited, depending on the animation-range you set). The entry and exit range keywords let you target just the entering or just the exiting phase.
