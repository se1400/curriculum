Ordered Lists and the type Attribute

When working with ordered lists in HTML, the type attribute allows you to specify the style of numbering used.

Numbering Styles Available:

  • Numerical (1, 2, 3, ...)
  • Uppercase alphabetical (A, B, C, ...)
  • Lowercase alphabetical (a, b, c, ...)
  • Uppercase Roman numerals (I, II, III, ...)
  • Lowercase Roman numerals (i, ii, iii, ...)
The type attribute controls the numbering format for ordered lists

The type Attribute Values

The type attribute can be set on an <ol> element to control numbering:

type="1"

Numerical lists

1, 2, 3, ...

type="A"

Uppercase alphabetical

A, B, C, ...

type="a"

Lowercase alphabetical

a, b, c, ...

type="I"

Uppercase Roman numerals

I, II, III, ...

type="i"

Lowercase Roman numerals

i, ii, iii, ...

Default: type="1"

By default, ordered lists use numerical numbering (type="1").

HTML Example:

<ol type="1">
  <li>First item</li>
  <li>Second item</li>
  <li>Third item</li>
</ol>
  1. First item
  2. Second item
  3. Third item

Uppercase Alphabetical: type="A"

Use type="A" for uppercase alphabetical numbering.

HTML Example:

<ol type="A">
  <li>First item</li>
  <li>Second item</li>
  <li>Third item</li>

  1. First item
  2. Second item
  3. Third item

Use Case:

Perfect for outlines, multiple choice questions, or section labeling.

Targeting with Attribute Selectors

Use CSS attribute selectors to style lists based on their type attribute.

Basic Syntax:

ol[type="value"] {
  property: value;
}

Benefits:

  • Style lists based on numbering format
  • Differentiate list types visually
  • Control appearance based on structure
  • No need for additional classes

Example: Targeting type="A"

Style uppercase alphabetical lists specifically:

CSS:

ol[type="A"] {
  color: purple;
  font-weight: bold;
  background-color: #f3e5f5;
}

Result:

  1. Item 1
  2. Item 2
  3. Item 3

Any ordered list with type="A" will have purple, bold text with a light purple background.

Example: Targeting type="i"

Style lowercase Roman numeral lists:

CSS:

ol[type="i"] {
  color: green;
  font-weight: bold;
  background-color: #e8f5e9;
}

Result:

  1. Item 1
  2. Item 2
  3. Item 3

This rule changes the text color to green for any ordered list that uses lowercase Roman numerals.

All Types Styled Differently

You can style each type uniquely to visually distinguish them:

CSS:

ol[type="1"] { background: #e3f2fd; }
ol[type="A"] { background: #f3e5f5; }
ol[type="a"] { background: #fff3e0; }
ol[type="I"] { background: #ffebee; }
ol[type="i"] { background: #e8f5e9; }

Benefits of Styling by type

Greater Control:

  • Customize appearance based on content structure
  • Visually distinguish different list types
  • No need for manual class additions
  • Automatic styling based on semantic markup

Common Use Cases:

  • Style exam answer choices (A, B, C, D)
  • Format formal document sections (I, II, III)
  • Distinguish outline levels
  • Create visually organized hierarchies
