What Are Variables?

In JavaScript, variables act as containers for storing data that you can access and modify throughout your program.

üí° Think of Variables as Boxes

Variables are like boxes that hold values. They let you keep track of things like numbers or text and refer to these values whenever you need them in your program.

What Can Variables Store?

  • Numbers (like scores, ages, prices)
  • Text (like names, messages, labels)
  • True/false values (like login status)
  • And much more!

Declaring Variables with let

One way to declare a variable in JavaScript is to use the let keyword.

Example: Declaring a Variable

let age;

What's Happening Here?

Right now, the age variable does not have a value assigned to it. If you try to use it, it will return undefined, which means it has no value.

Testing the Variable

let age;
console.log(age); // undefined

Note: console.log() is a function that outputs information to the console, which is a part of your web browser used for debugging code.

The // symbols are used to add comments in your code. Comments are notes for yourself or other programmers that are ignored when the code runs.

Assigning Values to Variables

To assign a value to a variable, you need to use the assignment operator:

let age = 25;

Now when you use the age variable, it will return the value of 25:

let age = 25;
console.log(age); // 25

About the Assignment Operator

The assignment operator looks like an equals sign (=), but it doesn't check for equality. Instead, it assigns a value to a variable.

This process of assigning a value to a variable is known as initialization.

Reassigning Values

One advantage of using the let keyword is that you can reassign values to variables.

Reassignment: Giving a new value to a variable that already has one.

Example of Reassignment

let age = 25;
console.log(age); // 25

age = 30;
console.log(age); // 30

Important Points:

  • The age variable now holds the value of 30
  • Notice the let keyword wasn't needed again because the variable was already declared
  • When reassigning, you only need to reference the variable name

Reassignment is useful because it allows you to update and change the value stored in a variable as your program runs. A good example would be updating points in a game.

Naming Variables: Best Practices

Naming variables may seem straightforward, but there are some rules and best practices to ensure your code is readable and functional.

Rule #1: Use Descriptive Names

Your variable names should describe what the data represents.

‚ùå Bad Variable Names

let x = 10;
let y = "John";

These names don't tell you what the data represents.

‚úì Good Variable Names

let age = 10;
let userName = "John";

These names clearly describe the data they hold.

Instead of using a name like x, a more descriptive name such as age or points makes your code easier to understand.

Variable Naming Rules

Rule #2: Starting Characters

Variables in JavaScript must begin with:

  • A letter (a-z or A-Z)
  • An underscore (_)
  • A dollar sign ($)

They cannot start with a number.

‚úì Valid Variable Names

let age;
let _score;
let $total;

‚ùå Invalid Variable Names

let 1stPlace; // starts with a number

Case Sensitivity

Rule #3: Case Matters!

Variable names are case-sensitive, meaning the word age in all lowercase and the word Age with a capital A are considered different variables.

Example

let age = 25;
let Age = 30;

console.log(age); // 25
console.log(Age); // 30

These are two completely separate variables with different values!

This is why it's important to stick with a consistent naming convention like camelCase.

camelCase Naming Convention

What is camelCase?

camelCase is where the first word is all lowercase and each subsequent word starts with an uppercase letter.

Examples of camelCase

let thisIsCamelCase;
let anotherExampleVariable;
let freeCodeCampStudents;

Why Use camelCase?

  • It's the standard convention in JavaScript
  • Makes multi-word variables easy to read
  • Keeps your code consistent with industry practices
  • Helps avoid naming confusion

What to Avoid in Variable Names

Rule #4: Avoid Reserved Keywords

There are certain keywords in JavaScript that you cannot use as variable names, as they are reserved for the language itself.

‚ùå Cannot Use These Keywords

let let;        // Reserved keyword
let const;      // Reserved keyword
let function;   // Reserved keyword
let return;     // Reserved keyword

Rule #5: Avoid Special Characters

You should avoid using special characters like exclamation points (!) or at symbols (@) in your variable names.

Best Practice: Keep variable names readable by using letters, numbers, underscores, or dollar signs only.

Variable Naming Guidelines Summary

Key Takeaways

‚úì DO:

  • Use descriptive names that explain what the data represents
  • Start variables with a letter, underscore, or dollar sign
  • Use camelCase for multi-word variable names
  • Be consistent with your naming conventions
  • Use letters, numbers, underscores, and dollar signs

‚úó DON'T:

  • Use single letters like x or y (unless in math contexts)
  • Start variable names with numbers
  • Use reserved JavaScript keywords
  • Use special characters like ! or @
  • Forget that variable names are case-sensitive

By following these guidelines, your code will be cleaner and more manageable as it grows in complexity.
