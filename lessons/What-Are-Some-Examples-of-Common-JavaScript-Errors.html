SyntaxError

Grammar Mistakes in Code

A SyntaxError happens when you write something incorrectly in your code, like missing a parenthesis, or a bracket.

Think of it like a grammar mistake in a sentence.

Common Example: Missing Comma in Array

Here is a common mistake developers make when creating arrays:

‚ùå Incorrect Code

const arr = ["Beau", "Quincy" "Tom"];

Error Message

SyntaxError: Unexpected string

The Problem

Each array element needs to be separated by a comma otherwise it will result in an error message.

Missing the comma between "Quincy" and "Tom" causes JavaScript to not understand what you mean.

‚úì Correct Code

const arr = ["Beau", "Quincy", "Tom"]; console.log(arr); // ["Beau", "Quincy", "Tom"]

More SyntaxError Examples

‚ùå Missing Closing Brace

if (true) { console.log("Hello"); // Missing closing brace!
SyntaxError: Unexpected end of input

‚ùå Missing Closing Parenthesis

console.log("Hello World"; // Missing closing parenthesis!
SyntaxError: missing ) after argument list

‚ùå Missing Closing Quote

const message = "Hello World; // Missing closing quote!
SyntaxError: Invalid or unexpected token

‚ùå Missing Closing Bracket

const numbers = [1, 2, 3; // Missing closing bracket!
SyntaxError: missing ] after element list

‚Ä¢ Carefully check all opening and closing symbols

‚Ä¢ Use a code editor with syntax highlighting

‚Ä¢ Pay attention to the error line number

‚Ä¢ Count your brackets, braces, parentheses, and quotes

ReferenceError: Not Defined

Variable Doesn't Exist

The second common JavaScript error is a ReferenceError.

There are several types of ReferenceErrors, triggered in different ways.

Type 1: Not Defined Variables

The first type of ReferenceError would be not defined variables.

‚ùå Incorrect Code

console.log(price); // Trying to use 'price' but it doesn't exist!

Error Message

ReferenceError: price is not defined

The Problem

In this example, we are trying to log the price variable to the console but it hasn't been defined.

This will result in a ReferenceError.

‚úì Correct Code

const price = 29.99; console.log(price); // 29.99

ReferenceError: Before Initialization

Type 2: Accessing Before Declaration

Another example of a ReferenceError is trying to access a variable, declared with let or const, before it has been defined.

‚ùå Incorrect Code

console.log(b); const b = 50;

Error Message

ReferenceError: Cannot access 'b' before initialization

The Problem

The example above will result in a "Cannot access 'b' before initialization" error.

Variables declared with let and const are in a "temporal dead zone" until the line where they're declared is executed.

‚úì Correct Code

const b = 50; console.log(b); // 50

‚ö†Ô∏è Important

Always declare and initialize variables before you try to use them!

TypeError

Wrong Type Operations

The third common error would be a TypeError.

These errors occur when you try to perform an operation on the wrong type.

Common Example: Array Method on Object

Here is an example of trying to use the map method on an object:

‚ùå Incorrect Code

const developerObj = { name: "Jessica", country: "USA", isEmployed: true }; developerObj.map(); // Trying to use array method on object!

Error Message

TypeError: developerObj.map is not a function

The Problem

This example will result in a "developerObj.map is not a function" error because the map method is used for arrays and not objects.

‚úì Correct Code

const developerArray = [ { name: "Jessica", country: "USA" }, { name: "Tom", country: "UK" } ]; const names = developerArray.map(dev => dev.name); console.log(names); // ["Jessica", "Tom"]

More TypeError Examples

‚ùå Calling Non-Function

const age = 25; age(); // Number is not a function!
TypeError: age is not a function

‚ùå Property of Undefined

let user; console.log(user.name); // Can't read property of undefined!
TypeError: Cannot read property 'name' of undefined

‚ùå Assignment to Constant

const PI = 3.14; PI = 3.15; // Can't reassign const!
TypeError: Assignment to constant variable

‚ùå Property of Null

const data = null; console.log(data.length); // Can't read property of null!
TypeError: Cannot read property 'length' of null

üí° How to Prevent

‚Ä¢ Check data types before operations

‚Ä¢ Use typeof to verify types

‚Ä¢ Check if variables exist before accessing properties

‚Ä¢ Use correct methods for each data type

RangeError

Values Outside Valid Range

The last common error we will look at is the RangeError.

A RangeError happens when your code tries to use a value that's outside the range of what JavaScript can handle.

Common Example: Invalid Array Length

Here is an example of assigning an invalid value to an array's length:

‚ùå Incorrect Code

const arr = []; arr.length = -1; // Negative length is invalid!

Error Message

RangeError: Invalid array length

The Problem

Since an array's length has to be a non-negative integer, setting it to -1 triggers a RangeError.

Array length must be between 0 and 232-1 (4,294,967,295).

‚úì Correct Code

const arr = []; arr.length = 5; // Valid positive length console.log(arr.length); // 5

More RangeError Examples

‚ùå Invalid toFixed Argument

const num = 3.14159; num.toFixed(101); // toFixed accepts 0-100!
RangeError: toFixed() digits argument must be between 0 and 100

‚ùå Invalid toPrecision Argument

const num = 123.456; num.toPrecision(0); // Must be between 1-100!
RangeError: toPrecision() argument must be between 1 and 100

‚ùå Maximum Call Stack

function infiniteLoop() { infiniteLoop(); } infiniteLoop(); // Never stops!
RangeError: Maximum call stack size exceeded

‚ùå Invalid String Repeat

"Hi".repeat(-1); // Can't repeat negative times!
RangeError: Invalid count value

üí° How to Prevent

‚Ä¢ Check parameter ranges before method calls

‚Ä¢ Ensure array lengths are non-negative

‚Ä¢ Avoid infinite recursion

‚Ä¢ Validate numeric inputs

Key Takeaways

Four Common Error Types

As you continue to program in JavaScript, just be aware of these different types of errors you will probably encounter and why they are happening.

SyntaxError

Writing something incorrectly in your code, like missing a parenthesis or semicolon.

ReferenceError

Trying to access a variable that doesn't exist or hasn't been initialized yet.

TypeError

Performing an operation on the wrong type, like calling a number as a function.

RangeError

Using a value that's outside the valid range, like setting array length to -1.
