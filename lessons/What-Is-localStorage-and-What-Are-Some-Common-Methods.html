localStorage and Web Storage API

What is the Web Storage API?

The Web Storage API provides a mechanism for browsers to store key-value pairs right within the browser.

What Can You Do?

The API allows developers to store information that can be used across:

  • Different page reloads
  • Multiple sessions
  • Various pages of the same site

Data Storage

All data is stored in the browser, making it accessible without server requests.

Two Main Components

Web Storage API Components

The two main components in the Web Storage API are localStorage and sessionStorage.

localStorage

  • Data persists permanently
  • Survives browser close
  • Survives page refresh
  • Remains until deleted

sessionStorage

  • Data for page session only
  • Cleared on tab close
  • Cleared on window close
  • Temporary storage

This lesson focuses on localStorage. You'll learn about sessionStorage in the next lesson.

What is localStorage?

Definition

localStorage is the part of the Web Storage API that allows data to persist even after the browser window is closed or the page is refreshed.

Data Persistence

This data remains available until it is explicitly removed by:

  • The application (using JavaScript)
  • The user (clearing browser data)

Key Characteristic

Permanent storage - Data stays in the browser indefinitely unless removed.

What is sessionStorage?

Definition

sessionStorage stores data for the duration of the page session, meaning the data is available as long as the browser tab or window is open.

Key Difference

Unlike localStorage, the data in sessionStorage is cleared when the tab or window is closed.

When It's Cleared

  • Tab is closed
  • Window is closed
  • Browser session ends

You will learn more about sessionStorage in the next lesson.

Common Use Cases for localStorage

When to Use localStorage

Typical Applications

1. User Settings

Store preferences like themes or language choices

2. Form Data

Remember form data across browser sessions (draft emails, form inputs)

3. Caching

Cache small pieces of information to improve the performance of web apps

4. User Experience

Remember user choices like "don't show this again" preferences

What is Caching?

Definition

Caching refers to storing frequently accessed data in a temporary storage location, known as a cache.

How It Works

Subsequent requests for that data can be served more quickly without having to:

  • Recompute the data
  • Fetch it from a slower data source
  • Query a database
  • Contact an external server

Benefits

  • Faster Load Times: Data is retrieved instantly from local storage
  • Reduced Server Load: Fewer requests to the server
  • Better Performance: Improved user experience

Common localStorage Methods

Four Main Methods

localStorage provides four essential methods for managing stored data.

The Methods

setItem()
getItem()
removeItem()
clear()

What They Do

  • setItem(): Store a key-value pair
  • getItem(): Retrieve a value by key
  • removeItem(): Delete a specific item
  • clear(): Remove all stored data

setItem() Method

Storing Data

The setItem() method stores a key-value pair in localStorage.

Syntax

localStorage.setItem(key, value);

Example

localStorage.setItem('username', 'JaneDoe');

What Happens?

  • Key: 'username'
  • Value: 'JaneDoe'
  • This data is now stored in the browser
  • It persists even after closing the browser

getItem() Method

Retrieving Data

The getItem() method retrieves the value of a given key from localStorage.

Syntax

localStorage.getItem(key);

Example

let username = localStorage.getItem('username');
console.log(username); // JaneDoe

What Happens?

  • Retrieves the value associated with key 'username'
  • Returns 'JaneDoe' (the stored value)
  • If the key doesn't exist, returns null

removeItem() Method

Removing Data

The removeItem() method removes an item from localStorage using its key.

Syntax

localStorage.removeItem(key);

Example

localStorage.removeItem('username');

What Happens?

  • Finds the item with key 'username'
  • Deletes it from localStorage
  • Future getItem('username') calls will return null

clear() Method

Clearing All Data

The clear() method removes all data from localStorage.

Syntax

localStorage.clear();

⚠️ Warning

This method removes ALL items from localStorage for your domain. Use with caution!

What Happens?

  • Every key-value pair is deleted
  • localStorage becomes empty
  • Cannot be undone

Complete Example: User Theme

Storing Theme Preference

Let's look at a complete example using localStorage to store a user's preferred theme.

Full Code

// Store the user's theme preference
localStorage.setItem('theme', 'dark');

// Retrieve the stored theme preference
const userTheme = localStorage.getItem('theme');
console.log(userTheme); // 'dark'

// Remove the theme preference
localStorage.removeItem('theme');

// Clear all localStorage data
localStorage.clear();

Example Breakdown

Step-by-Step Explanation

1. Store Theme

localStorage.setItem('theme', 'dark');

We store a theme choice ('dark') for the user

2. Retrieve Theme

const userTheme = localStorage.getItem('theme');

We retrieve that theme and output it to the console

3. Remove or Clear

We demonstrate how to remove a specific item or clear all stored data

Security Considerations

⚠️ Important Security Warning

localStorage should NOT be used to store sensitive information, such as passwords, because it can pose security risks.

Why Not Secure?

  • No Encryption: Data is stored in plain text
  • Accessible via JavaScript: Any script can read it
  • XSS Vulnerability: Cross-site scripting attacks can access it
  • Permanent Storage: Data persists even if compromised

What NOT to Store

  • Passwords
  • API keys
  • Credit card information
  • Personal identification numbers
  • Authentication tokens (use httpOnly cookies instead)

Summary

Key Concepts

Web Storage API

  • sessionStorage: Cleared when tab closes (temporary)
