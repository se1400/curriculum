The Universal Selector

The universal selector * is a special type of CSS selector that matches any element in the document.

Common Use Cases

  • Applying styles to all elements on the page
  • Resetting or normalizing styles across different browsers
  • Selecting all elements within a specific context
  • Creating global baseline styles

Syntax

The universal selector is represented by an asterisk: *

What It Does

It can be used to select all elements globally across the entire document or within a specific context.

CSS - Universal Selector
* {
  margin: 0;
  padding: 0;
  box-sizing: border-box;
}
What This Does

This applies the specified styles to every single element in the HTML document. It's commonly used to reset default browser styles.

CSS Reset Example

A common technique is using the universal selector to reset margin and padding to zero for all elements.

Why Reset?

Different browsers apply different default styles. Resetting creates a consistent baseline across all browsers.

CSS
* {
  margin: 0;
  padding: 0;
}

Specificity of Universal Selector

The universal selector has the lowest specificity value of any selector.

Specificity: (0, 0, 0, 0)

What This Means

It contributes 0 to all parts of the specificity value. Any other selector will override the styles set by the universal selector.

Specificity Comparison

The universal selector has the lowest specificity compared to all other selectors:

Inline styles
(1, 0, 0, 0)
ID selectors
(0, 1, 0, 0)
Class selectors
(0, 0, 1, 0)
Type selectors
(0, 0, 0, 1)
Universal (*)
(0, 0, 0, 0)

Override Behavior

Any other selector will override the universal selector's styles:

  • Type selectors (p, div, h1)
  • Class selectors (.class)
  • ID selectors (#id)
  • Inline styles
CSS
* {
  color: blue;
}

p {
  color: red;
}
Result

This paragraph is red Type selector wins

The type selector (p) overrides the universal selector (*) because it has higher specificity

Complete Cascade Example

Let's see how the universal selector interacts with all selector types:

The CSS Rules

Multiple selectors targeting the same element with different colors.

CSS
* {
  color: blue;
}

p {
  color: red;
}

.highlight {
  color: green;
}

#unique {
  color: purple;
}
HTML
<p id="unique" class="highlight">
  This text has multiple styles applied.
</p>

Which Style Wins?

* { color: blue; } (0,0,0,0) OVERRIDDEN
p { color: red; } (0,0,0,1) OVERRIDDEN
.highlight { color: green; } (0,0,1,0) OVERRIDDEN
#unique { color: purple; } (0,1,0,0) WINS!

The Result

The ID selector has the highest specificity, so it wins the cascade.

Final Result

This text has multiple styles applied.

Why Purple?

  1. Universal selector sets blue (0,0,0,0)
  2. Type selector overrides to red (0,0,0,1)
  3. Class selector overrides to green (0,0,1,0)
  4. ID selector overrides to purple (0,1,0,0) ✓

Key Takeaways

  • The universal selector * matches any element in the document
  • Commonly used for CSS resets to normalize browser default styles
  • Has the lowest specificity value: (0, 0, 0, 0)
  • Any other selector will override universal selector styles
  • Perfect for setting baseline styles that can be easily overridden

Remember

Universal selector = Lowest specificity = Easily overridden
