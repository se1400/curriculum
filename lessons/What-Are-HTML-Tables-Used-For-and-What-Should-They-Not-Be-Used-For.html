HTML Tables - Introduction

HTML tables aren't used as much these days as they used to be. But, as a frontend developer, you should still be familiar with them.
Tables are one of the earliest ways developers had for displaying data in a browser way back in the 1990s.
Historical Context: Tables were among the first structural elements available to web developers for organizing and presenting information in a grid-like format. While their usage has evolved, they remain important for their original purpose.
Even though modern web development has moved beyond using tables for layout, understanding tables is essential for properly displaying tabular data.

Table Structure & Elements

A table has a main table element with several semantic sections inside it:
<table>
<thead> - Table head element (header section)
<tbody> - Table body element (main content)
<tfoot> - Table foot element (footer section)
The table head, body, and foot elements can each contain some number of table rows (<tr>).
Each table row can contain:
<th> - Table header which labels the data in that row
<td> - Table data, individual data cells
Don't be intimidated: While that's a lot of HTML elements, these follow a simple hierarchy that becomes intuitive with practice.

Real-World Table Example

Here's an example from the U.S. Bureau of Labor Statistics showing software developer job data:
<table id="quickfacts">
  <thead>
    <tr>
      <th colspan="2">Quick Facts: Software Developers</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <th>2023 Median Pay</th>
      <td>$130,160 per year<br>$62.58 per hour</td>
    </tr>
    <tr>
      <th>Typical Entry-Level Education</th>
      <td>Bachelor's degree</td>
    </tr>
    <tr>
      <th>Work Experience in a Related Occupation</th>
      <td>None</td>
    </tr>
  </tbody>
</table>
Rendered Output:
Quick Facts: Software Developers, Quality Assurance Analysts, and Testers
2023 Median Pay $130,160 per year
$62.58 per hour
Typical Entry-Level Education Bachelor's degree
Work Experience in a Related Occupation None

Simple Table Hierarchy

Here's the simplest table we can create that includes all of these elements:
<table>
  <thead>
    <tr>
      <th>The title of this table</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <th>First Row</th>
      <td>First Data Cell</td>
    </tr>
    <tr>
      <th>Second Row</th>
      <td>Second Data Cell</td>
    </tr>
  </tbody>
  <tfoot>
    <tr>
      <td colspan="2">The footer of this table</td>
    </tr>
  </tfoot>
</table>
Rendered Output:
The title of this table
First Row First Data Cell
Second Row Second Data Cell
The footer of this table, which might contain date of publication, author credits, or other meta information
Key Pattern: The data itself is always within a <tr> - and within that <tr> element is a <th> element with a header, and a <td> element with data.

What NOT to Use Tables For

❌ DON'T Use Tables For:

  • Page layouts
  • Positioning non-data elements
  • Creating website structure
  • Design purposes

✓ DO Use Tables For:

  • Displaying tabular data
  • Presenting data in rows and columns
  • Statistical information
  • Comparison charts
Common Mistake: Some websites choose to use <div> elements to build their own tables instead of using the more appropriate <table> element. While it is possible to display tabular data using generic div elements, it is still better to use the table element instead.
Many years ago, developers might have used a table to position non-data elements on a webpage. This was never considered a best practice. But you may encounter some codebases where tables are still used like this.
Modern Approach: Nowadays, developers use CSS flexbox and grid to layout their designs. freeCodeCamp will cover these tools in depth later.
Bottom Line: Use HTML tables for their original intended purpose: displaying tabular data.
