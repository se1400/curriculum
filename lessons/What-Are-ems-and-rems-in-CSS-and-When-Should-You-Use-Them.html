Relative Units: em and rem

Moving Beyond Absolute Units

While absolute length units like pixels are helpful in certain situations, there are times when relative units are more appropriate.

This lesson covers two relative units:
  • em units
  • rem units

These units provide flexibility and scalability that absolute units cannot offer.

Understanding em Units

Definition

em units are relative to the font size of the element.

Key Behavior: When using ems for the font-size property, the size of the text will be relative to the font size of the parent element.

How It Works

If an element has a font size, em units will calculate based on that font size. If not, they'll calculate based on the parent's font size.

em Units in Action: Margin Example

Visual Demonstration

I am a paragraph element

Blue box
Calculation:
font-size: 20px
margin-bottom: 1.5em
Result: 1.5 × 20px = 30px margin

The margin is 1.5 times the font size of the paragraph element, creating clear space between elements.

HTML

<p class="para">
  I am a paragraph element
</p>
<div class="blue-box"></div>

CSS

.para {
  font-size: 20px;
  margin-bottom: 1.5em;
  border: 2px solid red;
}

.blue-box {
  background-color: blue;
  color: white;
  padding: 10px;
}

em Units: Inheriting from Parent

What If There's No font-size?

If you remove the font-size property from the element, the bottom margin will be relative to the font size of the parent element.

Default Behavior: The parent element is the body element, which has a default font size of 16px.
New Calculation:
No explicit font-size set
Parent (body) font-size: 16px
margin-bottom: 1.5em
Result: 1.5 × 16px = 24px margin

When to Use em Units

Ideal Use Cases

Good use cases for em units include working with modular components like buttons or cards.

Benefit: By using em units, you can ensure that all aspects of the component (padding, margin, borders) scale proportionally with the font size, keeping consistent proportions.

Example Scenarios

  • Button padding that scales with button text size
  • Card components where spacing relates to text size
  • Icon sizes relative to nearby text
  • Spacing within modular UI elements

The Challenge with Pixel-Based Typography

Accessibility Issues

Up until this point, we've been setting font sizes using pixels. But this presents challenges for users.

Browser Font Size Settings

Users can control the default font size in their browser settings. Those with visual impairments may increase the font size to make content easier to read.

The Problem: If you set pixels for font sizes, the text will NOT scale proportionally with the rest of the content when users adjust their browser settings.

rem Units: The Solution for Typography

What are rem Units?

A rem unit is relative to the font size of the root element, which is the html element.

Default: By default, the font size of the html element is 16px.

How rem Units Solve the Problem

If the user increases the font size in their browser settings, the font size of the html element will increase, and all rem units will scale proportionally.

Accessibility Win: rem units respect user preferences and make content accessible to users with visual impairments.

Using rem Units for Font Size

Visual Example

This is regular text.

This text is slightly larger.

Calculation:
Root (html) font-size: 16px (default)
font-size: 1.2rem
Result: 1.2 × 16px = 19.2px

The font size of the paragraph is 1.2 times the root element's font size. If the user changes browser settings, this scales accordingly!

HTML

<p>This is regular text.</p>
<p class="para">
  This text is slightly larger.
</p>

CSS

.para {
  font-size: 1.2rem;
}

When Should You Use rem Units?

Typography

rem units are preferred over pixels for typography because they scale proportionally with the user's browser settings.

Accessibility: This makes your content more accessible to users with visual impairments.

Additional Benefits

rem units can also help maintain consistent spacing and layout across different elements.

  • Font sizes that respect user preferences
  • Consistent spacing throughout the site
  • Predictable scaling behavior
  • Better accessibility compliance

em vs rem: Quick Comparison

em Units

Relative to: The font size of the element or parent element

Best for: Modular components where spacing should scale with component size

Example: Button padding, component spacing

rem Units

Relative to: The font size of the root (html) element

Best for: Typography and global spacing that should respect user settings

Example: Font sizes, consistent margins

Key Takeaways

About em Units

  • Relative to the font size of the element (or parent if not specified)
  • Great for modular components that need proportional scaling
  • Useful for padding, margins, and borders within components

About rem Units

  • Relative to the font size of the root (html) element
  • Default root font size is 16px
  • Preferred for typography because they scale with user settings
  • Improves accessibility for users with visual impairments
  • Helps maintain consistent spacing across elements
Remember: Use rem for typography and global spacing, use em for component-level proportional scaling.
