VS Code Workspaces

VS Code considers projects to be "workspaces." A workspace is simply the directory (folder) you opened in VS Code.

To create a new project, you'll need to create a new directory for it first.

A directory is another name for a folder on your computer.

Understanding Workspaces

When you open a folder in VS Code, that folder becomes your workspace. All files and subfolders within it are part of your project workspace.

Command Line Interfaces

Command-line interfaces (CLI) allow you to interact with your operating system through text-based commands.

Different operating systems have different CLI tools available.

CLI Tools by Operating System

Windows: Command Prompt or PowerShell

Mac: Terminal app

Linux: Terminal

Creating a New Directory

Use the command line to create a new directory for your project.

First, navigate to your home directory, then create the new directory using the mkdir command.

mkdir stands for "make directory" and is used to create new folders.

Navigate to Home Directory
cd ~
Create New Directory
mkdir my-project

Opening Your Project in VS Code

There are several ways to open your new directory with VS Code.

Choose whichever method feels most comfortable for your workflow.

Methods to Open VS Code

  • Right-click context menu from your file explorer
  • Double-click the VS Code application and use File → Open Folder
  • Use the command line with the code command
Command Line Method
code /path/to/folder

Creating a New File

Once your workspace is open in VS Code, you can create new files for your project.

Click on the File menu at the top-left corner and select "New File...", then name your file and save it.

Your new file will appear in the explorer tab on the left side of VS Code.

Creating index.html

  • Click File menu → New File...
  • Name your file index.html
  • Click Save
  • Add HTML content to your file
  • File appears in explorer tab

Why Use a Web Server?

You might be tempted to open the HTML file in your browser directly. While that can work, it can lead to unexpected bugs.

Common issues include CSS not loading properly, JavaScript errors, and other resource loading problems.

Instead, you should put your HTML file behind a proper web server.

Problems with Direct File Opening

  • CSS may not load properly
  • JavaScript security restrictions
  • File paths may not work correctly
  • CORS errors with external resources

Benefits of Web Server

  • Proper HTTP protocol handling
  • Correct MIME types
  • No unexpected bugs
  • Mimics production environment

Live Server Extension

The Live Server extension makes it easy to run a local web server right from VS Code.

Click on the extensions tab in the left corner and search for "Live Server" in the marketplace. It's available for free.

Once installed, you'll see a "Go Live" button in your status bar at the bottom of VS Code.

Installing Live Server

  • Click Extensions tab (left sidebar)
  • Type "Live Server" in search bar
  • Click Install on the extension
  • Look for "Go Live" button in status bar

Running Your Project

Make sure you have your HTML file open in VS Code, then click the "Go Live" button in the status bar.

Your page should open automatically in a new browser tab. If it doesn't, you can navigate directly to http://localhost:5500/

Congratulations! You've set up your very own project in VS Code and can view your changes with Live Server.

Using Live Server

  • Open your HTML file in VS Code
  • Click "Go Live" button
  • Browser opens automatically
  • Or visit http://localhost:5500/
  • Changes update in real-time
Local Server Address
http://localhost:5500/
