CSS Filter Property

The CSS filter property is a powerful tool that allows you to apply graphical effects to elements on a web page.

What It's Used For:

It's particularly useful for adjusting the visual presentation of images, backgrounds, and even text without modifying the original asset.

Common Effects:

  • Blurring - Create depth or obscure content
  • Color shifting - Change hues and tones
  • Contrast adjustments - Enhance or reduce contrast
  • Brightness - Make lighter or darker
  • And many more!

Basic Syntax

The basic syntax for the filter property is straightforward:

CSS Syntax
selector {
  filter: function(amount);
}

Components:

  • function - The specific filter effect you want to apply
  • amount - A value that determines the intensity of the effect
Example
img {
  filter: blur(5px);
}

Filter Function: blur()

The blur() function applies a Gaussian blur to the element.

CSS
img {
  filter: blur(2px);
}

Amount: Specified in pixels and represents the radius of the blur.

Filter Function: brightness()

The brightness() function adjusts the brightness of the element.

CSS
img {
  filter: brightness(150%);
}
  • 0% - Makes the element completely black
  • 100% - Original brightness (default)
  • >100% - Increases brightness

Filter Function: grayscale()

The grayscale() function converts the element to grayscale.

CSS
img {
  filter: grayscale(100%);
}
  • 0% - Leaves the image unchanged (full color)
  • 100% - Completely grayscale (no color)

Filter Function: sepia()

The sepia() function applies a sepia tone to the element.

CSS
img {
  filter: sepia(80%);
}

Like grayscale, it uses a percentage value from 0% to 100%.

Use Cases: Creating a vintage or old-timey look

Filter Function: hue-rotate()

The hue-rotate() function applies a hue rotation to the element.

CSS
img {
  filter: hue-rotate(90deg);
}

The value is defined in degrees and represents a rotation around the color circle.

  • 0deg - Original colors
  • 90deg - Quarter rotation
  • 180deg - Half rotation (opposite colors)
  • 360deg - Full rotation (back to original)

Combining Multiple Filters

One of the most powerful aspects of the filter property is the ability to combine multiple effects.

How to Combine:

You can apply several filters to the same element by separating them with spaces:

CSS
img {
  filter: contrast(120%) brightness(110%) sepia(30%);
}

This rule applies:

  • Increased contrast (120%)
  • Slightly increased brightness (110%)
  • A subtle sepia effect (30%)

Filter Syntax: How to Combine

When combining filters, order can matter in some cases:

✓ Correct Syntax (Space-Separated)
filter: blur(5px) sepia(50%);
❌ Wrong Syntax (Plus Sign)
filter: blur(5px) + sepia(50%); /* Doesn't work! */
❌ Wrong Syntax (Comma)
filter: blur(5px), sepia(50%); /* Doesn't work! */
❌ Wrong Syntax (&&)
filter: blur(5px) && sepia(50%); /* Doesn't work! */

Remember: Separate multiple filters with spaces only!

Common Use Cases

✓ Great Uses for Filters:

  • Image galleries: Hover effects with brightness or grayscale
  • Backgrounds: Blur backgrounds behind modals
  • Vintage effects: Sepia tones for old-style photos
  • Loading states: Blur content while loading
  • Focus indicators: Dim/desaturate inactive elements
  • Dark mode: Invert colors for dark themes
  • Artistic effects: Combine filters for unique looks

Practical Example: Hover Effect

CSS
img {
  filter: grayscale(100%);
  transition: filter 0.3s;
}

img:hover {
  filter: grayscale(0%);
}

Accessibility Considerations

⚠️ Important Warning:

As with any powerful feature, it's important to be careful with how you use filters to enhance your design without overwhelming your users or compromising accessibility.

Accessibility Concerns:

  • Contrast: Don't reduce contrast so much that text becomes unreadable
  • Color blindness: Be aware that hue-rotate might confuse color-blind users
  • Motion sensitivity: Animated filters can trigger motion sickness
  • Performance: Heavy filters can cause lag on slower devices
  • Content clarity: Don't blur or obscure important content

Accessibility: Do's and Don'ts

✓ Do These:

  • Use filters for decorative elements
  • Test filters with screen readers
  • Provide alternatives to filtered content
  • Keep text readable (good contrast)
  • Test on different devices
  • Use transitions for smooth changes
  • Respect prefers-reduced-motion

❌ Avoid These:

  • Filtering text to be barely readable
  • Heavy blur on interactive elements
  • Relying solely on color changes
  • Animating filters rapidly
  • Making navigation hard to see
  • Ignoring performance on mobile

Performance Considerations

Performance Impact:

Filters are GPU-accelerated in modern browsers, but they can still impact performance:

  • Blur: Most expensive filter (heavy computation)
  • Multiple filters: Combining many filters increases cost
  • Large elements: Filtering big images takes more resources
  • Animations: Animated filters can cause lag

✓ Optimization Tips:

  • Use will-change: filter for animated filters
  • Limit the number of filtered elements
  • Test on mobile devices
  • Consider using filtered images instead
  • Avoid filtering the entire page
