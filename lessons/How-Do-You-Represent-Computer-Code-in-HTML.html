The Inline Code Element

The inline code element (<code>) is used to represent short snippets of code inside text.
Common Use Cases:
Technical articles
Documentation pages
Inline code references within paragraphs

Example: CSS Code Snippet

<p>
  To set the text color to blue in CSS, use the following code:
  <code>color: blue;</code>
</p>
Output:
To set the text color to blue in CSS, use the following code: color: blue;

How the Code Element Works

By wrapping a code snippet inside <code> tags, it communicates to the browser that the text is a piece of inline code.
Default Browser Styling:
The browser applies default styles to content inside the code element. The default styling is a monospaced font.
A monospaced font means each character takes up the same amount of horizontal space, making code easier to read and align.
Important: The code element is meant to represent a single line of code. If you want to represent multiple lines of code, you will need to place a code element inside a preformatted text element.

The Preformatted Text Element

The preformatted text element (<pre>) is used to represent preformatted text. For multiple lines of code, you need to place a <code> element inside a <pre> element.

Example: Multi-line CSS Declaration

<pre>
  <code>
  body {
    color: red;
  }
  </code>
</pre>
Output:
body {
  color: red;
}

Spacing with the Pre Element

Critical: When using the <pre> element, you need to be mindful of spacing because it will display exactly as written inside the HTML document.
In the browser, the code will appear with the exact same indentation and spacing that you use in your HTML source code.
If you change the indentation in your code example, you will see a corresponding difference in the indentation displayed on the screen.
The <pre> element preserves:
  • Spaces
  • Line breaks
  • Indentation
  • All whitespace characters

Best Practices Summary

When including code examples in your HTML document, choose the appropriate element based on the length and format of your code:

Use <code> alone for:

Short inline code examples
Single-line code snippets within text
Variable names or function names in paragraphs

Use <pre> + <code> for:

Longer code snippets
Multi-line code blocks
Code that requires preserved formatting
Key Takeaway: The combination of semantic HTML elements (<code> and <pre>) helps create clear, accessible documentation and technical content.
