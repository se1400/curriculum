Audio and Video Elements

The <audio> and <video> elements allow you to add sound and video content to your HTML documents.

These elements provide native support for multimedia without requiring external plugins.

Audio Element

Supported formats:

  • MP3
  • WAV
  • OGG

Video Element

Supported formats:

  • MP4
  • OGG
  • WebM

Basic Audio Element

To include audio content on your web page, you can use the audio element with the src attribute pointing to the location of your audio file.

Without additional attributes, nothing shows up on the page - the audio element is present but invisible.

Basic Audio (No Controls)
<audio
  src="audio-file.mp3"
></audio>

Note

This audio element exists in the DOM but is not visible or interactive without the controls attribute.

Controls Attribute

If you want to see the audio player on the page, then you can add the controls attribute.

The controls attribute enables users to manage audio playback, including adjusting volume, and pausing or resuming playback.

The controls attribute is a boolean attribute - if omitted, no controls will be shown.

Audio with Controls
<audio
  src="audio-file.mp3"
  controls
></audio>

Control Features

  • Play/pause button
  • Volume adjustment
  • Progress bar
  • Time display

Loop Attribute

The loop attribute is a boolean attribute that makes the audio replay continuously.

When the audio reaches the end, it will automatically restart from the beginning.

Audio with Loop
<audio
  src="audio-file.mp3"
  loop
  controls
></audio>

Behavior

Audio will restart automatically when it reaches the end, playing continuously until stopped by the user.

Muted Attribute

When present in the audio element, the muted boolean attribute will start the audio in a muted state.

Users can unmute the audio by clicking on the volume icon in the audio player.

Audio with Muted
<audio
  src="audio-file.mp3"
  loop
  controls
  muted
></audio>

Use Cases

  • Background music that users can enable
  • Autoplay requirements (many browsers require muted for autoplay)
  • User-controlled audio experiences

Multiple Source Elements

When it comes to audio file types, there are differences in which browsers support which type.

To accommodate this, you can use <source> elements inside the audio element and the browser will select the first source that it understands.

Multiple Sources for Compatibility
<audio controls>
  <source src="audio.ogg" type="audio/ogg" />
  <source src="audio.wav" type="audio/wav" />
  <source src="audio.mp3" type="audio/mpeg" />
</audio>

How It Works

The browser starts with the first source type. If it can't play that audio format, it moves down to the next type in the list.

Video Element

All the attributes we have learned so far are also supported in the video element.

The <video> element works similarly to <audio> but displays video content.

Video Element Example
<video
  src="video.mp4"
  loop
  controls
  muted
  width="400"
></video>

Shared Attributes

  • src or source elements
  • controls
  • loop
  • muted
  • autoplay

Autoplay Attribute

The autoplay attribute is a boolean attribute that makes the video (or audio) play automatically when the page loads.

Most browsers require videos to be muted to allow autoplay due to user experience policies.

Video with Autoplay
<video
  src="video.mp4"
  autoplay
  muted
  controls
></video>

Important Note

Modern browsers often block autoplay with sound. To enable autoplay, include the muted attribute.

Poster Attribute

If you wanted to display an image while the video is downloading, you can use the poster attribute.

This attribute is not available for audio elements and is unique to the video element.

The poster image appears before the video loads or before the user presses play.

Video with Poster
<video
  src="video.mp4"
  controls
  poster="thumbnail.jpg"
  width="400"
></video>

Benefits

  • Shows a preview image before video loads
  • Provides visual context
  • Improves user experience
  • Can make the page more visually appealing
