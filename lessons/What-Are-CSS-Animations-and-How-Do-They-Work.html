They provide a way to smoothly transition elements between different styles over a specified duration.

Two Main Components

The Building Blocks

At its core, a CSS animation consists of two main components: the @keyframes rule and the animation property.

Component 1: @keyframes

  • Defines the stages and styles of the animation

Component 2: animation Property

  • Applies the animation to an element

The @keyframes Rule

Defining Animation Stages

The @keyframes rule defines the stages and styles of the animation. It specifies what styles the element should have at various points during the animation.

@keyframes animation-name { 0% { /* Starting styles */ } 50% { /* Midpoint styles */ } 100% { /* Ending styles */ } }

Percentages: The percentages represent the progress of the animation timeline.

Example: slide-in Animation

Moving Left to Right

Here's an example:

@keyframes slide-in { 0% { transform: translateX(-100%); } 100% { transform: translateX(0); } }

What it does: This @keyframes rule, named slide-in, defines an animation that moves an element from left to right.

The translateX Function

Horizontal Movement

The translateX function in your @keyframes animation is controlling the horizontal position of an element as it animates into view.

/* Start: 100% to the left (off-screen) */ 0% { transform: translateX(-100%); } /* End: 0% offset (original position) */ 100% { transform: translateX(0); }

Understanding Values:
• translateX(-100%) = Move 100% to the left
• translateX(0) = Original position
• translateX(100%) = Move 100% to the right

Applying the Animation

Using the animation Property

To apply this animation to an element, you use the animation property.

.sliding-element { animation: slide-in 2s ease-out; }

Breaking it down:
• slide-in = Name of @keyframes
• 2s = Duration (2 seconds)
• ease-out = Timing function

Result: This applies the slide-in animation to the element with a duration of 2 seconds and an ease-out timing function.

Animation Shorthand Property

All-in-One Syntax

The animation property is actually a shorthand for several individual properties:

/* Shorthand */ .element { animation: name duration timing-function delay iteration-count direction fill-mode play-state; } /* Example */ .element { animation: slide-in 2s ease-out 0s infinite alternate both running; }

8 Properties in 1: The animation property combines 8 individual properties!

animation-name

Property 1 of 8

animation-name

Specifies the @keyframes rule to use.

.element { animation-name: slide-in; }

Must Match: The name here must exactly match the name in your @keyframes rule!

animation-duration

Property 2 of 8

animation-duration

Sets how long the animation should take to complete.

.element { animation-duration: 2s; /* 2 seconds */ animation-duration: 500ms; /* 500 milliseconds */ }

Units: Use s for seconds or ms for milliseconds.

animation-timing-function

Property 3 of 8

animation-timing-function

Defines how the animation progresses over time.

.element { animation-timing-function: ease; /* Default */ animation-timing-function: linear; /* Constant speed */ animation-timing-function: ease-in; /* Slow start */ animation-timing-function: ease-out; /* Slow end */ animation-timing-function: ease-in-out; /* Slow start & end */ }

Common Values: ease, linear, ease-in, ease-out, ease-in-out

animation-delay

Property 4 of 8

animation-delay

Specifies a delay before the animation starts.

.element { animation-delay: 0s; /* Start immediately */ animation-delay: 1s; /* Wait 1 second */ animation-delay: 500ms; /* Wait 500 milliseconds */ }

Useful For: Staggering multiple animations or creating sequences!

animation-iteration-count

Property 5 of 8

animation-iteration-count

Sets how many times the animation should repeat.

.element { animation-iteration-count: 1; /* Once (default) */ animation-iteration-count: 3; /* 3 times */ animation-iteration-count: infinite; /* Forever */ }

Infinite Loops: Use infinite to repeat the animation endlessly!

animation-direction

Property 6 of 8

animation-direction

Determines whether the animation should play forwards, backwards, or alternate.

.element { animation-direction: normal; /* 0% → 100% */ animation-direction: reverse; /* 100% → 0% */ animation-direction: alternate; /* 0% → 100% → 0% */ animation-direction: alternate-reverse; /* 100% → 0% → 100% */ }

Alternate: Creates a back-and-forth effect when combined with iteration-count!

animation-fill-mode

Property 7 of 8

animation-fill-mode

Specifies how the element should be styled before and after the animation.

.element { animation-fill-mode: none; /* Default behavior */ animation-fill-mode: forwards; /* Keep end state */ animation-fill-mode: backwards; /* Use start state during delay */ animation-fill-mode: both; /* Both forwards & backwards */ }

Forwards: Keeps the element in its final animated state after completion!

animation-play-state

Property 8 of 8

animation-play-state

Allows you to pause and resume the animation.

.element { animation-play-state: running; /* Playing (default) */ animation-play-state: paused; /* Paused */ } .element:hover { animation-play-state: paused; /* Pause on hover */ }

Interactive: Great for giving users control over animations!

Using Individual Properties

Precise Control

You can use these properties individually for more precise control:

@keyframes color-change { 0% { background-color: red; } 50% { background-color: blue; } 100% { background-color: green; } } .complex-animation { animation-name: color-change; animation-duration: 3s; animation-timing-function: ease-in-out; animation-delay: 0s; animation-iteration-count: infinite; animation-direction: normal; animation-fill-mode: none; animation-play-state: running; }

Triggering Animations

Event-Based Animations

CSS animations can be triggered by various events, such as hovering over an element:

.button { transition: transform 0.3s ease; } .button:hover { transform: scale(1.1); }

Note: Transitions

Simpler Alternative

While this example uses the transition property, which is simpler for basic effects, it demonstrates how CSS can create interactive, animated elements.

Transitions:

  • Simpler syntax for two-state changes
  • Triggered by state changes (hover, focus, etc.)

Animations (@keyframes):

  • More complex, multi-stage effects
  • Can run automatically or be triggered

Use Animations Wisely

Performance & Accessibility

It's important to note that while CSS animations are powerful, they should be used in moderation.

Warning: Overuse of animations can lead to poor performance and may be distracting or problematic for users with certain accessibility needs.

Potential Issues:

  • Performance – Too many animations = slow page
  • Distraction – Can make content hard to read
  • Motion Sensitivity – Can trigger discomfort
  • Battery Drain – Continuous animations use power

Accessibility Considerations

Respecting User Preferences

Always consider providing options to reduce or disable animations for users who prefer less motion.

/* Respect user's motion preferences */ @media (prefers-reduced-motion: reduce) { * { animation-duration: 0.01ms !important; animation-iteration-count: 1 !important; transition-duration: 0.01ms !important; } }

Inclusive Design: This media query disables animations for users who have enabled "reduce motion" in their system settings!

Best Practices

Creating Great Animations

Follow These Guidelines:

  • Use animations purposefully, not decoratively
  • Keep animations subtle and quick (under 1 second)
  • Animate transform and opacity for best performance
  • Respect prefers-reduced-motion
  • Avoid animating width, height, or layout properties
  • Test on slower devices
  • Provide pause controls for long animations
