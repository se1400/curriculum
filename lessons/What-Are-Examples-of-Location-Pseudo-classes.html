What Are Location Pseudo-classes?

Location pseudo-classes are used for styling links and elements that are targeted within the current document.

They Apply Styles Based On:

  • Whether a link is visited
  • Whether an element is currently in focus
  • URL fragment identifiers (like #section1)
  • Link locations (internal vs external)
Location pseudo-classes help users understand link states and navigate documents more effectively

Location Pseudo-classes Overview

There are five main location pseudo-classes:

:link

Unvisited links

:visited

Visited links

:any-link

All links (visited + unvisited)

:local-link

Links to same document

:target

Element matching URL fragment

:link Pseudo-class

The :link pseudo-class allows you to target all unvisited links on a webpage.

Purpose:

  • Style links before the user clicks them
  • Distinguish unvisited from visited links
  • Use brand colors for new links
  • Provide visual consistency

CSS Example:

a:link {
  color: magenta;
}
Visit Example.com

This link appears magenta until clicked

:visited Pseudo-class

The :visited pseudo-class targets links the user has already clicked.

How It Works:

  • Applies after user clicks the link
  • Replaces the :link styling
  • Helps users distinguish visited pages
  • Prevents revisiting content unnecessarily

CSS Example:

a:visited {
  color: purple;
}
Visit Example.com

After clicking, this link turns purple

Privacy Note:

Due to privacy concerns, browsers limit which CSS properties can be modified with :visited (typically only color-related properties).

:link and :visited Flow

Understanding the relationship between :link and :visited:

1
Initial State: Link appears with :link styling (e.g., magenta)
2
User Action: User clicks the link
3
State Change: :link style no longer applies
4
New State: :visited pseudo-class takes over (e.g., purple)
The :visited pseudo-class helps users distinguish between links they have visited and those they have not

:any-link Pseudo-class

The :any-link pseudo-class is a combination of :link and :visited.

What It Matches:

  • Any anchor element with an href attribute
  • Applies regardless of visited state
  • Matches both :link and :visited links
  • Useful for applying base link styles

CSS Example:

a:any-link {
  color: crimson;
}
Visit Example.com

This link is crimson whether visited or not

:any-link Use Cases

The :any-link pseudo-class is particularly useful for applying base styles to all links.

Without :any-link

a:link,
a:visited {
  text-decoration: none;
  font-weight: bold;
}

Need to list both pseudo-classes

With :any-link

a:any-link {
  text-decoration: none;
  font-weight: bold;
}

Cleaner, more concise code

:local-link Pseudo-class

The :local-link pseudo-class targets links that point to the same document.

Intended Purpose:

  • Differentiate internal links from external ones
  • Style same-domain links differently
  • Help users understand navigation context
  • Improve internal site navigation

CSS Example (Theoretical):

a:local-link {
  color: blue;
  text-decoration: none;
}

⚠️ Browser Support:

Currently, no browser supports the :local-link pseudo-class. While it's part of the CSS specification, it hasn't been implemented yet.

:target Pseudo-class

The :target pseudo-class selects an element that matches the current URL fragment identifier.

What's a URL Fragment?

The part of a URL after the # symbol, like #section1

Example: https://example.com/page#section1

Common Uses:

  • In-page navigation (table of contents)
  • Highlighting the active section
  • Jump links within long documents
  • Accordion-style content

:target Example

Here's a practical example of :target for in-page navigation:

CSS:

section:target {
  background-color: lightgreen;
  border-color: green;
}

HTML:

<nav>
  <a href="#section1">Introduction</a>
  <a href="#section2">Features</a>
</nav>

<section id="section1">
  <h2>Introduction</h2>
</section>

How It Works:

When you click a navigation link, the URL changes to include the fragment identifier (like #intro). The :target pseudo-class then applies styles to the element with that matching ID.

Location Pseudo-classes Summary

Key Takeaways:

  • Location pseudo-classes style links and targeted elements
  • :link - Styles unvisited links
  • :visited - Styles visited links (limited properties)
  • :any-link - Matches all links (visited + unvisited)
  • :local-link - For same-document links (not supported yet)
  • :target - Matches element with current URL fragment
  • Best for: Navigation, link states, in-page jumping
Location pseudo-classes enhance navigation and help users understand their location within a document
