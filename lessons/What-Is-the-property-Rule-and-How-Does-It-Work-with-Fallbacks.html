The @property Rule

Advanced Custom Properties

The @property rule is a powerful CSS feature that allows developers to define custom properties with greater control over their behavior, including how they animate and their initial values.

What You'll Learn:

  • Understanding the @property syntax
  • The three key descriptors
  • How to animate custom properties
  • Working with fallbacks
  • Browser support strategies

Supercharging Variables: @property takes CSS custom properties to the next level!

Enhanced Functionality

Beyond Standard Custom Properties

This rule provides a way to enhance the functionality of CSS custom properties and offers more flexibility in their application.

Key Advantages:

  • Type Safety – Define what values are valid
  • Animation – Animate custom properties smoothly
  • Initial Values – Set default starting values
  • Inheritance Control – Choose if values inherit
  • Better Validation – Browser checks types

The Difference: Standard custom properties are just strings. @property makes them typed, validated values!

Basic Syntax

The Structure

The basic syntax of the @property rule is as follows:

@property --property-name { syntax: '<type>'; inherits: true | false; initial-value: <value>; }

Three Required Parts: Every @property rule must have syntax, inherits, and initial-value.

The Property Name

Naming Your Property

The --property-name is the name of the custom property you're defining. Like all custom properties, it must start with two dashes.

/* Valid property names */ @property --main-color { /* ... */ } @property --button-size { /* ... */ } @property --animation-speed { /* ... */ } /* Must start with -- */

Same Rule: Just like regular custom properties, always use the -- prefix!

The syntax Descriptor

Defining the Type

syntax defines the type of the property. --property-name can be things like <color>, <length>, <number>, <percentage>, or more complex types.

/* Common syntax types */ @property --my-color { syntax: '<color>'; /* ... */ } @property --my-size { syntax: '<length>'; /* ... */ } @property --my-opacity { syntax: '<number>'; /* ... */ }

Type Safety: The browser will validate that values match the specified type!

Common Syntax Types

Available Types

Frequently Used Types:

  • <color> – Color values (#fff, rgb(), hsl())
  • <length> – Length values (10px, 2rem, 50%)
  • <number> – Numbers (0.5, 100, -5)
  • <percentage> – Percentages (50%, 100%)
  • <angle> – Angles (45deg, 0.5turn)
  • <time> – Time values (1s, 500ms)

Plus More: You can also combine types with | or use * for "any value"!

The inherits Descriptor

Inheritance Control

inherits specifies whether the property should inherit its value from its parent element.

/* Inherits from parent */ @property --text-color { syntax: '<color>'; inherits: true; initial-value: black; } /* Does NOT inherit */ @property --button-bg { syntax: '<color>'; inherits: false; initial-value: blue; }

Two Options: true = inherits from parent, false = does not inherit.

The initial-value Descriptor

Setting Defaults

initial-value sets the default value of the property.

@property --main-color { syntax: '<color>'; inherits: false; initial-value: #3498db; /* Default: blue */ } @property --spacing { syntax: '<length>'; inherits: true; initial-value: 20px; /* Default: 20px */ }

Always There: Even if you don't set the property, it will have this initial value!

Practical Example

All Three Descriptors

Here's a practical example of using the @property rule:

@property --main-color { syntax: '<color>'; inherits: false; initial-value: #3498db; } .button { background-color: var(--main-color); }

In this example: We're defining a custom property --main-color as a color value, setting it to not inherit, and giving it an initial value of #3498db.

Animation Power

Game Changer

One of the key benefits of the @property rule is that it allows for animation of custom properties, which wasn't possible before:

Revolutionary: Before @property, you couldn't animate custom properties. Now you can!

❌ Without @property

Custom properties can't be animated smoothly

✅ With @property

Custom properties animate smoothly!

Animating a Gradient

Smooth Transitions

@property --gradient-position { syntax: '<percentage>'; inherits: false; initial-value: 0%; } .gradient-box { background: linear-gradient( 90deg, red var(--gradient-position), blue 100% ); transition: --gradient-position 1s; } .gradient-box:hover { --gradient-position: 100%; }

The Result: The code above creates a gradient that smoothly animates when the element is hovered over, something that wasn't achievable with standard custom properties.

Working with Fallbacks

Graceful Degradation

Now, let's discuss how the @property rule works with fallbacks.

Why Fallbacks? Fallbacks are crucial in CSS to ensure that styles degrade gracefully in browsers that don't support certain features.

Two Levels: With the @property, fallbacks work on two levels: for the rule itself and for the use of the custom property.

Two Levels of Fallbacks

Double Protection

Level 1: Rule Fallback

For browsers that don't support @property at all

Level 2: Value Fallback

For when the custom property isn't defined or is invalid

Belt and Suspenders: Protect both against unsupported browsers AND undefined values!

Level 1: Browser Fallback

For Unsupported Browsers

For browsers that don't support the @property rule, you can provide a fallback by declaring the custom property in the traditional way:

/* Fallback: Traditional custom property */ :root { --main-color: #3498db; } /* Enhanced: @property rule */ @property --main-color { syntax: '<color>'; inherits: false; initial-value: #3498db; }

How it works: In this case, browsers that support @property will use the more strictly defined version, while others will fall back to the standard custom property declaration.

Level 2: Value Fallback

Using var() Fallbacks

When using the custom property, you can provide a fallback value using the var() function, just as you would with standard custom properties:

.button { background-color: var(--main-color, blue); }

Safety Net: This ensures that even if the --main-color property is not defined or is invalid, the button will still have a background color.

Complete Fallback Strategy

Belt, Suspenders, AND a Rope

/* Level 1: Traditional declaration (browser fallback) */ :root { --main-color: #3498db; } /* Enhanced: @property rule */ @property --main-color { syntax: '<color>'; inherits: false; initial-value: #3498db; } /* Level 2: var() fallback (value fallback) */ .button { background-color: var(--main-color, blue); }

Triple Protection: This covers all scenarios – unsupported browsers, undefined properties, and invalid values!

Type-Safe Fallbacks

Advanced Protection

The @property rule also allows for more complex fallback scenarios. For instance, you can use it to provide type-safe fallbacks:

@property --padding { syntax: '<length>'; inherits: false; initial-value: 10px; } .box { padding: var(--padding); }

In this case: If someone tries to set --padding to an invalid value, like a color, the browser will fall back to the initial value of 10px, maintaining type safety.

Type Safety in Action

Invalid Values Rejected

@property --spacing { syntax: '<length>'; inherits: false; initial-value: 20px; } /* This works */ .element { --spacing: 30px; margin: var(--spacing); /* Uses 30px */ } /* This is INVALID - falls back to initial-value! */ .other { --spacing: red; /* Not a length! */ margin: var(--spacing); /* Uses 20px (initial value) */ }

Protection: The browser automatically validates and rejects invalid types!

Browser Support Status

Current Compatibility

It's worth noting that the @property rule itself is not yet universally supported.

As of mid-2023: It's supported in Chrome, Edge, and Opera, with Firefox support behind a flag.

Important: For broader browser support, it's important to provide fallbacks and use feature detection.

Support Strategy:

  • Always provide traditional custom property fallback
  • Use @supports for feature detection
  • Test across multiple browsers

Feature Detection

Using @supports

/* Check if @property is supported */ @supports (background: var(--test)) { /* Modern browsers with @property support */ @property --fancy-gradient { syntax: '<percentage>'; inherits: false; initial-value: 0%; } .element { /* Use animated custom property */ transition: --fancy-gradient 1s; } } /* Fallback for older browsers */ .element { /* Static styles */ }

Progressive Enhancement: Enhance for modern browsers while maintaining basic functionality for all!

Best Practices

Using @property Wisely

Follow These Guidelines:

  • Always provide fallback custom properties
  • Use initial-value as a sensible default
  • Choose inherits based on the property's purpose
  • Be specific with syntax types when possible
  • Test across multiple browsers
  • Use for animation where it adds value

Real-World Use Cases

When to Use @property

Perfect For:

  • Animated Gradients – Smooth color transitions
  • Dynamic Themes – Type-safe color systems
  • Interactive Elements – Hover effects with animations
  • Design Systems – Validated design tokens
  • Complex Animations – Custom timing functions
  • Type Safety – Ensuring valid values

@property vs Standard Custom Properties

Key Differences

  • ✗ No type checking
  • ✗ Can't animate
  • ✗ No initial value

@property

  • ✓ Type checking
  • ✓ Can animate
  • ✓ Initial value

Conclusion

Significant Advancement

Key Benefits:

  • Enables new possibilities like animation

Remember: Use it judiciously and always provide appropriate fallbacks to ensure a good experience across all browsers.

The Key: As with any cutting-edge web technology, enhance the experience for browsers that support it while ensuring basic functionality for those that don't.
