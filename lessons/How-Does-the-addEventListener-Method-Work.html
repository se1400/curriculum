The addEventListener() Method

What are Events?

When a user clicks on a button or there is a change in a form, this is known as an event.

In your programs, you will need to have a way to listen for these events and respond to them.

The addEventListener() Method

The addEventListener() method is used to listen for events.

It takes two arguments: the event you want to listen for and a function that will be called when the event occurs.

Basic Syntax

element.addEventListener(event, listener);

Parameters

element: The HTML element to monitor for events

event: The type of event to listen for (as a string)

listener: A function to call when the event occurs

Event Types (Strings)

‚Ä¢ "click" - Click events

‚Ä¢ "input" - Input changes

‚Ä¢ "mouseover" - Mouse enters

‚Ä¢ "keydown" - Key pressed

‚Ä¢ "submit" - Form submitted

What is a Listener?

The listener is an object that will receive the notification when the event occurs.

Most of the time, this will be a function that you define to handle the event.

How It Works

1. Select an element
‚Üì
2. Add event listener with event type
‚Üì
3. Define what happens when event occurs
‚Üì
4. Function runs automatically when event fires

Ways to Define the Listener Function

Option 1: Arrow Function (Inline)

element.addEventListener("click", () => { // code to run when the click event occurs console.log("Button clicked!"); });

The function is defined directly as the second argument.

Option 2: Function Reference

function handleClick() { // code to run when the click event occurs console.log("Button clicked!"); } element.addEventListener("click", handleClick);

Create a separate function and pass its name (without parentheses!).

‚ö†Ô∏è Important Note

When passing a function reference, do NOT include parentheses:

// ‚úì CORRECT - passes the function element.addEventListener("click", handleClick); // ‚ùå WRONG - calls the function immediately element.addEventListener("click", handleClick());

Choosing Your Function Style

‚úì Use Arrow Functions When:

‚Ä¢ The code is short and simple

‚Ä¢ You only use it once

‚Ä¢ It's closely related to the element

button.addEventListener("click", () => { console.log("Clicked!"); });

‚úì Use Named Functions When:

‚Ä¢ The code is longer or complex

‚Ä¢ You'll reuse the function

‚Ä¢ You want better code organization

Why Use Separate Functions?

Sometimes you may want to create a separate function to handle the event. This can make your code easier to read and maintain.

Benefits:

‚úì Can reuse the function for multiple elements

‚úì Easier to test the function independently

‚úì Makes code more organized and readable

‚úì Can remove the listener later if needed

The EventListener Interface

Advanced Option (Rarely Used)

The listener argument can also be:

‚Ä¢ null

‚Ä¢ An object that uses the EventListener interface

What is the EventListener Interface?

The EventListener interface defines a single method called handleEvent().

This method is automatically called whenever the event you're listening for occurs.

Using this interface allows the same object to handle multiple events if needed.

Example (Advanced)

const eventHandler = { handleEvent(event) { console.log("Event type:", event.type); } }; // Can use the same object for multiple events button.addEventListener("click", eventHandler); button.addEventListener("mouseover", eventHandler);

üí° For Most Cases...

Most of the time, you won't need to use this interface, and you can instead pass in a function that will be called when the event occurs.

Stick with regular functions or arrow functions for simplicity!

Example 1: Click Event

Scenario: Show Alert on Button Click

Let's create a button that shows an alert when clicked.

HTML

<button id="btn">Show alert</button>

Goal

We want to listen for the "click" event on this button and show an alert when the button is clicked.

1 Access the Button Element

const btn = document.getElementById("btn");

2 Add Event Listener

const btn = document.getElementById("btn"); btn.addEventListener("click", () => { alert("You clicked the button"); });

Example 2: Input Event

Another Event Type: "input"

Another type of event that you can listen for is the "input" event.

This event is triggered when the value of an input element changes.

HTML

<input type="text" id="input" placeholder="Type something" />

Goal

We want to listen for the input event on this input element and log the value to the console each time the user types something.

1 Access the Input Element

const input = document.getElementById("input");

2 Add Event Listener

const input = document.getElementById("input"); input.addEventListener("input", (event) => { console.log(event.target.value); });

Other Common Events

Many More Event Types

There are many more events that you can listen for using the addEventListener() method.

Some common events include mouseover, mouseout, keydown, keyup, and submit.

mouseover

Fires when the mouse pointer enters an element

mouseout

Fires when the mouse pointer leaves an element

element.addEventListener( "mouseout", handler );

keydown

Fires when a key is pressed down

element.addEventListener( "keydown", handler );

keyup

Fires when a key is released

element.addEventListener( "keyup", handler );

submit

Fires when a form is submitted

form.addEventListener( "submit", handler );

focus

Fires when an element receives focus

input.addEventListener( "focus", handler );
