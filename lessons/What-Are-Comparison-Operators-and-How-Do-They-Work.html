Comparison Operators

What Are Comparison Operators?

Comparison operators allow you to compare two values and return a true or false result.

What Can You Do With Them?

You can use the result to make a decision or control the flow of your program.

Common Uses

You use comparisons in:

  • if statements - Making decisions
  • Loops - Controlling iterations
  • Conditionals - Checking conditions
  • Many other situations where you need to make decisions based on certain conditions

Greater Than Operator (>)

>

What It Does

The greater than operator, represented by a right-angle bracket (>), checks if the value on the left is greater than the one on the right.

Example:

let a = 6;
let b = 9;

console.log(a > b); // false
console.log(b > a); // true

Greater Than or Equal Operator (>=)

>=

What It Does

The greater than or equal operator, represented by a right-angle bracket and equals sign (>=), checks if the value on the left is either greater than OR equal to the one on the right.

Example:

let a = 6;
let b = 9;
let c = 6;

console.log(a >= b); // false
console.log(b >= a); // true
console.log(a >= c); // true
false true true

  • a >= c ‚Üí 6 >= 6 ‚Üí true (6 IS = 6)

Real-World Examples

Example 1: Age Verification

let age = 18;
let minimumAge = 18;

if (age >= minimumAge) {
  console.log("Access granted!");
} else {
  console.log("Access denied!");
}
// Output: Access granted!

Example 2: Grade Checking

let score = 85;

if (score >= 90) {
  console.log("Grade: A");
} else if (score >= 80) {
  console.log("Grade: B");
} else if (score >= 70) {
  console.log("Grade: C");
}
// Output: Grade: B

Example 3: Loop Control

for (let i = 0; i < 5; i++) {
  console.log(i);
}
// Output: 0, 1, 2, 3, 4
// Stops when i is NOT less than 5

Common Mistakes to Avoid

‚ùå Mistake 1: Using = Instead of ==

let x = 5;

// WRONG - This is assignment, not comparison!
if (x = 10) {
  console.log("This will always run!");
}

// CORRECT - Use comparison operators
if (x == 10) {
  console.log("This checks if x equals 10");
}

‚ùå Mistake 2: Confusing > with >=

let age = 18;

// WRONG for "18 and older"
if (age > 18) {
  console.log("Access granted");  // Won't run for 18!
}

// CORRECT for "18 and older"
if (age >= 18) {
  console.log("Access granted");  // Includes 18!
}

‚ùå Mistake 3: Wrong Operator Direction

let min = 10;
let max = 100;
let value = 50;

// Checking if value is between min and max
// CORRECT
if (value >= min && value <= max) {
  console.log("Value is in range");
}

Type Coercion and Comparisons

üí° Important Note

Unlike == and !=, the comparison operators >, >=, <, and <= will attempt to convert values to numbers before comparing.

Examples:

console.log(5 > "3");    // true (string "3" becomes number 3)
console.log("10" < 20);  // true (string "10" becomes number 10)
console.log("5" >= 5);   // true (string "5" becomes number 5)

String Comparison

When comparing strings, JavaScript uses lexicographical (alphabetical) order:

console.log("apple" < "banana");  // true
console.log("z" > "a");           // true
console.log("10" < "9");          // true (string comparison, not numeric!)

‚úì Best Practice

When comparing numbers, make sure both values are actually numbers to avoid confusion!
