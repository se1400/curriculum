What Are Style Sheets?

  • A style sheet specifies fonts, colors, spacing, and other characteristics for a website
  • Link every page to the style sheet instead of specifying styles in each document
  • Change your corporate typeface or color scheme by modifying one or two entries
  • Style sheets bring a sophisticated level of display to the Web

Style sheets enable you to set typeface controls, letter and line spacing, margins, page borders, and precisely position graphics and text anywhere on a web page.

How CSS Works

  • CSS defines style constructs: fonts, colors, positioning
  • Styles can be stored in HTML or a separate .css file
  • A style rule is a formatting instruction applied to an element
  • Internal style sheet - placed within a web page
  • External style sheet - exists in a separate document

An element is a piece of information in a web page, such as an image, paragraph, or link. Tags are used to mark up elements.

The Cascade

The cascading part of CSS refers to how style rules are applied. More specific styles override more general styles:

  1. Specificity - More specific selectors override less specific ones (ID overrides class, class overrides element)
  2. Source order - When specificity is equal, later rules override earlier rules
  3. Importance - Inline styles override internal styles, which override external styles

Think of cascading like genetic inheritance: general traits pass from parents to child, but more specific traits are unique to the child.

The Cascade in Action

A Basic Style Sheet

Pages can share visual properties through a common style sheet:

  • Large, bold Verdana font for headings
  • Normal Verdana font for body text
  • Black text with purple subheadings
  • Margins and vertical spacing
  • Centered footer in small print

Style Sheet Syntax

HTML tags in the style sheet are called selectors. Curly braces contain the style rules:

body {
  font: 10pt Verdana, Geneva, Arial, sans-serif;
  color: black;
  line-height: 14pt;
  padding: 5pt;
}

h1 {
  font: bold 14pt Verdana;
}
  • Each style rule ends with a semicolon (;)
  • Font names with spaces need quotation marks
  • List multiple fonts as fallbacks
  • If the first font isn't available, the browser tries the next

Linking CSS to HTML

Include a <link> tag in the <head> section:

<link rel="stylesheet" href="styles.css">
  • Assumes style sheet is in the same folder as HTML
  • All modern browsers support style sheets
  • Separates content from formatting

The real benefit: easily create a site with multiple pages that maintains a consistent look and feel. One change impacts all pages.

A CSS Style Primer

CSS style properties can be grouped into two major categories:

Layout Properties

Affect the positioning of elements on a web page:

  • Margins
  • Padding
  • Alignment

Formatting Properties

Affect the visual display of elements:

  • Font type
  • Font size
  • Color

Basic Layout Properties

The display Property

  • block - Element on a new line
  • list-item - New line with bullet
  • inline - Displayed inline with paragraph
  • inline-block - Block element inline with surroundings
  • none - Element is hidden

The display property also has powerful values like flex, grid, and table for advanced layout functionality.

display: block;

CSS Units of Measurement

Fixed Lengths

  • in - Inches
  • cm - Centimeters
  • mm - Millimeters
  • px - Pixels
  • pt - Points (72 per inch)

Relative Lengths

  • em - Relative to parent font-size
  • rem - Relative to root font-size
  • % - Percentage of parent
  • vw, vh - 1% of viewport width/height
  • ch - Width of the character "0"

Best practices recommend relative font sizes (like rem) so designs scale gracefully and remain accessible.

Basic Formatting Properties

Border Properties

  • border-width - Width of the border
  • border-color - Color of the border
  • border-style - Style (solid, dotted, dashed, etc.)
  • border - All border properties at once
  • border-radius - Corner curves

Border Style Values

  • solid, double, dashed, dotted
  • groove, ridge, inset, outset
  • none, hidden
border: 10px double red;
border-radius: 1rem;

Color Properties

  • color - Sets text color (foreground)
  • background-color - Sets background color

Color Notations

  • Named colors: black, orange
  • Hexadecimal: #999999
  • RGB: rgb(0, 0, 255)
  • RGBa: rgba(0, 0, 0, 0.8)

HSL Colors

HSL separates color (hue) from lightness and saturation:

/* Pure blue */
background-color: hsl(240, 100%, 50%);

/* Light green, 80% opacity */
color: hsla(120, 60%, 70%, 0.8);

CSS Custom Properties (Variables)

Store values to reuse throughout your stylesheet:

:root {
  --primary-color: #593d87;
  --secondary-color: #8094d6;
  --base-spacing: 1rem;
  --heading-font: Verdana, Geneva, Arial, sans-serif;
}

h1 {
  color: var(--primary-color);
  font-family: var(--heading-font);
  margin-bottom: var(--base-spacing);
}
  • Defined using two dashes (--) followed by a name
  • Accessed using the var() function
  • Makes maintenance easier
  • Ensures consistency across design
  • Can define globally in :root or scope to specific elements

Logical Properties for Modern Layouts

Logical properties are relative to writing mode and text direction:

  • margin-inline-start instead of margin-left
  • padding-block-end instead of padding-bottom
  • border-inline for horizontal borders
  • text-align: start instead of text-align: left
/* Traditional */
.card {
  margin-left: 1rem;
  padding-right: 2rem;
}

/* Logical */
.card {
  margin-inline-start: 1rem;
  padding-inline-end: 2rem;
}

Logical properties provide better support for right-to-left languages and internationalization.

Responsive Design Considerations

Modern websites need to work on devices of all sizes. Key concepts:

  • Relative units (like rem, em, %, vw, vh) scale better than fixed units like pixels
  • Media queries allow different styles based on screen width, device capabilities, and user preferences
  • Flexible layouts using percentages, Flexbox, or Grid adapt naturally to different container sizes
  • Always test your designs on multiple devices and screen sizes

Build responsive design thinking into your style sheets from the start to ensure great experiences for all users.

Understanding Cascade Layers

The @layer rule gives explicit control over cascade order:

@layer reset, base, components, utilities;

@layer reset {
  * { margin: 0; padding: 0; }
}

@layer base {
  body { font-family: sans-serif; }
}

@layer components {
  .button { padding: 1rem; }
}
  • Layers cascade in the order declared
  • Later layers take precedence
  • Styles outside any layer have highest precedence
  • Useful for organizing larger projects

Using Style Classes

A style class is a custom set of formatting specifications that can be applied to any element:

h1.silly { font: 36pt Comic Sans; }
h1.serious { font: 36pt Arial; }

Use the class attribute in HTML:

<h1 class="silly">Marvin's Munchies</h1>
<h1 class="serious">MMI Investor Info</h1>
  • Selector includes a period (.) followed by class name
  • Classes are reusable across elements
  • Can create classes for any element or use . alone for any tag
p.subheader {
  font-weight: bold;
  color: #593d87;
}

Using Style IDs

Style IDs are similar to classes but use a hash mark (#) and can only be used once per page:

#header {
  font-size: 20pt;
  font-family: Verdana;
}

#footer {
  font-size: 8pt;
  text-align: center;
}

Apply with the id attribute:

<header id="header">
  ...
</header>

<footer id="footer">
  ...
</footer>

IDs have higher specificity than classes. Best practice is to use classes for styling and IDs for JavaScript hooks or anchor links.

Internal Style Sheets

Place styles directly in the HTML document using the <style> tag:

<head>
  <style>
    body {
      font-family: Arial;
    }
    .highlight {
      background: yellow;
    }
  </style>
</head>

When to Use

  • Single-page websites
  • Page-specific styles
  • Quick prototyping

Drawback: Styles can't be cached separately or shared across pages.

Inline Styles

Apply styles directly to an element using the style attribute:

<p style="color: red; font-size: 18px;">
  Styled inline
</p>

When to Use

  • Dynamic styles via JavaScript
  • One-off exceptions
  • Email HTML (required by some clients)

Avoid When Possible

  • Mixes content and presentation
  • Harder to maintain
  • Can't reuse styles
  • Overrides everything (high specificity)

Summary

Key Concepts

  • Style sheets specify fonts, colors, spacing for a website
  • CSS rules cascade: specific styles override general ones
  • External style sheets allow consistent styling across pages
  • Classes (.class) are reusable
  • IDs (#id) are unique per page

Modern CSS Features

  • Custom properties (CSS variables)
  • Logical properties for internationalization
  • Cascade layers for organizing large projects
  • Responsive design with relative units
