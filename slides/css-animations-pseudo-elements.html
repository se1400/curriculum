Overview

Transitions → Animations

Transitions — Reactive

  • Triggered by a state change (hover, focus, class toggle)
  • A → B only (two states)
  • Cannot loop, cannot auto-start
  • Best for: hover effects, interactive feedback

Animations — Proactive

  • Start automatically — on load, or when class is added
  • Any number of keyframe stops
  • Can loop, reverse, pause/resume
  • Best for: spinners, entrance effects, looping motion

When to choose animations

  • Element should animate on page load without user action
  • Effect should loop (spinner, pulse, marquee)
  • Motion passes through three or more states
  • Animation needs to reverse, hold end state, or pause
Before CSS animations: developers used JavaScript setInterval loops or Flash. CSS animations move that rendering logic where it belongs — the stylesheet.
@keyframes

Defining Animations with @keyframes

The from / to Shorthand

from = 0%  |  to = 100%

@keyframes fadeIn {
  from { opacity: 0; }
  to   { opacity: 1; }
}

.hero {
  animation: fadeIn 0.6s ease-out;
}

When the browser sees .hero, it immediately interpolates opacity from 0 to 1 over 0.6 seconds.

Give keyframes descriptive, intention-revealing names: slideInLeft, pulseRed, spin. camelCase and kebab-case are both valid.

@keyframes

Percentage-Based Keyframes

Use percentages for 3+ states. Percentages = progress through the cycle, not clock time.

@keyframes bounce {
  0%   { transform: translateY(0); }
  30%  { transform: translateY(-40px); }
  60%  { transform: translateY(-20px); }
  80%  { transform: translateY(-10px); }
  100% { transform: translateY(0); }
}

If duration = 2s, the 50% keyframe occurs at exactly 1 second.

You can share a keyframe stop across multiple percentages:
0%, 100% { opacity: 1; }
You can also mix from/to with percentage stops in the same rule.

▶ 5-stop bounce animation

The 5 keyframe stops create the realistic squash-and-stretch feel. Two stops would just go up and come straight back down.

Animation Properties

animation-delay

Positive delay — wait before starting.

Negative delay — start mid-cycle as if already running. This is the stagger trick:

.card:nth-child(1) { animation-delay: 0s; }
.card:nth-child(2) { animation-delay: 0.1s; }
.card:nth-child(3) { animation-delay: 0.2s; }

Negative delay for loopers

animation-delay: -1s on a 3s animation → starts at the 1-second mark. Items in a loading-dots row can all loop at different phases so they're never all in the same position.

Animation Properties

animation-fill-mode

What styles apply before the animation starts and after it ends? By default: none — element snaps back.

ValueBehavior
noneUses normal styles before & after — snaps back
forwardsHolds the 100% keyframe styles after ending
backwardsApplies 0% styles during the delay period
bothCombines forwards + backwards — most commonly needed
 
/* The most reliable pattern */
.notification {
  animation: slideInUp 0.4s
             ease-out both;
}

The snap-back bug is the #1 beginner animation frustration. Always add fill-mode: both to animations that slide, fade, or transform elements into their final position.
Animation Properties

animation-play-state

Switches a running animation between running and paused. Useful for accessibility and interactive media.

/* CSS: pause on hover */
.spinner:hover {
  animation-play-state: paused;
}

/* JS: toggle pause */
el.style.animationPlayState
  = 'paused';   // pause
el.style.animationPlayState
  = 'running';  // resume
Resuming from paused continues from exactly where it stopped — it does not restart from the beginning.

Shorthand

The animation Shorthand

Combines all 8 sub-properties. Only name and duration are required.

/* Full form (for learning): */
animation-name:             slideInUp;
animation-duration:         0.5s;
animation-timing-function:  ease-out;
animation-delay:            0.2s;
animation-iteration-count:  1;
animation-direction:        normal;
animation-fill-mode:        both;
animation-play-state:       running;

/* Shorthand equivalent: */
animation: slideInUp 0.5s ease-out 0.2s 1 normal both running;

/* Most common simplified form: */
animation: slideInUp 0.5s ease-out 0.2s both;

Order mnemonic:
name duration timing delay count direction fill state

⚠ Two time values: The first time value is always duration, the second is always delay.
animation: spin 1s 0.5s = duration 1s, delay 0.5s — not the other way around.
For multiple animations, comma-separate the entire shorthand:
animation: floatUp 3s ease-in-out infinite, colorShift 6s linear infinite;
Combining with Transforms

Combining Animations with transform

Animations are most powerful with transform + opacity. Multiple transform functions combine at each keyframe stop.

@keyframes launch {
  0% {
    transform: translateY(0)
               scale(1)
               rotate(0deg);
    opacity: 1;
  }
  50% {
    transform: translateY(-60px)
               scale(1.1)
               rotate(-5deg);
    opacity: 1;
  }
  100% {
    transform: translateY(-200px)
               scale(0.8)
               rotate(10deg);
    opacity: 0;
  }
}

.rocket {
  animation: launch 1.2s ease-in forwards;
}
Order matters! translateY(-60px) scale(1.1) first moves, then scales — swapping them changes the result.

Three transform functions run simultaneously: translateY moves up, scale shrinks it, rotate adds a tilting wobble — all in a single keyframe.

Multiple Animations

Layering Multiple Animations

Comma-separate names (and all their corresponding values) to stack animations on one element. Each runs on its own independent timeline.

@keyframes floatUp {
  0%, 100% { transform: translateY(0); }
  50%      { transform: translateY(-12px); }
}
@keyframes colorShift {
  0%   { color: #4f6df5; }
  50%  { color: #ec4899; }
  100% { color: #4f6df5; }
}

.floating-title {
  animation:
    floatUp    3s ease-in-out infinite,
    colorShift 6s linear     infinite;
}
/* floatUp completes 2 cycles
   while colorShift completes 1 */
Values map by position: first duration → first animation, second duration → second animation, and so on.

Performance

Animation Performance

The browser's rendering pipeline — animating the wrong properties forces expensive work on every frame:

🎨 Style
Compute styles
→
📐 Layout
width · height · margin · top/left
Triggers reflow!
→
🖌 Paint
background · color · box-shadow
Triggers repaint
→
⚡ Composite
transform · opacity
GPU only ✓

❌ Avoid — triggers layout every frame

@keyframes moveLeft-bad {
  from { left: 0; }
  to   { left: 300px; }
}

✓ Prefer — GPU compositor only

@keyframes moveLeft-good {
  from { transform: translateX(0); }
  to   { transform: translateX(300px); }
}
Gold standard: animate only transform and opacity. They bypass layout and paint entirely, handing work to the GPU.
Performance

The will-change Property

A hint to the browser that this element is about to be animated — promotes it to its own GPU layer before the animation starts, eliminating the first-frame startup cost.

.animated-card {
  will-change: transform, opacity;
}

/* Remove the hint once done (JS) */
card.addEventListener('animationend',
  () => {
    card.style.willChange = 'auto';
  }
);
⚠ Frequently misused. Do not apply it to every element as a blanket optimization. Each promoted layer consumes GPU memory. Only use it where you have measured a real performance problem, and always remove it after the animation completes.

When to use will-change

  • Element animates immediately on page load (hero entrance)
  • High-frequency animation (real-time game, particle effect)
  • You've profiled and confirmed a jank issue

When NOT to use it

  • On hover (too early — element may never animate)
  • On every card, image, or button as a "precaution"
  • On static elements
The browser already does a good job of promoting animated elements automatically. will-change is an override, not a default.
Accessibility

Respecting prefers-reduced-motion

Many people experience dizziness or nausea from on-screen motion. OS settings expose this preference to CSS:

OS settings: Reduce Motion on macOS/iOS, Remove Animations on Android, Show Animations toggle on Windows.
Never skip this check for looping animations. A continuously spinning element is one of the most commonly reported accessibility barriers on the web.
/* Default: full animation */
.hero-text {
  animation: slideInUp 0.6s ease-out both;
}

/* Reduced motion: keep fade,
   remove spatial movement */
@media (prefers-reduced-motion: reduce) {
  .hero-text {
    animation: fadeIn 0.2s ease both;
  }
}

/* Nuclear option: disable everything */
@media (prefers-reduced-motion: reduce) {
  * {
    animation-duration: 0.01ms !important;
    animation-iteration-count: 1 !important;
  }
}

Modern CSS

Scroll-Driven Animations

Traditionally, scroll animations required JavaScript — attaching a scroll event listener, computing scroll position, updating styles on every frame.

animation-timeline brings this natively to CSS (Chrome 115+, 2023):

  • scroll() — ties animation to scroll progress of a scrollable container
  • view() — triggers animation based on how much the element is visible in the viewport
Browser support (early 2025): Chrome & Edge only. Not yet in Firefox or Safari stable. Check caniuse.com
Always wrap in: @supports (animation-timeline: scroll()) { }

The old way (JavaScript)

window.addEventListener('scroll', () => {
  const progress = window.scrollY
    / document.body.scrollHeight;
  progressBar.style.width
    = (progress * 100) + '%';
});

CSS scroll-driven animations eliminate this pattern entirely — no JS, no event listeners, no performance risk of heavy scroll handlers.

  (animation-timeline: view()) {
  .card { opacity: 1; transform: none; }
}
Pseudo-elements

Pseudo-elements

Pseudo-elements style a part of an element, or generate new content — without touching HTML. They use double-colon syntax to distinguish from pseudo-classes:

SelectorTypeWhat it does
::beforepseudo-elementInserts generated child at start of content
::afterpseudo-elementInserts generated child at end of content
::placeholderpseudo-elementStyles input placeholder text
::selectionpseudo-elementStyles highlighted/selected text
::markerpseudo-elementStyles list bullets or numbers
::first-letterpseudo-elementTargets the first rendered character
:hoverpseudo-classState — not a pseudo-element

Three key rules

  • ::before/::after MUST have a content property — without it, the browser won't render them at all (use content: "" for pure shapes)
  • Generated content is not in the DOM — invisible to most screen readers — ideal for decorative additions only
  • Positioning pseudo-elements absolutely requires position: relative on the parent
Older browsers support single-colon :before and :after. Most still accept it, but the double-colon :: is the modern standard.

Practical Patterns

Pattern: CSS Spinner / Loading Indicator

A pure-CSS spinner uses a border and rotate animation — no images, no JavaScript.

<div class="spinner"
     aria-label="Loading"
     role="status"></div>

.spinner {
  width: 40px; height: 40px;
  border: 4px solid #e2e8f0; /* track */
  border-top-color: #4f6df5; /* segment */
  border-radius: 50%;
  animation: spin 0.8s linear infinite;
}

@keyframes spin {
  to { transform: rotate(360deg); }
}

/* Accessibility: respect motion prefs */
@media (prefers-reduced-motion: reduce) {
  .spinner { animation: none; }
}
Accessibility: role="status" and aria-label="Loading" are required — screen readers need these to announce that loading is in progress.

Practical Patterns

Pattern: Staggered List Entrance

Give each item a slightly later animation-delay for a cascading reveal that draws the eye down the list.

@keyframes itemIn {
  from {
    opacity: 0;
    transform: translateX(-20px);
  }
  to {
    opacity: 1;
    transform: translateX(0);
  }
}

.menu-item {
  animation: itemIn 0.4s ease-out both;
}

/* Each child gets a slightly later start */
.menu-item:nth-child(1) { animation-delay: 0s; }
.menu-item:nth-child(2) { animation-delay: 0.05s; }
.menu-item:nth-child(3) { animation-delay: 0.1s; }
.menu-item:nth-child(4) { animation-delay: 0.15s; }
.menu-item:nth-child(5) { animation-delay: 0.2s; }
Combine with fill-mode: both so items start invisible (during their delay) and hold their final position after appearing.

Practical Patterns

Pattern: Tooltip Arrow with ::before

The CSS Border Triangle Trick

A triangle is drawn by making an element's width and height 0, and coloring only one border — adjacent borders become transparent, forming a triangle.

.tooltip::after {
  content: "";
  position: absolute;
  top: 100%; /* below the tooltip */
  left: 50%;
  transform: translateX(-50%);
  border-width: 6px;
  border-style: solid;
  border-color:
    #1e293b  /* top    — visible */
    transparent  /* right  — hidden */
    transparent  /* bottom — hidden */
    transparent; /* left   — hidden */
  /* Result: ▼ pointing down */
}
Rotate the visible border side to point in any direction: border-bottom-color for ▲ pointing up, border-right-color for ◀ pointing left.

Q & A

Q & A

Q. What is the difference between a CSS transition and a CSS animation?
A. A transition reacts to a state change — it only plays when a property changes (hover, focus, class toggle). An animation is proactive: it starts automatically, can loop, reverse, and pass through any number of keyframe stops. Transitions are best for interactive feedback; animations are best for motion that needs to play on its own schedule.
Q. My animation runs once and the element snaps back to its original style. How do I fix it?
A. Add animation-fill-mode: forwards (or both if you also have a delay and want starting styles applied immediately). forwards tells the browser to hold the styles from the last keyframe after the animation completes.
Q. Can I animate a pseudo-element independently from the element itself?
A. Yes. Attach an animation directly to the pseudo-element selector: .card::before { animation: spin 1s linear infinite; }. The pseudo-element's animation runs entirely separately from any animation on .card.
Q & A

Q & A (continued)

Q. Do screen readers read content generated by ::before and ::after?
A. Browser behavior is inconsistent, but most modern screen readers do read generated content. However, generated content should only be used for purely decorative purposes. Any meaningful text belongs in your HTML. Use content: "" / "" (the second empty string is an accessibility alt-text value) to explicitly hide decorative pseudo-content from assistive technology.
Q. How does a negative animation-delay work?
A. A negative delay starts the animation as if it already started that many seconds ago. animation-delay: -1s on a 3-second animation begins playback at the 1-second mark. This technique is used to stagger looping animations so they do not all start at 0% simultaneously — useful for a wave of loading dots.
Q. What does animation-timeline: view() actually measure?
A. It measures the element's progress through the viewport — specifically, how much of the element has entered the scrollport. The animation's 0% keyframe corresponds to when the element begins entering the viewport from below, and 100% when it has fully entered. The animation-range property with entry and exit keywords lets you target just the entering or just the exiting phase.
