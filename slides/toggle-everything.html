Core Concept

The Central Principle

CSS does all the visual work. JavaScript just flips a switch.

The golden rule: JavaScript does not create visual effects. CSS does. JavaScript's only job is to add or remove a class name. The CSS transitions you already know handle the rest.

CSS defines two states

/* State 1: Default — hidden */
.panel {
  max-height: 0;
  overflow: hidden;
  transition: max-height 0.4s ease;
}

/* State 2: Open — visible */
.panel.open {
  max-height: 500px;
}

JS toggles between them

panel.classList.toggle('open');

That's it. One line. The CSS transition fires automatically the moment the class changes.

Why? This is called separation of concerns. CSS handles appearance. JS handles state. Your code stays clean, easy to debug, and easy to hand off to another developer.
Core Concept

One Pattern Powers Everything

This exact same pattern — CSS defines two states, JS toggles the class — powers almost every interactive element you've ever seen:

🌙 Dark Mode

Toggle .dark-mode on body — CSS custom properties change every color at once

📂 Accordion

Toggle .open on a panel — max-height transitions reveal content smoothly

☰ Hamburger Menu

Toggle .nav-open on nav — slides in from the side on mobile

🗂 Tabs

Toggle .active on tab + panel — one visible at a time

💬 Modal Dialog

Toggle .open on overlay — fades in over the page

🔔 Toast Notification

Toggle .visible on toast — slides up, auto-dismisses with setTimeout

Learn this pattern once → build everything.
Reference

classList Methods

These are the four methods you'll use to manage CSS classes from JavaScript:

MethodWhat it doesExample
.toggle('name') Adds the class if missing, removes it if present el.classList.toggle('open')
.add('name') Adds the class (no effect if already there) el.classList.add('active')
.remove('name') Removes the class (no effect if not there) el.classList.remove('open')
.contains('name') Returns true or false el.classList.contains('dark-mode')

Dark Mode Toggle

Dark Mode — The CSS Side

Dark mode uses CSS custom properties with two selectors — the default and a .dark-mode class on <body>:

Define two color schemes

/* Light mode (default) */
:root {
  --bg:   #ffffff;
  --text: #111111;
  --card: #f5f5f5;
}

/* Dark mode (toggled by JS) */
body.dark-mode {
  --bg:   #1a1a2e;
  --text: #eeeeee;
  --card: #16213e;
}

Use the variables everywhere

body {
  background: var(--bg);
  color: var(--text);
  transition: background 0.3s,
              color 0.3s;
}
Why custom properties? Because --bg, --text, and --card cascade to every element using them, that single class toggle updates the entire page at once.
Dark Mode Toggle

Dark Mode — The JavaScript

One line of JavaScript toggles the entire page's color scheme: 
document.body.classList.toggle('dark-mode');

Update the button label with the ternary operator:

const isDark = document.body.classList.contains('dark-mode');
btn.textContent = isDark ? '☀ Light Mode' : '☾ Dark Mode';

Accordion

Accordion — Expand & Collapse

An accordion is a list of sections where clicking a heading reveals or hides its content.

Why use an accordion?

  • Reduce visual clutter — show content only when needed
  • FAQ pages, settings panels, product features
  • Used on nearly every website
  • The max-height trick makes it animate smoothly

The max-height trick

You can't transition height: auto. But you can transition max-height from 0 to a value larger than the content. The browser animates the change smoothly.

Why not display: none? Because display can't be transitioned. The element just pops in/out with no animation. max-height: 0 + overflow: hidden gives you a smooth reveal.
Accordion

Accordion — The Code

CSS — Two states

.accordion-body {
  max-height: 0;
  overflow: hidden;
  transition: max-height 0.35s ease;
}

.accordion-item.open .accordion-body {
  max-height: 400px;
}

JavaScript — Toggle + close others

headers.forEach(function(header) {
  header.addEventListener('click', function() {
    const item = header.parentElement;

    // Close all others
    document.querySelectorAll('.accordion-item')
      .forEach(function(el) {
        if (el !== item)
          el.classList.remove('open');
      });

    // Toggle this one
    item.classList.toggle('open');
  });
});
Accessibility: Always toggle aria-expanded on the heading button — screen readers announce "expanded" or "collapsed" as users navigate.

Hamburger Menu

Real Hamburger Menu

Replace the CSS-only checkbox hack with clean, readable JavaScript.

The old way (CSS-only hack)

  • Hidden checkbox + :checked pseudo-class
  • Uses the sibling selector (~) to show/hide nav
  • Clever but hard to read and maintain
  • No way to update aria-expanded

The right way (JavaScript)

  • Toggle a class on the nav element
  • CSS handles the slide-in animation
  • JavaScript updates button label and aria-expanded
  • Cleaner, more readable, accessible
Same visual result. Better code. Better accessibility. The checkbox hack was a creative workaround. Now that you know JavaScript, you can do it properly.
Hamburger Menu

Hamburger Menu — The Code

CSS — slide from the right

.nav-links {
  position: fixed;
  top: 0;
  right: -100%;
  width: 75%;
  height: 100vh;
  background: var(--card);
  transition: right 0.3s ease;
  padding-top: 5rem;
}

.nav-links.nav-open {
  right: 0;
}

JavaScript — three lines

menuBtn.addEventListener('click',
  function() {
    navLinks.classList.toggle('nav-open');

    const isOpen = navLinks
      .classList.contains('nav-open');

    menuBtn.setAttribute(
      'aria-expanded', isOpen
    );

    menuBtn.textContent =
      isOpen ? '✕' : '☰';
});

Tabs

Tab System

Shows one content panel at a time. Clicking a tab activates it and hides the rest.

The pattern: Remove .active from all, add to the one that was clicked

tabs.forEach(function(tab) {
  tab.addEventListener('click', function() {
    // Deactivate all tabs and panels
    tabs.forEach(function(t)   { t.classList.remove('active'); });
    panels.forEach(function(p) { p.classList.remove('active'); });

    // Activate clicked tab + matching panel
    tab.classList.add('active');
    const targetId = tab.dataset.tab;
    document.querySelector('#' + targetId).classList.add('active');
  });
});
Connection to data-* attributes: The data-tab attribute on each button stores the ID of the panel it controls. This is the same dataset pattern from the previous lessons.

Modal Dialog

Modal Dialog

An overlay that appears on top of everything, requiring the user's attention before continuing.

When to use a modal

  • Confirmation dialogs ("Are you sure?")
  • Login/signup forms
  • Image lightboxes
  • Cookie consent banners

Three ways to close it

  • Click the ✕ button
  • Click the dark backdrop
  • Press the Escape key

Why opacity instead of display: none?

display: none removes the element from layout instantly — there is nothing to transition. The element just pops in/out.
opacity: 0 + pointer-events: none keeps the element present but invisible and non-interactive. Now you can transition opacity for a smooth fade in/out.
Modal Dialog

Modal — The Code

CSS — hidden by default

.modal-overlay {
  position: fixed;
  inset: 0;
  background: rgba(0,0,0,0.6);
  display: flex;
  align-items: center;
  justify-content: center;
  opacity: 0;
  pointer-events: none;
  transition: opacity 0.2s;
}

.modal-overlay.open {
  opacity: 1;
  pointer-events: auto;
}

JavaScript

function openModal() {
  overlay.classList.add('open');
}
function closeModal() {
  overlay.classList.remove('open');
}

// Close on backdrop click
overlay.addEventListener('click',
  function(event) {
    if (event.target === overlay)
      closeModal();
  });
Why event.target === overlay? Clicking inside the dialog box also fires the overlay's click handler (event bubbling). This check ensures we only close when the backdrop itself is clicked, not the dialog content.

Timing

setTimeout — Delayed Actions

Schedule a function to run once after a delay (in milliseconds).

The syntax

setTimeout(function() {
  // This runs after 3 seconds
  toast.classList.remove('visible');
}, 3000);  // 3000ms = 3 seconds

Common use: Toast notification

function showToast(message) {
  toast.textContent = message;
  toast.classList.add('visible');

  setTimeout(function() {
    toast.classList.remove('visible');
  }, 3000);
}

Cancel with clearTimeout

const id = setTimeout(myFunc, 5000);
// User dismisses manually — cancel timer
clearTimeout(id);

Timing

setInterval — Repeated Actions

Schedule a function to run repeatedly at a fixed interval.

The syntax

const clockId = setInterval(function() {
  updateClock();
}, 1000);  // Every 1 second

Always cancel when done

clearInterval(clockId);
Memory leak warning: A running interval that is never cancelled keeps executing even after the user navigates away. Every setInterval must have a matching clearInterval.

Classic use: Live clock

function updateClock() {
  const now = new Date();
  const h = String(now.getHours())
    .padStart(2, '0');
  const m = String(now.getMinutes())
    .padStart(2, '0');
  const s = String(now.getSeconds())
    .padStart(2, '0');
  clock.textContent =
    h + ':' + m + ':' + s;
}

Accessibility

aria-expanded — Accessible Toggles

One line of JavaScript makes your interactive components usable by everyone.

What it does

When a button controls a collapsible section, aria-expanded tells screen readers whether the content is currently visible or hidden.

  • aria-expanded="true" → screen reader says "expanded"
  • aria-expanded="false" → screen reader says "collapsed"

Where to use it

  • Accordion headers
  • Hamburger menu buttons
  • Dropdown toggles
  • Any button that shows/hides content

The code (one line)

// After toggling the class:
const isOpen = el.classList.contains('open');
btn.setAttribute(
  'aria-expanded', isOpen
);

In the HTML

<button
  aria-expanded="false"
  aria-controls="panel-1">
  Section Title
</button>
Tiny effort, huge impact. Not everyone uses a mouse. Users with visual impairments rely on screen readers to navigate. One attribute makes your components work for them.
Best Practices

Best Practices

Let CSS do the animating
Define both states in CSS with a transition. JavaScript only adds or removes the class. Never write animation timings in JavaScript.
Toggle aria-expanded on interactive components
Buttons that show/hide content need aria-expanded="true" when open and "false" when closed. One line makes your components accessible.
Always clear intervals you no longer need
Every setInterval should have a corresponding clearInterval. Forgetting this is a common source of subtle bugs and memory leaks.
Use opacity + pointer-events for animated show/hide
display: none makes transitions impossible. Use opacity: 0; pointer-events: none; as the hidden state.
Keep modal close logic in one function
Multiple triggers close the same modal (✕ button, backdrop click, Escape key). One closeModal() function called by all three is cleaner than duplicating logic.
