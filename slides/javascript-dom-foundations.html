Setup

How JavaScript Fits Into Your Page

JavaScript lives in .js files and connects via the <script> tag. Place it in <head> with the defer attribute:

<head>
  <meta charset="UTF-8">
  <link rel="stylesheet" href="styles.css">
  <!-- defer = wait for full HTML -->
  <script defer src="script.js"></script>
</head>
defer — downloads the script in the background, then runs it after the entire HTML has loaded. Your JavaScript will always have access to every element on the page.

Inline scripts (for CodePen)

<script>
  // Code runs directly in the page
  console.log('Hello!');
</script>
Avoid (legacy patterns):
document.write() — can corrupt your page
alert() — intrusive dialog boxes
Instead, always update elements in the DOM.

JavaScript Syntax Basics

Statements & Semicolons

const name = 'Ada Lovelace';
document.querySelector('h1').textContent = name;
console.log(name);

One instruction per line. Semicolons prevent subtle bugs — make them a habit.

Comments

// Single-line comment

   Explain complex logic here. */

Case Sensitivity & Naming

// These are THREE different variables:
const myVariable = 1;
const MyVariable = 2;
const MYVARIABLE = 3;
Convention: camelCase
First word lowercase, each new word capitalized.
✅ accentColor, userName, isLoggedIn
❌ accent_color, c, x1
  • Names can contain letters, numbers, _, $
  • Must start with a letter, _, or $ — not a number
  • Cannot use reserved words: const, let, if…
Variables

Variables: const and let

const — values that won't change

const siteName = 'Portfolio';
const primaryColor = '#2d3436';
const maxScore = 100;

// ❌ This throws an error:
siteName = 'New Name';

Your default choice. The error protection is a feature.

let — values that will change

let score = 0;
score = score + 10; // ✅ works

let activeTab = 'home';
activeTab = 'about';   // ✅ works

Use when you need to update a value over time.

Practical rule: start everything with const. If you get an assignment error, change it to let. This catches accidental overwrites.
Avoid var — it's the old way (pre-2015) with confusing scope rules. You'll see it in legacy tutorials. Treat it like let but don't use it yourself.

Data Types

The Three Data Types You'll Use Most

📝 Strings

const title = 'JavaScript';
const name  = "Grace Hopper";

Any text. Use single quotes for most strings, backticks for template literals.

🔢 Numbers

const age  = 28;
const price = 49.99;
const temp = -5;
const rate = 0.08;
// Math works directly:
price * (1 + rate); // 53.99

No quotes needed. Integers and decimals are the same type.

💡 Booleans

const isLoggedIn = true;
const hasSubmitted = false;
const isDarkMode = true;
// Used in decisions:
if (isLoggedIn) { /*...*/ }

A light switch — only true or false.

Template Literals

What does "literal" mean?
A literal is a value written directly in your code. 42 is a number literal — you typed the actual number. 'hello' is a string literal — you typed the actual text. A template literal is just a special kind of string that has fill-in-the-blank slots built in.

The Mad Libs analogy: imagine a sentence with blanks — "Hello, ___! You are a ___." You write the fixed text once, and JavaScript fills in the blanks automatically.

const name = 'Sofia';
const role = 'UX Designer';

// ❌ Old way — glued together with +
const msg = 'Hello, ' + name +
  '! Your role is ' + role + '.';

// ✅ Template literal — read like English
const msg = `Hello, ${name}! Your role is ${role}.`;
// → "Hello, Sofia! Your role is UX Designer."

Three rules: use backticks `` ` `` (not quotes), put variables inside ${}, and line breaks are preserved automatically.

// Great for building HTML strings:
const card = `
  <div class="card">
    <h3>${name}</h3>
    <p class="role">${role}</p>
  </div>
`;
Feels familiar? The ${} syntax is the same idea as CSS var(--color) — a slot that says "insert a value here." You've already been thinking this way.
The ${} slot runs JavaScript.
You can put any expression inside — not just variable names:
`Price: $${(9.99 * 1.08).toFixed(2)}`
→ "Price: $10.79"

Math

Math: Random Numbers

// Random decimal: 0 to <1
Math.random()   // 0.4823...

// Round DOWN to whole number
Math.floor(4.9) // → 4
Math.floor(0.9) // → 0

// Random integer 0–359 (HSL hue)
const hue = Math.floor(Math.random() * 360);

// Dice roll: 1–6
const roll = Math.floor(Math.random() * 6) + 1;
Pattern: Math.floor(Math.random() * N) gives a random integer from 0 to N-1. Add 1 if you want 1 to N.

Here's how the dice roll works end-to-end — HTML, then JavaScript:

<!-- HTML -->
<button id="roll-btn">🎲 Roll</button>
<p id="result">?</p>
// JavaScript
const btn    = document.querySelector('#roll-btn');
const result = document.querySelector('#result');

btn.addEventListener('click', function() {
  const roll = Math.floor(Math.random() * 6) + 1;
  result.textContent = roll;
});

The DOM

The DOM: Your Page as a Data Structure

When the browser loads HTML, it builds the Document Object Model (DOM) — a tree of connected objects in memory. Every element is a node JavaScript can find and change.

<!-- Your HTML -->
<div class="card">
  <h3 id="title">My First Card</h3>
  <p>A description.</p>
</div>
The fundamental pattern:
1. Find an element in the DOM
2. Change it (content, style, classes, attributes)
3. The page updates instantly

The DOM

Selecting Elements with querySelector

querySelector() is like a search bar for your page — you hand it the exact same CSS selector syntax you already write in stylesheets, and it returns that element so JavaScript can work with it.

Step 1 — Your HTML:

<h2 id="page-title">Products</h2>

<div class="card">Sneakers</div>
<div class="card">T-Shirt</div>
<div class="card" data-flavor="mint">Hat</div>
<span>Sale ends tonight</span>

Step 2 — Your JavaScript:

// Same selectors you use in CSS!
document.querySelector('#page-title')  // by ID
document.querySelector('.card')       // first match
document.querySelector('[data-flavor]') // by attribute
document.querySelector('span')         // by tag

// ALL matches — returns a list:
document.querySelectorAll('.card')

// Always save to a variable:
const title = document.querySelector('#page-title');
Tip: Store the result in a variable. Think of it like bookmarking a page — you only have to look it up once, then you can use the bookmark any time.

DOM Manipulation

Changing Content: textContent vs innerHTML

Once you've selected an element, you can change what's inside it. Think of an HTML element like a picture frame — textContent puts a plain label on it, innerHTML puts actual artwork inside.

Your HTML:

<h1 id="page-heading">Welcome!</h1>

<div class="card-grid">
  <!-- cards will go here -->
</div>

textContent — plain text only:

const heading = document.querySelector('#page-heading');

heading.textContent = 'Welcome to JavaScript!';
// h1 now shows: Welcome to JavaScript!

// HTML tags treated as plain characters:
heading.textContent = '<em>italic?</em>';
// Shows literally: <em>italic?</em>  ← safe!

innerHTML — builds real HTML:

const grid = document.querySelector('.card-grid');

grid.innerHTML = `
  <div class="card"><h3>Sneakers</h3></div>
  <div class="card"><h3>T-Shirt</h3></div>
`;
// Real styled cards appear in the grid!
⚠️ Security Rule: XSS
Never use innerHTML with text a user typed. If someone types <script>stealData()</script> in a form and you inject it — that code runs.

Simple rule:
User's words → textContent (safe)
Your own HTML → innerHTML (ok)

Changing Styles & CSS Custom Properties

Your CSS (what's already set up):

/* CSS custom properties on :root */
:root {
  --accent: #6c5ce7;
  --bg: #f5f5f5;
}
.card {
  background: var(--accent);  /* uses the variable */
  border-radius: 12px;
  padding: 1.5rem;
}

Your JavaScript (changing things):

// Option 1: inline style (quick but messy)
const card = document.querySelector('.card');
card.style.backgroundColor = '#6c5ce7';
// CSS hyphen → JS camelCase!

// Option 2: CSS variable (changes EVERYTHING)
document.documentElement.style
  .setProperty('--accent', '#6c5ce7');

// Every element using var(--accent) updates instantly
The power move: a single CSS custom property might feed 10 elements — the button, the border, the heading highlight. Changing it once from JavaScript updates all 10 at the same time.

classList — The Bridge Between JS and CSS

Think of classList like a wardrobe for your HTML element. Your CSS designs the outfits (states). JavaScript uses add(), remove(), and toggle() to dress or undress the element — and CSS transitions handle the animation automatically.

Your CSS — two states defined:

/* Default state */
.card {
  background: #1a1a2e;
  border: 2px solid #2e2e4a;
  transform: scale(1);
  transition: all .3s ease;  /* animation built-in */
}

/* Active state — JS adds this class */
.card.active {
  background: linear-gradient(135deg, #a78bfa, #6ee7b7);
  color: #0f0f1a;
  transform: scale(1.04);
  box-shadow: 0 0 20px rgba(167,139,250,.4);
}

Your JavaScript — one line switches the state:

const card = document.querySelector('.card');

card.classList.add('active');      // default → active
card.classList.remove('active');   // active → default
card.classList.toggle('active');   // flip back and forth
card.classList.contains('active'); // → true or false
The Key Insight: CSS defines both states. JavaScript flips the switch. One line of JS + CSS you already wrote = fully interactive UI.

Creating and Removing Elements

Think of this like building a LEGO piece in your hands before snapping it onto the board. You create the element, decorate it, then attach it to the page — and CSS styles it instantly the moment it lands.

Your HTML — the container already exists:

<!-- This container is in your HTML file -->
<div class="card-grid" id="grid">
  <!-- JavaScript will add cards here -->
</div>

Your JavaScript — 4 steps to add a card:

// Step 1: Create a blank element
const newCard = document.createElement('div');

// Step 2: Give it content and classes
newCard.classList.add('card');
newCard.textContent = 'Sneakers';

// Step 3: Add extra data (optional)
newCard.setAttribute('data-id', '42');

// Step 4: Snap it onto the page
const grid = document.querySelector('#grid');
grid.appendChild(newCard);
// ↑ Now a real .card appears, styled by CSS
// Removing is even simpler:
const card = document.querySelector('.card');
card.remove(); // Gone immediately

data-* Attributes and dataset

Think of data-* attributes like sticky notes on an element — you write any extra information you want right on the HTML tag, and JavaScript can read it any time. Perfect for storing IDs, colors, labels, or states without cluttering your class names.

Your HTML — sticky notes on each element:

<!-- data-color and data-name are YOUR custom attributes -->
<div class="swatch"
     data-color="#6c5ce7"
     data-name="Purple"></div>

<div class="swatch"
     data-color="#00b894"
     data-name="Teal"></div>

Your JavaScript — read the sticky notes:

const swatch = document.querySelector('.swatch');

// .dataset reads any data-* attribute
swatch.dataset.color  // → "#6c5ce7"
swatch.dataset.name   // → "Purple"

// You can write sticky notes from JS too:
swatch.dataset.selected = 'true';
// Adds data-selected="true" to the element
Naming: data-background-color → dataset.backgroundColor (hyphens become camelCase in JS)
Class names vs data attributes:
Use class names for styling — they control CSS.
Use data attributes for data — IDs, values, metadata. JavaScript reads both, but they serve different purposes.
