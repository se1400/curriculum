Setup

How JavaScript Fits Into Your Page

JavaScript lives in .js files and connects via the <script> tag. Place it in <head> with the defer attribute:

<head>
  <meta charset="UTF-8">
  <link rel="stylesheet" href="styles.css">
  <!-- defer = wait for full HTML -->
  <script defer src="script.js"></script>
</head>
defer — downloads the script in the background, then runs it after the entire HTML has loaded. Your JavaScript will always have access to every element on the page.

Inline scripts (for CodePen)

<script>
  // Code runs directly in the page
  console.log('Hello!');
</script>
Avoid (legacy patterns):
document.write() — can corrupt your page
alert() — intrusive dialog boxes
Instead, always update elements in the DOM.

The Browser Console — Your Testing Playground

Open DevTools right now:

  • Mac: Option + Cmd + J
  • Windows: F12, then Console tab

Type this and press Enter on any webpage:

document.querySelector('h1').textContent = 'It works!'

That's JavaScript modifying the DOM — live, instantly, no files needed.

Refresh to undo. The console is completely safe to experiment in. Nothing you type there is permanent.

JavaScript Syntax Basics

Statements & Semicolons

const name = 'Ada Lovelace';
document.querySelector('h1').textContent = name;
console.log(name);

One instruction per line. Semicolons prevent subtle bugs — make them a habit.

Comments

// Single-line comment

   Explain complex logic here. */

const price = 9.99; // inline comment

Case Sensitivity & Naming

// These are THREE different variables:
const myVariable = 1;
const MyVariable = 2;
const MYVARIABLE = 3;
Convention: camelCase
First word lowercase, each new word capitalized.
✅ accentColor, userName, isLoggedIn
❌ accent_color, c, x1
  • Names can contain letters, numbers, _, $
  • Must start with a letter, _, or $ — not a number
  • Cannot use reserved words: const, let, if…
Variables

Variables: const and let

const — values that won't change

const siteName = 'Portfolio';
const primaryColor = '#2d3436';
const maxScore = 100;

// ❌ This throws an error:
siteName = 'New Name';

Your default choice. The error protection is a feature.

let — values that will change

let score = 0;
score = score + 10; // ✅ works

let activeTab = 'home';
activeTab = 'about';   // ✅ works

Use when you need to update a value over time.

Practical rule: start everything with const. If you get an assignment error, change it to let. This catches accidental overwrites.
Avoid var — it's the old way (pre-2015) with confusing scope rules. You'll see it in legacy tutorials. Treat it like let but don't use it yourself.

Data Types

The Three Data Types You'll Use Most

📝 Strings

const title = 'JavaScript';
const name  = "Grace Hopper";
const msg   = `Hello, world!`;

Any text. Use single quotes for most strings, backticks for template literals.

🔢 Numbers

const age  = 28;
const price = 49.99;
const temp = -5;
const rate = 0.08;
// Math works directly:
price * (1 + rate); // 53.99

No quotes needed. Integers and decimals are the same type.

💡 Booleans

const isLoggedIn = true;
const hasSubmitted = false;
const isDarkMode = true;
// Used in decisions:
if (isLoggedIn) { /*...*/ }

A light switch — only true or false.

Template Literals

Template literals use backtick characters and embed values with ${}:

const name = 'Sofia';
const role = 'UX Designer';

// ❌ Old way — messy
const msg = 'Hello, ' + name +
  '! Your role is ' + role + '.';

// ✅ Template literal — clean
const msg = `Hello, ${name}! Your role is ${role}.`;

They also preserve line breaks, making HTML generation clean:

const card = `
  <div class="card">
    <h3>${name}</h3>
    <p class="role">${role}</p>
  </div>
`;
Looks familiar? The ${} syntax is the same idea as CSS var() — insert a value here. If you've used CSS custom properties, template literals will feel natural.

Math: Random Numbers

// Random decimal: 0 to <1
Math.random()   // 0.4823...

Math.floor(0.9) // → 0

// Random integer 0–359 (HSL hue)
const hue = Math.floor(Math.random() * 360);

// Dice roll: 1–6
const roll = Math.floor(Math.random() * 6) + 1;
Pattern: Math.floor(Math.random() * N) gives a random integer from 0 to N-1. Add 1 if you want 1 to N.

The DOM

The DOM: Your Page as a Data Structure

When the browser loads HTML, it builds the Document Object Model (DOM) — a tree of connected objects in memory. Every element is a node JavaScript can find and change.

<!-- Your HTML -->
<div class="card">
  <h3 id="title">My First Card</h3>
  <p>A description.</p>
</div>
The fundamental pattern:
1. Find an element in the DOM
2. Change it (content, style, classes, attributes)
3. The page updates instantly

The DOM

Selecting Elements with querySelector

querySelector() takes any CSS selector — because you already know CSS selectors:

// Same syntax as CSS!
document.querySelector('.card')
document.querySelector('#title')
document.querySelector('h2')
document.querySelector('.nav li')
document.querySelector('[data-id]')

// ALL matches (not just first):
document.querySelectorAll('.card')

// Store result — don't repeat the query!
const heading = document.querySelector('h1');
const cards   = document.querySelectorAll('.card');
Always store in a variable. Searching the DOM is slow if done repeatedly. Query once, reuse the variable.

DOM Manipulation

Changing Content: textContent vs innerHTML

textContent — plain text only

const heading = document.querySelector('h1');
heading.textContent = 'Welcome!';

// HTML tags are treated as literal text
heading.textContent = '<strong>Bold?</strong>';
// Displays: <strong>Bold?</strong> — literally

✅ Safe for user-provided content. Tags won't execute.

innerHTML — parses HTML

const grid = document.querySelector('.grid');
grid.innerHTML = `<div class="card">
  <h3>New Card</h3>
</div>`;
// A real styled card now exists in the DOM

✅ Use when building HTML from your own data.

⚠️ XSS Warning
Never use innerHTML with user-typed content. A user could type <script>stealYourData()</script> into a form, and if you inject it with innerHTML, it runs.

Rule: user data → textContent. Your own data → innerHTML.

Changing Styles & CSS Custom Properties

style property (inline)

const card = document.querySelector('.card');
// CSS hyphen → JavaScript camelCase
card.style.backgroundColor = '#6c5ce7';
card.style.borderRadius    = '12px';
card.style.padding         = '1.5rem';

Works but creates hard-to-undo inline styles. Use sparingly.

CSS custom properties (powerful)

// Changes ONE variable → updates EVERYTHING
document.documentElement.style
  .setProperty('--accent', '#6c5ce7');

// Generate a random color theme:
const hue = Math.floor(Math.random() * 360);
document.documentElement.style
  .setProperty('--hue', hue);

If your CSS uses var(--accent) on buttons, links, and borders, one line updates them all.

DOM Manipulation

classList — The Bridge Between JS and CSS

const card = document.querySelector('.card');

// Add a class
card.classList.add('featured');

// Remove a class
card.classList.remove('featured');

// Toggle: adds if absent, removes if present
card.classList.toggle('featured');

// Check if a class exists
card.classList.contains('featured');
// → true or false
The Key Insight:
CSS defines what both states look like.
JavaScript simply switches between them.
One line of JS + CSS you already wrote = interactive UI.

Creating and Removing Elements

4-step process to create

// Step 1: Create an empty element
const newCard = document.createElement('div');

// Step 2: Configure it
newCard.classList.add('card');
newCard.textContent = 'A New Card';

// Step 3: Set attributes (optional)
newCard.setAttribute('data-id', '42');

// Step 4: Add to the page
const grid = document.querySelector('.grid');
grid.appendChild(newCard);

Removing

const card = document.querySelector('.card');
card.remove(); // Gone immediately

DOM Manipulation

data-* Attributes and dataset

<!-- HTML: embed extra data on elements -->
<div class="swatch"
     data-color="#6c5ce7"
     data-name="Purple"></div>

<div class="swatch"
     data-color="#00b894"
     data-name="Teal"></div>
// JavaScript reads via .dataset
const swatch = document.querySelector('.swatch');
console.log(swatch.dataset.color); // "#6c5ce7"
console.log(swatch.dataset.name);  // "Purple"

// You can also set data attributes from JS:
swatch.dataset.selected = 'true';
// Adds: data-selected="true" in the HTML
Naming rule: data-background-color → dataset.backgroundColor (hyphen → camelCase)
Why data-*?
Class names are for styling. data-* attributes carry data — IDs, colors, states, metadata. JavaScript reads them naturally. This is how you link a rendered card back to its source data object.
