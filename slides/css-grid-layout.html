Background

Fixed vs. Liquid Layouts — A Brief History

Fixed Layouts

  • Set to a specific pixel width
  • Predictable design control
  • Horizontal scroll on small screens
  • Dead space on very wide screens
.wrapper { width: 960px; }

Liquid (Fluid) Layouts

  • Uses percentage widths
  • Stretches to fill any window
  • Long lines hard to read on wide screens
  • Collapses awkwardly on narrow screens
.wrapper { width: 100%; }
The old solution: Float-based hybrids — negative margins, padding hacks, and float pushed far beyond its purpose. Brittle and hard to maintain.
CSS Grid solves this cleanly. Rather than fighting the browser to achieve layout, Grid gives you a direct, expressive way to describe exactly what you want.
Introduction to CSS Grid

CSS Grid — The Two-Dimensional Layout System

Two-dimensional = Grid controls both rows and columns simultaneously.
Every previous layout method (floats, inline-block, Flexbox) works in only one direction at a time.

Key Vocabulary

Grid Container Element with display: grid. All layout control starts here.
Grid Items Direct children only. Grandchildren do not participate.
Grid Tracks The columns and rows. Columns run vertically; rows run horizontally.
Grid Lines Dividers between tracks. Numbered automatically starting at 1.
Grid Cell Intersection of one row track and one column track — like a table cell.
Grid Area A rectangle spanning multiple cells. Items can span rows and columns.

Introduction to CSS Grid

Activating a Grid & Implicit vs. Explicit

Activating a Grid

One declaration turns any element into a grid container:

.container {
  display: grid;
}

All direct children become grid items. Nothing dramatic visually yet — items stack in one column by default (the implicit grid).

Define structure to take control:

.container {
  display: grid;
  grid-template-columns: 1fr 1fr;
}

Explicit vs. Implicit Grid

Explicit Grid

The grid you define with CSS. Full control over columns and rows.

Implicit Grid

What the browser creates when content overflows your explicit grid. New rows added automatically.

Define your grid explicitly so the browser doesn't have to guess. Use grid-auto-rows to control implicit row sizing.

Defining the Grid

Defining Columns and Rows

grid-template-columns / rows

Space-separated values — each defines one track:

.container {
  display: grid;
  grid-template-columns: 200px 400px 200px;
  grid-template-rows: 100px 300px;
}

Accepted units: px, %, auto, fr

Controlling Implicit Rows

When items overflow your defined rows, use grid-auto-rows:

.container {
  display: grid;
  grid-template-columns: 1fr 1fr 1fr;
  grid-auto-rows: 150px;
}
Great for card grids — every auto-created row gets a consistent height.

The fr Unit

fr = fractional unit. Assigns a fraction of available free space after fixed items are placed.

Powerful Grid Functions

repeat() and minmax()

repeat()

Avoid repetitive column definitions:

/* Tedious */
grid-template-columns: 1fr 1fr 1fr 1fr 1fr;

/* Clean */
grid-template-columns: repeat(5, 1fr);

/* Repeating a pattern */
grid-template-columns: repeat(3, 200px 1fr);
/* → 6 columns: 200px 1fr 200px 1fr 200px 1fr */

minmax()

Sets a minimum and maximum size for a track:

grid-template-columns:
  repeat(3, minmax(200px, 1fr));
  • Each column: at least 200px
  • Can grow to an equal share of space
  • Flexible but never too small

Combining Both

.container {
  display: grid;
  grid-template-columns:
    repeat(3, minmax(200px, 1fr));
  gap: 1rem;
}

Responsive Grids Without Media Queries

auto-fill vs. auto-fit

Pass a keyword instead of a number to repeat() — the browser decides how many columns fit automatically. 
.container {
  display: grid;
  grid-template-columns: repeat(auto-fit, minmax(200px, 1fr));
  /* "Make as many columns as will fit at ≥200px wide" — no media queries needed */
}

auto-fill

Creates as many column tracks as will fit — even if some are empty. Empty columns still take up space.

auto-fit

Also creates as many tracks as fit, but collapses empty columns to zero width. Filled columns expand to use the space.

In most real-world cases (enough items to fill a row), both behave identically. Use auto-fit when you want items to expand rather than leave gaps.

Spacing

Controlling Spacing with gap

What gap does

Adds space between grid tracks — not around the outer edges of the grid.

/* Equal gap on all sides between items */
.container {
  display: grid;
  grid-template-columns: repeat(3, 1fr);
  gap: 20px;
}

/* Different row and column gaps */
.container {
  row-gap: 30px;
  column-gap: 15px;
}

/* Shorthand: row column */
.container {
  gap: 30px 15px;
}
Deprecated names: grid-row-gap / grid-column-gap still work but are outdated. Use row-gap / column-gap (also works in Flexbox).

Placing Items in the Grid

Auto-Placement & grid-auto-flow

Auto-Placement (default)

Items fill left → right, top → bottom in source order.

Control direction with grid-auto-flow:

/* Default: items flow across rows */
grid-auto-flow: row;

/* Items flow down columns instead */
grid-auto-flow: column;

Dense Packing

When items span multiple cells, gaps can appear. dense backfills them:

.container {
  display: grid;
  grid-template-columns: repeat(4, 1fr);
  grid-auto-flow: row dense;
}
Accessibility note: dense can reorder items visually — items may appear before their source-order position, which affects keyboard navigation and screen readers.

How Grid Lines Work

A 3-column grid has 4 vertical lines; a 2-row grid has 3 horizontal lines — numbered from 1.

Placing with Lines

Named Grid Areas

Named Grid Areas — Layout as a Map

Step 1: Name each item

header  { grid-area: header; }
nav     { grid-area: navigation; }
main    { grid-area: content; }
aside   { grid-area: sidebar; }
footer  { grid-area: footer; }

Step 2: Define the layout map

.container {
  display: grid;
  grid-template-columns: 200px 1fr;
  grid-template-rows: auto 1fr auto;
  grid-template-areas:
    "header     header"
    "navigation content"
    "footer     footer";
}

Use . to leave a cell empty:

  grid-template-areas:
    "header  header"
    "sidebar content"
    ".       footer";

Alignment

Alignment Inside the Grid

Key Concept: Two Axes

Inline axis (horizontal): justify-* properties
Block axis (vertical): align-* properties

Items Within Cells

On the container — applies to all items:

.container {
  justify-items: center; /* horizontal in cell */
  align-items: end;      /* vertical in cell */

  /* Shorthand: align / justify */
  place-items: center;
}

Values: stretch (default) · start · end · center

Override on Individual Items

.special-item {
  justify-self: end;
  align-self: start;
}

Aligning the Grid in its Container

When tracks are smaller than the container:

.container {
  grid-template-columns: repeat(3, 200px);
  justify-content: space-between; /* horizontal */
  align-content: center;          /* vertical */

  /* Shorthand */
  place-content: center space-between;
}

Advanced Placement

Overlapping Grid Items & z-index

How Overlapping Works

Place two items in the same grid cells — they layer on top of each other.

.container {
  display: grid;
  grid-template-columns: repeat(3, 1fr);
  grid-template-rows: 200px 200px;
}

.background-item {
  grid-column: 1 / -1; /* full width */
  grid-row: 1 / 2;
}

.overlay-item {
  grid-column: 2 / 3;
  grid-row: 1 / 2; /* same row! */
}

Controlling Stack Order

.background-item { z-index: 1; }
.overlay-item    { z-index: 2; }
Grid items automatically get a stacking context — z-index works without setting position: relative. This is different from regular block elements.

Step 1: Grid container

.page-grid {
  display: grid;
  grid-template-columns: 200px 1fr 250px;
  grid-template-rows: auto 1fr auto;

}

Advanced Grid

Subgrid — Nested Grids That Align

The Problem

Normally, a grid item's children don't participate in the parent grid — they create their own independent layout.

<div class="parent-grid">
  <div class="card">
    <h2>Title</h2>
    <p>Content...</p>
    <button>Action</button>
  </div>
  <div class="card">...</div>
</div>

Cards with different content lengths → misaligned titles, content, and buttons across columns.

Title
Shorter content
Button
A Much Longer Card Title Here
Much longer content that wraps to multiple lines
Button

The Subgrid Solution

Use grid-template-columns: subgrid or grid-template-rows: subgrid on the child to inherit the parent's tracks:

.parent-grid {
  display: grid;
  grid-template-columns: 1fr 1fr;
  grid-template-rows: auto 1fr auto;
}

.card {
  /* Span all 3 row tracks */
  grid-row: span 3;
  display: grid;

  /* Inherit parent row tracks */
  grid-template-rows: subgrid;
}

Now titles, content areas, and buttons all align across cards — regardless of content length.

Browser support: All modern browsers support subgrid (Chrome 117+, Firefox 71+, Safari 16+).

Decision Guide

Grid vs. Flexbox — When to Use Which

They are not competitors — they are complementary tools that solve different problems.

CSS Grid — Two-Dimensional

Use when you need to control layout in both rows and columns simultaneously:

  • Full page structure (header, sidebar, main, footer)
  • Card galleries and dashboards
  • Data tables and any layout where elements must align both horizontally and vertically at the same time
  • Overlapping elements
"Am I thinking about rows and columns together?" → If yes, Grid.

Flexbox — One-Dimensional

Use when you need to control layout in a single direction:

  • A row of navigation links
  • A group of buttons
  • A column of stacked form elements
  • A horizontal media object (image next to text)
  • Any component where content size should drive the layout
Content-first: items determine their own space.
Use both together! A common pattern: Grid for the overall page structure, then Flexbox inside each area to arrange its internal components. Grid structures the page; Flexbox arranges the pieces within each section.

Common Questions

Q & A

What's the difference between implicit and explicit grid?

The explicit grid is what you define with grid-template-columns and grid-template-rows. The implicit grid is what the browser creates automatically when items overflow. Control implicit row size with grid-auto-rows.

When should I use fr instead of percentages?

fr distributes available space — after fixed tracks and gaps are accounted for. Percentages are based on the total container size, so you have to manually subtract gaps. Use fr for almost everything; use % only when you need a track to be an exact percentage of the container.

Named areas vs. line-based placement — which to use?

Use named areas for full page layouts with distinct sections — the template reads like a visual map. Use line-based placement for precise individual item positioning, overlapping items, or when the number of tracks is dynamic.

What does -1 mean as a grid line number?

Negative numbers count from the end of the explicit grid. Line -1 is always the last line — the right edge for columns, bottom edge for rows. grid-column: 1 / -1 always spans the full width, no matter how many columns you have.

What's the difference between auto-fill and auto-fit?

Both create as many columns as fit. The difference appears with fewer items than columns: auto-fill keeps empty column tracks (they take up space); auto-fit collapses empty tracks so filled items expand. In most cases auto-fit is what you want.
