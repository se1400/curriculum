The CSS Box Model

Every HTML element is a rectangular box with four layers, whether visible or not:

Figure 10.1 — The CSS box model

From Outside In:

  • Margin — Space outside the element (always transparent)
  • Border — Surrounds the padding; can be styled with width, style, and color
  • Padding — Space inside the element; inherits the background color
  • Content — The actual text, images, or child elements
These layers are always present on every element, even when you don't explicitly set them.

Box Model: How Size Is Calculated

In the default box model, the rendered size includes all layers:

A Simple div

div {
  width: 250px;
  height: 100px;
  background-color: #E74C3C;
  border: 1px solid #000;
}

Figure 10.2 — A simple div

Same div + margin + padding

div#d2 {
  width: 250px;
  height: 100px;
  background-color: #E74C3C;
  border: 5px solid #000;
  margin: 10px;
  padding: 10px;
}

Figure 10.3 — Padding and border add to the size

Box Model Math

The actual space an element takes up on the page:

Total Width

width
+ padding-left + padding-right
+ border-left  + border-right
+ margin-left  + margin-right

Total Height

height
+ padding-top    + padding-bottom
+ border-top     + border-bottom
+ margin-top     + margin-bottom

Example Calculation

For the second div on the previous slide:

Width:  250 + 10 + 10 + 5 + 5 + 10 + 10
      = 300px actual width

Height: 100 + 10 + 10 + 5 + 5 + 10 + 10
      = 150px actual height
The trap: You set width: 250px but the element takes up 300px of horizontal space!

Changing the Box Model: box-sizing

The default box model adds padding and border outside your specified width. box-sizing fixes this:

content-box (default)

  • Width/height apply to content only
  • Padding and border are added on top
  • Element ends up larger than specified
div { width: 250px; padding: 10px; }
/* Actual width: 270px */

border-box (intuitive)

  • Width/height include padding and border
  • Element is exactly the size you specify
  • Content area shrinks to accommodate padding/border
div { width: 250px; padding: 10px;
     box-sizing: border-box; }
/* Actual width: 250px ✓ */
Best practice: Most developers apply this globally: * { box-sizing: border-box; }

The Outline: A Special Case

CSS also has outline properties that look like borders but behave differently:

Border vs Outline

FeatureBorderOutline
Takes up space?YesNo
Affects layout?YesNo
Can be rounded?Yes (border-radius)No
Per-side control?YesNo

Outline Properties

element {
  outline: 2px solid blue;
  outline-offset: 5px;
}
Common use: Browser focus indicators (the ring you see when tabbing through links) use outlines so they don't shift the layout.

The Whole Scoop on Positioning

The position property controls how an element is placed on the page:

static

Default. Normal document flow. top/left have no effect.

relative

Offsets from its normal position. Still takes up original space.

absolute

Positioned relative to nearest positioned parent. Removed from flow.

fixed

Positioned relative to the browser window. Stays put on scroll.

sticky

Hybrid: acts relative until a scroll threshold, then acts fixed.

Position offsets: top, right, bottom, left — these work on all types except static.

Relative Positioning

Elements flow normally, stacking block elements vertically and inline elements horizontally:

Element Types

  • Block (<div>, <p>) — starts on a new line, full width
  • Inline (<span>, <code>) — flows with text, no width/height
  • Inline-block (<img>) — flows inline but can have width/height
div {
  position: relative;
  left: 25px;  /* shifts 25px from
                  its normal spot */
}

Absolute Positioning

Elements are placed at exact coordinates relative to their nearest positioned parent:

Key Behaviors

  • Removed from the normal flow
  • Positioned relative to nearest ancestor with position: relative (or the page)
  • Can overlap other elements
  • left/top take priority over right/bottom
#d1 { position: absolute;
      left: 0; top: 0; }
#d2 { position: absolute;
      left: 50px; top: 30px; }

Fixed & Sticky Positioning

fixed

  • Positioned relative to the browser window
  • Stays in place as the page scrolls
  • Common uses: sticky headers, floating buttons, watermarks

sticky

  • Acts relative until you scroll past a threshold
  • Then acts fixed until its container ends
  • Common use: table headers that "stick" while scrolling
.sticky-header {
  position: sticky;
  top: 0; /* sticks at top */
}

Controlling the Way Things Stack Up: z-index

When elements overlap, z-index controls which appears on top — like layers in Photoshop:

How It Works

  • Higher z-index = closer to the viewer (on top)
  • Default is 0 if not set
  • Can be negative to push behind other elements
  • Only works on positioned elements (not static)
  • An element always appears on top of its parent
#box1 { z-index: 1; }  /* behind */
#box2 { z-index: 10; } /* on top */
#box3 { z-index: 5; }  /* middle */
Without z-index, elements later in the HTML appear on top of earlier ones.

Modern Layout with Flexbox

Flexbox makes creating flexible, responsive layouts dramatically easier than older methods:

What Makes Flexbox Different

  • Direction-agnostic — easily switch between horizontal and vertical
  • Flexible sizing — items grow to fill space or shrink to fit
  • Powerful alignment — center anything in one line
  • Order control — rearrange visually without changing HTML

Getting Started

.container {
  display: flex;
}

/* That's it! Direct children
   become flex items. */
Flexbox = one dimension. It arranges items in a single row or column. For two-dimensional layouts (rows AND columns), use CSS Grid (Lesson 12).

Main Axis & Cross Axis

This is the most important concept in Flexbox. Every flex container has two axes:

flex-direction: row (default)

  • Main axis = horizontal (left → right)
  • Cross axis = vertical (top → bottom)

flex-direction: column

  • Main axis = vertical (top → bottom)
  • Cross axis = horizontal (left → right)
Key:
justify-content → main axis
align-items → cross axis

Flex Container: Direction & Wrapping

flex-direction

Sets the main axis direction:

flex-direction: row;           /* → default */
flex-direction: row-reverse;   /* ← */
flex-direction: column;        /* ↓ */
flex-direction: column-reverse;/* ↑ */

flex-wrap

Controls whether items wrap to new lines:

flex-wrap: nowrap;       /* default: one line */
flex-wrap: wrap;         /* wraps to new lines */
flex-wrap: wrap-reverse; /* wraps upward */

flex-flow (shorthand)

Combines direction and wrap:

flex-flow: row wrap;

gap (modern spacing)

Cleaner than margins for spacing between items:

.container {
  display: flex;
  gap: 1rem;       /* all gaps */
  row-gap: 1rem;   /* between rows */
  column-gap: 2rem;/* between columns */
}
gap replaces the old approach of margins + removing margins on first/last items.

justify-content — Main Axis Spacing

Controls how items are distributed along the main axis:

justify-content: flex-start;     /* items at start (default) */
justify-content: flex-end;       /* items at end */
justify-content: center;         /* items centered */
justify-content: space-between;  /* first/last at edges, equal space between */
justify-content: space-around;   /* equal space around each item */
justify-content: space-evenly;   /* perfectly equal gaps everywhere */

align-items & align-content

align-items — Cross Axis

Aligns items within each line:

align-items: stretch;    /* default */
align-items: flex-start; /* top */
align-items: flex-end;   /* bottom */
align-items: center;     /* middle */
align-items: baseline;   /* text baselines */

align-content — Multiple Lines

Aligns the lines themselves (only when flex-wrap: wrap):

align-content: stretch;       /* default */
align-content: flex-start;
align-content: flex-end;
align-content: center;
align-content: space-between;
align-content: space-around;
Common confusion: align-items aligns items within a single line. align-content aligns the lines themselves when items wrap to multiple lines.

Flex Item Properties

Individual flex items have their own properties that control how they behave:

flex-grow

How much an item grows to fill space (0 = don't grow):

flex-grow: 0; /* default */
flex-grow: 1; /* fills space */
flex-grow: 2; /* grows 2× as much */

flex-shrink

How much an item shrinks when space is tight:

flex-shrink: 1; /* default: can shrink */
flex-shrink: 0; /* never shrink */

flex-basis

The starting size before growing/shrinking:

flex-basis: auto; /* use content size */
flex-basis: 200px; /* start at 200px */
flex-basis: 50%;   /* start at 50% */

flex (shorthand) — use this!

flex: 0 1 auto; /* default */
flex: 1;        /* grow equally */
flex: 2;        /* grow 2× more */
flex: 0 0 200px;/* fixed 200px */
Most common: flex: 1; makes items equal-width and flexible.

Order & Individual Alignment

order

Changes the visual order without changing HTML:

.item-1 { order: 2; } /* appears 2nd */
.item-2 { order: 1; } /* appears 1st */
.item-3 { order: 3; } /* appears 3rd */
Accessibility warning: order only changes visual order — screen readers and Tab key still follow the HTML source order.

align-self

Override the container's align-items for one item:

.container {
  align-items: flex-start;
}

.special {
  align-self: flex-end;
  /* just this item aligns
     to the bottom */
}

Practical Flexbox Patterns

Navigation Bar

.nav {
  display: flex;
  gap: 2rem;
  padding: 1rem;
  background: #2C3E50;
}

Perfect Centering

.center {
  display: flex;
  justify-content: center;
  align-items: center;
  min-height: 100vh;
}

Equal-Height Cards

.cards {
  display: flex;
  gap: 1rem;
}
.card { flex: 1; }
/* All cards same height! */

Sticky Footer

body {
  display: flex;
  flex-direction: column;
  min-height: 100vh;
}
main { flex: 1; }
/* Footer pushed to bottom */

Responsive Patterns with Flexbox

Flexbox can create responsive layouts without media queries:

The Magic Pattern

.responsive-grid {
  display: flex;
  flex-wrap: wrap;
  gap: 1rem;
}

.responsive-grid > * {
  flex: 1 1 300px;
  /* grow, shrink, min 300px */
}

What happens:

  • Items try to be at least 300px wide
  • They grow to fill available space
  • They wrap to new lines when they don't fit
  • All without media queries!

Common Flexbox Pitfalls

1. Percentage widths in flex items

/* Instead of: */
.item { width: 50%; }

/* Use: */
.item { flex: 0 0 50%; }

2. min-width: auto overflow

Flex items won't shrink below their content by default. Fix:

.item { min-width: 0; }

3. Margins don't collapse

Unlike block elements, flex item margins never collapse. Use gap instead.

4. Auto margins are powerful

Auto margins consume all available space — useful for pushing items:

.nav-item:last-child {
  margin-left: auto;
  /* pushes to far right */
}

5. flex: 1 vs flex: auto

flex: 1 ignores content size (basis: 0). flex: auto respects it (basis: auto).

When to Use What

Flexbox

  • Navigation menus
  • Cards in a row/column
  • Centering content
  • Equal-height columns
  • Responsive layouts
  • Distributing space

Positioning

  • Tooltips / popovers
  • Modals / overlays
  • Precise placement
  • Sticky headers
  • Layering with z-index
  • Floating buttons

CSS Grid (Lesson 12)

  • Full page layouts
  • Rows AND columns
  • Complex 2D layouts
  • Magazine-style layouts
  • Dashboard grids
  • Template areas
Rule of thumb: Start with Flexbox for most things. Reach for positioning when you need overlays. Use Grid when you need rows and columns.

Summary: Key Takeaways

Box Model

  • Every element = content + padding + border + margin
  • Default model adds padding/border outside width
  • Use box-sizing: border-box globally
  • Outlines don't affect layout

Positioning

  • static (default), relative, absolute, fixed, sticky
  • z-index controls stacking order
  • Higher z-index = on top

Flexbox

  • display: flex on the container
  • Main axis vs cross axis is key
  • justify-content = main axis
  • align-items = cross axis
  • flex: 1 for flexible equal items
  • gap for clean spacing

Common Questions

Q: When do I use relative vs absolute positioning?

Use relative for slight nudges from normal position. Use absolute when you need pixel-perfect placement inside a container. For most layouts, use Flexbox or Grid instead.

Q: Should I always use border-box?

Yes, for nearly all projects. It makes sizing intuitive — 250px means 250px. Apply it globally with * { box-sizing: border-box; }.

Q: What if elements overlap without z-index?

Elements later in the HTML appear on top of earlier ones. Think of the browser painting them in order — the last one painted is on top.

Q: justify-content vs align-items?

justify-content = main axis (horizontal by default). align-items = cross axis (vertical by default). When flex-direction: column, they swap!

Q: Why doesn't flex: 1 respect content width?

flex: 1 is shorthand for flex: 1 1 0 — the 0 basis ignores content size. Use flex: auto (1 1 auto) to respect content.

Q: Can I use Flexbox for full page layouts?

Yes, but CSS Grid is better for 2D page layouts. Flexbox excels at one-dimensional arrangements (rows or columns).
