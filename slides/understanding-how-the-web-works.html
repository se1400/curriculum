A Brief History of HTML & the Web

Key Milestones

  • Pre-1990: Internet existed but was technical and difficult to use
  • 1990: Tim Berners-Lee created HTML for easy cross-referencing
  • 1993: ~100 web servers existed worldwide
  • Today: Billions of websites serve billions of files

Key Terms

Hypertext: Text with cross-reference links between pages

HTML: Hypertext Markup Language - describes how content is organized and linked

Learn more: Wikipedia: History of the Web

Creating Web Content

The term "web content" is more accurate than "web page" because multiple file types work together:

The Three Core Technologies

  • HTML - Structure (headings, paragraphs, lists)
  • CSS - Presentation (colors, fonts, layout)
  • JavaScript - Behavior (interactions, animations)

HTML = What content is

CSS = How it looks

JavaScript = How it behaves

Writing standards-compliant code ensures your content works across all browsers and devices!

Understanding Web Content Delivery

A browser request and server response

Basic Flow:

  1. Browser sends a request to the web server
  2. Server finds the requested file
  3. Server sends a response back to the browser
  4. Browser renders the content for display

The Content Delivery Process

When you visit www.google.com:

  1. Browser requests index.html
  2. Server finds and sends HTML file
  3. Browser reads HTML, finds image tag: 
    <img alt="Google" src="/images/logo.png">
  4. Browser requests the image separately
  5. Server sends the image
  6. Browser assembles and displays everything

Visiting www.google.com

Browsers do more than display - they assemble content from multiple requests!

Local vs. Remote Viewing

Local Viewing

  • Files on your own hard drive
  • No web server needed
  • Only you can see the content
  • Great for development/testing

Remote Viewing

  • Files on a web server
  • Accessible to anyone online
  • Requires hosting account
  • Needed to share your work

You can complete all lessons locally, but you'll need a web server for others to see your work.

Selecting a Web Hosting Provider

Key Selection Criteria

  • Reliability/Uptime - Is your site always accessible?
  • Customer Service - Phone, email, chat support
  • Server Space - Room for your files
  • Bandwidth - Handle your traffic load
  • Domain Management - Include domain name?
  • Price - Don't overpay! ~$75/year max for basic hosting

Using FTP to Transfer Files

FTP (File Transfer Protocol) or SFTP (Secure FTP) moves files from your computer to your web server.

You'll need from your hosting provider:

  • Hostname - The server address (e.g., ftp.yourdomain.com)
  • Username - Your account name
  • Password - Your account password

Tip: Use SFTP when available - it encrypts your connection for security!

Using an FTP Client

  4. Drag files from your computer (left) to server (right)

Understanding the Document Root

The document root is the starting point for your website files.

File Location = URL

If sample.html is in document root:

http://yourdomain.com/sample.html

If it's in a subdirectory:

http://yourdomain.com/newdir/sample.html

Important: Files outside the document root are NOT accessible via web browser!

The sample.html file accessed via web browser

File Organization Best Practices

Common Directory Structure

document_root/
    index.html
    images/
        logo.png
        banner.jpg
    css/
        styles.css
    js/
        script.js
    downloads/
        portfolio.zip

Linking to Files

<a href="/downloads/artfolio.zip">Download my art portfolio!</a>

Using an Index Page

The index.html file is the default page for any directory.

Both URLs show same page:

http://apple.com/iphone/
http://apple.com/iphone/index.html

Without index.html:

  • Directory listing shown, OR
  • "Directory Listing Denied" error

Always name your home page index.html and place it in the document root!

Modern Deployment: Git & GitHub

While FTP works, modern development often uses version control:

What is Git?

  • Tracks changes to files over time
  • See what changed, when, and by whom
  • Roll back to previous versions

What is GitHub?

  • Hosts Git repositories online
  • Easy collaboration
  • GitHub Pages - Free web hosting!

Continuous Deployment: Push to GitHub, your site updates automatically. Used by Netlify, Vercel, Cloudflare Pages.

Summary

Key Takeaways

  • HTML, CSS, and JavaScript create web content
  • Browsers request files from web servers
  • Web servers deliver files to browsers
  • FTP/SFTP transfers files to servers
  • Document root is your website's starting point
  • index.html is the default page

Best Practices

  • Test in multiple browsers
  • Write standards-compliant code
  • Organize files in logical directories
  • Consider Git for version control
  • Use SFTP for secure transfers
