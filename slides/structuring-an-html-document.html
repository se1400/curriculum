Listing 2.1: Basic HTML Structure

Every web page needs these essential elements:

HTML Tags Every Page Must Have

Document Structure

  • <!doctype html> - Declares HTML5
  • <html lang="en"> - Root element with language
  • <head> - Metadata (not displayed)
  • <body> - Visible content

Inside <head>

  • <meta charset="utf-8"> - Character encoding (security!)
  • <meta name="viewport"...> - Mobile responsiveness
  • <title> - Browser tab & bookmarks

Remember: <head> content = NOT displayed | <body> content = displayed

Three Types of HTML Tags

Opening Tag

<tagname>

Begins with < and ends with >

Example: <p>

Closing Tag

</tagname>

Has a / after the <

Example: </p>

Empty Tag

<tagname>

No closing tag needed

Examples: <br> <hr> <img>

Best Practice: Always use lowercase for HTML tags.

Listing 2.2: Paragraphs and Line Breaks

Browsers ignore extra whitespace - use tags to control formatting:

Paragraphs, Breaks & Rules

Key Tags

  • <p>...</p> - Paragraph (adds space before/after)
  • <br> - Line break (no extra space)
  • <hr> - Horizontal rule (line across page)

Both <br> and <hr> are empty tags - no closing tag needed!

Whitespace Collapse

Extra spaces, tabs, and line breaks in your HTML code are reduced to a single space by browsers.

You must use HTML tags to control formatting!

Listing 2.3: HTML Headings

Use <h1> through <h6> to create content hierarchy:

Heading Best Practices

Think Outline, Not Size

  • <h1> - Main page heading (one per page)
  • <h2> - Major sections
  • <h3> - Subsections within h2
  • <h4> - <h6> - Deeper levels (rarely needed)

Don't skip levels! Go h1 → h2 → h3, not h1 → h3

Title vs Heading

<title> = Browser tab (in <head>, not visible on page)

<h1> = Page heading (in <body>, visible on page)

Accessibility

Screen readers use heading levels to help users navigate. Proper hierarchy = accessible content!

Understanding Semantic Elements

HTML5 introduced elements that describe the meaning of content:

<header>

Introductory content

<nav>

Navigation links

<main>

Main content

<section>

Thematic grouping

<article>

Standalone content

<aside>

Secondary content

<footer>

Footer information

Key Principle: HTML = Structure (what it is) | CSS = Presentation (how it looks) | JavaScript = Behavior (what it does)

Why Use Semantic HTML?

Benefits

  • Accessibility: Screen readers understand page structure
  • SEO: Search engines better understand content
  • Maintainability: Code is easier to read and update
  • Consistency: Standard structure across projects

Accessibility Matters

  • Screen readers can navigate effectively
  • Users can skip to specific sections
  • Works better across devices
  • Respects user preferences (dark mode, etc.)

Semantic elements don't change appearance by default - that's CSS's job! They provide meaning, not style.

Common Structure

  • <header> - Top of page, logo/title
  • <nav> - Site navigation

Key Insight

You can have multiple <header> and <footer> elements:

  • One for the whole page
  • One for each <article>
  • One for each <section>

There's no single "correct" way to structure content - organize it in a way that makes sense for your content!

Listing 2.4: Semantic Markup Example

A complete example using semantic elements:

The <header> Element

Purpose

Contains introductory content for its parent element.

Can Include:

  • Logo or site title
  • Navigation menu
  • Search form
  • Introductory text
<header>
  <img src="logo.jpg" alt="Company">
  <h1>Welcome to Our Site</h1>
</header>

Key Point: You can have multiple <header> elements - one for the page, and one for each section or article!

The <nav> Element

Purpose

Contains navigation links to other pages or sections.

Best Used For:

  • Main site navigation
  • Table of contents
  • Breadcrumb navigation
  • Pagination links
<nav>
  <ul>
    <li><a href="/">Home</a></li>
    <li><a href="/about">About</a></li>
    <li><a href="/contact">Contact</a></li>
  </ul>
</nav>

Not every group of links needs <nav> - use it for major navigation blocks.

The <section> Element

Purpose

Groups thematically related content together.

When to Use:

  • Chapters of content
  • Tabbed content panels
  • Numbered sections
  • Homepage content areas
<section>
  <header>
    <h2>Features</h2>
  </header>
  <p>Our product features...</p>
</section>

Rule of thumb: If content would have its own heading in an outline, it might be a <section>.

The <article> Element

Purpose

Contains standalone, self-contained content that could be distributed independently.

Examples:

  • Blog posts
  • News articles
  • Forum posts
  • Product cards
  • User comments
<article>
  <header>
    <h2>Blog Post Title</h2>
    <p>By Author | Jan 2026</p>
  </header>
  <p>Article content...</p>
  <footer>Tags: HTML, CSS</footer>
</article>

Article vs Section: An article is standalone; a section is a thematic grouping within something larger.

The <aside> Element

Purpose

Contains secondary content related but not essential to the main content.

Common Uses:

  • Sidebars
  • Pull quotes
  • Advertising
  • Related links
  • Author bio
<aside>
  <h3>Related Articles</h3>
  <ul>
    <li><a href="#">Article 1</a></li>
    <li><a href="#">Article 2</a></li>
  </ul>
</aside>

Think of <aside> as a sidebar - related information that supplements the main content.

The <footer> Element

Purpose

Contains footer information about its parent element.

Typically Includes:

  • Copyright information
  • Contact details
  • Related links
  • Social media links
  • Site map
<footer>
  <p>&copy; 2026 My Company</p>
  <nav>
    <a href="/privacy">Privacy</a>
    <a href="/terms">Terms</a>
  </nav>
</footer>

Like <header>, you can have multiple footers - one for the page and one for each section/article!

Summary

Essential Tags

  • <!doctype html> - Document type
  • <html> - Root element
  • <head> - Metadata
  • <body> - Content
  • <p>, <br>, <hr> - Text formatting
  • <h1>-<h6> - Headings

Semantic Elements

  • <header> - Introductory content
  • <nav> - Navigation
  • <main> - Primary content
  • <section> - Thematic grouping
  • <article> - Standalone content
  • <aside> - Secondary content
  • <footer> - Footer info
