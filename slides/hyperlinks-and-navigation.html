Lesson 7: Hyperlinks and Navigation

The Foundation of the Web

Use arrow keys, buttons, or swipe to navigate

What Are Hyperlinks?

  • Hyperlinks are the defining feature of the World Wide Web
  • They connect documents, pages, and resources across the internet
  • Created with the <a> (anchor) element
  • The href attribute specifies the destination
<a href="https://example.com">Visit Example</a>
Fun fact: "Hypertext" in HTML stands for text with links — the ability to jump from one document to another is what made the web revolutionary.

Key Attributes

  • href — the URL or path to link to
  • target — where to open the link
  • title — tooltip text on hover
  • download — forces a file download
  • rel — relationship to the linked resource

Types of Web Addresses

There are three ways to specify a link destination:

Relative

href="about.html"
href="../images/pic.jpg"
href="pages/contact.html"

Relative to the current file's location

Relative-Root

href="/about.html"
href="/images/pic.jpg"
href="/pages/contact.html"

Starts from the website root (leading /)

Absolute (FQDN)

href="https://example.com"
href="https://example.com/about"
href="http://api.example.com"

Full URL including protocol and domain

Understanding Relative Paths

mywebsite/
  index.html
  pages/
    about.html ← you are here
    contact.html
  images/
    logo.png
  css/
    style.css

From pages/about.html:

Target Path
contact.html contact.html
index.html ../index.html
logo.png ../images/logo.png
style.css ../css/style.css
.. means "go up one directory"

Relative-Root vs Absolute Paths

Relative-Root (starts with /)

<!-- Always from website root -->
<a href="/about.html">About</a>
<a href="/pages/contact.html">Contact</a>
<img src="/images/logo.png">
Works the same no matter what page you're on. Great for site navigation.
Won't work opening files directly with file:// protocol — needs a web server.

Absolute (Full URL)

<!-- Complete web address -->
<a href="https://www.google.com">
  Google
</a>

<a href="https://example.com/page">
  External Page
</a>
Use absolute URLs when linking to external websites. Use relative or relative-root paths for your own content.

Linking Within a Page (Anchor Links)

Use the # symbol with an element's id to jump to a specific section:

Step 1: Add IDs to sections

<h2 id="introduction">Introduction</h2>
<p>Welcome to my page...</p>

<h2 id="chapter1">Chapter 1</h2>
<p>This is the first chapter...</p>

<h2 id="conclusion">Conclusion</h2>
<p>In summary...</p>

Step 2: Link to those IDs

<!-- Table of contents -->
<nav>
  <a href="#introduction">Intro</a>
  <a href="#chapter1">Chapter 1</a>
  <a href="#conclusion">Conclusion</a>
</nav>

<!-- Back to top link -->
<a href="#top">Back to Top</a>
The browser scrolls smoothly (or jumps) to the element with the matching id.

Linking to Non-HTML Files

Links can point to any file type — PDFs, images, documents, and more.

The download Attribute

<!-- Opens in browser (default) -->
<a href="report.pdf">
  View Report
</a>

<!-- Forces download -->
<a href="report.pdf" download>
  Download Report
</a>

<!-- Download with custom filename -->
<a href="report-v3-final.pdf"
   download="Annual-Report.pdf">
  Download Report
</a>

Common File Types

TypeBrowser Behavior
PDFUsually opens in viewer
ImagesDisplays directly
ZIP/RARAuto-downloads
DOCX/XLSXUsually downloads
MP3/MP4May play in browser
The download attribute lets you override the default behavior and force a file download.

Linking to External Websites

Use fully qualified domain names (FQDNs) — complete URLs including the protocol:

<!-- Always include the protocol (https://) -->
<a href="https://www.w3schools.com">W3Schools</a>
<a href="https://developer.mozilla.org">MDN Web Docs</a>
<a href="https://github.com">GitHub</a>

Don't forget the protocol!

<!-- WRONG - treated as relative path -->
<a href="www.google.com">Google</a>

<!-- CORRECT -->
<a href="https://www.google.com">
  Google
</a>

HTTPS vs HTTP

  • https:// — Encrypted, secure (preferred)
  • http:// — Unencrypted, insecure

Always use https:// when available.

Opening Links in New Windows

The target attribute controls where the link opens:

Common target values

ValueBehavior
_selfSame tab (default)
_blankNew tab/window
_parentParent frame
_topTop-level frame
<a href="https://google.com"
   target="_blank">
  Open Google in new tab
</a>

When to use target="_blank"

  • External links (leaving your site)
  • PDF documents
  • Help documentation
  • Media that shouldn't interrupt the flow
Warning: Always add security attributes when using target="_blank"! (See next slide)

Link Security: Protecting Your Users

When using target="_blank", the opened page can access your page via window.opener:

The Risk

<!-- INSECURE -->
<a href="https://example.com"
   target="_blank">
  Click here
</a>

Vulnerable

  • External page can read window.opener
  • Could redirect your page to a phishing site
  • Known as tabnapping

The Fix

<!-- SECURE -->
<a href="https://example.com"
   target="_blank"
   rel="noopener noreferrer">
  Click here
</a>

Secure

  • noopener — Prevents window.opener access
  • noreferrer — Hides the referring URL
  • Modern browsers auto-add noopener, but include both for compatibility

Good use cases

  • Describing where a link goes

Don't rely on titles for

  • Critical information (not accessible on mobile)
  • Tooltips don't appear on touch devices
  • Screen readers may not announce them

Styling Links with CSS

Links have special pseudo-class selectors that respond to user interaction:

Pseudo-class When it applies Example
a:link Unvisited link Blue link
a:visited Previously visited link Purple link
a:hover Mouse cursor is over it Red on hover
a:active Being clicked (mouse down) Dark red
a:focus Has keyboard focus (Tab key) Outlined 
a:link    { color: blue; }
a:visited { color: purple; }
a:hover   { color: red; text-decoration: none; }
a:active  { color: darkred; }
a:focus   { outline: 2px solid blue; }

The LoVe HAte Rule

CSS pseudo-classes for links must be declared in a specific order to work correctly:

L :link Unvisited links
V :visited Visited links
H :hover Mouse over
A :active Being clicked

LoVe   HAte

Why does order matter? CSS specificity rules mean later declarations override earlier ones of equal specificity. If :hover comes before :visited, the visited style will override the hover effect.

Visited Link Privacy

Browsers limit what CSS properties can be changed for :visited links to protect user privacy:

Allowed properties

  • color
  • background-color
  • border-color
  • outline-color
  • column-rule-color

NOT allowed

  • font-size
  • padding / margin
  • background-image
  • display
  • Any property that changes layout
Why? Without these restrictions, a malicious site could use JavaScript to check the computed styles of links, determining which sites you've visited — a serious privacy concern. This protection has been in place since about 2010.

Skip Links for Accessibility

Skip links let keyboard and screen reader users bypass navigation and jump to main content:

<body>
  <a href="#main-content" class="skip-link">
    Skip to main content
  </a>
  <nav>...long navigation...</nav>
  <main id="main-content">
    ...page content...
  </main>
</body>

CSS for Skip Links

.skip-link {
  position: absolute;
  top: -100px;
  left: 0;
  background: #000;
  color: #fff;
  padding: 8px 16px;
  z-index: 100;
}

.skip-link:focus {
  top: 0;  /* Appears on Tab */
}

Why Skip Links Matter

  • Required by WCAG 2.1 Level A
  • Keyboard users would otherwise Tab through every nav link
  • Screen readers announce navigation on every page
  • Hidden visually until focused via Tab key
Try pressing Tab on any major website — you'll often see a "Skip to content" link appear!

Using Links Effectively

Use descriptive link text

<a href="report.pdf">
  Download the 2024 Annual Report
</a>

Avoid vague text

<a href="report.pdf">
  Click here
</a>

Keep links concise

<a href="/pricing">
  View pricing plans
</a>

Don't make entire paragraphs links

<a href="/pricing">
  Our pricing page has all the
  information about plans and
  pricing for every tier we offer
</a>

Warn about external links

<a href="https://example.com"
   target="_blank"
   rel="noopener noreferrer">
  Visit Example (opens new tab)
</a>

Don't surprise users

<!-- No warning about behavior -->
<a href="https://example.com"
   target="_blank">
  Example
</a>

Key Takeaways

Linking Basics

  • Use relative paths for your own content
  • Use absolute URLs for external sites
  • Use #id for in-page navigation
  • Add download to force file downloads
  • Use mailto: for email links

Security & Accessibility

  • Always add rel="noopener noreferrer" with target="_blank"
  • Include skip links for keyboard users
  • Write descriptive link text

Styling

  • Remember LoVe HAte order for pseudo-classes
  • Visited link styling is limited for privacy
  • Always style :focus for keyboard users
