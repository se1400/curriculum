Using Margins

Margins add empty space around the outside of an element's rectangular area.

Margin Properties

  • margin-top — Sets the top margin
  • margin-right — Sets the right margin
  • margin-bottom — Sets the bottom margin
  • margin-left — Sets the left margin
  • margin — Shorthand for all four margins

Values

  • auto — Browser calculates the margin
  • Length units — px, em, rem, etc.
  • Percentages — Relative to container width
/* Individual properties */
margin-top: 20px;
margin-right: 10px;

/* Shorthand */
margin: 20px 10px;

Understanding Margin Shorthand: TRBL

Remember the order with the TRBL mnemonic (pronounced "trouble" or "tribble" for Star Trek fans!):

T Top
R Right
B Bottom
L Left
Think of it clockwise: start at the top and work your way around the rectangle
This TRBL order also applies to padding and border properties!

Margin Shorthand Patterns

The margin property accepts 1 to 4 values with different meanings:

One Value

margin: 15px;

Applies 15px to all four sides

Two Values

margin: 10px 20px;

10px top/bottom, 20px left/right

Three Values

margin: 10px 20px 15px;

10px top, 20px left/right, 15px bottom

Four Values

margin: 10px 20px 15px 5px;

Top, Right, Bottom, Left (TRBL)

Pro tip: Use 1 value or 4 values consistently for clarity in your code

Removing Default Body Margins

Browsers add default margins to the <body> element, which can cause unexpected spacing.

The Problem

Even with margin: 0 on your elements, there's still space from the browser edge:

div {
  margin: 0px;
}
/* Still has space! */
Without removing body margins, element placement calculations will be off by 8-10 pixels!

The Solution

Reset the body margin to ensure elements start flush with the browser window:

body {
  margin: 0px;
}
Many developers use a CSS reset or normalize.css to standardize default browser styles across all browsers.

Understanding Margin Collapse

Vertical margins collapse between adjacent elements — they don't add up!

How It Works

.box1 {
  margin-bottom: 20px;
}
.box2 {
  margin-top: 30px;
}
/* Space between = 30px, NOT 50px! */

The space between the elements will be the larger of the two margins (30px), not their sum.

Why This Matters

  • Prevents excessive spacing between paragraphs and headings
  • Only affects vertical margins (top/bottom)
  • Horizontal margins (left/right) never collapse
  • Padding never collapses (it's inside the element)
Pro tip: Use margin-bottom OR margin-top consistently, but not both on stacked elements

Modern Units for Responsive Spacing

While px is simple, modern web development uses more flexible units:

Relative Units

  • rem — Relative to root element's font size (most consistent)
  • em — Relative to parent element's font size
  • % — Percentage of container width
  • vw/vh — Viewport width/height units

Example

/* Scales with user's font size */
.container {
  margin: 1rem;
  padding: 2rem;
}

/* Responsive to viewport */
.hero {
  margin: 5vh 0;
}
Using rem improves accessibility — spacing scales when users adjust browser font size!

Logical Properties for Internationalization

Modern CSS includes logical properties that adapt to different writing directions (LTR vs RTL):

Physical Properties (Old)

margin-left: 20px;
margin-right: 10px;
margin-top: 15px;
margin-bottom: 15px;

Fixed to physical directions — breaks for right-to-left languages like Arabic or Hebrew

Logical Properties (Modern)

margin-inline-start: 20px;
margin-inline-end: 10px;
margin-block-start: 15px;
margin-block-end: 15px;

Adapts automatically — inline-start is left in LTR, right in RTL!

When to use: If your site supports multiple languages, logical properties save you from maintaining separate stylesheets

Padding Elements

Padding adds space inside the rectangular area of an element.

Key Difference

  • Margin: Space outside the element
  • Padding: Space inside the element
  • Padding assumes the element's background color
  • Padding increases clickable area for buttons/links

Padding Properties

  • padding-top
  • padding-right
  • padding-bottom
  • padding-left
  • padding — Shorthand (uses TRBL order)
/* Two values */
padding: 12px 8px;
/* vertical  horizontal */
Remember: Padding follows the same TRBL shorthand rules as margins!

Understanding Box Sizing

One of the most confusing aspects for beginners: how browsers calculate element dimensions.

Default: content-box model

Margin (outside)
Border
Padding
Content
width: 200px
height: 100px
Problem: width/height only applies to content box!
Actual width: 200px (content) + 40px (padding) + 10px (border) = 250px

The Box-Sizing Solution

Use box-sizing: border-box; to make width/height include padding and border:

Without border-box

.box {
  width: 200px;
  padding: 20px;
  border: 5px solid black;
}
/* Actual width: 250px */
Calculations are difficult — you must manually account for padding and borders!

With border-box

.box {
  box-sizing: border-box;
  width: 200px;
  padding: 20px;
  border: 5px solid black;
}
/* Actual width: 200px ✓ */
The element is exactly 200px — content area shrinks to fit padding and border! 
/* Apply to all elements (best practice) */
* {
  box-sizing: border-box;
}

Margin vs Padding: When to Use Which?

Use Margin When:

  • You need space between elements
  • Centering with margin: 0 auto;
  • Creating consistent vertical rhythm
  • Pushing elements away from each other

Use Padding When:

  • You need space inside an element
  • Increasing clickable area on buttons/links
  • Creating breathing room around text
  • Background color should extend to the space

Key Differences

Aspect Margin Padding
Location Outside Inside
Background Transparent Inherits element
Clickable No Yes
Collapse Vertical: Yes Never

Keeping Everything Aligned

text-align (Horizontal)

  • left — Left-justified (default)
  • right — Right-justified
  • center — Centered
  • justify — Straight edges on both sides
.centered {
  text-align: center;
}

vertical-align (Inline Elements)

  • top — Aligns with top of line
  • middle — Aligns middle with parent's middle
  • bottom — Aligns with bottom of line
  • baseline — Aligns baselines
  • text-top / text-bottom
img {
  vertical-align: middle;
}
Important: vertical-align only works on inline or inline-block elements!

Centering Blocks of Content

Centering block-level elements requires a different approach than centering text:

Traditional Method

.centered-block {
  width: 50%;
  margin-left: auto;
  margin-right: auto;
}

/* Shorthand */
.centered-block {
  width: 50%;
  margin: 0 auto;
}
The element must have a width smaller than its container for this to work!

Modern Methods (Preview)

/* Flexbox */
.container {
  display: flex;
  justify-content: center;
}

/* Grid */
.container {
  display: grid;
  place-items: center;
}
You'll learn these powerful centering techniques in Lesson 12!
Never use the deprecated <center> tag — it's been removed from HTML!

Understanding the float Property

The float property moves elements to one side, allowing content to wrap around them.

How It Works

  • Elements float horizontally, not vertically
  • Values: left or right
  • Element moves as far as the container allows
  • Following content wraps around it
img {
  float: left;
  margin: 0 15px 15px 0;
}

Modern Usage

Use float for: Text wrapping around images (magazine-style layout)
Don't use float for: Page layouts — use Flexbox or Grid instead!
While float was historically used for entire page layouts, modern CSS (Flexbox and Grid) has replaced that use case

The clear Property

Use clear to control what appears after floated elements:

Clear Values

  • clear: left; — No floating elements on the left
  • clear: right; — No floating elements on the right
  • clear: both; — No floating elements on either side
  • clear: none; — Allow floating (default)

Example

img {
  float: left;
}

h2 {
  clear: left;
  /* Heading appears BELOW image,
     not wrapped around it */
}
Common use case: You have a floated image followed by paragraphs, but want a heading to appear below the image (not wrapped around it)

Modern Layout: The gap Property

Flexbox and Grid introduce gap, which simplifies spacing between items:

The Old Way (Margins)

.container {
  display: flex;
}

.item {
  margin-right: 20px;
}

/* But the last item also
   has margin! Need :last-child
   to fix it */

The Modern Way (Gap)

.container {
  display: flex;
  gap: 20px;
  /* That's it! */
}
gap creates space between items, but not around the edges! 
/* Different row and column gaps */
.grid-container {
  display: grid;
  gap: 20px 40px; /* 20px rows, 40px columns */
}

Summary: Key Takeaways

Margins & Padding

  • Margin = space outside element
  • Padding = space inside element
  • Both use TRBL shorthand order
  • Vertical margins collapse
  • Reset body margins for precise layouts
  • Use rem for responsive spacing

Alignment

  • text-align for horizontal alignment
  • vertical-align for inline elements only
  • Center blocks with margin: 0 auto;

Box Sizing

  • Use box-sizing: border-box; globally
  • Makes width/height include padding & border
  • Essential for predictable layouts

Float

  • Use for text wrapping around images
  • Don't use for page layouts anymore
  • Use clear to control flow after floats

Modern Techniques

  • Logical properties for internationalization
  • gap property for Flexbox/Grid spacing
  • Flexbox & Grid for layouts (Lesson 12)
