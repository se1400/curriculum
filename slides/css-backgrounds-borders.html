Background & Border Basics

The background Shorthand

One property sets all background sub-properties in order:

background: url('img.png') center / cover no-repeat padding-box #dfdfdf;

Sub-properties

  • background-image — image URL or gradient
  • background-position — placement keywords or lengths
  • background-size — after /, or keywords like cover
  • background-repeat — repeat, no-repeat, space, round
  • background-origin — border-box / padding-box / content-box
  • background-clip — same three values + text
  • background-attachment — scroll / fixed / local
  • background-color — fallback solid color

Common Examples

/* Color only */
background: #dfdfdf;

/* Image + fallback color */
background: url('paper.png') #dfdfdf;

/* Full shorthand */
background: url('hero.jpg')
  center / cover no-repeat;
Tip: You can use background on any HTML element — not just <body> or <div>.

Using Multiple Borders & Backgrounds

Faking Multiple Borders with box-shadow

CSS only supports one border, but box-shadow accepts a comma-separated list and can simulate stacked rings using a spread radius with zero blur:

.multi-border {
  border: 3px solid tomato;
  box-shadow:
    0 0 0 6px gold,
    0 0 0 10px steelblue;
}
  • Innermost: real border (tomato)
  • Middle: 6px gold "ring"
  • Outer: 10px steel blue "ring"
  • Does not affect layout — shadows don't push other elements
Note: box-shadow is outside the border-box. Design around it using margin if spacing matters.

Using Multiple Borders & Backgrounds

Layering Multiple Backgrounds

How It Works

  • List backgrounds separated by commas
  • First in list = top layer, last = bottom
  • Images with transparent areas let layers below show through
  • Final item can be a background-color (always bottommost)

Wolf + Moon Example

.box {
  width: 500px;
  height: 400px;
  background:
    url('wolf.svg') center bottom / cover no-repeat,
    url('moon.png') right top / 45% no-repeat;
}
The wolf is on top (first), moon below it. Transparent areas in each SVG let the other show through.
Forgotten Background Properties

background-clip & background-origin

background-clip

Controls where the background is painted (where it ends):

  • border-box — extends under the border (default)
  • padding-box — stops at the border edge
  • content-box — stops at the padding edge
  • text — clips background to text shape

background-origin

Controls where the background starts (position reference point):

  • border-box — starts at border edge
  • padding-box — starts at padding edge (default)
  • content-box — starts at content edge

Easy Way to Remember

Think of a painting:

  • background-origin = where the brush starts
  • background-clip = where the painter's tape ends
For multiple backgrounds, separate values with commas — first value = top layer: 
background-clip: content-box, border-box;
background-origin: content-box, padding-box;
MDN reference: background-clip  |  background-origin
Forgotten Background Properties

Gradient Text with background-clip: text

Clips a gradient (or image) background to the shape of the text — combined with a transparent text color, the gradient becomes the text fill:

.gradient-text {
  background: linear-gradient(
    to right, #ff6b6b, #feca57
  );
  -webkit-background-clip: text;
  -webkit-text-fill-color: transparent;
  background-clip: text;
}
Include both the -webkit- prefixed and unprefixed versions for maximum browser compatibility.
Works best on large, bold text. Thin or small text may not show the effect clearly. Ensure sufficient contrast for accessibility.

Forgotten Background Properties

background-attachment

Controls how the background moves relative to the viewport when the page scrolls:

scroll

Background stays fixed relative to the element but scrolls with the page. Default for most cases.

fixed

Background stays fixed relative to the viewport. Creates a "parallax" effect as the page scrolls over it.

local

Background scrolls with both the page and the element's own scroll. Useful for scrollable containers.

.scroll { background-attachment: scroll; }
.fixed  { background-attachment: fixed;  }
.local  { background-attachment: local;  }
Mobile Warning: background-attachment: fixed does not work as expected on iOS Safari or many mobile browsers — it often renders as scroll or causes performance issues. For parallax effects, prefer CSS scroll-driven animations.

Blending Backgrounds

background-blend-mode

Blends background layers within one element — like Photoshop layer modes:

.blended {
  background-image: url('photo.jpg');
  background-color: steelblue;
  background-blend-mode: multiply;
}

mix-blend-mode

Blends an entire element (including content) with whatever is visually behind it on the page:

.title-overlay {
  mix-blend-mode: screen;
}

/* "cut-out" text effect */
.cutout-text {
  mix-blend-mode: multiply;

The Frosted Glass Effect

Frosted Glass: filter vs backdrop-filter

filter — affects the element itself

Applies graphic effects to the element and all its content:

/* Desaturate image on hover */
img:hover {
  filter: grayscale(1);
}

/* Common filter values */
filter: blur(5px);
filter: brightness(0.8);
filter: contrast(1.5);
filter: grayscale(1);
filter: saturate(2);
filter: sepia(0.6);
Using filter: blur() blurs the element's text too — not ideal for frosted glass panels.

backdrop-filter — blurs what's behind

Applies effects only to the content behind the element — the element's own text stays sharp:

.glass-panel {
  background: rgba(255,255,255,0.25);
  backdrop-filter: blur(12px);
  border: 1px solid rgba(255,255,255,0.4);
  border-radius: 1rem;
}
Frosted glass panel
backdrop-filter: blur(8px)
Requires semi-transparent background. Fully opaque backgrounds hide the blur effect entirely.

Alternating Background Colors

Zebra-Stripe Tables with :nth-child()

No classes on every row needed — CSS auto-selects by position:

tbody tr:nth-child(odd) {
  background-color: #dfdfdf;
}

Selector Values

  • odd — rows 1, 3, 5, 7…
  • even — rows 2, 4, 6, 8…
  • 4 — exactly the 4th row
  • 2n+5 — every other, starting at 5th
  • 6n+2 — every 6th, starting at 2nd
Works on any element, not just tables: li:nth-child(odd), div:nth-child(3n), etc.

:nth-child() — Advanced Selectors

The of Keyword (Modern CSS)

Filter counting to only elements that match a specific type:

/* Count only <li> items, ignore others */
li:nth-child(odd of li) {
  background-color: #dfdfdf;
}

Supported in all modern browsers. Fixes problems when mixed content (like headings inside lists) would throw off counting.

Related Pseudo-Classes

  • :nth-of-type(n) — counts by element type
  • :nth-last-child(n) — counts from the bottom up
  • :nth-last-of-type(n) — type-based, bottom up
  • :first-child / :last-child — first & last
  • :only-child — sole child of its parent

Using Gradients as Backgrounds

CSS Gradients — Overview

Gradients are added as background images — no actual image files required. Three types:

linear-gradient()
radial-gradient()
conic-gradient()

Linear

Color changes along a straight line — horizontal, vertical, or diagonal.

background-image:
  linear-gradient(
    to right, red, blue
  );

Radial

Color radiates outward from a center point — creates circles or ellipses.

background-image:
  radial-gradient(
    circle, yellow, orange
  );

Conic

Color sweeps around a center point — like a pie chart or color wheel.

background-image:
  conic-gradient(
    red, yellow, green, red
  );
Always also set a fallback background-color for older browsers that might not support the gradient function.
Using Gradients as Backgrounds

Linear Gradients

Syntax

/* Minimum: two colors (top → bottom) */
background-image:
  linear-gradient(red, blue);

/* With direction */
background-image:
  linear-gradient(to right, red, blue);

/* Multiple color stops */
background-image:
  linear-gradient(
    45deg,
    #ff00d5,
    rgba(39,164,0,0.5) 35%,
    blue
  );

Direction Keywords

  • to right — left → right
  • to bottom — top → bottom (default)
  • to top left — diagonal to corner
  • 45deg — any angle (0° = up, 90° = right)

Using Gradients as Backgrounds

Repeating Gradients — CSS Patterns

repeating-linear-gradient()

Color stops define one tile — the browser repeats it to fill the element:

/* Diagonal blue & white stripes */
background-image:
  repeating-linear-gradient(
    45deg,
    #3a86ff,
    #3a86ff 10px,
    #ffffff 10px,
    #ffffff 20px
  );
  • Adjust the angle for different stripe directions
  • Adjust the pixel values for stripe width
  • Use 3+ colors for more complex patterns

repeating-radial-gradient()

background-image:
  repeating-radial-gradient(
    circle,
    #3a86ff 0px,
    #3a86ff 10px,
    transparent 10px,
    transparent 20px
  );

Rounded Corners & Aspect Ratio

Guaranteed Squares with aspect-ratio

The aspect-ratio property maintains a fixed width-to-height ratio regardless of content or container size:

/* Always a square */
.square {
  width: 200px;
  aspect-ratio: 1 / 1;  /* or just: 1 */
}

/* 16:9 video thumbnail */
.video-thumb {
  width: 100%;
  aspect-ratio: 16 / 9;
}

/* 4:3 card */
.card-image {
  width: 100%;
  aspect-ratio: 4 / 3;
}

/* Responsive circle with border-radius */
.avatar {
  width: 20%;
  aspect-ratio: 1;
  border-radius: 50%;
}
Supported in all modern browsers. Replaces the old "padding-top hack" for maintaining aspect ratios.

Image Borders with border-image

Syntax

/* border-image shorthand */
border-image:
  source  slice  width  outset  repeat;

/* Common pattern */
.fancy-border {
  border: 10px solid transparent;
  border-image:
    url('border.png') 30 round;
}

/* Gradient border! */

  border: 4px solid transparent;

Exploring Outlines

Accessible Focus with :focus-visible

The Problem

Browsers show a focus ring on every clicked element. Designers often removed it entirely with outline: none, breaking keyboard navigation for all users.

The Solution

:focus-visible shows the outline only when the browser determines keyboard navigation is being used — not on mouse clicks:

/* Remove focus ring for mouse clicks */
button:focus:not(:focus-visible) {
  outline: none;
}

/* Show a strong ring for keyboard nav */
button:focus-visible {
  outline: 3px solid #ffd166;
  outline-offset: 4px;
}
This is the modern, accessible way to handle focus styles. Supported in all current browsers.
