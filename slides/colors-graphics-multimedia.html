Best Practices for Choosing Colors

  • Natural palette — Use colors you'd see on a casual stroll; avoid ultra-bright colors that cause eye strain
  • Accessible combinations — WCAG 2.1 requires 4.5:1 contrast for normal text, 3:1 for large text (AA)
  • Good contrast — Dark text on light backgrounds is popular because it works
  • Small palette — Stick to 3–4 main colors + 1–2 accents
  • Consider demographics — Color preferences vary by age, culture, and region
  • Consider dark mode — Use prefers-color-scheme media query to adapt

Understanding Web Colors

CSS supports multiple ways to specify colors:

Color Formats

  • #ff0000 — Hexadecimal
  • rgb(255, 0, 0) — RGB
  • hsl(0, 100%, 50%) — HSL
  • rgba(255, 0, 0, 0.5) — RGB + alpha
  • hsla(0, 100%, 50%, 0.5) — HSL + alpha

Hexadecimal Color Values

Format: #rrggbb — each pair is 00 (none) to ff (full brightness)

How It Works

  • ff = full brightness (255)
  • cc = 80% brightness (204)
  • 99 = 60% brightness (153)
  • 66 = 40% brightness (102)
  • 33 = 20% brightness (51)
  • 00 = none (0)

HSL and HSLa Values

Many designers prefer HSL because it's intuitive — adjust hue, saturation, and lightness independently.

Format: hsl(hue, saturation, lightness)

  • Hue — 0–360 on the color wheel (0=red, 120=green, 240=blue)
  • Saturation — 0% (gray) to 100% (full color)
  • Lightness — 0% (black) to 100% (white), 50% = normal
/* Pure red */
color: hsl(0, 100%, 50%);

/* Muted dark blue */
color: hsl(240, 50%, 30%);

/* Pastel green */
color: hsl(120, 40%, 80%);

Why HSL?

  • Want it darker? Decrease lightness
  • Want it muted? Decrease saturation
  • Easy to create cohesive color schemes

HSLa — With Transparency

/* Semi-transparent red */
background-color: hsla(0, 100%, 50%, 0.5);

CSS Custom Properties for Colors

Define colors once, use everywhere — makes theme changes easy.

Define Variables

:root {
  --primary: #3498db;
  --secondary: #2ecc71;
  --text: #333333;
  --bg: #ffffff;
}

body {
  background: var(--bg);
  color: var(--text);
}

h1 { color: var(--primary); }
a  { color: var(--secondary); }

Dark Mode with Variables

:root {
  --text: #333333;
  --bg: #ffffff;
}

@media (prefers-color-scheme: dark) {
  :root {
    --text: #f0f0f0;
    --bg: #1a1a1a;
  }
}

All elements using these variables automatically adapt when the user switches between light and dark mode.

  • Lossless (no quality loss)
  • Full alpha transparency
  • .png

Quick guide: Use JPEG for photos, PNG for graphics with transparency or sharp text, GIF only for simple animations.

Preparing Images for the Web

Cropping

  • Apply the rule of thirds
  • Place subjects at intersections
  • Remove distracting edges

Resizing

  • Always resize down, never up
  • Keep aspect ratio locked
  • 2000–2500px max width

Color Correction

  • Adjust brightness & contrast
  • Fix color balance
  • Keep original as backup

Placing Images with <img>

Basic Syntax

<img src="photo.jpg"
     alt="A sunset over the ocean"
     width="800"
     height="600">

Key Attributes

  • src — Path to the image (required)
  • alt — Text description (required for accessibility)
  • width / height — Prevents layout shift
  • loading="lazy" — Defer offscreen images

Why alt Text Matters

  • Screen readers announce it for blind users
  • Displays when image fails to load
  • Helps search engines understand images
  • Use alt="" for purely decorative images
<!-- Decorative: empty alt -->
<img src="divider.png" alt="">

<!-- Informative: describe it -->
<img src="chart.png"
  alt="Sales increased 30% in Q4">

Responsive Images

Serve the right image size for each device using <picture> and srcset.

The <picture> Element

<picture>
  <source srcset="photo.avif"
    type="image/avif">
  <source srcset="photo.webp"
    type="image/webp">
  <img src="photo.jpg"
    alt="Fallback for older browsers">
</picture>

Browser picks the first supported format.

The srcset Attribute

<img
  srcset="photo-400.jpg 400w,
          photo-800.jpg 800w,
          photo-1200.jpg 1200w"
  sizes="(max-width: 600px) 400px,
         (max-width: 1000px) 800px,
         1200px"
  src="photo-800.jpg"
  alt="Responsive photo">

Browser picks the best size for the viewport.

Background Images with CSS

Key Properties

.hero {
  background-image:
    url('hero.jpg');
  background-size: cover;
  background-position: center;
  background-repeat: no-repeat;
}

/* Shorthand */
.hero {
  background:
    url('hero.jpg')
    center / cover
    no-repeat;
}

background-size

  • cover — Fill container, may crop
  • contain — Fit inside, may leave gaps
  • 100% auto — Specific dimensions

background-repeat

  • no-repeat — Show once
  • repeat — Tile in both directions
  • repeat-x / repeat-y — Tile one direction

Image Maps

Define clickable regions within a single image, each linking to a different destination.

<img src="workspace.jpg"
  usemap="#workspace"
  alt="Office workspace">

<map name="workspace">
  <area shape="rect"
    coords="0,0,200,150"
    href="monitor.html"
    alt="Monitor">
  <area shape="circle"
    coords="350,200,50"
    href="coffee.html"
    alt="Coffee mug">
  <area shape="poly"
    coords="500,100,600,50,
            700,100,600,200"
    href="plant.html"
    alt="Plant">
</map>

Shapes Available

  • rect — Rectangle (x1, y1, x2, y2)
  • circle — Circle (centerX, centerY, radius)
  • poly — Polygon (x1,y1, x2,y2, ...)

Image maps aren't responsive by default. For modern sites, consider using CSS-positioned links over an image instead, or use a library like image-map.

HTML5 Audio

Basic Usage

<audio controls>
  <source src="song.mp3"
    type="audio/mpeg">
  <source src="song.ogg"
    type="audio/ogg">
  Your browser does not
  support the audio element.
</audio>

Attributes

  • controls — Show play/pause UI
  • autoplay — Start playing (often blocked)
  • loop — Repeat when finished
  • muted — Start muted
  • preload — none, metadata, or auto

Supported Formats

Provide multiple <source> elements as fallbacks. The browser uses the first format it supports.

HTML5 Video

Basic Usage

<video controls width="640"
  height="360" poster="thumb.jpg">
  <source src="video.mp4"
    type="video/mp4">
  <source src="video.webm"
    type="video/webm">
  Your browser does not
  support the video element.
</video>

Extra Attributes

  • poster — Thumbnail before playing
  • width / height — Dimensions
  • playsinline — Inline playback on mobile

Embedding YouTube/Vimeo

<iframe width="560" height="315"
  src="https://www.youtube.com/
       embed/VIDEO_ID"
  title="Video title"
  frameborder="0"
  allow="accelerometer;
    autoplay; encrypted-media"
  allowfullscreen>
</iframe>

Tip: For external videos, embedding via <iframe> is usually better than self-hosting — saves bandwidth and handles format support automatically.

Accessibility with Images & Multimedia

Images

  • Always provide meaningful alt text
  • Use alt="" for decorative images
  • Don't use color alone to convey info
  • Ensure sufficient contrast (4.5:1 minimum)

Video & Audio

  • Provide captions for video
  • Provide transcripts for audio
  • Never autoplay with sound

Summary

Colors

  • Use accessible, high-contrast palettes
  • Hex, RGB, HSL — pick your format
  • CSS custom properties simplify themes
  • Support dark mode with prefers-color-scheme

Graphics

  • JPEG for photos, PNG for transparency, SVG for icons
  • WebP/AVIF for smaller modern files
  • Always include alt text

Images in CSS & HTML

  • <img> with alt, width, height
  • <picture> and srcset for responsive images
  • CSS background-image for decorative use

Multimedia

  • <audio> and <video> with fallback sources
  • Embed YouTube/Vimeo via <iframe>
  • Always provide captions and transcripts
