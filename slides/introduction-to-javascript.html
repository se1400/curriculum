Learning Web Scripting Basics

  • Web pages would be boring without scripts
  • Scripting adds interactivity to otherwise static pages
  • Complex functionality could once only be done with compiled languages
  • JavaScript now enables this functionality within web browsers

Client-side means scripts run in the browser on the user's computer, not on the web server.

Scripts and Programs

Programs

  • Written in compiled languages (C, C++, Java)
  • Must be compiled before running
  • Platform-specific
  • Source code is not visible to users

Scripts

  • Written in interpreted languages (JavaScript)
  • Run directly without compilation
  • Cross-platform (runs in any browser)
  • Source code is visible in the browser

Scripts rely on another application (the browser) to run. Programs are standalone applications.

Introducing JavaScript

  • Created by Netscape in 1995 as "LiveScript"
  • Renamed to JavaScript due to Java's popularity
  • JavaScript is NOT Java (despite the similar name)
  • Microsoft created JScript (a JavaScript clone) for Internet Explorer
  • ECMA International standardized JavaScript as ECMAScript
  • Modern versions are ES6 (2015) and newer

JavaScript is the standard client-side scripting language supported by all major browsers.

How JavaScript Fits into a Web Page

There are several ways to include JavaScript in your web pages:

  • In the body - Scripts in <script> tags execute when the page loads
  • In the head - Functions defined for later use
  • Event handlers - Code that responds to user actions
  • External files - Separate .js files linked to the page

Script in the Body

Scripts in the body execute immediately when the browser reaches them.

Script in the Head

Scripts in the <head> are typically functions called later:

<head>
  <script>
    function greetUser() {
      alert("Welcome!");
    }
  </script>
</head>
<body>
  <button onclick="greetUser()">
    Click Me
  </button>
</body>
  • Functions are defined but not executed immediately
  • Called when a user action triggers them
  • Ensures functions are available before the page content loads

Modern practice often places scripts at the end of the body or uses defer attribute.

Event Handlers

Event handlers respond to user actions directly in HTML attributes.

Using Separate JavaScript Files

Store JavaScript in external .js files:

<script src="myscript.js"></script>

Advantages

  • Keeps HTML cleaner and more readable
  • Browser can cache the script file
  • Same script can be reused across multiple pages
  • Easier to maintain and debug

Example: myscript.js

function showMessage() {
  alert("Hello from external file!");
}

function calculateSum(a, b) {
  return a + b;
}

External files should not include <script> tags - just the JavaScript code.

Understanding JavaScript Events

Events are actions that happen in the browser:

Mouse Events

  • onclick
  • ondblclick
  • onmouseover
  • onmouseout

Keyboard Events

  • onkeydown
  • onkeyup
  • onkeypress

Page Events

  • onload
  • onunload
  • onresize
  • onscroll

Form Events

Events specific to form elements:

  • onfocus - Element receives focus
  • onblur - Element loses focus
  • onchange - Value changes
  • onsubmit - Form is submitted
  • oninput - Input value changes
<input type="text"
  onfocus="this.style.background='yellow'"
  onblur="this.style.background='white'">

<form onsubmit="return validateForm()">
  ...
</form>

Exploring JavaScript's Capabilities

JavaScript can do many things in web pages:

  • Navigation - Browser history, redirects, new windows
  • Form Validation - Check user input before submitting
  • DOM Manipulation - Change page content dynamically
  • Animations - Create visual effects and transitions
  • AJAX - Load data without refreshing the page
  • Storage - Save data in the browser
  • Multimedia - Control audio and video
  • Graphics - Draw with Canvas and SVG

JavaScript Variables

Declaring Variables

// Modern way (preferred)
const name = "John";  // Cannot be reassigned
let age = 25;         // Can be reassigned

// Older way (avoid)
var score = 100;      // Function scoped

When to Use

  • const - Value won't change (default choice)
  • let - Value will change
  • var - Avoid in modern code

Use const by default, let when needed.

The Date Object

JavaScript's built-in Date object handles dates and times:

const now = new Date();

// Get components
now.getFullYear();  // 2024
now.getMonth();     // 0-11
now.getDate();      // 1-31
now.getDay();       // 0-6 (Sun-Sat)
now.getHours();     // 0-23
now.getMinutes();   // 0-59
now.getSeconds();   // 0-59

Display Methods

now.toDateString();
// "Mon Jan 15 2024"

now.toTimeString();
// "14:30:00 GMT-0700"

now.toLocaleString();
// "1/15/2024, 2:30:00 PM"

Using Browser Developer Tools

Opening Dev Tools

  • Chrome/Edge: F12 or Ctrl+Shift+I (Cmd+Option+I on Mac)
  • Firefox: F12 or Ctrl+Shift+I
  • Safari: Cmd+Option+I (enable in Preferences first)

Key Panels

  • Console - View errors and log messages
  • Elements/Inspector - Examine HTML and CSS
  • Sources/Debugger - Step through code
  • Network - Monitor requests

The Console is your best friend for debugging JavaScript!

Debugging with console.log()

Output messages to the browser console:

// Log text
console.log("Hello, console!");

// Log variables
const name = "Alice";
console.log("User:", name);

// Log multiple values
console.log("Sum:", 5 + 3);

// Log objects
console.log({ name: "Bob", age: 30 });

Other Console Methods

// Information
console.info("Info message");

// Warnings
console.warn("Warning message");

// Errors
console.error("Error message");

// Tables
console.table([
  { name: "Alice", age: 25 },
  { name: "Bob", age: 30 }
]);

// Timing
console.time("timer");
// ... code to time ...
console.timeEnd("timer");
  • console.info() - Informational messages
  • console.warn() - Warnings (yellow)
  • console.error() - Errors (red)
  • console.table() - Display data as a table
  • console.time() / timeEnd() - Measure execution time
  • console.clear() - Clear the console

Dealing with JavaScript Errors

Common Error Types

  • SyntaxError - Code has invalid syntax
  • ReferenceError - Variable doesn't exist
  • TypeError - Value is wrong type
  • RangeError - Number out of range

Finding Errors

  • Check the browser console (F12)
  • Errors show file name and line number
  • Stack trace shows where error occurred
  • Click the filename to jump to source code

JavaScript Error Examples

SyntaxError

// Missing closing parenthesis
console.log("Hello";

ReferenceError

// Variable not defined
console.log(undefinedVar);

TypeError

// Calling non-function
const num = 42;
num();

Error message in the browser console

Debugging: Finding the Error Line

Browser Error Highlighting

  • Click the filename/line in the console
  • Browser opens the Sources/Debugger panel
  • Error line is highlighted
  • Set breakpoints to pause execution
  • Step through code line by line

Firefox highlighting the line with the error

Handling Errors with Try-Catch

try {
  // Code that might cause an error
  const result = riskyOperation();
  console.log(result);
} catch (error) {
  // Handle the error
  console.error("An error occurred:", error.message);
} finally {
  // Always runs (optional)
  console.log("Cleanup complete");
}
  • try - Wrap potentially problematic code
  • catch - Handle errors gracefully
  • finally - Code that always runs
  • Prevents script from crashing
  • Provides meaningful error messages to users

JavaScript Best Practices

  • Use const and let instead of var
  • Use meaningful variable and function names
  • Add comments to explain complex logic
  • Use strict equality (===) instead of (==)
  • Keep functions small and focused
  • Use external JavaScript files
  • Test in multiple browsers
  • Use the console for debugging
  • Handle errors with try-catch
  • Keep learning - JavaScript evolves constantly

Q&A

Q: What is JavaScript used for?

A: Adding interactive elements to web pages, form validation, dynamic content updates, animations, and communicating with servers.

Q: Is JavaScript the same as Java?

A: No. They are completely different languages. The similar name was a marketing decision in 1995.

Q: Where do I put JavaScript code?

A: In the body, head, inline event handlers, or external .js files. External files are preferred for larger scripts.

Q: How do I debug JavaScript?

A: Use browser developer tools (F12), check the console for errors, and use console.log() to trace your code.
